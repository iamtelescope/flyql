`tokenize()` — это чистая функция, которая превращает выражение FlyQL в плоский список типизированных токенов. Это общий фундамент, на котором можно построить любой подсветчик синтаксиса, pretty-printer или форматтер FlyQL. Функция доступна во всех трёх языках с побайтно идентичной семантикой — один и тот же вход даёт одни и те же токены в JavaScript, Python и Go.

Используйте её, когда нужен контроль рендеринга на уровне токенов (вывод в консоль с ANSI, свой HTML, декорации в редакторе, линтеры, работающие с токенами). Если нужна только HTML-строка, используйте более высокоуровневый хелпер [`flyql/highlight`](/ru/getting-started/javascript/#доступные-подпути-импорта) (только JavaScript).

## Форма токена

Каждый токен — это простая запись с четырьмя полями:

| Поле | Тип | Описание |
|------|-----|----------|
| `text` | `string` | Фрагмент исходного входа, который покрывает этот токен. |
| `type` | `string` | Одно из значений `CharType` (см. [ниже](#типы-токенов)). |
| `start` | `int` | Смещение первого символа в `text` (с нуля). |
| `end` | `int` | Смещение, следующее сразу за последним символом — `end == start + len(text)`. |

Токены удовлетворяют двум инвариантам, на которые можно опираться:

- **Круговой обход:** конкатенация всех `text` по порядку точно воспроизводит исходный ввод.
- **Смежные смещения без пропусков:** `tokens[0].start == 0`, `tokens[i].start == tokens[i-1].end` для всех `i > 0`, и `tokens[-1].end == len(input)` (для непустого входа).

Вместе эти два свойства позволяют форматтеру пройтись по списку токенов, разметить каждый `text` и никогда не пропустить символ и не пересечь границу.

## API

`tokenize` принимает входную строку и опциональный режим. Только режим `"query"` реализован во всех трёх языках; режим `"columns"` доступен только в JavaScript.

### JavaScript

```javascript
import { tokenize } from 'flyql/tokenize'

const tokens = tokenize("status=200 and region='us-east'")
// [
//   { text: 'status',    type: 'flyqlKey',      start:  0, end:  6 },
//   { text: '=',         type: 'flyqlOperator', start:  6, end:  7 },
//   { text: '200',       type: 'number',        start:  7, end: 10 },
//   { text: ' ',         type: 'space',         start: 10, end: 11 },
//   { text: 'and',       type: 'flyqlOperator', start: 11, end: 14 },
//   { text: ' ',         type: 'space',         start: 14, end: 15 },
//   { text: 'region',    type: 'flyqlKey',      start: 15, end: 21 },
//   { text: '=',         type: 'flyqlOperator', start: 21, end: 22 },
//   { text: "'us-east'", type: 'string',        start: 22, end: 31 },
// ]

// Режим columns (только JavaScript)
const columnTokens = tokenize('id as user_id', { mode: 'columns' })
```

### Python

```python
from flyql import tokenize

tokens = tokenize("status=200 and region='us-east'")
for t in tokens:
    print(t.text, t.type.value, t.start, t.end)

# Python Token — это frozen dataclass с полями: text, type, start, end
```

Режим columns не поддерживается в Python и вызывает `ValueError`:

```python
tokenize("id", mode="columns")
# ValueError: columns mode is only available in the JavaScript package
```

### Go

```go
import "github.com/iamtelescope/flyql/golang"

tokens, err := flyql.Tokenize("status=200 and region='us-east'", "query")
if err != nil {
    // не-nil для неподдерживаемых режимов; "query" никогда не возвращает ошибку
}
for _, t := range tokens {
    fmt.Println(t.Text, t.Type, t.Start, t.End)
}
```

Режим columns возвращает не-nil ошибку: `columns mode is only available in the JavaScript package`.

## Типы токенов

Токенизатор выдаёт константы `CharType` из базового парсера. В режиме query наиболее интересны следующие типы:

| Тип | Пример | Когда он появляется |
|-----|--------|---------------------|
| `flyqlKey` | `status`, `meta.region` | Имена полей слева от оператора |
| `flyqlOperator` | `=`, `!=`, `>=`, `and`, `or`, `not` | Операторы сравнения и логические |
| `number` | `200`, `-42`, `3.14`, `1e5` | Числовые литералы (см. [каноническое правило чисел](#каноническое-правило-чисел)) |
| `string` | `'ok'`, `"hello"` | Строковые литералы в кавычках |
| `flyqlBoolean` | `true`, `false` | Булевы литералы |
| `flyqlNull` | `null` | Литерал null |
| `flyqlColumn` | `alice`, `Infinity` | Некавыченное значение, не являющееся литералом — трактуется как ссылка на колонку |
| `flyqlPipe` | `\|` | Разделитель трансформеров |
| `flyqlTransformer` | `upper`, `chars` | Имена трансформеров |
| `flyqlArgument` | `25` (внутри `chars(25)`) | Аргумент трансформера |
| `flyqlParameter` | `$name`, `$1` | Плейсхолдеры параметров |
| `flyqlWildcard` | `*` | Подстановочный знак в значениях |
| `space` | ` ` | Пробельные символы между токенами |
| `flyqlError` | `y` в `x!y` | Хвост ввода, который парсер не смог прочитать |

Режим columns использует собственный набор: `column`, `alias`, `aliasOperator`, `operator`, `transformer`, `argument`, `space`, `error`.

Обратите внимание, что `flyqlValue` **никогда не появляется** в выходе — каждый сырой токен значения из парсера повышается правилом ниже до того, как `tokenize` вернёт результат.

## Правило повышения VALUE

Парсер изначально помечает каждый символ значения типом `flyqlValue`. Перед возвратом `tokenize` повышает каждый токен значения до более конкретного типа:

1. `"true"` или `"false"` → `flyqlBoolean`
2. `"null"` → `flyqlNull`
3. соответствует [каноническому правилу чисел](#каноническое-правило-чисел) → `number`
4. начинается с `'` или `"` → `string`
5. иначе → `flyqlColumn`

Эта логика работает только в режиме query. В режиме columns понятия «значение» нет.

## Каноническое правило чисел

Токен значения классифицируется как `number` только когда он точно соответствует регулярному выражению:

```
^-?\d+(\.\d+)?([eE][+-]?\d+)?$
```

Оно **намеренно уже**, чем `parseFloat` в JavaScript, `float()` в Python или `strconv.ParseFloat` в Go. Следующие входы **не являются** числами и проваливаются в `flyqlColumn`:

- `Infinity`, `-Infinity`
- `NaN`
- шестнадцатеричные литералы вроде `0x1F`
- числа с пробелами по краям, например `  42  `

Все три языка разделяют один и тот же регекс, поэтому `val=Infinity` токенизируется одинаково в JavaScript, Python и Go.

## Хвостовой ERROR-токен

Если парсер останавливается до того, как прочитал весь вход, `tokenize` добавляет один финальный токен, покрывающий непрочитанный хвост:

```javascript
tokenize('x!y')
// [
//   { text: 'x', type: 'flyqlKey',      start: 0, end: 1 },
//   { text: '!', type: 'flyqlOperator', start: 1, end: 2 },
//   { text: 'y', type: 'flyqlError',    start: 2, end: 3 },
// ]
```

Это сохраняет инвариант кругового обхода даже для некорректного ввода. Форматтеры могут рендерить хвостовой `flyqlError` (или `error` в режиме columns) отдельным стилем — JavaScript-хелпер `highlight()` использует CSS-класс `flyql-error`.

Пустой вход возвращает пустой список, а не хвостовой error-токен.

## Ограничения

- **Режим columns только в JavaScript.** Только JavaScript-парсер колонок пишет вывод типизированных символов; парсеры колонок в Python и Go этого не делают, поэтому `tokenize()` в этих языках не может быть расширен до режима columns.
- **Гарантии только для ASCII.** Параллельность смещений между языками гарантирована только для ASCII-входа. Не-ASCII-вход может давать разные `start`/`end` между языками из-за разницы байт/кодпойнт/UTF-16.
- **Нет номеров строк и колонок.** Токены содержат только символьные смещения. Если нужны пары `(line, column)` для редактора, вычисляйте их из `start` относительно исходного текста.
