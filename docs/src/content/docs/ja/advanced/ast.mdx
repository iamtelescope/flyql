FlyQL の parser は **抽象構文木（AST）** を生成します。これはクエリを構造化して表現したもので、これをたどることで任意のターゲット向けに出力を生成できます。組み込みの SQL generator（ClickHouse、PostgreSQL、StarRocks）はすべて、この同じ AST を走査して実装されています。Elasticsearch クエリ、MongoDB フィルタ、Prometheus セレクタ、ログ検索用 `grep` コマンドなど、任意のターゲット向けに独自 generator を構築できます。

## AST ノード型

AST は 3 種類の型で構成されます。**Node**（木構造）、**Expression**（葉の条件）、**Key**（フィールド参照）です。

### Node

`Node` は **branch**（2 つの部分木をブール演算子で結合するノード）か、**leaf**（単一の expression を保持するノード）のいずれかです。AST の基本的な構成要素です。

| フィールド | 型 | 説明 |
|------------|----|------|
| `boolOperator` | `string` | <FlyqlToken type="operator">and</FlyqlToken> または <FlyqlToken type="operator">or</FlyqlToken>。`left` と `right` の結合方法を示します。leaf ノードでは空です |
| `expression` | `Expression` or `null` | leaf ノードの場合の条件です。branch ノードでは `null` です |
| `left` | `Node` or `null` | 左側の子部分木。leaf ノードでは `null` です |
| `right` | `Node` or `null` | 右側の子部分木。leaf ノードでは `null` です |
| `negated` | `boolean` | このノードが <FlyqlToken type="operator">not</FlyqlToken> で包まれている場合は `true` |

ノードは、`expression` が設定され `left` / `right` が `null` の **leaf** か、`left` / `right` が設定され `expression` が `null` の **branch** のどちらかです。両方になることはありません。

### Expression

`Expression` は、キー、演算子、値からなる単一条件を表します。

| フィールド | 型 | 説明 |
|------------|----|------|
| `key` | `Key` | 評価対象のフィールド。例: <FlyqlToken type="key">status</FlyqlToken>、<FlyqlToken type="key">user.metadata.role</FlyqlToken> |
| `operator` | `string` | <FlyqlToken type="operator">=</FlyqlToken>, <FlyqlToken type="operator">!=</FlyqlToken>, <FlyqlToken type="operator">{'~'}</FlyqlToken>, <FlyqlToken type="operator">{'!~'}</FlyqlToken>, <FlyqlToken type="operator">{'>'}</FlyqlToken>, <FlyqlToken type="operator">{'<'}</FlyqlToken>, <FlyqlToken type="operator">{'>='}</FlyqlToken>, <FlyqlToken type="operator">{'<='}</FlyqlToken>, <FlyqlToken type="operator">truthy</FlyqlToken>, <FlyqlToken type="operator">in</FlyqlToken>, <FlyqlToken type="operator">not in</FlyqlToken>, <FlyqlToken type="operator">has</FlyqlToken>, <FlyqlToken type="operator">not has</FlyqlToken> のいずれか |
| `value` | `any` | 比較値。スカラー演算子では string、number、boolean、null のいずれかです。<FlyqlToken type="operator">in</FlyqlToken> と <FlyqlToken type="operator">not in</FlyqlToken> では空です |
| `valueType` | `string` or `null` | `value` の明示的な型。`"int"`, `"bigint"`, `"float"`, `"string"`, `"bool"`, `"null"` のいずれか。<FlyqlToken type="operator">in</FlyqlToken> / <FlyqlToken type="operator">not in</FlyqlToken> では `null` |
| `values` | `array` or `null` | <FlyqlToken type="operator">in</FlyqlToken> / <FlyqlToken type="operator">not in</FlyqlToken> 用の値リスト。異種型も可。他の演算子では `null` |
| `valuesTypes` | `string[]` or `null` | `values` と対応する要素ごとの型配列。`values` を使わない場合は `null` |

### Key

`Key` は、問い合わせ対象となるフィールドパスを表します。

| フィールド | 型 | 説明 |
|------------|----|------|
| `segments` | `string[]` | パスセグメント。単純キーなら `["status"]`、ネストキーなら `["user", "metadata", "role"]` |
| `raw` | `string` | クエリで書かれた元のキー文字列 |
| `isSegmented` | `boolean` | キーが複数セグメントを持つ、つまりドット記法を使っている場合に `true` |

### Parameter

`Parameter` は未解決のプレースホルダ（`$name` または `$1`）を表します。SQL 生成やマッチングの前に `bindParams()` によって置き換えられます。

| フィールド | 型 | 説明 |
|------------|----|------|
| `name` | `string` | `$` プレフィックスを除いたプレースホルダ識別子。名前付きの場合は `"code"`、位置指定の場合は `"1"` |
| `positional` | `boolean` | パラメータが位置指定（数字のみの名前）の場合は `true` |

`Parameter` は次のいずれかの位置に現れます。

- `expression.value`（`valueType === "parameter"`、スカラー位置）
- `expression.values` 内の要素（IN リスト位置）
- `FunctionCall.parameterArgs` 内の要素（テンポラル関数の引数位置）

`bindParams()` で解決されると、AST には `Parameter` インスタンスは残りません。

### FunctionCall

`FunctionCall` はテンポラル関数値（`ago(...)`、`now()`、`today(...)`、`startOf(...)`）を表します。

| フィールド | 型 | 説明 |
|------------|----|------|
| `name` | `string` | `"ago"`、`"now"`、`"today"`、`"startOf"` のいずれか |
| `durationArgs` | `Duration[]` | parse 済みの duration 引数（例: `[{value: 5, unit: "m"}]`） |
| `unit` | `string` | `startOf` の unit 引数（`"day"`、`"week"`、`"month"`） |
| `timezone` | `string` | `today` および `startOf` のタイムゾーン引数 |
| `parameterArgs` | `Parameter[]` | バインド時に `durationArgs` / `unit` / `timezone` を埋めるパラメータプレースホルダ |

## AST を走査する

AST は二分木です。処理するときは再帰走査を使います。

1. ノードが `expression` を持つ場合（leaf ノード）は、その条件を処理します。
2. ノードが `left` / `right` を持つ場合（branch ノード）は、子ノードを再帰的に処理し、`boolOperator` で結合します。
3. `negated` が true の場合は、結果を否定で包みます。

## カスタム Generator を作る

以下は、FlyQL AST を **Elasticsearch Query DSL** オブジェクトへ変換するカスタム generator の完全な例です。

### JavaScript

```javascript
import { parse } from 'flyql'
import { Operator } from 'flyql'

function generateElasticsearch(node) {
  if (!node) return { match_all: {} }

  let result

  if (node.expression) {
    // Leaf node — convert the expression
    result = expressionToES(node.expression)
  } else {
    // Branch node — recurse and combine
    const left = generateElasticsearch(node.left)
    const right = generateElasticsearch(node.right)

    if (node.boolOperator === 'and') {
      result = { bool: { must: [left, right] } }
    } else {
      result = { bool: { should: [left, right], minimum_should_match: 1 } }
    }
  }

  // Handle negation
  if (node.negated) {
    result = { bool: { must_not: [result] } }
  }

  return result
}

function expressionToES(expr) {
  const field = expr.key.raw

  switch (expr.operator) {
    case Operator.EQUALS:
      return { term: { [field]: expr.value } }

    case Operator.NOT_EQUALS:
      return { bool: { must_not: [{ term: { [field]: expr.value } }] } }

    case Operator.REGEX:
      return { regexp: { [field]: String(expr.value) } }

    case Operator.NOT_REGEX:
      return { bool: { must_not: [{ regexp: { [field]: String(expr.value) } }] } }

    case Operator.GREATER_THAN:
      return { range: { [field]: { gt: expr.value } } }

    case Operator.GREATER_OR_EQUALS_THAN:
      return { range: { [field]: { gte: expr.value } } }

    case Operator.LOWER_THAN:
      return { range: { [field]: { lt: expr.value } } }

    case Operator.LOWER_OR_EQUALS_THAN:
      return { range: { [field]: { lte: expr.value } } }

    case Operator.IN:
      return { terms: { [field]: expr.values || [] } }

    case Operator.NOT_IN:
      return { bool: { must_not: [{ terms: { [field]: expr.values || [] } }] } }

    case Operator.TRUTHY:
      return { exists: { field } }

    case Operator.HAS:
      return { wildcard: { [field]: `*${expr.value}*` } }

    case Operator.NOT_HAS:
      return { bool: { must_not: [{ wildcard: { [field]: `*${expr.value}*` } }] } }

    default:
      throw new Error(`unsupported operator: ${expr.operator}`)
  }
}

// Usage
const result = parse('status = 200 and message ~ "error.*" and env in ["prod", "staging"]')
const esQuery = generateElasticsearch(result.root)
console.log(JSON.stringify(esQuery, null, 2))
```

### Go

```go
package main

import (
	"encoding/json"
	"fmt"
	flyql "github.com/iamtelescope/flyql/golang"
)

func generateElasticsearch(node *flyql.Node) map[string]any {
	if node == nil {
		return map[string]any{"match_all": map[string]any{}}
	}

	var result map[string]any

	if node.Expression != nil {
		// Leaf node — convert the expression
		result = expressionToES(node.Expression)
	} else {
		// Branch node — recurse and combine
		left := generateElasticsearch(node.Left)
		right := generateElasticsearch(node.Right)

		if node.BoolOperator == flyql.BoolOpAnd {
			result = map[string]any{
				"bool": map[string]any{"must": []any{left, right}},
			}
		} else {
			result = map[string]any{
				"bool": map[string]any{"should": []any{left, right}, "minimum_should_match": 1},
			}
		}
	}

	// Handle negation
	if node.Negated {
		result = map[string]any{
			"bool": map[string]any{"must_not": []any{result}},
		}
	}

	return result
}

func expressionToES(expr *flyql.Expression) map[string]any {
	field := expr.Key.Raw

	switch expr.Operator {
	case flyql.OpEquals:
		return map[string]any{"term": map[string]any{field: expr.Value}}
	case flyql.OpNotEquals:
		return map[string]any{"bool": map[string]any{"must_not": []any{map[string]any{"term": map[string]any{field: expr.Value}}}}}
	case flyql.OpGreater:
		return map[string]any{"range": map[string]any{field: map[string]any{"gt": expr.Value}}}
	case flyql.OpLess:
		return map[string]any{"range": map[string]any{field: map[string]any{"lt": expr.Value}}}
	case flyql.OpIn:
		return map[string]any{"terms": map[string]any{field: expr.Values}}
	case flyql.OpNotIn:
		return map[string]any{"bool": map[string]any{"must_not": []any{map[string]any{"terms": map[string]any{field: expr.Values}}}}}
	case flyql.OpTruthy:
		return map[string]any{"exists": map[string]any{"field": field}}
	case flyql.OpRegex:
		return map[string]any{"regexp": map[string]any{field: fmt.Sprintf("%v", expr.Value)}}
	default:
		panic(fmt.Sprintf("unsupported operator: %s", expr.Operator))
	}
}

func main() {
	result, err := flyql.Parse(`status = 200 and env in ["prod", "staging"]`)
	if err != nil {
		panic(err)
	}

	esQuery := generateElasticsearch(result.Root)
	data, _ := json.MarshalIndent(esQuery, "", "  ")
	fmt.Println(string(data))
}
```

### Python

```python
from flyql import parse
from flyql.core.constants import Operator, BoolOperator

def generate_elasticsearch(node):
    if node is None:
        return {"match_all": {}}

    if node.expression is not None:
        # Leaf node — convert the expression
        result = expression_to_es(node.expression)
    else:
        # Branch node — recurse and combine
        left = generate_elasticsearch(node.left)
        right = generate_elasticsearch(node.right)

        if node.bool_operator == BoolOperator.AND.value:
            result = {"bool": {"must": [left, right]}}
        else:
            result = {"bool": {"should": [left, right], "minimum_should_match": 1}}

    # Handle negation
    if node.negated:
        result = {"bool": {"must_not": [result]}}

    return result

def expression_to_es(expr):
    field = expr.key.raw
    op = expr.operator  # operator is stored as a string value, e.g. "=", "!=", "~"

    if op == Operator.EQUALS.value:
        return {"term": {field: expr.value}}
    elif op == Operator.NOT_EQUALS.value:
        return {"bool": {"must_not": [{"term": {field: expr.value}}]}}
    elif op == Operator.REGEX.value:
        return {"regexp": {field: str(expr.value)}}
    elif op == Operator.NOT_REGEX.value:
        return {"bool": {"must_not": [{"regexp": {field: str(expr.value)}}]}}
    elif op == Operator.GREATER_THAN.value:
        return {"range": {field: {"gt": expr.value}}}
    elif op == Operator.LOWER_THAN.value:
        return {"range": {field: {"lt": expr.value}}}
    elif op == Operator.IN.value:
        return {"terms": {field: expr.values or []}}
    elif op == Operator.NOT_IN.value:
        return {"bool": {"must_not": [{"terms": {field: expr.values or []}}]}}
    elif op == Operator.TRUTHY.value:
        return {"exists": {"field": field}}
    elif op == Operator.HAS.value:
        return {"wildcard": {field: f"*{expr.value}*"}}
    elif op == Operator.NOT_HAS.value:
        return {"bool": {"must_not": [{"wildcard": {field: f"*{expr.value}*"}}]}}
    else:
        raise ValueError(f"unsupported operator: {op}")


# Usage
result = parse('status = 200 and message ~ "error.*" and env in ["prod", "staging"]')
es_query = generate_elasticsearch(result.root)
print(es_query)
```

## 組み込み Generator の仕組み

組み込み SQL generator も、上で示したのと同じ再帰パターンに従います。各 generator は次の処理を行います。

1. **ルート `Node`** とカラム schema マップを受け取る
2. **木を再帰的に走査する**。branch ノードは `(left AND right)` や `(left OR right)` に、leaf ノードは SQL 条件に変換する
3. **各演算子を処理する**。`expression.operator` に応じて分岐し、方言ごとの SQL を出力する
4. **否定されたノードを** `NOT (...)` で包む

カスタム generator との主な違いは、SQL generator がカラム schema に対する検証も行うことです。型、許可値、カラムの存在などを確認します。カスタム generator では、この検証を省略することも独自実装することもできます。
