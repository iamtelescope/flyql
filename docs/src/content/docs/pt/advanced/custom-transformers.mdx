O sistema de transformadores do FlyQL é extensível. Pode registar transformadores personalizados que funcionam de forma idêntica aos integrados (`upper`, `lower`, `len`) nos parsers, geradores SQL, matcher em memória e autocompletar do editor.

## Interface do Transformador

Cada transformador implementa 6 membros:

| Membro | Tipo | Descrição |
|--------|------|-------------|
| `name` | string | Identificador único em minúsculas (por exemplo, `"firstoctet"`) |
| `inputType` | Type | Tipo de entrada esperado (`string`, `int`, `float`, `bool`, `array`, ou `any` para entrada universal que aceita qualquer tipo a montante — ver [Transformers com entrada Any](#transformers-com-entrada-any)) |
| `outputType` | Type | Tipo de saída após a transformação |
| `argSchema` | ArgSpec[] | Tipos de argumento declarados e aridade (por omissão: vazio = sem argumentos) |
| `sql(dialect, columnRef, args)` | string | Gerar SQL específico do dialeto envolvendo a referência da coluna |
| `apply(value, args)` | any | Transformar um valor em tempo de execução em memória |

O método `sql()` recebe o nome do dialeto SQL (`"clickhouse"`, `"postgresql"`, `"starrocks"`), uma string de referência de coluna e a lista de argumentos analisados. Devolve uma expressão SQL que envolve essa referência. O método `apply()` recebe um valor em tempo de execução e a lista de argumentos, e devolve o valor transformado.

### ArgSpec

Cada entrada em `argSchema` declara um argumento esperado:

| Campo | Tipo | Por omissão | Descrição |
|-------|------|-------------|-------------|
| `type` | Type | — | Tipo de argumento esperado (`string`, `int`, `float`, `bool`) |
| `required` | boolean | `true` | Se o argumento deve ser fornecido |

A aridade é derivada do comprimento do esquema: argumentos obrigatórios devem estar presentes, argumentos opcionais finais podem ser omitidos. Transformadores sem argumentos (como `upper`, `lower`, `len`) herdam o esquema vazio por omissão.

## API do Registo

| Método | Descrição |
|--------|-------------|
| `defaultRegistry()` | Devolve um registo novo pré-carregado com transformadores integrados |
| `registry.register(transformer)` | Adicionar um transformador personalizado (gera erro se o nome já estiver registado) |
| `registry.get(name)` | Procurar um transformador pelo nome (devolve null/None/nil se não encontrado) |
| `registry.names()` | Listar todos os nomes de transformadores registados |

O padrão: obter um registo padrão, registar os seus transformadores personalizados, passar o registo aos geradores e ao matcher.

## Python

Defina um transformador personalizado criando uma subclasse de `Transformer`:

```python
from flyql.transformers import Transformer, Type, default_registry

class FirstOctetTransformer(Transformer):
    @property
    def name(self): return "firstoctet"
    @property
    def input_type(self): return Type.STRING
    @property
    def output_type(self): return Type.INT
    def sql(self, dialect, column_ref, args=None):
        if dialect == "clickhouse":
            return f"toUInt8(splitByChar('.', {column_ref})[1])"
        return f"CAST(SPLIT_PART({column_ref}, '.', 1) AS INTEGER)"
    def apply(self, value, args=None):
        return int(str(value).split('.')[0])
```

O atributo de classe `arg_schema` tem como valor por omissão `()` (sem argumentos). Para declarar argumentos, substitua-o por um tuplo de `ArgSpec`:

```python
from typing import ClassVar, Tuple
from flyql.transformers.base import ArgSpec

class SubstringTransformer(Transformer):
    arg_schema: ClassVar[Tuple[ArgSpec, ...]] = (
        ArgSpec(type=Type.INT),                    # start (required)
        ArgSpec(type=Type.INT, required=False),    # length (optional)
    )
    # ... name, input_type, output_type, sql, apply ...
```

Registar e utilizar:

```python
from flyql import parse
from flyql.generators.postgresql.generator import to_sql_where, Column

registry = default_registry()
registry.register(FirstOctetTransformer())

result = parse("src_ip|firstoctet > 192")
columns = {"src_ip": Column("src_ip", "text")}
sql = to_sql_where(result.root, columns, registry=registry)
# CAST(SPLIT_PART(src_ip, '.', 1) AS INTEGER) > 192
```

Para correspondência em memória:

```python
from flyql.matcher.evaluator import Evaluator
from flyql.matcher.record import Record

evaluator = Evaluator(registry=registry)
record = Record({"src_ip": "193.0.0.1"})
evaluator.evaluate(result.root, record)  # True (193 > 192)
```

## JavaScript

Defina um transformador personalizado estendendo `Transformer`:

```javascript
import { Transformer, Type, ArgSpec, defaultRegistry } from 'flyql/transformers'

class FirstOctetTransformer extends Transformer {
    get name() { return 'firstoctet' }
    get inputType() { return Type.STRING }
    get outputType() { return Type.INT }
    sql(dialect, columnRef, args = []) {
        if (dialect === 'clickhouse') return `toUInt8(splitByChar('.', ${columnRef})[1])`
        return `CAST(SPLIT_PART(${columnRef}, '.', 1) AS INTEGER)`
    }
    apply(value, args = []) { return parseInt(String(value).split('.')[0], 10) }
}
```

O getter `argSchema` tem como valor por omissão `[]` (sem argumentos). Para declarar argumentos, substitua-o:

```javascript
class SubstringTransformer extends Transformer {
    get argSchema() {
        return [
            new ArgSpec(Type.INT, true),     // start (required)
            new ArgSpec(Type.INT, false),    // length (optional)
        ]
    }
    // ... name, inputType, outputType, sql, apply ...
}
```

Registar e utilizar:

```javascript
import { parse } from 'flyql'
import { generateWhere, newColumn } from 'flyql/generators/postgresql'

const registry = defaultRegistry()
registry.register(new FirstOctetTransformer())

const result = parse("src_ip|firstoctet > 192")
const columns = { src_ip: newColumn({ name: 'src_ip', type: 'text' }) }
const sql = generateWhere(result.root, columns, registry)
// CAST(SPLIT_PART(src_ip, '.', 1) AS INTEGER) > 192
```

Para correspondência em memória:

```javascript
import { match } from 'flyql/matcher'

match("src_ip|firstoctet > 192", { src_ip: "193.0.0.1" }, registry) // true
```

Para autocompletar do editor — os transformadores personalizados aparecem ao lado dos integrados:

```javascript
import { EditorEngine } from 'flyql-vue'

const engine = new EditorEngine(columns, { registry })
// Após escrever "src_ip|", as sugestões incluem "firstoctet", "upper", "lower", "len"
```

## Go

Defina um transformador personalizado implementando a interface `Transformer`:

```go
package main

import (
    "fmt"
    "strconv"
    "strings"
    "github.com/iamtelescope/flyql/golang/transformers"
)

type FirstOctet struct{}

func (f FirstOctet) Name() string               { return "firstoctet" }
func (f FirstOctet) InputType() transformers.Type  { return transformers.TransformerTypeString }
func (f FirstOctet) OutputType() transformers.Type { return transformers.TransformerTypeInt }
func (f FirstOctet) ArgSchema() []transformers.ArgSpec        { return []transformers.ArgSpec{} }
func (f FirstOctet) SQL(dialect, colRef string, args []any) string {
    if dialect == "clickhouse" {
        return fmt.Sprintf("toUInt8(splitByChar('.', %s)[1])", colRef)
    }
    return fmt.Sprintf("CAST(SPLIT_PART(%s, '.', 1) AS INTEGER)", colRef)
}
func (f FirstOctet) Apply(value interface{}, args []any) interface{} {
    parts := strings.SplitN(fmt.Sprint(value), ".", 2)
    n, _ := strconv.Atoi(parts[0])
    return n
}
```

Para declarar argumentos, devolva entradas `ArgSpec` a partir de `ArgSchema()`:

```go
func (s Substring) ArgSchema() []transformers.ArgSpec {
    return []transformers.ArgSpec{
        {Type: transformers.TransformerTypeInt, Required: true},   // start (required)
        {Type: transformers.TransformerTypeInt, Required: false},  // length (optional)
    }
}
```

Registar e utilizar:

```go
import (
    flyql "github.com/iamtelescope/flyql/golang"
    postgresqlgen "github.com/iamtelescope/flyql/golang/generators/postgresql"
)

registry := transformers.DefaultRegistry()
registry.Register(FirstOctet{})

result, _ := flyql.Parse("src_ip|firstoctet > 192")
columns := map[string]*postgresqlgen.Column{
    "src_ip": postgresqlgen.NewColumn("src_ip", "text", nil),
}
sql, _ := postgresqlgen.ToSQLWhere(result.Root, columns, registry)
// CAST(SPLIT_PART(src_ip, '.', 1) AS INTEGER) > 192
```

Para correspondência em memória:

```go
import "github.com/iamtelescope/flyql/golang/matcher"

matched, _ := matcher.Match("src_ip|firstoctet > 192", map[string]any{"src_ip": "193.0.0.1"}, registry)
// matched == true
```

## Transformers com entrada `Any`

Um transformer pode declarar `inputType = Type.Any` (valor de string `"any"`) para aceitar qualquer tipo a montante. O caso de uso motivador é um transformer `type` que devolve o nome do tipo da coluna, independentemente do tipo concreto dessa coluna.

Regras:

1. `Any` é válido apenas em `inputType`. Declarar `Any` como `outputType` ou em qualquer `ArgSpec.type` é rejeitado em tempo de `register()`. Cada transformer deve declarar um tipo de saída concreto.
2. Quando um transformer com entrada Any é usado numa cadeia, a verificação de igualdade do tipo de entrada é ignorada. A estritude a jusante é preservada — por exemplo, `col|accepts_any_returning_array|upper` continua a dar erro em `upper` porque `array` não é uma entrada válida para `upper`.
3. `parseFlyQLType("any")` gera um erro: `any` não é um tipo de coluna ou de valor válido.

from flyql.flyql_type import Type
from flyql.transformers import Transformer

class TypeTransformer(Transformer):
    @property
    def name(self): return "type"
    @property
    def input_type(self): return Type.Any      # aceita qualquer tipo a montante
    @property
    def output_type(self): return Type.String  # deve ser concreto
    def sql(self, dialect, column_ref, args=None):
        if dialect == "clickhouse":
            return f"toTypeName({column_ref})"
        return f"pg_typeof({column_ref})::text"
    def apply(self, value, args=None):
        return type(value).__name__

## Compatibilidade de Tipos

Os transformadores personalizados participam na mesma validação de cadeia de tipos que os integrados. Se encadear `field|firstoctet|upper`, o sistema deteta que `firstoctet` produz `int` mas `upper` requer `string`.

Tipos disponíveis: consulte [`flyql.Type`](/advanced/column-types/). A única exceção é `Any`, que é um carácter universal de entrada de transformer e não é um tipo de coluna ou valor válido.

## Validação com `diagnose()`

A função `diagnose()` percorre um AST analisado contra um esquema de colunas e um registo de transformadores, retornando registos `Diagnostic` posicionados sem lançar exceções. Verifica:

- **Coluna desconhecida** — chave base ausente da lista de colunas
- **Transformador desconhecido** — nome do transformador ausente do registo
- **Aridade incorreta** — poucos ou demasiados argumentos em relação ao `argSchema`
- **Tipo de argumento incorreto** — o tipo do valor do argumento não corresponde ao tipo declarado no `ArgSpec` (alargamento int para float é permitido)
- **Conflito de tipo na cadeia** — o tipo de saída de um transformador não corresponde ao tipo de entrada do seguinte, ou o tipo da coluna não corresponde ao tipo de entrada do primeiro transformador

Cada diagnóstico contém um `range` (posição no código), `message`, `severity` (`"error"`) e `code` (`"unknown_column"`, `"unknown_transformer"`, `"arg_count"`, `"arg_type"`, `"chain_type"`).

```python
# Python
from flyql import parse, Column, ColumnSchema, Type, diagnose

ast = parse("hstt|upper=X").root
columns = ColumnSchema.from_columns([Column(name="host", column_type=Type.String)])
diags = diagnose(ast, columns)  # columns is a ColumnSchema
# [Diagnostic(range=Range(0, 4), message="column 'hstt' is not defined", severity='error', code='unknown_column')]
```

```javascript
// JavaScript
import { parse, Column, ColumnSchema, Type, diagnose } from 'flyql'

const ast = parse('hstt|upper=X').root
const columns = ColumnSchema.fromColumns([new Column('host', false, Type.String)])
const diags = diagnose(ast, columns)  // columns is a ColumnSchema
// [Diagnostic { range: Range(0, 4), message: "column 'hstt' is not defined", severity: 'error', code: 'unknown_column' }]
```

```go
// Go
import flyql "github.com/iamtelescope/flyql/golang"

result, _ := flyql.Parse("hstt|upper=X")
schema := flyql.FromColumns([]flyql.Column{flyql.NewColumn("host", flyql.TypeString)})
diags := flyql.Diagnose(result.Root, schema, nil) // nil = default registry
// []Diagnostic{{Range: {0, 4}, Message: "column 'hstt' is not defined", Severity: "error", Code: "unknown_column"}}
```

Passe um registo personalizado como terceiro argumento se tiver transformadores personalizados registados. Passe `nil`/`None`/`null` para utilizar o registo padrão.

## Segurança

Os métodos `sql()` dos transformadores personalizados recebem apenas referências de colunas (do esquema), nunca valores fornecidos pelo utilizador. Os valores do utilizador são escapados/parametrizados pelo gerador. Isto preserva a garantia de proteção contra injeção do FlyQL — o SQL personalizado envolve nomes de colunas, não dados do utilizador.
