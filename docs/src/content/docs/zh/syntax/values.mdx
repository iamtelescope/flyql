---
title: 值与类型
description: FlyQL 中的值类型、字面量与表达式
---

FlyQL 支持 7 种值类型：**整数**、**大整数**、**浮点数**、**字符串**、**布尔值**、**空值** 和 **数组**。AST 中的每个值都会带有明确的类型标记。

## 值类型

### 整数

在 int64 范围内的整数：

```
count = 42
status != 200
age >= 18
```

### 大整数

超出 int64（Go/Python）或 `MAX_SAFE_INTEGER`（JavaScript）范围的整数：

```
id = 18446744073709551615
```

### 浮点数

带小数点的数字：

```
price > 9.99
ratio = 3.14
temperature<=-0.5
```

### 字符串

文本值可以加引号，也可以不加：

```
level = error
user = "John Doe"
message = 'hello world'
count = '42'
```

带引号的数字仍然会被视为字符串。

可使用反斜杠转义引号：

```
user = 'John
message="said \"hello\""
```

### 布尔值

布尔字面量为小写的 `true` 和 `false`：

```
active = true
debug = false
active != true
```

像 `True`、`FALSE` 这样的变体会被当作字符串。

### 空值

空值使用小写 `null`：

```
field = null
field != null
```

对于 `null`，只允许使用 `=` 与 `!=`。使用 `>`, `<`, `>=`, `<=`, `~`, `!~` 都会触发解析错误。

SQL 生成时：

- `field = null` → `field IS NULL`
- `field != null` → `field IS NOT NULL`

### 数组

数组通常与 `in` / `not in` 搭配使用：

```
status in [200, 201, 202]
name in ['alice', 'bob']
field in [1, "hello", true, null]
```

每个元素都有自己的类型信息。

## 真值 / 假值表达式

一个没有操作符的字段表示对该字段执行真值检查：

```
active
message and status
not archived
```

假值通常包括：

- `null`
- 空字符串
- `0`
- `false`
- 空数组（取决于具体运行时实现）
