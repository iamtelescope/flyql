Les requetes FlyQL sont composees d'une ou plusieurs conditions reliees par des operateurs booleens (<FlyqlToken type="operator">and</FlyqlToken>, <FlyqlToken type="operator">or</FlyqlToken>, <FlyqlToken type="operator">not</FlyqlToken>). Chaque condition est soit une comparaison, soit une verification de veracite.

## Structure de base d'une requete

<FlyqlCode code={`status = 200 and active and not archived`} />

- <FlyqlCode code={`status = 200`} inline /> — le champ <FlyqlToken type="key">status</FlyqlToken> est egal a <FlyqlToken type="value">200</FlyqlToken>
- <FlyqlCode code={`active`} inline /> — le champ <FlyqlToken type="key">active</FlyqlToken> a une valeur vraie (truthy)
- <FlyqlCode code={`not archived`} inline /> — le champ <FlyqlToken type="key">archived</FlyqlToken> est faux (null, vide, zero ou false)

## Autres exemples

<FlyqlCode code={`service != 'api' or user = "john doe"`} />

<FlyqlCode code={`message ~ "error.*" and not debug`} />

<FlyqlCode code={`(a = 1 or b = 2) and not (c = 3 and d = 4)`} />

<FlyqlCode code={`status in [200, 201] and method not in ['DELETE', 'PUT']`} />

<FlyqlCode code={`message has 'error' and tags not has 'debug'`} />

## Regles generales

- **Cles autonomes** — Une cle sans operateur est traitee comme une verification de veracite : <FlyqlCode code={`active`} inline />
- **Comparaisons** — Une cle avec un operateur doit avoir une valeur correspondante : <FlyqlCode code={`status = 200`} inline />
- **Espaces** — Les espaces autour des operateurs sont facultatifs : <FlyqlCode code={`status=200`} inline /> et <FlyqlCode code={`status = 200`} inline /> sont equivalents
- **Precedence des operateurs** — <FlyqlToken type="operator">and</FlyqlToken> a une priorite plus elevee que <FlyqlToken type="operator">or</FlyqlToken> : <FlyqlCode code={`a = 1 or b = 2 and c = 3`} inline /> est interprete comme <FlyqlCode code={`a = 1 or (b = 2 and c = 3)`} inline />
- **Parentheses** — Utilisez <FlyqlToken type="operator">( )</FlyqlToken> pour modifier la precedence : <FlyqlCode code={`(a = 1 or b = 2) and c = 3`} inline />

- [Listes](/fr/syntax/lists/) — <FlyqlToken type="operator">in</FlyqlToken> et <FlyqlToken type="operator">not in</FlyqlToken> avec des valeurs de liste
