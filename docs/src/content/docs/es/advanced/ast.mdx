El parser de FlyQL produce un **árbol de sintaxis abstracta (AST)**, una representación estructurada de la consulta que puedes recorrer para generar salida para cualquier destino. Los generadores SQL integrados de ClickHouse, PostgreSQL y StarRocks se implementan recorriendo este mismo AST. Puedes construir tu propio generador para consultas de Elasticsearch, filtros de MongoDB, selectores de Prometheus, comandos `grep` sobre logs o cualquier otro destino.

## Tipos de nodos del AST

El AST está formado por tres tipos: **Node** para la estructura del árbol, **Expression** para las condiciones hoja y **Key** para las referencias a campos.

### Node

Un `Node` puede ser una **rama**, que combina dos subárboles con un operador booleano, o una **hoja**, que contiene una única expresión. Es el bloque básico del AST.

| Campo | Tipo | Descripción |
|-------|------|-------------|
| `boolOperator` | `string` | `"and"` o `"or"`, indica cómo se combinan `left` y `right`. Está vacío en nodos hoja |
| `expression` | `Expression` o `null` | La condición, si este nodo es hoja. `null` en nodos rama |
| `left` | `Node` o `null` | Subárbol hijo izquierdo. `null` en nodos hoja |
| `right` | `Node` o `null` | Subárbol hijo derecho. `null` en nodos hoja |
| `negated` | `boolean` | `true` si este nodo está envuelto en `not` |

Un nodo es **o bien** una hoja, donde `expression` está definido y `left` y `right` son `null`, **o bien** una rama, donde `left` y `right` están definidos y `expression` es `null`. Nunca puede ser ambas cosas a la vez.

### Expression

Una `Expression` representa una condición individual: una clave, un operador y un valor.

| Campo | Tipo | Descripción |
|-------|------|-------------|
| `key` | `Key` | El campo que se evalúa, por ejemplo `status` o `user.metadata.role` |
| `operator` | `string` | Uno de: `=`, `!=`, `~`, `!~`, `>`, `<`, `>=`, `<=`, `truthy`, `in`, `not in`, `has`, `not has` |
| `value` | `any` | El valor de comparación. Es un string, número, boolean o null para operadores escalares. Está vacío para `in` y `not in` |
| `valueType` | `string` o `null` | El tipo explícito de `value`: `"integer"`, `"bigint"`, `"float"`, `"string"`, `"boolean"` o `"null"`. Es `null` para `in` y `not in` |
| `values` | `array` o `null` | Lista de valores para `in` y `not in`. Puede ser heterogénea. `null` para el resto de operadores |
| `valuesTypes` | `string[]` o `null` | Array paralelo a `values` con el tipo de cada elemento. Cada elemento es una de las constantes de tipo. `null` cuando `values` no se usa |

### Key

Una `Key` representa la ruta del campo consultado.

| Campo | Tipo | Descripción |
|-------|------|-------------|
| `segments` | `string[]` | Segmentos de la ruta. `["status"]` para una clave simple o `["user", "metadata", "role"]` para una anidada |
| `raw` | `string` | La clave original tal como se escribió en la consulta |
| `isSegmented` | `boolean` | `true` si la clave tiene más de un segmento, es decir, si usa notación por puntos |

## Ejemplos de estructura del AST

### Condición simple

Consulta: `status = 200`

```
Node (leaf)
  expression:
    key: { segments: ["status"], raw: "status" }
    operator: "="
    value: 200
```

### Combinación booleana

Consulta: `status = 200 and method = "GET"`

```
Node (branch, boolOperator: "and")
  left: Node (leaf)
    expression:
      key: { segments: ["status"], raw: "status" }
      operator: "="
      value: 200
  right: Node (leaf)
    expression:
      key: { segments: ["method"], raw: "method" }
      operator: "="
      value: "GET"
```

### Negación

Consulta: `not status = 500`

```
Node (leaf, negated: true)
  expression:
    key: { segments: ["status"], raw: "status" }
    operator: "="
    value: 500
```

### Booleanos anidados con paréntesis

Consulta: `env=prod and (status=500 or level = error)`

```
Node (branch, boolOperator: "and")
  left: Node (leaf)
    expression:
      key: { segments: ["env"], raw: "env" }
      operator: "="
      value: "prod"
  right: Node (branch, boolOperator: "or")
    left: Node (leaf)
      expression:
        key: { segments: ["status"], raw: "status" }
        operator: "="
        value: 500
    right: Node (leaf)
      expression:
        key: { segments: ["level"], raw: "level" }
        operator: "="
        value: "error"
```

### Pertenencia a listas

Consulta: `status in [200, 201, 204]`

```
Node (leaf)
  expression:
    key: { segments: ["status"], raw: "status" }
    operator: "in"
    value: ""
    values: [200, 201, 204]
    valuesType: "number"
```

### Comprobación truthy

Consulta: `active`

```
Node (leaf)
  expression:
    key: { segments: ["active"], raw: "active" }
    operator: "truthy"
    value: ""
```

### Clave anidada

Consulta: `user.metadata.role = "admin"`

```
Node (leaf)
  expression:
    key: { segments: ["user", "metadata", "role"], raw: "user.metadata.role", isSegmented: true }
    operator: "="
    value: "admin"
```

## Recorrer el AST

El AST es un árbol binario. Para procesarlo, usa un recorrido recursivo:

1. Si el nodo tiene `expression`, es un nodo hoja: procesa la condición.
2. Si el nodo tiene `left` y `right`, es un nodo rama: recorre ambos hijos y combínalos con `boolOperator`.
3. Si `negated` es true, envuelve el resultado con una negación.

## Crear un generador personalizado

A continuación tienes un ejemplo completo de un generador personalizado que convierte un AST de FlyQL en un objeto del **Elasticsearch Query DSL**.

### JavaScript

```javascript
import { parse } from 'flyql'
import { Operator } from 'flyql'

function generateElasticsearch(node) {
  if (!node) return { match_all: {} }

  let result

  if (node.expression) {
    // Leaf node — convert the expression
    result = expressionToES(node.expression)
  } else {
    // Branch node — recurse and combine
    const left = generateElasticsearch(node.left)
    const right = generateElasticsearch(node.right)

    if (node.boolOperator === 'and') {
      result = { bool: { must: [left, right] } }
    } else {
      result = { bool: { should: [left, right], minimum_should_match: 1 } }
    }
  }

  // Handle negation
  if (node.negated) {
    result = { bool: { must_not: [result] } }
  }

  return result
}

function expressionToES(expr) {
  const field = expr.key.raw

  switch (expr.operator) {
    case Operator.EQUALS:
      return { term: { [field]: expr.value } }

    case Operator.NOT_EQUALS:
      return { bool: { must_not: [{ term: { [field]: expr.value } }] } }

    case Operator.REGEX:
      return { regexp: { [field]: String(expr.value) } }

    case Operator.NOT_REGEX:
      return { bool: { must_not: [{ regexp: { [field]: String(expr.value) } }] } }

    case Operator.GREATER_THAN:
      return { range: { [field]: { gt: expr.value } } }

    case Operator.GREATER_OR_EQUALS_THAN:
      return { range: { [field]: { gte: expr.value } } }

    case Operator.LOWER_THAN:
      return { range: { [field]: { lt: expr.value } } }

    case Operator.LOWER_OR_EQUALS_THAN:
      return { range: { [field]: { lte: expr.value } } }

    case Operator.IN:
      return { terms: { [field]: expr.values || [] } }

    case Operator.NOT_IN:
      return { bool: { must_not: [{ terms: { [field]: expr.values || [] } }] } }

    case Operator.TRUTHY:
      return { exists: { field } }

    case Operator.HAS:
      return { wildcard: { [field]: `*${expr.value}*` } }

    case Operator.NOT_HAS:
      return { bool: { must_not: [{ wildcard: { [field]: `*${expr.value}*` } }] } }

    default:
      throw new Error(`unsupported operator: ${expr.operator}`)
  }
}

// Usage
const result = parse('status = 200 and message ~ "error.*" and env in ["prod", "staging"]')
const esQuery = generateElasticsearch(result.root)
console.log(JSON.stringify(esQuery, null, 2))
```

### Go

```go
package main

import (
	"encoding/json"
	"fmt"
	flyql "github.com/iamtelescope/flyql/golang"
)

func generateElasticsearch(node *flyql.Node) map[string]any {
	if node == nil {
		return map[string]any{"match_all": map[string]any{}}
	}

	var result map[string]any

	if node.Expression != nil {
		// Leaf node — convert the expression
		result = expressionToES(node.Expression)
	} else {
		// Branch node — recurse and combine
		left := generateElasticsearch(node.Left)
		right := generateElasticsearch(node.Right)

		if node.BoolOperator == flyql.BoolOpAnd {
			result = map[string]any{
				"bool": map[string]any{"must": []any{left, right}},
			}
		} else {
			result = map[string]any{
				"bool": map[string]any{"should": []any{left, right}, "minimum_should_match": 1},
			}
		}
	}

	// Handle negation
	if node.Negated {
		result = map[string]any{
			"bool": map[string]any{"must_not": []any{result}},
		}
	}

	return result
}

func expressionToES(expr *flyql.Expression) map[string]any {
	field := expr.Key.Raw

	switch expr.Operator {
	case flyql.OpEquals:
		return map[string]any{"term": map[string]any{field: expr.Value}}
	case flyql.OpNotEquals:
		return map[string]any{"bool": map[string]any{"must_not": []any{map[string]any{"term": map[string]any{field: expr.Value}}}}}
	case flyql.OpGreater:
		return map[string]any{"range": map[string]any{field: map[string]any{"gt": expr.Value}}}
	case flyql.OpLess:
		return map[string]any{"range": map[string]any{field: map[string]any{"lt": expr.Value}}}
	case flyql.OpIn:
		return map[string]any{"terms": map[string]any{field: expr.Values}}
	case flyql.OpNotIn:
		return map[string]any{"bool": map[string]any{"must_not": []any{map[string]any{"terms": map[string]any{field: expr.Values}}}}}
	case flyql.OpTruthy:
		return map[string]any{"exists": map[string]any{"field": field}}
	case flyql.OpRegex:
		return map[string]any{"regexp": map[string]any{field: fmt.Sprintf("%v", expr.Value)}}
	default:
		panic(fmt.Sprintf("unsupported operator: %s", expr.Operator))
	}
}

func main() {
	result, err := flyql.Parse(`status = 200 and env in ["prod", "staging"]`)
	if err != nil {
		panic(err)
	}

	esQuery := generateElasticsearch(result.Root)
	data, _ := json.MarshalIndent(esQuery, "", "  ")
	fmt.Println(string(data))
}
```

### Python

```python
from flyql.core.parser import parse
from flyql.core.constants import Operator, BoolOperator

def generate_elasticsearch(node):
    if node is None:
        return {"match_all": {}}

    if node.expression is not None:
        # Leaf node — convert the expression
        result = expression_to_es(node.expression)
    else:
        # Branch node — recurse and combine
        left = generate_elasticsearch(node.left)
        right = generate_elasticsearch(node.right)

        if node.bool_operator == BoolOperator.AND.value:
            result = {"bool": {"must": [left, right]}}
        else:
            result = {"bool": {"should": [left, right], "minimum_should_match": 1}}

    # Handle negation
    if node.negated:
        result = {"bool": {"must_not": [result]}}

    return result

def expression_to_es(expr):
    field = expr.key.raw
    op = expr.operator  # operator is stored as a string value, e.g. "=", "!=", "~"

    if op == Operator.EQUALS.value:
        return {"term": {field: expr.value}}
    elif op == Operator.NOT_EQUALS.value:
        return {"bool": {"must_not": [{"term": {field: expr.value}}]}}
    elif op == Operator.REGEX.value:
        return {"regexp": {field: str(expr.value)}}
    elif op == Operator.NOT_REGEX.value:
        return {"bool": {"must_not": [{"regexp": {field: str(expr.value)}}]}}
    elif op == Operator.GREATER_THAN.value:
        return {"range": {field: {"gt": expr.value}}}
    elif op == Operator.LOWER_THAN.value:
        return {"range": {field: {"lt": expr.value}}}
    elif op == Operator.IN.value:
        return {"terms": {field: expr.values or []}}
    elif op == Operator.NOT_IN.value:
        return {"bool": {"must_not": [{"terms": {field: expr.values or []}}]}}
    elif op == Operator.TRUTHY.value:
        return {"exists": {"field": field}}
    elif op == Operator.HAS.value:
        return {"wildcard": {field: f"*{expr.value}*"}}
    elif op == Operator.NOT_HAS.value:
        return {"bool": {"must_not": [{"wildcard": {field: f"*{expr.value}*"}}]}}
    else:
        raise ValueError(f"unsupported operator: {op}")


# Usage
result = parse('status = 200 and message ~ "error.*" and env in ["prod", "staging"]')
es_query = generate_elasticsearch(result.root)
print(es_query)
```

## Cómo funcionan los generadores integrados

Los generadores SQL integrados siguen el mismo patrón recursivo mostrado arriba. Cada generador:

1. **Recibe el `Node` raíz** y un mapa con el esquema de columnas.
2. **Recorre el árbol**: los nodos rama se convierten en `(left AND right)` o `(left OR right)` y los nodos hoja en condiciones SQL.
3. **Procesa cada operador** consultando `expression.operator` y generando SQL específico del dialecto.
4. **Envuelve los nodos negados** en `NOT (...)`.

La diferencia principal con un generador personalizado es que los generadores SQL también validan las expresiones contra el esquema de columnas, comprobando tipos, valores permitidos y existencia de columnas. Un generador personalizado puede omitir esta parte o implementar su propia validación.
