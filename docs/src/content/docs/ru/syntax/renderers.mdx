---
title: Рендереры
description: Прикрепление метаданных отображения к SELECT-колонкам с алиасами без влияния на генерируемый SQL
---

Рендереры прикрепляют метаданные отображения к SELECT-колонкам с алиасами. Они
разбираются и валидируются flyql, но **никогда не влияют на SQL или результат
матчера**. В отличие от [преобразователей](/syntax/transformers/), которые изменяют
то, что возвращает база данных, рендереры описывают, как ваше приложение должно
отображать значение -- интерпретация полностью на вашей стороне.

## Синтаксис пайпа после алиаса

Цепочка рендереров идёт **после** алиаса, через пайп (`|`):

```
url as link|href("http://{{value}}")
```

Алиас обязателен. Колонка без алиаса не может нести рендереры.

## Цепочки рендереров

Объединяйте несколько рендереров дополнительными пайпами:

```
url as link|href("http://{{value}}")|badge("info")
```

Рендереры разбираются слева направо в упорядоченный список. Семантика цепочки
полностью определяется вашим приложением -- flyql не навязывает систему типов
ввода/вывода для рендереров.

## Рендереры и преобразователи вместе

Преобразователи выполняются до алиаса; рендереры -- после:

```
url|upper as u|href("https://{{value}}")
```

Преобразователь `upper` влияет на генерируемый SQL. Рендерер `href` -- нет.

## Рендереры без аргументов

Рендереры без аргументов разбираются так же, как преобразователи без аргументов:

```
url as link|plain
```

## Включение рендереров

Разбор рендереров включается через флаг возможностей парсера:

**Python**
```python
from flyql.columns import parse
cols = parse('url as link|href("/{{value}}")',
             capabilities={"transformers": True, "renderers": True})
```

**Go**
```go
import "github.com/iamtelescope/flyql/golang/columns"
caps := columns.Capabilities{Transformers: true, Renderers: true}
cols, _ := columns.Parse(`url as link|href("/{{value}}")`, caps)
```

**JavaScript**
```javascript
import { parse } from 'flyql/columns'
const cols = parse('url as link|href("/{{value}}")',
    { transformers: true, renderers: true })
```

Когда `renderers` равно `false` (по умолчанию), пайп `|` после алиаса вызывает
`ParserError` с errno `11` и сообщением `"renderers are not enabled"`.

## Пустой реестр по умолчанию

В отличие от [`TransformerRegistry.default_registry()`](/advanced/custom-transformers/),
который содержит четыре встроенных преобразователя (`upper`, `lower`, `len`, `split`),
`RendererRegistry.default_registry()` **пуст по дизайну**. Ядро flyql
не делает предположений о вашей технологии рендеринга (HTML, markdown,
React, Vue и т.д.). Вы должны [зарегистрировать собственные рендереры](/advanced/custom-renderers/)
прежде чем валидатор их распознает.

Это означает, что для использования рендереров в валидаторе или редакторе вы
должны предоставить реестр; иначе каждый использованный рендерер создаст
диагностику `unknown_renderer`.

## Рендереры vs. преобразователи

| Аспект                    | Преобразователи      | Рендереры            |
| ------------------------- | -------------------- | -------------------- |
| Позиция в синтаксисе     | до алиаса            | **после** алиаса     |
| Влияет на генерируемый SQL | да                  | **нет**              |
| Влияет на результат матчера | да                 | **нет**              |
| Реестр по умолчанию       | 4 встроенных         | пустой               |
| Система типов ввода/вывода | да (типизация цепочки) | нет              |
| Интерпретация             | выполняет flyql      | выполняет ваше приложение |

См. [Пользовательские рендереры](/advanced/custom-renderers/) для регистрации и
хуков валидации.
