## 리스트 포함 여부

값이 리스트에 포함되는지 확인하려면 <FlyqlToken type="operator">in</FlyqlToken>, 포함되지 않는지 확인하려면 <FlyqlToken type="operator">not in</FlyqlToken>을 사용합니다.

<FlyqlCode code={`status in [200, 201, 204]`} />
<FlyqlCode code={`env not in ['prod', 'staging']`} />

## 구문 규칙

- 값은 대괄호 <FlyqlToken type="operator">[ ]</FlyqlToken>로 감싸고 쉼표로 구분합니다
- 값은 서로 다른 타입이 섞여도 됩니다: <FlyqlCode code={`[200, 'ok', true, null]`} inline />
- 문자열 값은 따옴표로 감싸야 합니다: <FlyqlCode code={`['a', 'b', 'c']`} inline />
- 숫자 값은 따옴표 없이 작성합니다: <FlyqlCode code={`[1, 2, 3]`} inline />
- 쉼표와 대괄호 주변 공백은 허용됩니다

### 문자열 리스트

<FlyqlCode code={`method in ['GET', 'POST', 'PUT']`} />
<FlyqlCode code={`env not in ['dev', 'test', 'local']`} />

문자열 값에는 작은따옴표와 큰따옴표를 모두 사용할 수 있습니다.

### 숫자 리스트

<FlyqlCode code={`status in [200, 201, 204]`} />
<FlyqlCode code={`port not in [80, 443]`} />

### 빈 리스트

빈 리스트도 허용됩니다.

<FlyqlCode code={`status in []`} />

빈 <FlyqlToken type="operator">in</FlyqlToken> 리스트는 항상 false입니다. 빈 리스트와 일치하는 것은 없습니다.

<FlyqlCode code={`status not in []`} />

빈 <FlyqlToken type="operator">not in</FlyqlToken> 리스트는 항상 true입니다. 모든 것이 빈 리스트에 "not in"입니다.

## 혼합 타입

리스트는 서로 다른 타입의 값을 지원합니다. 문자열, 숫자, boolean, null을 자유롭게 섞어 사용할 수 있습니다.

<FlyqlCode code={`field in [200, 'ok', true, null]`} />

각 요소는 parse된 expression의 `valuesTypes` 배열에 자신의 타입을 가집니다(예: `["int", "string", "bool", "null"]`). 빈 리스트의 경우 `valuesTypes`는 `null`입니다.

SQL generator가 리스트를 처리할 때는, 리스트 값 타입이 대상 컬럼 타입과 호환되는지 검증합니다. 예를 들어 숫자 컬럼에 문자열 값을 넘기면 SQL 생성 중 validation error가 발생합니다.

## IN 리스트 타입 일관성

파서는 **항상** 이종 리스트를 받아들입니다 — 타입 검증은 파서가 아니라 SQL 생성기에 있습니다. 각 생성기(ClickHouse, PostgreSQL, StarRocks)는 오류를 발생시키기 전에 동일한 두 단계 스킵 로직을 적용합니다.

### 1단계 — 호출 지점 게이트

검증기를 호출하기 전에 생성기는 세 가지 조건을 확인합니다:

1. 컬럼이 **선언된 타입**을 가짐 (`column.flyqlType()`이 비어 있지 않고 `Unknown`이 아님).
2. 키가 **분할되지 않음** (`category` 같은 단일 세그먼트, `meta.category`가 아님).
3. 리스트가 **동질**임 — 모든 요소가 같은 값 타입을 가짐 (Python에서 `len(set(values_types)) == 1`; JavaScript에서 `new Set(valuesTypes).size === 1`; Go에서 동등).

이 조건들 중 **어느 하나라도** 거짓이면 검증기는 완전히 스킵되고 SQL은 그대로 생성됩니다. 이종 리스트는 통과합니다. 타입이 없는 컬럼에 대한 리스트는 통과합니다. 분할된 키(JSON 경로, 배열 인덱스)에 대한 리스트도 통과합니다.

### 2단계 — 요소 검사

호출 지점 게이트를 통과한 경우, 검증기는 각 요소를 반복하면서 다음 호환성 맵에 따라 첫 번째 비호환 값에서 오류를 발생시킵니다:

| 컬럼 타입 | 허용되는 요소 타입 |
|----------|------------------|
| `String` | `string`         |
| `Int`    | `int`, `float`   |
| `Float`  | `int`, `float`   |
| `Bool`   | `bool`, `int`    |
| `Date`   | `string`         |

다른 컬럼 타입(`Array`, `Map`, `JSON`, `JSONString`, `Struct`, `Duration`, `Unknown`)은 맵에 없으며 요소 검사를 조용히 건너뜁니다.

### 오류 텍스트

검증이 실패하면 세 구현 모두 **동일한** 오류 메시지를 발생시킵니다:

<FlyqlCode code={`type mismatch in IN list: {column_type} column cannot contain {value_type} values`} />

구체적으로:

- Python: `FlyqlError("type mismatch in IN list: int column cannot contain string values")`
- Go: `errors.New("type mismatch in IN list: int column cannot contain string values")`
- JavaScript: `new Error("type mismatch in IN list: int column cannot contain string values")`

부분 문자열 `type mismatch in IN list:`는 세 언어와 세 SQL 방언에서 안정적이므로 grep으로 안전하게 검색할 수 있습니다.

세 가지 경우(동질 일치, 이종 우회, 동질 잘못된 타입 오류)에 대한 실행 가능한 엔드투엔드 데모는 [레시피 — IN 리스트 타입 일관성](/ko/syntax/recipes/#in-리스트-타입-일관성)을 참조하세요.

## 리스트 안의 파라미터

파라미터 placeholder는 IN 리스트 안에서도 사용할 수 있으며, 리터럴 값과 자유롭게 섞어서 쓸 수 있습니다.

<FlyqlCode code={`status in [$x, $y, 200]`} />
<FlyqlCode code={`env not in [$current, 'dev']`} />

`Parameter` 객체는 `bindParams()`가 해석하기 전까지 `values` 배열에 남아 있습니다. 전체 레퍼런스는 [파라미터](/ko/syntax/parameters/)를 참고하세요.

## 다른 연산자와의 조합

리스트 연산자는 다른 조건과 마찬가지로 불리언 연산자와 결합할 수 있습니다.

<FlyqlCode code={`status in [200, 201] and method in ['GET', 'POST']`} />
<FlyqlCode code={`level = 'error' or status not in [200, 201, 204]`} />
<FlyqlCode code={`(status in [500, 502, 503] or level = 'critical') and env='prod'`} />

## 예제

<FlyqlCode code={`status in [200, 201, 204]`} />
<FlyqlCode code={`env not in ['dev', 'test', 'staging']`} />
<FlyqlCode code={`method in ['GET', 'POST'] and status not in [401, 403, 404]`} />
