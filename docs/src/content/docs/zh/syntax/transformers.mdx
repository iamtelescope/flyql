转换器会在比较执行前对列值进行转换，并在生成 SQL 时输出对应的函数调用。转换器使用管道符 `|`，从左到右依次应用。

## 管道语法

通过 `|` 把转换器作用到列上：

<FlyqlCode code={`message|upper = "ERROR"`} />

这表示先对 <FlyqlToken type="key">message</FlyqlToken> 执行 <FlyqlToken type="transformer">upper</FlyqlToken>，然后再与 `"ERROR"` 比较。

## 链式调用

可以通过多个管道连续应用多个转换器：

<FlyqlCode code={`message|lower|len > 100`} />

转换顺序是从左到右：先 <FlyqlToken type="transformer">lower</FlyqlToken>，再 <FlyqlToken type="transformer">len</FlyqlToken>。

## 类型系统

每个转换器都声明输入类型和输出类型。链式调用必须保证前一个输出类型与后一个输入类型兼容。

| 类型 | 说明 |
|------|------|
| `string` | 文本 |
| `int` | 整数 |
| `float` | 浮点数 |
| `bool` | 布尔值 |
| `array` | 数组（通常结合 <FlyqlToken type="operator">has</FlyqlToken> 使用） |

## 内置转换器

| 转换器 | 输入 | 输出 | 说明 |
|--------|------|------|------|
| <FlyqlToken type="transformer">upper</FlyqlToken> | string | string | 转为大写 |
| <FlyqlToken type="transformer">lower</FlyqlToken> | string | string | 转为小写 |
| <FlyqlToken type="transformer">len</FlyqlToken> | string | int | 获取字符串长度 |
| <FlyqlToken type="transformer">split</FlyqlToken> | string | array | 按分隔符拆分，例如 `split(".")` |

## 各数据库生成结果

转换器会生成各方言对应的 SQL 函数调用。链式转换会嵌套输出。

**单个转换器：** <FlyqlCode code={`status|upper = "ERROR"`} inline />

| 方言 | 生成 SQL |
|------|----------|
| ClickHouse | `equals(upper(status), 'ERROR')` |
| PostgreSQL | `UPPER(status) = 'ERROR'` |
| StarRocks | `UPPER(\`status\`) = 'ERROR'` |

**链式转换：** <FlyqlCode code={`message|lower|len > 100`} inline />

| 方言 | 生成 SQL |
|------|----------|
| ClickHouse | `length(lower(message)) > 100` |
| PostgreSQL | `LENGTH(LOWER(message)) > 100` |
| StarRocks | `LENGTH(LOWER(\`message\`)) > 100` |

**与 <FlyqlToken type="operator">has</FlyqlToken> 配合：** <FlyqlCode code={`message|split(" ") has "hello"`} inline />

| 方言 | 生成 SQL |
|------|----------|
| ClickHouse | `has(splitByChar(' ', message), 'hello')` |
| PostgreSQL | `'hello' = ANY(STRING_TO_ARRAY(message, ' '))` |
| StarRocks | `` array_contains(SPLIT(`message`, ' '), 'hello') `` |

## 类型错误

如果链式调用的类型不兼容，编辑器会提示错误：

<FlyqlCode code={`message|len|upper`} />

这里 <FlyqlToken type="transformer">len</FlyqlToken> 输出 `int`，而 <FlyqlToken type="transformer">upper</FlyqlToken> 需要 `string` 输入，因此会报错。

## 示例

<FlyqlCode code={`status|upper = "ERROR"`} />
<FlyqlCode code={`message|lower|len > 100`} />
<FlyqlCode code={`host|lower != "localhost"`} />
<FlyqlCode code={`path|len > 0`} />
<FlyqlCode code={`tags|split(",") has "urgent"`} />

## 转换器与渲染器的区别

转换器在别名**之前**运行，会改变 flyql 在 SQL 中输出或匹配的值。[渲染器](/syntax/renderers/)在别名**之后**运行，附加显示元数据，但不会影响生成的 SQL 或匹配器结果。如果你想说"将此值转为大写返回"，请使用转换器。如果你想说"在我的应用中将此列渲染为可点击的链接"，请使用渲染器。

| 关注点                    | 转换器               | 渲染器               |
| ------------------------- | -------------------- | -------------------- |
| 语法中的位置              | 别名之前             | 别名**之后**         |
| 影响生成的 SQL            | 是                   | **否**               |
| 影响匹配器输出            | 是                   | **否**               |
| 默认注册器                | 4 个内置             | 空                   |

后置别名管道语法请参见[渲染器](/syntax/renderers/)，注册 API 请参见[自定义渲染器](/advanced/custom-renderers/)。
