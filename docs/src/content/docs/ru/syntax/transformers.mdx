---
title: Преобразователи
description: Преобразование значений колонок перед сравнением с помощью функций, объединённых через `|`
---

Преобразователи изменяют значения колонок перед сравнением в запросах или генерацией SQL. Они используют синтаксис пайпа (`|`) и применяются слева направо.

## Синтаксис пайпа

Примените преобразователь к колонке с помощью `|`:

```
message|upper = "ERROR"
```

Это преобразует значение `message` с помощью `upper` перед сравнением с `"ERROR"`.

## Цепочки

Объединяйте несколько преобразователей дополнительными пайпами:

```
message|lower|len > 100
```

Преобразователи применяются слева направо: сначала выполняется `lower` (строка в строку), затем `len` (строка в int). Результат сравнивается с `> 100`.

## Система типов

Каждый преобразователь объявляет входной и выходной тип. Цепочки преобразователей должны быть типово совместимыми — выходной тип одного должен совпадать со входным типом следующего.

| Тип | Описание |
|-----|----------|
| `string` | Текстовые значения |
| `int` | Целые числа |
| `float` | Числа с плавающей точкой |
| `array` | Массив значений (используйте оператор `has` для проверки принадлежности) |

## Встроенные преобразователи

| Преобразователь | Вход | Выход | Описание |
|-----------------|------|-------|----------|
| `upper` | string | string | Перевод в верхний регистр |
| `lower` | string | string | Перевод в нижний регистр |
| `len` | string | int | Длина строки |
| `split` | string | array | Разделение по разделителю: `split(".")` |

## SQL-вывод по диалектам

Преобразователи генерируют вызовы функций, специфичные для каждого диалекта SQL. Цепочки преобразователей создают вложенные вызовы (самый внутренний выполняется первым):

**Один преобразователь:** `status|upper = "ERROR"`

| Диалект | Сгенерированный SQL |
|---------|---------------------|
| ClickHouse | `equals(upper(status), 'ERROR')` |
| PostgreSQL | `UPPER(status) = 'ERROR'` |
| StarRocks | `UPPER(\`status\`) = 'ERROR'` |

**Цепочка преобразователей:** `message|lower|len > 100`

| Диалект | Сгенерированный SQL |
|---------|---------------------|
| ClickHouse | `length(lower(message)) > 100` |
| PostgreSQL | `LENGTH(LOWER(message)) > 100` |
| StarRocks | `LENGTH(LOWER(\`message\`)) > 100` |

**Split с has:** `message|split(" ") has "hello"`

| Диалект | Сгенерированный SQL |
|---------|---------------------|
| ClickHouse | `has(splitByChar(' ', message), 'hello')` |
| PostgreSQL | `'hello' = ANY(STRING_TO_ARRAY(message, ' '))` |
| StarRocks | `` array_contains(SPLIT(`message`, ' '), 'hello') `` |

## Ошибки типов

Если цепочка содержит несовместимые типы, редактор выводит ошибку:

```
message|len|upper
```

Это вызовет ошибку, потому что `len` возвращает `int`, а `upper` требует `string` на входе. Редактор покажет: "len outputs int, upper requires string input".

## Примеры

```
status|upper = "ERROR"           # Сравнение в верхнем регистре
message|lower|len > 100          # Цепочка: нижний регистр, затем длина
host|lower != "localhost"        # Регистронезависимое неравенство
path|len > 0                     # Проверка на непустоту
tags|split(",") has "urgent"     # Разделить и проверить принадлежность
```
