FlyQL-Abfragen bestehen aus einer oder mehreren Bedingungen, die mit booleschen Operatoren (<FlyqlToken type="operator">and</FlyqlToken>, <FlyqlToken type="operator">or</FlyqlToken>, <FlyqlToken type="operator">not</FlyqlToken>) verbunden sind. Jede Bedingung ist entweder ein Vergleich oder eine Truthy-Prüfung.

## Grundstruktur einer Abfrage

<FlyqlCode code={`status = 200 and active and not archived`} />

- <FlyqlCode code={`status = 200`} inline /> — <FlyqlToken type="key">status</FlyqlToken> Feld ist gleich <FlyqlToken type="value">200</FlyqlToken>
- <FlyqlCode code={`active`} inline /> — <FlyqlToken type="key">active</FlyqlToken> Feld hat einen truthy Wert
- <FlyqlCode code={`not archived`} inline /> — <FlyqlToken type="key">archived</FlyqlToken> Feld ist falsy (null, leer, nullwertig oder false)

## Weitere Beispiele

<FlyqlCode code={`service != 'api' or user = "john doe"`} />

<FlyqlCode code={`message ~ "error.*" and not debug`} />

<FlyqlCode code={`(a = 1 or b = 2) and not (c = 3 and d = 4)`} />

<FlyqlCode code={`status in [200, 201] and method not in ['DELETE', 'PUT']`} />

<FlyqlCode code={`message has 'error' and tags not has 'debug'`} />

## Allgemeine Regeln

- **Einzelne Schlüssel** — Ein Schlüssel ohne Operator wird als Truthy-Prüfung behandelt: <FlyqlCode code={`active`} inline />
- **Vergleiche** — Ein Schlüssel mit Operator muss einen zugehörigen Wert haben: <FlyqlCode code={`status = 200`} inline />
- **Leerzeichen** — Leerzeichen um Operatoren sind optional: <FlyqlCode code={`status=200`} inline /> und <FlyqlCode code={`status = 200`} inline /> sind äquivalent
- **Operator-Priorität** — <FlyqlToken type="operator">and</FlyqlToken> bindet stärker als <FlyqlToken type="operator">or</FlyqlToken>: <FlyqlCode code={`a = 1 or b = 2 and c = 3`} inline /> wird als <FlyqlCode code={`a = 1 or (b = 2 and c = 3)`} inline /> geparst
- **Klammern** — Verwende <FlyqlToken type="operator">( )</FlyqlToken>, um Prioritäten zu überschreiben: <FlyqlCode code={`(a = 1 or b = 2) and c = 3`} inline />
