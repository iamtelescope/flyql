El sistema de transformers de FlyQL es extensible. Puedes registrar transformers personalizados que funcionen igual que los integrados como `upper`, `lower` o `len` en los parsers, generadores SQL, matcher en memoria y autocompletado del editor.

## Interfaz de transformer

Todo transformer implementa 6 miembros:

| Miembro | Tipo | Descripción |
|---------|------|-------------|
| `name` | string | Identificador único en minúsculas, por ejemplo `"firstoctet"` |
| `inputType` | Type | Tipo de entrada esperado (`string`, `int`, `float`, `bool`, `array`, o `any` para una entrada comodín que acepta cualquier tipo upstream — consulta [Transformers con entrada `Any`](#transformers-con-entrada-any)) |
| `outputType` | Type | Tipo de salida tras la transformación |
| `argSchema` | ArgSpec[] | Tipos de argumento declarados y aridad (por defecto: vacío = sin argumentos) |
| `sql(dialect, columnRef, args)` | string | Genera SQL específico del dialecto envolviendo la referencia de columna |
| `apply(value, args)` | any | Transforma un valor en memoria durante la ejecución |

El método `sql()` recibe el nombre del dialecto SQL (`"clickhouse"`, `"postgresql"`, `"starrocks"`), una cadena con la referencia de columna y la lista de argumentos parseados. Devuelve una expresión SQL que envuelve esa referencia. El método `apply()` recibe un valor en tiempo de ejecución y la lista de argumentos, y devuelve el valor transformado.

### ArgSpec

Cada entrada en `argSchema` declara un argumento esperado:

| Campo | Tipo | Por defecto | Descripción |
|-------|------|-------------|-------------|
| `type` | Type | — | Tipo de argumento esperado (`string`, `int`, `float`, `bool`) |
| `required` | boolean | `true` | Si el argumento debe proporcionarse |

La aridad se deriva de la longitud del esquema: los argumentos requeridos deben estar presentes, los argumentos opcionales finales pueden omitirse. Los transformers sin argumentos (como `upper`, `lower`, `len`) heredan el esquema vacío por defecto.

## API del registro

| Método | Descripción |
|--------|-------------|
| `defaultRegistry()` | Devuelve un registro nuevo precargado con los transformers integrados |
| `registry.register(transformer)` | Añade un transformer personalizado y lanza error si el nombre ya está registrado |
| `registry.get(name)` | Busca un transformer por nombre y devuelve null, None o nil si no existe |
| `registry.names()` | Lista todos los nombres de transformers registrados |

El patrón es: obtener un registro por defecto, registrar transformers personalizados y pasar ese registro a los generadores y al matcher.

## Python

Define un transformer personalizado heredando de `Transformer`:

```python
from flyql.transformers import Transformer, Type, default_registry

class FirstOctetTransformer(Transformer):
    @property
    def name(self): return "firstoctet"
    @property
    def input_type(self): return Type.STRING
    @property
    def output_type(self): return Type.INT
    def sql(self, dialect, column_ref, args=None):
        if dialect == "clickhouse":
            return f"toUInt8(splitByChar('.', {column_ref})[1])"
        return f"CAST(SPLIT_PART({column_ref}, '.', 1) AS INTEGER)"
    def apply(self, value, args=None):
        return int(str(value).split('.')[0])
```

El atributo de clase `arg_schema` tiene como valor por defecto `()` (sin argumentos). Para declarar argumentos, sobreescríbalo con una tupla de `ArgSpec`:

```python
from typing import ClassVar, Tuple
from flyql.transformers.base import ArgSpec

class SubstringTransformer(Transformer):
    arg_schema: ClassVar[Tuple[ArgSpec, ...]] = (
        ArgSpec(type=Type.INT),                    # start (required)
        ArgSpec(type=Type.INT, required=False),    # length (optional)
    )
    # ... name, input_type, output_type, sql, apply ...
```

Regístralo y úsalo:

```python
from flyql import parse
from flyql.generators.postgresql.generator import to_sql_where, Column

registry = default_registry()
registry.register(FirstOctetTransformer())

result = parse("src_ip|firstoctet > 192")
columns = {"src_ip": Column("src_ip", "text")}
sql = to_sql_where(result.root, columns, registry=registry)
# CAST(SPLIT_PART(src_ip, '.', 1) AS INTEGER) > 192
```

Para evaluación en memoria:

```python
from flyql.matcher.evaluator import Evaluator
from flyql.matcher.record import Record

evaluator = Evaluator(registry=registry)
record = Record({"src_ip": "193.0.0.1"})
evaluator.evaluate(result.root, record)  # True (193 > 192)
```

## JavaScript

Define un transformer personalizado extendiendo `Transformer`:

```javascript
import { Transformer, Type, ArgSpec, defaultRegistry } from 'flyql/transformers'

class FirstOctetTransformer extends Transformer {
    get name() { return 'firstoctet' }
    get inputType() { return Type.STRING }
    get outputType() { return Type.INT }
    sql(dialect, columnRef, args = []) {
        if (dialect === 'clickhouse') return `toUInt8(splitByChar('.', ${columnRef})[1])`
        return `CAST(SPLIT_PART(${columnRef}, '.', 1) AS INTEGER)`
    }
    apply(value, args = []) { return parseInt(String(value).split('.')[0], 10) }
}
```

El getter `argSchema` tiene como valor por defecto `[]` (sin argumentos). Para declarar argumentos, sobreescríbalo:

```javascript
class SubstringTransformer extends Transformer {
    get argSchema() {
        return [
            new ArgSpec(Type.INT, true),     // start (required)
            new ArgSpec(Type.INT, false),    // length (optional)
        ]
    }
    // ... name, inputType, outputType, sql, apply ...
}
```

Regístralo y úsalo:

```javascript
import { parse } from 'flyql'
import { generateWhere, newColumn } from 'flyql/generators/postgresql'

const registry = defaultRegistry()
registry.register(new FirstOctetTransformer())

const result = parse("src_ip|firstoctet > 192")
const columns = { src_ip: newColumn({ name: 'src_ip', type: 'text' }) }
const sql = generateWhere(result.root, columns, registry)
// CAST(SPLIT_PART(src_ip, '.', 1) AS INTEGER) > 192
```

Para evaluación en memoria:

```javascript
import { match } from 'flyql/matcher'

match("src_ip|firstoctet > 192", { src_ip: "193.0.0.1" }, registry) // true
```

Para autocompletado del editor, los transformers personalizados aparecen junto a los integrados:

```javascript
import { EditorEngine } from 'flyql-vue'

const engine = new EditorEngine(columns, { registry })
// After typing "src_ip|", suggestions include "firstoctet", "upper", "lower", "len"
```

## Go

Define un transformer personalizado implementando la interfaz `Transformer`:

```go
package main

import (
    "fmt"
    "strconv"
    "strings"
    "github.com/iamtelescope/flyql/golang/transformers"
)

type FirstOctet struct{}

func (f FirstOctet) Name() string               { return "firstoctet" }
func (f FirstOctet) InputType() transformers.Type  { return transformers.TransformerTypeString }
func (f FirstOctet) OutputType() transformers.Type { return transformers.TransformerTypeInt }
func (f FirstOctet) ArgSchema() []transformers.ArgSpec        { return []transformers.ArgSpec{} }
func (f FirstOctet) SQL(dialect, colRef string, args []any) string {
    if dialect == "clickhouse" {
        return fmt.Sprintf("toUInt8(splitByChar('.', %s)[1])", colRef)
    }
    return fmt.Sprintf("CAST(SPLIT_PART(%s, '.', 1) AS INTEGER)", colRef)
}
func (f FirstOctet) Apply(value interface{}, args []any) interface{} {
    parts := strings.SplitN(fmt.Sprint(value), ".", 2)
    n, _ := strconv.Atoi(parts[0])
    return n
}
```

Para declarar argumentos, devuelva entradas `ArgSpec` desde `ArgSchema()`:

```go
func (s Substring) ArgSchema() []transformers.ArgSpec {
    return []transformers.ArgSpec{
        {Type: transformers.TransformerTypeInt, Required: true},   // start (required)
        {Type: transformers.TransformerTypeInt, Required: false},  // length (optional)
    }
}
```

Regístralo y úsalo:

```go
import (
    flyql "github.com/iamtelescope/flyql/golang"
    postgresqlgen "github.com/iamtelescope/flyql/golang/generators/postgresql"
)

registry := transformers.DefaultRegistry()
registry.Register(FirstOctet{})

result, _ := flyql.Parse("src_ip|firstoctet > 192")
columns := map[string]*postgresqlgen.Column{
    "src_ip": postgresqlgen.NewColumn("src_ip", "text", nil),
}
sql, _ := postgresqlgen.ToSQLWhere(result.Root, columns, registry)
// CAST(SPLIT_PART(src_ip, '.', 1) AS INTEGER) > 192
```

Para evaluación en memoria:

```go
import "github.com/iamtelescope/flyql/golang/matcher"

matched, _ := matcher.Match("src_ip|firstoctet > 192", map[string]any{"src_ip": "193.0.0.1"}, registry)
// matched == true
```

## Transformers con entrada `Any`

Un transformer puede declarar `inputType = Type.Any` (valor de string `"any"`) para aceptar cualquier tipo upstream. Caso de uso motivador: un transformer `type` que devuelve el nombre del tipo de la columna, independientemente de cuál sea ese tipo.

Reglas:

- `Any` solo es válido en `inputType`. Declarar `Any` como `outputType` o en cualquier `ArgSpec.type` se rechaza en el momento de `register()`. Todo transformer debe declarar un tipo de salida concreto.
- Cuando un transformer con entrada `Any` se usa en una cadena, se omite la comprobación de igualdad del tipo de entrada para ese paso. La estrictez downstream se conserva — por ejemplo, `col|accepts_any_returning_array|upper` sigue dando error en `upper` porque `array` no es una entrada válida para `upper`.
- `parseFlyQLType("any")` lanza un error: `any` no es un tipo válido de columna ni de valor.

```python
from flyql.flyql_type import Type
from flyql.transformers import Transformer

class TypeTransformer(Transformer):
    @property
    def name(self): return "type"
    @property
    def input_type(self): return Type.Any      # acepta cualquier tipo upstream
    @property
    def output_type(self): return Type.String  # debe ser concreto
    def sql(self, dialect, column_ref, args=None):
        if dialect == "clickhouse":
            return f"toTypeName({column_ref})"
        return f"pg_typeof({column_ref})::text"
    def apply(self, value, args=None):
        return type(value).__name__
```

## Compatibilidad de tipos

Los transformers personalizados participan en la misma validación de cadenas de tipos que los integrados. Si encadenas `field|firstoctet|upper`, el sistema detecta que `firstoctet` devuelve `int`, pero `upper` requiere `string`.

Tipos disponibles: consulta [`flyql.Type`](/advanced/column-types/). La única excepción es `Any`, que es un comodín de entrada de transformer y no es un tipo válido de columna ni de valor.

## Validación con `diagnose()`

La función `diagnose()` recorre un AST parseado contra un schema de columnas y un registro de transformers, devolviendo registros `Diagnostic` posicionados sin lanzar excepciones. Verifica:

- **Columna desconocida** — clave base no está en la lista de columnas
- **Transformer desconocido** — nombre del transformer no está en el registro
- **Aridad incorrecta** — demasiados pocos o demasiados argumentos respecto a `argSchema`
- **Tipo de argumento incorrecto** — el tipo de valor del argumento no coincide con el tipo `ArgSpec` declarado (la ampliación de int a float está permitida)
- **Conflicto de tipo en cadena** — el tipo de salida de un transformer no coincide con el tipo de entrada del siguiente, o el tipo de la columna no coincide con el tipo de entrada del primer transformer

Cada diagnóstico contiene un `range` (posición en el código fuente), `message`, `severity` (`"error"`) y `code` (`"unknown_column"`, `"unknown_transformer"`, `"arg_count"`, `"arg_type"`, `"chain_type"`).

```python
# Python
from flyql import parse, Column, ColumnSchema, Type, diagnose

ast = parse("hstt|upper=X").root
columns = ColumnSchema.from_columns([Column(name="host", column_type=Type.String)])
diags = diagnose(ast, columns)  # columns is a ColumnSchema
# [Diagnostic(range=Range(0, 4), message="column 'hstt' is not defined", severity='error', code='unknown_column')]
```

```javascript
// JavaScript
import { parse, Column, ColumnSchema, Type, diagnose } from 'flyql'

const ast = parse('hstt|upper=X').root
const columns = ColumnSchema.fromColumns([new Column('host', false, Type.String)])
const diags = diagnose(ast, columns)  // columns is a ColumnSchema
// [Diagnostic { range: Range(0, 4), message: "column 'hstt' is not defined", severity: 'error', code: 'unknown_column' }]
```

```go
// Go
import flyql "github.com/iamtelescope/flyql/golang"

result, _ := flyql.Parse("hstt|upper=X")
schema := flyql.FromColumns([]flyql.Column{flyql.NewColumn("host", flyql.TypeString)})
diags := flyql.Diagnose(result.Root, schema, nil) // nil = default registry
// []Diagnostic{{Range: {0, 4}, Message: "column 'hstt' is not defined", Severity: "error", Code: "unknown_column"}}
```

Pasa un registro personalizado como tercer argumento si tienes transformers personalizados registrados. Pasa `nil`/`None`/`null` para usar el registro por defecto.

## Seguridad

Los métodos `sql()` de los transformers personalizados solo reciben referencias de columna procedentes del esquema, nunca valores proporcionados por el usuario. Los valores del usuario los escapa o parametriza el generador. Esto mantiene la garantía de resistencia a inyecciones de FlyQL: el SQL personalizado envuelve nombres de columnas, no entrada de usuario.
