FlyQL のクエリは、1 つ以上の条件を <FlyqlToken type="operator">and</FlyqlToken> / <FlyqlToken type="operator">or</FlyqlToken> でつなぎ、必要に応じて <FlyqlToken type="operator">not</FlyqlToken> で否定して構成します。各条件は比較式または truthy チェックです。

## 基本的なクエリ構造

<FlyqlCode code={`status = 200 and active and not archived`} />

- <FlyqlCode code={`status = 200`} inline /> — <FlyqlToken type="key">status</FlyqlToken> フィールドが <FlyqlToken type="value">200</FlyqlToken> と等しい
- <FlyqlCode code={`active`} inline /> — <FlyqlToken type="key">active</FlyqlToken> フィールドが truthy な値を持つ
- <FlyqlCode code={`not archived`} inline /> — <FlyqlToken type="key">archived</FlyqlToken> フィールドが falsy である（null、空、0、または false）

## 追加の例

<FlyqlCode code={`service != 'api' or user = "john doe"`} />

<FlyqlCode code={`message ~ "error.*" and not debug`} />

<FlyqlCode code={`(a = 1 or b = 2) and not (c = 3 and d = 4)`} />

<FlyqlCode code={`status in [200, 201] and method not in ['DELETE', 'PUT']`} />

<FlyqlCode code={`message has 'error' and tags not has 'debug'`} />

## 一般ルール

- **単独のキー** — 演算子なしのキーは [truthy チェック](/ja/syntax/values/#truthyfalsy-式)として扱われます: <FlyqlCode code={`active`} inline />
- **比較式** — 演算子付きのキーには、対応する値が必要です: <FlyqlCode code={`status = 200`} inline />
- **空白** — 演算子の前後のスペースは任意です: <FlyqlCode code={`status=200`} inline /> と <FlyqlCode code={`status = 200`} inline /> は同じ意味です
- **標準の演算子優先順位** — <FlyqlToken type="operator">not</FlyqlToken> が最も強く結合し、次に <FlyqlToken type="operator">and</FlyqlToken>、最後に <FlyqlToken type="operator">or</FlyqlToken>（SQL、Python、JavaScript、Go と同じ）: <FlyqlCode code={`a = 1 or b = 2 and c = 3`} inline /> は <FlyqlCode code={`a = 1 or (b = 2 and c = 3)`} inline /> として parse されます
- **括弧** — デフォルトの優先順位を上書きするには <FlyqlToken type="operator">( )</FlyqlToken> を使います: <FlyqlCode code={`(a = 1 or b = 2) and c = 3`} inline />
