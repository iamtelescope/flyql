## 布尔操作符

FlyQL 使用三个布尔操作符组合条件：

| 操作符 | 含义 | 示例 |
|--------|------|------|
| <FlyqlToken type="operator">and</FlyqlToken> | 两个条件都必须为真 | <FlyqlCode code={`status = 200 and active`} inline /> |
| <FlyqlToken type="operator">or</FlyqlToken> | 任意一个条件为真即可 | <FlyqlCode code={`level = 'error' or level = 'critical'`} inline /> |
| <FlyqlToken type="operator">not</FlyqlToken> | 对紧随其后的表达式或分组取反 | <FlyqlCode code={`not archived`} inline /> |

### 与取反组合

<FlyqlToken type="operator">not</FlyqlToken> 可以与 <FlyqlToken type="operator">and</FlyqlToken>、<FlyqlToken type="operator">or</FlyqlToken> 组合使用：

<FlyqlCode code={`active and not archived`} />
<FlyqlCode code={`level = 'error' or not healthy`} />

## 求值顺序

:::caution
在 FlyQL 中，<FlyqlToken type="operator">and</FlyqlToken> 与 <FlyqlToken type="operator">or</FlyqlToken> 之间**没有优先级**——这是与 SQL 有意的不同。条件会严格按出现的顺序**从左到右**组合。当分组很重要时，请始终使用括号。
:::

<FlyqlCode code={`a = 1 or b = 2 and c = 3`} />

会被解析为：

<FlyqlCode code={`(a = 1 or b = 2) and c = 3`} />

而不是 `a = 1 or (b = 2 and c = 3)`。如果你需要 SQL 风格的“and 优先于 or”行为，请使用括号显式表达。

相比之下，<FlyqlToken type="operator">not</FlyqlToken> 是一个**一元**前缀运算符，会与紧随其后的表达式或括号分组紧密结合。

## 括号分组

由于操作符之间没有优先级，括号是控制分组的唯一方式：

<FlyqlCode code={`status >= 400 or (level = 'error' and service = 'api')`} />

如果不加括号，相同的源代码

<FlyqlCode code={`status >= 400 or level = 'error' and service = 'api'`} />

会按从左到右的顺序解析为

<FlyqlCode code={`(status >= 400 or level = 'error') and service = 'api'`} />

这是一个完全不同的查询。

### 嵌套分组

括号可以继续嵌套：

<FlyqlCode code={`(a = 1 or b = 2) and not (c = 3 and d = 4)`} />

### 对整组取反

<FlyqlToken type="operator">not</FlyqlToken> 会作用于整个分组：

<FlyqlCode code={`not (status = 200 and healthy)`} />

这表示对组合条件整体取反——只有当 <FlyqlToken type="key">status</FlyqlToken> 不等于 200 或 <FlyqlToken type="key">healthy</FlyqlToken> 为假时才为真。

## 取反

<FlyqlToken type="operator">not</FlyqlToken> 可以作用于任何表达式——真值检查、比较表达式或整个分组：

<FlyqlCode code={`not active`} />
<FlyqlCode code={`not status = 200`} />
<FlyqlCode code={`not (a = 1 and b = 2)`} />
<FlyqlCode code={`active and not archived`} />

双重取反会互相抵消：<FlyqlCode code={`not not active`} inline /> 等价于 <FlyqlCode code={`active`} inline />。

## 示例

<FlyqlCode code={`status = 200 and service = 'api'`} />
<FlyqlCode code={`level = 'error' or level = 'critical'`} />
<FlyqlCode code={`status >= 400 or level = 'error' and service = 'api'`} />
<FlyqlCode code={`(status >= 400 or level = 'error') and service = 'api'`} />
<FlyqlCode code={`(status >= 500 and host = 'prod') or (level = 'error' and not service = 'healthcheck')`} />
