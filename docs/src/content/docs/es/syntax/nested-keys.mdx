## Notación por puntos

Usa rutas separadas por puntos para acceder a campos anidados en JSON, Map o datos estructurados:

<FlyqlCode code={`request.url = "/api/users"`} />
<FlyqlCode code={`response.headers.content_type = "application/json"`} />
<FlyqlCode code={`metadata.labels.env = 'prod'`} />

Cada segmento separado por puntos recorre un nivel dentro de la estructura anidada. Los generadores SQL traducen estas rutas a la sintaxis correcta para cada dialecto de base de datos: acceso a rutas JSON, búsquedas en Map, etc.

## Segmentos de clave entre comillas

Cuando un segmento de clave contiene un punto, enciérralo entre comillas para que ese punto no se interprete como separador:

<FlyqlCode code={`"foo.bar".baz = value`} />

Aquí:
- `"foo.bar"` es un único segmento de clave, el punto es literal
- <FlyqlToken type="key">baz</FlyqlToken> es un hijo de `"foo.bar"`

Sin comillas, <FlyqlToken type="key">foo.bar.baz</FlyqlToken> serían tres segmentos: <FlyqlToken type="key">foo</FlyqlToken> → <FlyqlToken type="key">bar</FlyqlToken> → <FlyqlToken type="key">baz</FlyqlToken>.

### Reglas de comillas

Se pueden usar comillas simples o dobles para los segmentos de clave:

<FlyqlCode code={`"foo.bar".baz = value`} />
<FlyqlCode code={`'foo.bar'.baz = value`} />

Solo hay que entrecomillar los segmentos que contienen puntos:

<FlyqlCode code={`metadata."dotted.key".value = 123`} />

Los identificadores entrecomillados también permiten usar [palabras reservadas](/es/syntax/reserved-words/) como nombres de columna — por ejemplo, <FlyqlCode code={`"not" = 1`} inline />.

## Profundidad de ruta

No hay límite de profundidad de anidación:

<FlyqlCode code={`a.b.c.d.e = value`} />

Cada segmento se recorre en orden al generar SQL o al evaluar en memoria.

## Combinación con operadores

Las claves anidadas funcionan con todos los operadores:

<FlyqlCode code={`request.status >= 400`} />
<FlyqlCode code={`response.body ~ "error.*"`} />
<FlyqlCode code={`metadata.tags in ['prod', 'staging']`} />
<FlyqlCode code={`request.headers.content_type != "text/html"`} />
<FlyqlCode code={`not request.authenticated`} />

## Acceso a elementos de matriz

Para columnas de tipo matriz, utiliza segmentos de ruta numéricos basados en cero para acceder a los elementos:

<FlyqlCode code={`tags.0 = 'important'`} />
<FlyqlCode code={`tags.1 = 'second'`} />

La indexación de matrices en FlyQL es **basada en 0 en todos los dialectos SQL y en el matcher en memoria**. Los generadores traducen el índice de flyql a la convención SQL nativa de cada destino: las matrices en ClickHouse, StarRocks y PostgreSQL están indexadas a partir de 1 a nivel de SQL, por lo que `tags.0` en flyql se emite como `tags[1]` en cada dialecto (con el entrecomillado de identificadores y la forma de función específicos del dialecto). Esto significa que la misma expresión flyql produce distintas cadenas SQL por dialecto pero **se evalúa al mismo resultado** sobre los mismos datos subyacentes.
