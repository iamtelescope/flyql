## Булевы операторы

FlyQL использует три булевых оператора для комбинирования условий:

| Оператор | Значение | Пример |
|----------|----------|--------|
| `and` | оба условия должны быть истинны | `status = 200 and active` |
| `or` | достаточно истинности одного условия | `level = error or level = critical` |
| `not` | отрицает выражение | `not archived` |

### Комбинированное отрицание

`not` можно комбинировать с `and` и `or`:

```
active and not archived
level = error or not healthy
```

## Приоритет операторов

`and` имеет более высокий приоритет, чем `or`. Это означает:

```
a = 1 or b = 2 and c = 3
```

будет разобрано как:

```
a = 1 or (b = 2 and c = 3)
```

Чтобы переопределить приоритет, используйте скобки явно.

## Группировка скобками

Используйте скобки для управления порядком вычисления:

```
(status >= 400 or level = error) and service = api
```

Без скобок `and` связывается сильнее:

```
status >= 400 or level = error and service = api
→ status >= 400 or (level = error and service = api)
```

### Вложенная группировка

Скобки могут быть вложенными:

```
(a = 1 or b = 2) and not (c = 3 and d = 4)
```

### Отрицание групп

`not` применяется ко всей группе:

```
not (status = 200 and healthy)
```

Это отрицает составное условие — истинно только когда `status` не равен 200 или `healthy` ложно.

## Отрицание

`not` отрицает любое выражение — проверку на истинность, сравнение или группу:

```
not active                  # поле ложно
not status = 200              # status не равен 200
not (a = 1 and b = 2)           # отрицание группы
active and not archived     # комбинация с другими условиями
```

Двойное отрицание взаимоуничтожается: `not not active` эквивалентно `active`.

## Примеры

```
# Простое AND
status = 200 and service = api

# Простое OR
level = error or level = critical

# Смешанное с учётом приоритета
status >= 400 or level = error and service = api

# Явная группировка
(status >= 400 or level = error) and service = api

# Сложный запрос с отрицанием и группировкой
(status >= 500 and host = prod*) or (level = error and not service = healthcheck)
```
