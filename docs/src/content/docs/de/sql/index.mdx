FlyQL-Generatoren erzeugen ein WHERE-Klausel-Fragment als einfachen SQL-String mit allen eingebetteten (escaped) Werten. Dieselbe FlyQL-Abfrage erzeugt je nach Zieldialekt unterschiedliches SQL.

## Unterstützte Dialekte

- [ClickHouse](/de/sql/clickhouse/) – keine Anführungszeichen für Spalten, `match()` für Regex, `JSONExtract`-Funktionen für JSON-Strings
- [PostgreSQL](/de/sql/postgresql/) – doppelt quotierte Bezeichner, `~` für Regex, JSONB-Operatoren
- [StarRocks](/de/sql/starrocks/) – mit Backticks quotierte Bezeichner, `regexp()` für Regex, `parse_json()` für JSON-Strings

## Zentrale Unterschiede

| Feature | ClickHouse | PostgreSQL | StarRocks |
|---------|------------|------------|-----------|
| Spaltenquotierung | Keine | `"name"` | `` `name` `` |
| Regex | `match(col, 'pat')` | `"col" ~ 'pat'` | `` regexp(`col`, 'pat') `` |
| Negiertes Regex | `NOT match(col, 'pat')` | `"col" !~ 'pat'` | `` NOT regexp(`col`, 'pat') `` |
| Wildcard | `col LIKE 'val%'` | `"col" LIKE 'val%'` | `` `col` LIKE 'val%' `` |
| Boolesche Operatoren | `AND` / `OR` | `AND` / `OR` | `AND` / `OR` |
| Leeres `IN []` | `0` | `FALSE` | `0` |
| Leeres `NOT IN []` | `1` | `TRUE` | `1` |
| String-<FlyqlToken type="operator">has</FlyqlToken> | `position(col, 'v') > 0` | `position('v' in "col") > 0` | `` INSTR(`col`, 'v') > 0 `` |
| Array-<FlyqlToken type="operator">has</FlyqlToken> | `has(col, 'v')` | `'v' = ANY("col")` | `` array_contains(`col`, 'v') `` |
| Map-<FlyqlToken type="operator">has</FlyqlToken> | `mapContains(col, 'k')` | `"col" ? 'k'` | `` array_contains(map_keys(`col`), 'k') `` |
| JSON-<FlyqlToken type="operator">has</FlyqlToken> | `JSONHas(toJSONString(col), 'k')` | `"col" ? 'k'` | `json_exists(...)` |

## JSON-Zugriff – der größte Unterschied

Jeder Dialekt behandelt verschachtelte JSON-Spalten sehr unterschiedlich:

**FlyQL-Eingabe:** <FlyqlCode code={`data.user.name = 'john'`} inline />

**ClickHouse** (native JSON-Spalte):
```sql
data.`user`.`name` = 'john'
```

**ClickHouse** (JSON als String gespeichert):
```sql
multiIf(JSONType(data, 'user', 'name') = 'String',
  equals(JSONExtractString(data, 'user', 'name'), 'john'), 0)
```

**PostgreSQL** (JSONB):
```sql
(jsonb_typeof("data"->'user'->'name') = 'string'
  AND "data"->'user'->>'name' = 'john')
```

**StarRocks** (natives JSON):
```sql
`data`->'\"user\"'->'\"name\"' = 'john'
```

Vollständige Beispiele findest du auf den jeweiligen Dialektseiten.

FlyQL-Array-Indizes sind in allen Dialekten einheitlich 0-basiert. Alle drei Dialekte verwenden auf der SQL-Ebene 1-basierte Arrays, sodass die Generatoren vor dem Erzeugen 1 zum flyql-Index addieren. Derselbe FlyQL-Ausdruck erzeugt pro Dialekt unterschiedliche SQL-Strings (aufgrund von Bezeichner-Quotierung und Funktionssyntax), wird aber gegen dieselben zugrunde liegenden Daten zum gleichen Ergebnis ausgewertet.

## Ausgabeformat

Generatoren geben ein WHERE-Klausel-Fragment zurück – ohne `WHERE`-Keyword und ohne parametrisierte Platzhalter. Alle Werte werden escaped und direkt eingebettet:

```
message = 'hello' and count > 10
```

Nicht:

```
WHERE message = ? AND count > ?
```
