## IN-सूची प्रकार संगति

पार्सर **हमेशा** <FlyqlCode code={`[200, 'ok', 404]`} inline /> जैसी विषम सूचियाँ स्वीकार करता है। SQL जनरेटर प्रकार संगति जाँच **केवल तभी** चलाते हैं जब (a) कॉलम का घोषित प्रकार है, (b) की खंडित नहीं है, और (c) सूची सजातीय है (`len(set(values_types)) == 1`)। विषम सूचियाँ, खंडित कीज़, और बिना प्रकार वाले कॉलम सभी कॉल साइट पर जाँच को बायपास करते हैं।

रेसिपी टाइप किए गए कॉलम पर तीन मामले प्रदर्शित करती है: एक सजातीय मिलान, एक विषम बायपास, और एक सजातीय गलत-प्रकार मामला जो विहित त्रुटि `type mismatch in IN list: ...` फेंकता है। त्रुटि पाठ Python, Go और JavaScript में समान है।
