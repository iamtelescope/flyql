Редактор FlyQL — это компонент для Vue 3, который предоставляет ввод запросов на основе схемы с автодополнением, подсветкой синтаксиса и клавиатурной навигацией.

## Быстрое подключение

```vue
<script setup>
import { ref } from 'vue'
import { FlyqlEditor } from 'flyql/editor'

const query = ref('')
const columns = {
    status: { type: 'number', suggest: true },
    level: { type: 'enum', suggest: true, autocomplete: true, values: ['debug', 'info', 'error'] },
    service: { type: 'string', suggest: true, autocomplete: true },
    host: { type: 'string', suggest: true },
}

function onSubmit() {
    console.log('Query submitted:', query.value)
}

function onParseError(error) {
    console.log('Parse error:', error)
}
</script>

<template>
    <FlyqlEditor
        v-model="query"
        :columns="columns"
        placeholder="Введите запрос FlyQL..."
        @submit="onSubmit"
        @parse-error="onParseError"
    />
</template>
```

Это отрисует текстовое поле с автодополнением для столбцов, типозависимыми операторами и логическими операторами — всё управляется схемой `columns`.

## Установка

```bash
npm install flyql
```

Редактор требует Vue 3 в качестве peer-зависимости:

```bash
npm install vue
```

Импортируйте из подпути `flyql/editor`:

```javascript
import { FlyqlEditor, FlyqlColumns } from 'flyql/editor'
```

Оба компонента внутренне импортируют `flyql.css`, который содержит переменные темы, стили панели подсказок и подсветку токенов. При использовании стандартных сборщиков (Vite, webpack и т.д.) это работает автоматически.

Если ваша конфигурация не обрабатывает CSS-импорты из JS, импортируйте стили вручную:

```javascript
import 'flyql/editor/flyql.css'
```

## Props

| Prop | Тип | По умолчанию | Описание |
|------|-----|--------------|----------|
| `modelValue` | `String` | `''` | Текст запроса (v-model) |
| `columns` | `Object` | `{}` | [Схема столбцов](/ru/editor/schema/) |
| `onAutocomplete` | `Function` | `null` | Асинхронная функция для подсказок значений |
| `onKeyDiscovery` | `Function` | `null` | Функция для удалённого обнаружения ключей |
| `placeholder` | `String` | `''` | Текст-подсказка |
| `autofocus` | `Boolean` | `false` | Автоматический фокус при монтировании |
| `debug` | `Boolean` | `false` | Отладочное логирование |
| `debounceMs` | `Number` | `300` | Задержка (мс) для асинхронных запросов автодополнения. Установите `0` для отключения |
| `dark` | `Boolean` | `false` | Включить тёмную тему ([см. темы](/ru/editor/theming/)) |

**Сигнатуры функций обратного вызова:**
- `onAutocomplete(key, value)` — возвращает `Promise<{ items: string[], incomplete?: boolean }>`
- `onKeyDiscovery(columnName, segments)` — возвращает `Promise<Array<{ name, type?, hasChildren? }>>`

Когда `incomplete` равно `true` в ответе, редактор понимает, что возвращённый список неполный, и повторно запросит данные при следующем нажатии клавиши. Асинхронные вызовы дебаунсятся (по умолчанию 150 мс), а текущие запросы автоматически отменяются при запуске нового.

## События

| Событие | Данные | Описание |
|---------|--------|----------|
| `update:modelValue` | `string` | Текст запроса изменился (v-model) |
| `submit` | — | Пользователь нажал Shift+Enter |
| `parse-error` | `string` | Сообщение об ошибке при невалидном запросе |
| `focus` | — | Редактор получил фокус |
| `blur` | — | Редактор потерял фокус |

## Доступные методы

Доступ через template ref:

```vue
<script setup>
import { ref } from 'vue'
import { FlyqlEditor } from 'flyql/editor'

const editorRef = ref(null)

function checkStatus() {
    const status = editorRef.value.getQueryStatus()
    console.log(status.valid, status.message)
}
</script>

<template>
    <FlyqlEditor ref="editorRef" v-model="query" :columns="columns" />
    <button @click="checkStatus">Check</button>
</template>
```

| Метод | Возвращает | Описание |
|-------|------------|----------|
| `focus()` | — | Установить фокус на редактор |
| `blur()` | — | Снять фокус с редактора |
| `getQueryStatus()` | `{ valid: boolean, message: string }` | Проверить текущий запрос |

## Асинхронное автодополнение значений

Передайте функцию `onAutocomplete` для получения подсказок значений из вашего API:

```vue
<script setup>
import { ref } from 'vue'
import { FlyqlEditor } from 'flyql/editor'

const query = ref('')
const columns = {
    service: { type: 'string', suggest: true, autocomplete: true },
    level: { type: 'enum', suggest: true, autocomplete: true, values: ['info', 'error'] },
}

async function onAutocomplete(key, value) {
    const resp = await fetch(`/api/autocomplete?key=${encodeURIComponent(key)}`)
    const data = await resp.json()
    return { items: data.values }
}
</script>

<template>
    <FlyqlEditor v-model="query" :columns="columns" :on-autocomplete="onAutocomplete" />
</template>
```

Функция получает `key` (имя столбца) и текущий префикс `value`. Возвращает `{ items: string[], incomplete?: boolean }`. Индикатор загрузки появляется автоматически через 200 мс.

### Потоковое автодополнение с состоянием

Редактор использует потоковое автодополнение с состоянием, которое минимизирует обращения к серверу:

- **Первоначальная загрузка** — При входе в контекст значения (например, `service=`) редактор немедленно выполняет запрос без дебаунса. Полный спиннер отображается до получения результатов.
- **Полный список** (`incomplete: false`) — Редактор сохраняет полученные элементы и переключается на фильтрацию на стороне клиента. Последующие нажатия клавиш фильтруют мгновенно без обращения к серверу, пока пользователь не перейдёт к другому столбцу.
- **Неполный список** (`incomplete: true`) — Редактор продолжает показывать текущие подсказки, пока в фоне выполняется повторный запрос с дебаунсом. В заголовке появляется встроенный спиннер. Когда приходят новые результаты, они заменяют текущие подсказки.

Если сервер вернул 0 элементов, редактор отображает «Совпадений не найдено». При клиентской фильтрации полного списка пустой результат просто скрывает подсказки (пользователь может удалить символы, чтобы увидеть больше).

Столбцы с `autocomplete: true` и статическими `values` используют эти значения напрямую без вызова функции. Столбцы с `autocomplete: true`, но без `values` вызывают функцию обратного вызова.

## Горячие клавиши

| Клавиша | Действие |
|---------|----------|
| Стрелка вверх/вниз | Навигация по подсказкам |
| Enter / Tab | Выбрать подсказку |
| Escape | Закрыть панель подсказок |
| Shift+Enter | Отправить запрос (вызывает событие `submit`) |
