## インストール

Python 版 FlyQL は PyPI で公開されています。

```bash
pip install flyql
```

これにより、`flyql` パッケージと依存関係の `google-re2` がインストールされます。

**要件:** Python 3.10+

## クエリを parse する

```python
from flyql import parse

result = parse("status = 200 and active")
print(result.root)
```

`parse()` は、AST を表す `root` ノードを持つ `ParseResult` オブジェクトを返します。入力が不正な場合は `ParserError` を送出します。

## SQL を生成する

generator を使うと、AST をパラメータ化 WHERE 句へ変換できます。データベース方言ごとに専用 generator があります。

### ClickHouse

```python
from flyql import parse
from flyql.generators.clickhouse.generator import to_sql_where
from flyql.generators.clickhouse.column import Column

result = parse("status >= 400 and host like 'prod%'")

columns = {
    "status": Column("status", False, "UInt32"),
    "host": Column("host", False, "String"),
}

sql = to_sql_where(result.root, columns)
print(sql)
```

PostgreSQL と StarRocks の generator も同じパターンです。`flyql.generators.postgresql` または `flyql.generators.starrocks` を import してください。

## メモリ上でマッチさせる

SQL を生成せずに、データレコードに対してクエリを評価できます。

```python
from flyql import parse
from flyql.matcher import Evaluator, Record


result = parse("status = 200 and active")

data = {
    "status": 200,
    "active": True,
    "host": "prod-api-01",
}

evaluator = Evaluator()
matches = evaluator.evaluate(result.root, Record(data))
print(f"Matches: {matches}")  # True
```

## カラム式を parse する

カラム式を、segments、transformers、aliases を持つ構造化データに parse できます。

```python
from flyql.columns import parse, parse_to_json

# 基本的なカラムを parse（transformers はデフォルトで無効）
parsed = parse("message, status")
for col in parsed:
    print(f"{col.name} (display: {col.display_name!r}, segments: {col.segments})")

# capabilities で transformers を有効化
with_transforms = parse("message|chars(25) as msg, status", capabilities={"transformers": True})

# または API レスポンス用に直接 JSON へシリアライズ
json_str = parse_to_json("message, status|upper", capabilities={"transformers": True})
print(json_str)
```

## パラメータ化クエリ

`$name` や `$1` のプレースホルダを使い、実行時に `bind_params()` で解決できます。

```python
from flyql import parse, bind_params
from flyql.generators.clickhouse.generator import to_sql_where
from flyql.generators.clickhouse.column import Column

# パラメータプレースホルダを含むクエリを parse
result = parse("status = $code and env in [$env, 'staging']")

# 具体的な値をパラメータにバインド
bind_params(result.root, {"code": 200, "env": "prod"})

# SQL を生成
columns = {
    "status": Column("status", False, "Int32"),
    "env": Column("env", False, "String"),
}
sql = to_sql_where(result.root, columns)
print(sql)
```

完全なリファレンスは [パラメータ](/ja/syntax/parameters/) を参照してください。

## クエリ内で Transformers を使う

transformer を含むクエリを parse し、SQL を生成できます。

```python
from flyql import parse
from flyql.generators.clickhouse.generator import to_sql_where, Column

# transformer を含むクエリを parse
result = parse("message|upper = 'ERROR'")

# ClickHouse 用の SQL を生成
columns = {"message": Column("message", False, "String")}
sql = to_sql_where(result.root, columns)
print(sql)  # equals(upper(message), 'ERROR')
```

構文の詳細は [Transformers](/ja/syntax/transformers/) を参照してください。
