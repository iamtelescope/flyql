Парсер FlyQL создаёт **абстрактное синтаксическое дерево (AST)** — структурированное представление запроса, которое можно обходить для генерации вывода в любой целевой формат. Встроенные SQL-генераторы (ClickHouse, PostgreSQL, StarRocks) реализованы обходом этого же AST. Вы можете создать собственный генератор для запросов Elasticsearch, фильтров MongoDB, селекторов Prometheus, команд grep по логам или любого другого формата.

## Типы узлов AST

AST состоит из трёх типов: **Node** (древовидная структура), **Expression** (листовые условия) и **Key** (ссылки на поля).

### Node

`Node` — это либо **ветвь** (объединяющая два поддерева логическим оператором), либо **лист** (содержащий одно выражение). Это фундаментальный строительный блок AST.

| Поле | Тип | Описание |
|------|-----|----------|
| `boolOperator` | `string` | <FlyqlToken type="operator">and</FlyqlToken> или <FlyqlToken type="operator">or</FlyqlToken> — способ объединения `left` и `right`. Пустая строка для листовых узлов. |
| `expression` | `Expression` или `null` | Условие, если это листовой узел. `null` для узлов-ветвей. |
| `left` | `Node` или `null` | Левое поддерево. `null` для листовых узлов. |
| `right` | `Node` или `null` | Правое поддерево. `null` для листовых узлов. |
| `negated` | `boolean` | `true`, если узел обёрнут в <FlyqlToken type="operator">not</FlyqlToken>. |

Узел является **либо** листом (`expression` задано, `left`/`right` равны null), **либо** ветвью (`left`/`right` заданы, `expression` равно null). Никогда и тем, и другим одновременно.

### Expression

`Expression` представляет одиночное условие — ключ, оператор и значение.

| Поле | Тип | Описание |
|------|-----|----------|
| `key` | `Key` | Поле, которое проверяется (например, <FlyqlToken type="key">status</FlyqlToken>, <FlyqlToken type="key">user.metadata.role</FlyqlToken>). |
| `operator` | `string` | Один из: <FlyqlToken type="operator">=</FlyqlToken>, <FlyqlToken type="operator">!=</FlyqlToken>, <FlyqlToken type="operator">{'~'}</FlyqlToken>, <FlyqlToken type="operator">{'!~'}</FlyqlToken>, <FlyqlToken type="operator">{'>'}</FlyqlToken>, <FlyqlToken type="operator">{'<'}</FlyqlToken>, <FlyqlToken type="operator">{'>='}</FlyqlToken>, <FlyqlToken type="operator">{'<='}</FlyqlToken>, <FlyqlToken type="operator">truthy</FlyqlToken>, <FlyqlToken type="operator">in</FlyqlToken>, <FlyqlToken type="operator">not in</FlyqlToken>, <FlyqlToken type="operator">has</FlyqlToken>, <FlyqlToken type="operator">not has</FlyqlToken>. |
| `value` | `any` | Значение для сравнения. Строка, число, Boolean, Null, `FunctionCall` или `Parameter` для скалярных операторов. Пустое для <FlyqlToken type="operator">in</FlyqlToken> и <FlyqlToken type="operator">not in</FlyqlToken>. |
| `valueType` | `string` или `null` | Явный тип `value`: `"int"`, `"bigint"`, `"float"`, `"string"`, `"bool"`, `"null"`, `"function"` или `"parameter"`. `null` для <FlyqlToken type="operator">in</FlyqlToken>/<FlyqlToken type="operator">not in</FlyqlToken>. |
| `values` | `array` или `null` | Список значений для операторов <FlyqlToken type="operator">in</FlyqlToken> и <FlyqlToken type="operator">not in</FlyqlToken>. Может быть гетерогенным и может содержать элементы `Parameter`. `null` для остальных операторов. |
| `valuesTypes` | `string[]` или `null` | Массив типов для каждого элемента, параллельный `values`. Каждый элемент — одна из констант типов (включая `"parameter"`). `null`, если `values` не используется. |

### Key

`Key` представляет путь к полю в запросе.

| Поле | Тип | Описание |
|------|-----|----------|
| `segments` | `string[]` | Сегменты пути. `["status"]` для простого ключа, `["user", "metadata", "role"]` для вложенного. |
| `raw` | `string` | Исходная строка ключа в том виде, в котором она записана в запросе. |
| `isSegmented` | `boolean` | `true`, если ключ содержит более одного сегмента (то есть использует точечную нотацию). |

### Parameter

`Parameter` представляет неразрешённый плейсхолдер (`$name` или `$1`), который будет подставлен `bindParams()` перед генерацией SQL или сопоставлением.

| Поле | Тип | Описание |
|------|-----|----------|
| `name` | `string` | Идентификатор плейсхолдера без префикса `$`. Именованный: `"code"`. Позиционный: `"1"`. |
| `positional` | `boolean` | `true`, если параметр позиционный (имя состоит только из цифр). |

`Parameter` может появляться как:
- `expression.value` с `valueType === "parameter"` (скалярная позиция)
- элемент внутри `expression.values` (позиция в IN-списке)
- элемент внутри `FunctionCall.parameterArgs` (позиция аргумента временной функции)

После того как `bindParams()` их разрешит, в AST не остаётся ни одного экземпляра `Parameter`.

### FunctionCall

`FunctionCall` представляет значение временной функции (`ago(...)`, `now()`, `today(...)`, `startOf(...)`).

| Поле | Тип | Описание |
|------|-----|----------|
| `name` | `string` | Одно из `"ago"`, `"now"`, `"today"`, `"startOf"`. |
| `durationArgs` | `Duration[]` | Разобранные аргументы продолжительности (например, `[{value: 5, unit: "m"}]`). |
| `unit` | `string` | Аргумент единицы для `startOf` (`"day"`, `"week"` или `"month"`). |
| `timezone` | `string` | Аргумент часового пояса для `today` и `startOf`. |
| `parameterArgs` | `Parameter[]` | Плейсхолдеры параметров, которые заполнят `durationArgs`/`unit`/`timezone` во время связывания. |

## Примеры структуры AST

### Простое условие

Запрос: <FlyqlCode code="status = 200" inline />

```
Node (leaf)
  expression:
    key: { segments: ["status"], raw: "status" }
    operator: "="
    value: 200
```

### Логическое объединение

Запрос: <FlyqlCode code='status = 200 and method = "GET"' inline />

```
Node (branch, boolOperator: "and")
  left: Node (leaf)
    expression:
      key: { segments: ["status"], raw: "status" }
      operator: "="
      value: 200
  right: Node (leaf)
    expression:
      key: { segments: ["method"], raw: "method" }
      operator: "="
      value: "GET"
```

### Отрицание

Запрос: <FlyqlCode code="not status = 500" inline />

```
Node (leaf, negated: true)
  expression:
    key: { segments: ["status"], raw: "status" }
    operator: "="
    value: 500
```

### Вложенная логика со скобками

Запрос: <FlyqlCode code="env='prod' and (status=500 or level = 'error')" inline />

```
Node (branch, boolOperator: "and")
  left: Node (leaf)
    expression:
      key: { segments: ["env"], raw: "env" }
      operator: "="
      value: "prod"
  right: Node (branch, boolOperator: "or")
    left: Node (leaf)
      expression:
        key: { segments: ["status"], raw: "status" }
        operator: "="
        value: 500
    right: Node (leaf)
      expression:
        key: { segments: ["level"], raw: "level" }
        operator: "="
        value: "error"
```

### Проверка принадлежности к списку

Запрос: <FlyqlCode code="status in [200, 201, 204]" inline />

```
Node (leaf)
  expression:
    key: { segments: ["status"], raw: "status" }
    operator: "in"
    value: ""
    values: [200, 201, 204]
    valuesType: "number"
```

### Проверка truthy

Запрос: <FlyqlCode code="active" inline />

```
Node (leaf)
  expression:
    key: { segments: ["active"], raw: "active" }

    value: ""
```

Запрос: <FlyqlCode code="status = $code" inline />

```

```

Запрос: <FlyqlCode code="status in [$x, 'ok']" inline />

```
Node (leaf)
  expression:
    key: { segments: ["status"], raw: "status" }
    operator: "in"
    values: [{ name: "x", positional: false }, "ok"]
    valuesTypes: ["parameter", "string"]
```

### Параметр внутри временной функции

Запрос: <FlyqlCode code="created > ago($d)" inline />

```

    key: { segments: ["created"], raw: "created" }
    operator: ">"
    value:
      name: "ago"
      durationArgs: []

    valueType: "function"
```

### Вложенный ключ

Запрос: <FlyqlCode code='user.metadata.role = "admin"' inline />

```
Node (leaf)
  expression:
    key: { segments: ["user", "metadata", "role"], raw: "user.metadata.role", isSegmented: true }
    operator: "="
    value: "admin"
```

## Обход AST

AST — это бинарное дерево. Для его обработки используется рекурсивный обход:

1. Если узел содержит `expression` (листовой узел) — обрабатываем условие.
2. Если узел содержит `left`/`right` (узел-ветвь) — рекурсивно обходим потомков и объединяем с помощью `boolOperator`.
3. Если `negated` равно true — оборачиваем результат в отрицание.

## Создание пользовательского генератора

Ниже приведён полный пример пользовательского генератора, который преобразует AST FlyQL в объект **Elasticsearch Query DSL**.

### JavaScript

```javascript
import { parse } from 'flyql'
import { Operator } from 'flyql'

function generateElasticsearch(node) {
  if (!node) return { match_all: {} }

  let result

  if (node.expression) {
    // Leaf node — convert the expression
    result = expressionToES(node.expression)
  } else {
    // Branch node — recurse and combine
    const left = generateElasticsearch(node.left)
    const right = generateElasticsearch(node.right)

    if (node.boolOperator === 'and') {
      result = { bool: { must: [left, right] } }
    } else {
      result = { bool: { should: [left, right], minimum_should_match: 1 } }
    }
  }

  // Handle negation
  if (node.negated) {
    result = { bool: { must_not: [result] } }
  }

  return result
}

function expressionToES(expr) {
  const field = expr.key.raw

  switch (expr.operator) {
    case Operator.EQUALS:
      return { term: { [field]: expr.value } }

    case Operator.NOT_EQUALS:
      return { bool: { must_not: [{ term: { [field]: expr.value } }] } }

    case Operator.REGEX:
      return { regexp: { [field]: String(expr.value) } }

    case Operator.NOT_REGEX:
      return { bool: { must_not: [{ regexp: { [field]: String(expr.value) } }] } }

    case Operator.GREATER_THAN:
      return { range: { [field]: { gt: expr.value } } }

    case Operator.GREATER_OR_EQUALS_THAN:
      return { range: { [field]: { gte: expr.value } } }

    case Operator.LOWER_THAN:
      return { range: { [field]: { lt: expr.value } } }

    case Operator.LOWER_OR_EQUALS_THAN:
      return { range: { [field]: { lte: expr.value } } }

    case Operator.IN:
      return { terms: { [field]: expr.values || [] } }

    case Operator.NOT_IN:
      return { bool: { must_not: [{ terms: { [field]: expr.values || [] } }] } }

    case Operator.TRUTHY:
      return { exists: { field } }

    case Operator.HAS:
      return { wildcard: { [field]: `*${expr.value}*` } }

    case Operator.NOT_HAS:
      return { bool: { must_not: [{ wildcard: { [field]: `*${expr.value}*` } }] } }

    default:
      throw new Error(`unsupported operator: ${expr.operator}`)
  }
}

// Usage
const result = parse('status = 200 and message ~ "error.*" and env in ["prod", "staging"]')
const esQuery = generateElasticsearch(result.root)
console.log(JSON.stringify(esQuery, null, 2))
```

### Go

```go
package main

import (
	"encoding/json"
	"fmt"
	flyql "github.com/iamtelescope/flyql/golang"
)

func generateElasticsearch(node *flyql.Node) map[string]any {
	if node == nil {
		return map[string]any{"match_all": map[string]any{}}
	}

	var result map[string]any

	if node.Expression != nil {
		// Leaf node — convert the expression
		result = expressionToES(node.Expression)
	} else {
		// Branch node — recurse and combine
		left := generateElasticsearch(node.Left)
		right := generateElasticsearch(node.Right)

		if node.BoolOperator == flyql.BoolOpAnd {
			result = map[string]any{
				"bool": map[string]any{"must": []any{left, right}},
			}
		} else {
			result = map[string]any{
				"bool": map[string]any{"should": []any{left, right}, "minimum_should_match": 1},
			}
		}
	}

	// Handle negation
	if node.Negated {
		result = map[string]any{
			"bool": map[string]any{"must_not": []any{result}},
		}
	}

	return result
}

func expressionToES(expr *flyql.Expression) map[string]any {
	field := expr.Key.Raw

	switch expr.Operator {
	case flyql.OpEquals:
		return map[string]any{"term": map[string]any{field: expr.Value}}
	case flyql.OpNotEquals:
		return map[string]any{"bool": map[string]any{"must_not": []any{map[string]any{"term": map[string]any{field: expr.Value}}}}}
	case flyql.OpGreater:
		return map[string]any{"range": map[string]any{field: map[string]any{"gt": expr.Value}}}
	case flyql.OpLess:
		return map[string]any{"range": map[string]any{field: map[string]any{"lt": expr.Value}}}
	case flyql.OpIn:
		return map[string]any{"terms": map[string]any{field: expr.Values}}
	case flyql.OpNotIn:
		return map[string]any{"bool": map[string]any{"must_not": []any{map[string]any{"terms": map[string]any{field: expr.Values}}}}}
	case flyql.OpTruthy:
		return map[string]any{"exists": map[string]any{"field": field}}
	case flyql.OpRegex:
		return map[string]any{"regexp": map[string]any{field: fmt.Sprintf("%v", expr.Value)}}
	default:
		panic(fmt.Sprintf("unsupported operator: %s", expr.Operator))
	}
}

func main() {
	result, err := flyql.Parse(`status = 200 and env in ["prod", "staging"]`)
	if err != nil {
		panic(err)
	}

	esQuery := generateElasticsearch(result.Root)
	data, _ := json.MarshalIndent(esQuery, "", "  ")
	fmt.Println(string(data))
}
```

### Python

```python
from flyql import parse
from flyql.core.constants import Operator, BoolOperator

def generate_elasticsearch(node):
    if node is None:
        return {"match_all": {}}

    if node.expression is not None:
        # Leaf node — convert the expression
        result = expression_to_es(node.expression)
    else:
        # Branch node — recurse and combine
        left = generate_elasticsearch(node.left)
        right = generate_elasticsearch(node.right)

        if node.bool_operator == BoolOperator.AND.value:
            result = {"bool": {"must": [left, right]}}
        else:
            result = {"bool": {"should": [left, right], "minimum_should_match": 1}}

    # Handle negation
    if node.negated:
        result = {"bool": {"must_not": [result]}}

    return result

def expression_to_es(expr):
    field = expr.key.raw
    op = expr.operator  # operator is stored as a string value, e.g. "=", "!=", "~"

    if op == Operator.EQUALS.value:
        return {"term": {field: expr.value}}
    elif op == Operator.NOT_EQUALS.value:
        return {"bool": {"must_not": [{"term": {field: expr.value}}]}}
    elif op == Operator.REGEX.value:
        return {"regexp": {field: str(expr.value)}}
    elif op == Operator.NOT_REGEX.value:
        return {"bool": {"must_not": [{"regexp": {field: str(expr.value)}}]}}
    elif op == Operator.GREATER_THAN.value:
        return {"range": {field: {"gt": expr.value}}}
    elif op == Operator.LOWER_THAN.value:
        return {"range": {field: {"lt": expr.value}}}
    elif op == Operator.IN.value:
        return {"terms": {field: expr.values or []}}
    elif op == Operator.NOT_IN.value:
        return {"bool": {"must_not": [{"terms": {field: expr.values or []}}]}}
    elif op == Operator.TRUTHY.value:
        return {"exists": {"field": field}}
    elif op == Operator.HAS.value:
        return {"wildcard": {field: f"*{expr.value}*"}}
    elif op == Operator.NOT_HAS.value:
        return {"bool": {"must_not": [{"wildcard": {field: f"*{expr.value}*"}}]}}
    else:
        raise ValueError(f"unsupported operator: {op}")


# Usage
result = parse('status = 200 and message ~ "error.*" and env in ["prod", "staging"]')
es_query = generate_elasticsearch(result.root)
print(es_query)
```

## Как работают встроенные генераторы

Встроенные SQL-генераторы следуют тому же рекурсивному шаблону, что показан выше. Каждый генератор:

1. **Получает корневой `Node`** и карту схемы столбцов.
2. **Рекурсивно обходит дерево:** узлы-ветви превращаются в `(left AND right)` или `(left OR right)`, листовые узлы — в SQL-условия.
3. **Обрабатывает каждый оператор**, переключаясь по `expression.operator` и генерируя SQL, специфичный для диалекта.
4. **Оборачивает отрицённые узлы** в `NOT (...)`.

Ключевое отличие от пользовательского генератора в том, что SQL-генераторы также валидируют выражения по схеме столбцов (проверяя типы, допустимые значения и существование столбца). Пользовательский генератор может пропустить этот шаг или реализовать собственную логику валидации.

## Справочник экспортируемых типов

### JavaScript

```javascript
import {
  parse,               // Parse a query string into an AST
  bindParams,          // Resolve parameter placeholders: bindParams(node, params)
  Parser,              // Parser class (advanced usage)
  Node,                // AST branch/leaf node
  Expression,          // Leaf condition
  Parameter,           // Parameter placeholder ($name / $1)
  FunctionCall,        // Temporal function value (ago, now, today, startOf)
  Key,                 // Field path
  Range,               // Source position range [start, end)
  Column,              // Core column type for validator
  ColumnSchema,        // Column schema container with nested resolution
  Operator,            // Comparison operator constants
  BoolOperator,        // Boolean operator constants ("and", "or")
  FlyqlError,          // Base error class
  ParserError,         // Parse error (extends FlyqlError, includes errno)
  diagnose,            // AST validator: diagnose(ast, schema, registry?) → Diagnostic[]
  Diagnostic,          // Positioned diagnostic record
} from 'flyql'
```

### Go

```go
import flyql "github.com/iamtelescope/flyql/golang"

// flyql.Parse(query string) (*ParseResult, error)
// flyql.BindParams(node *Node, params map[string]any) error — resolve parameter placeholders
// flyql.Node         — AST node struct
// flyql.Expression   — leaf condition struct
// flyql.Parameter    — parameter placeholder struct ($name / $1)
// flyql.FunctionCall — temporal function value struct
// flyql.Key          — field path struct
// flyql.Range       — source position range [Start, End)
// flyql.Column      — core column type for validator
// flyql.ColumnSchema — column schema container with nested resolution
// flyql.FromColumns(cols) *ColumnSchema — build schema from flat column slice
// flyql.Diagnose(ast, schema, registry) []Diagnostic — AST validator
// flyql.Diagnostic  — positioned diagnostic record
```
