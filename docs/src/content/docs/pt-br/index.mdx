Bem-vindo à documentação do projeto **FlyQL**. Versão atual: **{flyqlVersion}** ([registro de alterações](/pt-br/changelog)).

FlyQL é uma linguagem de consultas leve e legível projetada para filtrar dados. Transpila para SQL parametrizado e à prova de injeção para **ClickHouse**, **PostgreSQL** e **StarRocks** — com implementações completas em **Go**, **Python** e **JavaScript**. Escreva uma consulta uma vez, e o FlyQL gera SQL específico do dialeto para seu banco de dados — ou a avalia em memória contra registros de dados.

<FlyqlCode code={landingIntroQuery} />

Esta consulta funciona de forma idêntica independentemente de seu backend ser ClickHouse, PostgreSQL, StarRocks ou um conjunto de dados em memória.

## Principais Funcionalidades

- **À Prova de Injeção por Design** — A gramática não pode expressar subconsultas, JOINs, UNIONs ou DDL. A segurança é estrutural, não defensiva. Todo o SQL gerado usa valores parametrizados.
- **SQL Multi-Dialeto** — Uma linguagem de consultas, saída específica por dialeto. Gera SQL correto para ClickHouse, PostgreSQL e StarRocks com tratamento adequado de tipos (caminhos JSON, acesso a Map, tratamento de Array).
- **Multi-Linguagem** — Implementações completas em Go, Python e JavaScript. Parseie no frontend, valide no backend, gere SQL em qualquer lugar — mesma consulta, mesmos resultados.
- **Zero Dependências** — Parser escrito à mão como máquina de estados sem dependências externas em nenhuma linguagem. Trivial de integrar, nada para conflitar.
