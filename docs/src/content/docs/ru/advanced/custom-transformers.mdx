---
title: Пользовательские трансформеры
description: Регистрация собственных трансформеров для расширения FlyQL
---

Система трансформеров FlyQL расширяема. Вы можете регистрировать собственные трансформеры, которые будут работать так же, как встроенные (`upper`, `lower`, `len`) в парсере, SQL-генераторах, механизме сопоставления в памяти и автодополнении редактора.

## Интерфейс трансформера

Каждый трансформер реализует пять элементов:

| Элемент | Тип | Описание |
|---------|-----|----------|
| `name` | string | уникальный идентификатор |
| `inputType` | TransformerType | ожидаемый входной тип |
| `outputType` | TransformerType | тип после преобразования |
| `sql(dialect, columnRef)` | string | SQL-обёртка для целевого диалекта |
| `apply(value)` | any | преобразование значения в памяти |

## API реестра

| Метод | Описание |
|-------|----------|
| `defaultRegistry()` | возвращает новый реестр со встроенными трансформерами |
| `registry.register(transformer)` | добавляет трансформер в реестр |
| `registry.get(name)` | возвращает трансформер по имени |
| `registry.names()` | возвращает список доступных имён |

## Совместимость типов

Пользовательские трансформеры участвуют в той же типовой валидации, что и встроенные.

## Безопасность

Метод `sql()` получает только ссылку на колонку, а не пользовательский ввод. Поэтому защита FlyQL от инъекций сохраняется.
