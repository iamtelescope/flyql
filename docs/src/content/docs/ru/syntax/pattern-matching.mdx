## Регулярные выражения

Используйте <FlyqlToken type="operator">{'~'}</FlyqlToken> для проверки соответствия регулярному выражению и <FlyqlToken type="operator">{'!~'}</FlyqlToken> для отрицания:

<FlyqlCode code={`message ~ "error.*timeout"`} />
<FlyqlCode code={`path !~ "^/health"`} />

| Оператор | Значение |
|----------|----------|
| <FlyqlToken type="operator">{'~'}</FlyqlToken> | Значение соответствует регулярному выражению |
| <FlyqlToken type="operator">{'!~'}</FlyqlToken> | Значение не соответствует регулярному выражению |

### Синтаксис регулярных выражений

Синтаксис зависит от целевого бэкенда:

- **ClickHouse** — функция `match()` (синтаксис RE2)
- **PostgreSQL** — операторы <FlyqlToken type="operator">{'~'}</FlyqlToken> и <FlyqlToken type="operator">{'!~'}</FlyqlToken> (POSIX-регулярные выражения)
- **StarRocks** — функция `regexp`
- **Сопоставление в памяти** — движок регулярных выражений языка выполнения (Go `regexp`, Python `re2`, JS `RegExp`)

Заключайте шаблон в кавычки, если он содержит пробелы или специальные символы:

<FlyqlCode code={`message ~ "connection (refused|reset)"`} />
<FlyqlCode code={`host ~ "^prod-[0-9]+"`} />

### Флаги регулярных выражений

Вы можете изменить поведение регулярного выражения с помощью инлайн-флагов в начале шаблона, используя синтаксис `(?...)`. FlyQL поддерживает три флага, которые работают во всех бэкендах:

#### `(?i)` — Регистронезависимый поиск

Шаблон сопоставляется без учёта регистра:

<FlyqlCode code={`message ~ "(?i)error"`} />
<FlyqlCode code={`level ~ "(?i)^(warn|error|fatal)$"`} />

`(?i)error` совпадает с «Error», «ERROR», «error» и т. д.

#### `(?m)` — Многострочный режим

Изменяет поведение `^` и `$` так, что они совпадают с началом и концом каждой **строки**, а не всей строки целиком:

<FlyqlCode code={`log ~ "(?m)^ERROR"`} />

Без `(?m)` выражение `^ERROR` совпадает только если вся строка начинается с «ERROR». С `(?m)` оно также совпадает с «INFO first\nERROR second», потому что «ERROR» находится в начале строки.

#### `(?s)` — Dotall

Делает так, что `.` совпадает с **любым** символом, включая перевод строки. По умолчанию `.` не совпадает с `\n`:

<FlyqlCode code={`log ~ "(?s)start.*end"`} />

Без `(?s)` выражение `start.*end` не совпадает со «start\nend», так как `.` не пересекает перевод строки. С `(?s)` — совпадает.

#### Комбинирование флагов

Флаги можно комбинировать в одной группе:

<FlyqlCode code={`log ~ "(?im)^ERROR"`} />
<FlyqlCode code={`log ~ "(?ims)^START.*END$"`} />

#### Поддержка бэкендов

Все три флага работают во всех бэкендах FlyQL:

| Бэкенд | `(?i)` | `(?m)` | `(?s)` |
|--------|--------|--------|--------|
| **ClickHouse** | Yes | Yes | Yes |
| **PostgreSQL** | Yes | Yes | Yes |
| **StarRocks** | Yes | Yes | Yes |
| **Go matcher** | Yes | Yes | Yes |
| **Python matcher** | Yes | Yes | Yes |
| **JavaScript matcher** | Yes | Yes | Yes |

Регистронезависимый поиск можно сочетать с отрицанием, чтобы исключать шаблоны независимо от регистра:

<FlyqlCode code={`message ~ "(?i)error" and message !~ "(?i)warning"`} />

## Сопоставление LIKE

Используйте <FlyqlToken type="operator">like</FlyqlToken> и <FlyqlToken type="operator">ilike</FlyqlToken> для SQL-подобного сопоставления по шаблону с подстановочными знаками:

<FlyqlCode code={`host like 'prod%'`} />
<FlyqlCode code={`service like 'api-%'`} />
<FlyqlCode code={`message ilike '%Error%'`} />

| Оператор | Значение |
|----------|----------|
| <FlyqlToken type="operator">like</FlyqlToken> | Значение соответствует шаблону (с учётом регистра) |
| <FlyqlToken type="operator">not like</FlyqlToken> | Значение не соответствует шаблону |
| <FlyqlToken type="operator">ilike</FlyqlToken> | Значение соответствует шаблону (без учёта регистра) |
| <FlyqlToken type="operator">not ilike</FlyqlToken> | Значение не соответствует шаблону (без учёта регистра) |

### Подстановочные знаки

В шаблонах <FlyqlToken type="operator">like</FlyqlToken> / <FlyqlToken type="operator">ilike</FlyqlToken> доступны два подстановочных знака:

| Знак | Значение | Пример | Совпадает с |
|------|----------|--------|-------------|
| `%` | Любая последовательность символов (включая пустую) | <FlyqlCode code={`host like 'prod%'`} inline /> | `prod`, `prod-1`, `production` |
| `_` | Ровно один символ | <FlyqlCode code={`code like 'E__'`} inline /> | `E01`, `Exx`, но не `E1` и не `Error` |

Подстановочные знаки могут располагаться в любом месте шаблона — в начале, середине, конце или в комбинации:

<FlyqlCode code={`message like '%timeout%'`} />
<FlyqlCode code={`host like 'web-_-prod'`} />
<FlyqlCode code={`path like '/api/%/status'`} />

### Экранирование подстановочных знаков

Для литерального совпадения с `%` или `_` экранируйте их обратной косой чертой:

<FlyqlCode code={`value like '100\\%'`} />
<FlyqlCode code={`col like 'foo\\_bar'`} />

`100\%` совпадает с литеральной строкой «100%», а не с «100» и далее произвольным текстом.

### Регистронезависимое сопоставление

Используйте <FlyqlToken type="operator">ilike</FlyqlToken> для сопоставления без учёта регистра:

<FlyqlCode code={`message ilike '%error%'`} />
<FlyqlCode code={`host not ilike 'TEST%'`} />

### LIKE и регулярные выражения

| Сценарий | Синтаксис | Пример |
|----------|-----------|--------|
| Совпадение по префиксу | <FlyqlCode code={`key like 'prefix%'`} inline /> | <FlyqlCode code={`host like 'prod%'`} inline /> |
| Содержит | <FlyqlCode code={`key like '%word%'`} inline /> | <FlyqlCode code={`msg like '%error%'`} inline /> |
| Сложный шаблон | <FlyqlCode code={`key ~ "pattern"`} inline /> | <FlyqlCode code={`host ~ "^prod-[0-9]+"`} inline /> |

Используйте <FlyqlToken type="operator">like</FlyqlToken> для простого сопоставления с подстановочными знаками. Используйте регулярные выражения (<FlyqlToken type="operator">{'~'}</FlyqlToken>), когда нужны классы символов, чередование или другие сложные шаблоны.

### Поддержка бэкендов

Все SQL-бэкенды (ClickHouse, PostgreSQL, StarRocks) генерируют нативные SQL-операторы `LIKE` / `ILIKE` / `NOT LIKE` / `NOT ILIKE`. Matcher в памяти внутренне конвертирует шаблон в регулярное выражение.

:::note
StarRocks `ILIKE` требует версию 2.5 или выше.
:::

## Примеры

<FlyqlCode code={`message ~ "error|fail"`} />
<FlyqlCode code={`path ~ "^/api/v2"`} />
<FlyqlCode code={`path !~ "^/health"`} />
<FlyqlCode code={`host like 'prod%'`} />
<FlyqlCode code={`host like 'prod%' and message ~ "timeout.*" and path !~ "^/health"`} />
