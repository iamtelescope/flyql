## Instalación

```bash
npm install flyql
```

**Requisitos:** Node.js 16+. Todas las importaciones usan módulos ES.

## Parsear una consulta

```javascript
import { parse } from 'flyql'

const result = parse("status = 200 and active")
console.log(result.root)
```

`parse()` devuelve un objeto `Parser` con un nodo `root` que representa el AST. Lanza `ParserError` si la entrada no es válida:

```javascript
import { parse, ParserError } from 'flyql'

try {
    const result = parse("status = 200 and active")
    console.log(result.root)
} catch (err) {
    if (err instanceof ParserError) {
        console.error(`Parse error: ${err.message}`)
    }
}
```

## Generar SQL

Usa un generador para convertir el AST en una cláusula WHERE parametrizada. Cada dialecto de base de datos se importa desde una subruta distinta.

### ClickHouse

```javascript
import { parse } from 'flyql'
import { generateWhere, newColumn } from 'flyql/generators/clickhouse'

const result = parse("status >= 400 and host like 'prod%'")

const columns = {
    status: newColumn("status", false, "UInt32", null),
    host: newColumn("host", false, "String", null),
}

const sql = generateWhere(result.root, columns)
console.log(sql)
```

### Otros dialectos

Los generadores de PostgreSQL y StarRocks siguen el mismo patrón:

```javascript
import { generateWhere, newColumn } from 'flyql/generators/postgresql'
import { generateWhere, newColumn } from 'flyql/generators/starrocks'
```

Nota: `newColumn` de PostgreSQL tiene una firma diferente: `newColumn(name, type, values)`, sin el parámetro `jsonString`.

## Evaluar en memoria

Evalúa una consulta sobre un registro de datos sin generar SQL:

```javascript
import { match } from 'flyql/matcher'

const data = {
    status: 200,
    active: true,
    host: "prod-api-01",
}

const matches = match("status = 200 and active", data)
console.log(`Matches: ${matches}`) // true
```

## Parsear expresiones de columnas

Parsea una expresión de columnas en datos estructurados con segmentos, transformers y alias:

```javascript
import { parse, parseToJson } from 'flyql/columns'

// Parsea columnas básicas (los transformers están desactivados por defecto)
const parsed = parse("message, status")
for (const col of parsed) {
    console.log(`${col.name} (display: ${JSON.stringify(col.displayName)}, segments: ${col.segments})`)
}

// Activa transformers mediante capabilities
const withTransforms = parse("message|chars(25) as msg, status", { transformers: true })

// O serializa directamente a JSON para respuestas de API
const json = parseToJson("message, status|upper", { transformers: true })
console.log(json)
```

## Consultas parametrizadas

Usa marcadores `$name` y `$1`, y resuélvelos en tiempo de ejecución con `bindParams()`:

```javascript
import { parse, bindParams } from 'flyql'

const ast = parse('status = $code and env = $env')
bindParams(ast.root, { code: 200, env: 'prod' })
```

Consulta [Parámetros](/es/syntax/parameters/) para ver la referencia completa.

## Transformers en consultas

Parsea una consulta con transformers y genera SQL:

```javascript
import { parse } from 'flyql'
import { generateWhere, newColumn } from 'flyql/generators/clickhouse'

// Parsea una consulta con transformers
const result = parse("message|upper = 'ERROR'")

// Genera SQL para ClickHouse
const columns = { message: newColumn('message', false, 'String') }
const sql = generateWhere(result.root, columns)
console.log(sql) // equals(upper(message), 'ERROR')
```

Consulta [Transformers](/es/syntax/transformers/) para ver la referencia completa de sintaxis.

## Subrutas de importación disponibles

| Ruta de importación | Contenido |
|---------------------|-----------|
| `flyql` | Parser principal (`parse`, `ParserError`, operadores, constantes) |
| `flyql/core` | Módulo del parser directamente |
| `flyql/columns` | Parser de expresiones de columnas |
| `flyql/generators/clickhouse` | Generador SQL para ClickHouse |
| `flyql/generators/postgresql` | Generador SQL para PostgreSQL |
| `flyql/generators/starrocks` | Generador SQL para StarRocks |
| `flyql/matcher` | Matcher de registros en memoria |
| `flyql/editor` | Componente de editor web, requiere Vue 3 |
