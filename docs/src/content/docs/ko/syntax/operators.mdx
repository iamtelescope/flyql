FlyQL은 다음 비교 연산자를 지원합니다.

## 동등 비교

| 연산자 | 이름 | 예제 |
|--------|------|------|
| <FlyqlToken type="operator">=</FlyqlToken> | 같음 | <FlyqlCode code={`status = 200`} inline /> |
| <FlyqlToken type="operator">!=</FlyqlToken> | 같지 않음 | <FlyqlCode code={`env != prod`} inline /> |

<FlyqlCode code={`status = 200`} />
<FlyqlCode code={`service != "api-gateway"`} />

연산자의 양쪽에는 <FlyqlToken type="key">키</FlyqlToken>와 <FlyqlToken type="value">값</FlyqlToken>이 모두 필요합니다.

## 숫자 비교

| 연산자 | 이름 | 예제 |
|--------|------|------|
| <FlyqlToken type="operator">{'>'}</FlyqlToken> | 초과 | <FlyqlCode code={`status > 399`} inline /> |
| <FlyqlToken type="operator">{'>='}</FlyqlToken> | 이상 | <FlyqlCode code={`status >= 400`} inline /> |
| <FlyqlToken type="operator">{'<'}</FlyqlToken> | 미만 | <FlyqlCode code={`duration < 1000`} inline /> |
| <FlyqlToken type="operator">{'<='}</FlyqlToken> | 이하 | <FlyqlCode code={`duration <= 500`} inline /> |

<FlyqlCode code={`status >= 400 and duration < 5000`} />
<FlyqlCode code={`response_time > 100 and response_time <= 2000`} />

숫자 비교 연산자는 숫자 값에 대해 동작합니다. 문자열 값과 함께 사용하면 사전식 비교가 적용됩니다.

## Regex 매칭

| 연산자 | 이름 | 예제 |
|--------|------|------|
| <FlyqlToken type="operator">{'~'}</FlyqlToken> | regex 일치 | <FlyqlCode code={`message ~ "error.*"`} inline /> |
| <FlyqlToken type="operator">{'!~'}</FlyqlToken> | regex 불일치 | <FlyqlCode code={`path !~ "^/health"`} inline /> |

<FlyqlCode code={`message ~ "timeout|connection refused"`} />
<FlyqlCode code={`host !~ "^test-.*"`} />

Regex 패턴은 대상 데이터베이스 또는 런타임의 문법을 따릅니다. regex 플래그와 상세 정보는 [패턴 매칭](/ko/syntax/pattern-matching/)을 참고하세요.

## 리스트 포함 여부

| 연산자 | 이름 | 예제 |
|--------|------|------|
| <FlyqlToken type="operator">in</FlyqlToken> | 리스트에 포함됨 | <FlyqlCode code={`status in [200, 201]`} inline /> |
| <FlyqlToken type="operator">not in</FlyqlToken> | 리스트에 포함되지 않음 | <FlyqlCode code={`env not in ['dev', 'test']`} inline /> |

리스트 구문 규칙은 [리스트](/ko/syntax/lists/) 페이지를 참고하세요.

## 포함 검사

| 연산자 | 이름 | 예제 |
|--------|------|------|
| <FlyqlToken type="operator">has</FlyqlToken> | 포함함 | <FlyqlCode code={`message has 'error'`} inline /> |
| <FlyqlToken type="operator">not has</FlyqlToken> | 포함하지 않음 | <FlyqlCode code={`message not has 'test'`} inline /> |

<FlyqlCode code={`message has 'timeout'`} />
<FlyqlCode code={`tags not has 'deprecated'`} />

<FlyqlToken type="operator">has</FlyqlToken> 연산자는 타입에 따라 다른 포함 검사를 수행합니다.

- **String** — 부분 문자열 매칭: <FlyqlCode code={`message has 'error'`} inline />는 <FlyqlToken type="key">message</FlyqlToken>에 <FlyqlToken type="string">"error"</FlyqlToken>가 포함되면 true
- **Map** — 키 존재 검사: <FlyqlCode code={`metadata has 'region'`} inline />은 map에 <FlyqlToken type="string">"region"</FlyqlToken> 키가 있으면 true
- **Array/List** — 항목 포함 검사: <FlyqlCode code={`tags has 'web'`} inline />은 array에 <FlyqlToken type="string">"web"</FlyqlToken>가 포함되면 true
- **JSON** — 키 존재 검사: <FlyqlCode code={`data has 'name'`} inline />은 JSON 객체에 <FlyqlToken type="string">"name"</FlyqlToken> 키가 있으면 true
- **Null/빈 값** — <FlyqlToken type="operator">has</FlyqlToken>에서는 항상 false, <FlyqlToken type="operator">not has</FlyqlToken>에서는 항상 true

<FlyqlToken type="operator">has</FlyqlToken>는 단일 값만 받으며 리스트는 받지 않습니다. 값 집합에 대한 포함 여부를 확인하려면 <FlyqlToken type="operator">in</FlyqlToken>을 사용하세요.
