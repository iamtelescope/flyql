Bienvenido a la documentación del proyecto **FlyQL**. Versión actual: **{flyqlVersion}** ([registro de cambios](/es/changelog)).

FlyQL es un lenguaje de consultas ligero y legible diseñado para filtrar datos. Transpila a SQL parametrizado y resistente a inyecciones para **ClickHouse**, **PostgreSQL** y **StarRocks** — con implementaciones completas en **Go**, **Python** y **JavaScript**. Escribe una consulta una vez, y FlyQL genera SQL específico del dialecto para tu base de datos — o la evalúa en memoria contra registros de datos.

<FlyqlCode code={landingIntroQuery} />

Esta consulta funciona igual tanto si tu backend usa ClickHouse, PostgreSQL, StarRocks o un conjunto de datos en memoria.

## Características principales

- **Resistente a inyecciones por diseño**: la gramática no puede expresar subconsultas, `JOIN`, `UNION` ni DDL. La seguridad es estructural, no defensiva. Todo el SQL generado usa valores parametrizados.
- **SQL para varios dialectos**: un solo lenguaje de consultas con salida específica por dialecto. Genera SQL correcto para ClickHouse, PostgreSQL y StarRocks con el manejo de tipos adecuado para JSON, Map y Array.
- **Multilenguaje**: implementaciones completas en Go, Python y JavaScript. Puedes parsear en el frontend, validar en el backend y generar SQL en cualquier parte: misma consulta, mismos resultados.
- **Sin dependencias**: parser escrito a mano como máquina de estados, sin dependencias externas en ningún lenguaje. Es fácil de integrar y no introduce conflictos.
