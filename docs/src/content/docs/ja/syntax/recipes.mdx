## `event_type` が `'login'` または `'logout'` のレコード

同じカラムに対する 2 つの等価チェックを <FlyqlToken type="operator">or</FlyqlToken> で結合します。<FlyqlCode code={`'login'`} inline /> と <FlyqlCode code={`'logout'`} inline /> を囲む**シングルクォートは必須**です。クォートが無いと FlyQL は <FlyqlCode code={`login`} inline /> をカラム参照として扱い、文字列リテラルとは見なしません。

<FlyqlCode code={`event_type = 'login' or event_type = 'logout'`} />

複数値のリストの場合は <FlyqlToken type="operator">in</FlyqlToken> 形式の方が好ましいです: <FlyqlCode code={`event_type in ['login', 'logout']`} inline />。

## IN リストの型整合性

parser は <FlyqlCode code={`[200, 'ok', 404]`} inline /> のような異種型リストを **常に** 受け付けます。SQL ジェネレータが型整合性チェックを実行するのは (a) カラムに型が宣言されており、(b) key が分割されておらず、(c) リストが同質（`len(set(values_types)) == 1`）の場合 **のみ** です。異種型リスト、分割 key、型のないカラムはすべて呼び出し側でチェックをバイパスします。

このレシピは型付きカラムに対して 3 つのケースを示します: 同質マッチ、異種型バイパス、そして同質だが型が誤っているケース（標準エラー `type mismatch in IN list: ...` をスローする）。エラー文字列は Python・Go・JavaScript で同一です。
