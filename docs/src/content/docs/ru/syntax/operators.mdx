FlyQL поддерживает следующие операторы сравнения:

## Равенство

| Оператор | Название | Пример |
|----------|----------|--------|
| `=` | равно | `status = 200` |
| `!=` | не равно | `env != prod` |

```
status = 200
service != "api-gateway"
```

Обе стороны оператора обязательны — ключ и значение.

## Числовые сравнения

| Оператор | Название | Пример |
|----------|----------|--------|
| `>` | больше | `status > 399` |
| `>=` | больше или равно | `status >= 400` |
| `<` | меньше | `duration < 1000` |
| `<=` | меньше или равно | `duration <= 500` |

```
status >= 400 and duration < 5000
response_time > 100 and response_time <= 2000
```

Операторы числового сравнения работают с числовыми значениями. При использовании со строками применяется лексикографическое сравнение.

## Регулярные выражения

| Оператор | Название | Пример |
|----------|----------|--------|
| `~` | совпадение по регулярному выражению | `message ~ "error.*"` |
| `!~` | несовпадение по регулярному выражению | `path !~ "^/health"` |

```
message ~ "timeout|connection refused"
host !~ "^test-.*"
```

Синтаксис регулярных выражений определяется целевой базой данных или средой выполнения.

## Проверка принадлежности списку

| Оператор | Название | Пример |
|----------|----------|--------|
| `in` | входит в список | `status in [200, 201]` |
| `not in` | не входит в список | `env not in ['dev', 'test']` |

Подробности — в разделе [Списки](/ru/syntax/lists/).

## Проверка на вхождение

| Оператор | Название | Пример |
|----------|----------|--------|
| `has` | содержит | `message has 'error'` |
| `not has` | не содержит | `message not has 'test'` |

```
message has 'timeout'
tags not has 'deprecated'
```

Оператор `has` выполняет проверку вхождения в зависимости от типа:

- **String** — поиск подстроки: `message has 'error'` истинно, если `message` содержит `"error"`
- **Map** — наличие ключа: `metadata has 'region'` истинно, если в карте есть ключ `"region"`
- **Array/List** — наличие элемента: `tags has 'web'` истинно, если массив содержит `"web"`
- **JSON** — наличие ключа: `data has 'name'` истинно, если JSON-объект имеет ключ `"name"`
- **Null/пусто** — всегда false для `has`, всегда true для `not has`

`has` принимает одно значение (не список). Для проверки принадлежности к множеству значений используйте `in`.

## Сводка операторов

| Оператор | Название | Пример |
|----------|----------|--------|
| `=` | равно | `status = 200` |
| `!=` | не равно | `env != prod` |
| `>` | больше | `count > 10` |
| `>=` | больше или равно | `count >= 10` |
| `<` | меньше | `count < 100` |
| `<=` | меньше или равно | `count <= 100` |
| `~` | совпадение по регулярному выражению | `msg ~ "err.*"` |
| `!~` | несовпадение по регулярному выражению | `msg !~ "^ok"` |
| `in` | в списке | `s in [1, 2]` |
| `not in` | не в списке | `s not in [3]` |
| `has` | содержит | `msg has 'err'` |
| `not has` | не содержит | `msg not has 'ok'` |
