## Correspondência Regex

Use <FlyqlToken type="operator">{'~'}</FlyqlToken> para verificar se um valor corresponde a uma expressão regular, e <FlyqlToken type="operator">{'!~'}</FlyqlToken> para negar a correspondência:

<FlyqlCode code={`message ~ "error.*timeout"`} />
<FlyqlCode code={`path !~ "^/health"`} />

| Operador | Significado |
|----------|-------------|
| <FlyqlToken type="operator">{'~'}</FlyqlToken> | O valor corresponde ao padrão regex |
| <FlyqlToken type="operator">{'!~'}</FlyqlToken> | O valor não corresponde ao padrão regex |

### Sintaxe do Padrão Regex

A sintaxe regex depende do backend de destino:

- **ClickHouse** — usa a função `match()` (sintaxe RE2)
- **PostgreSQL** — usa os operadores <FlyqlToken type="operator">{'~'}</FlyqlToken> e <FlyqlToken type="operator">{'!~'}</FlyqlToken> (regex POSIX)
- **StarRocks** — usa a função `regexp`
- **Matcher em memória** — usa o motor de regex da linguagem hospedeira (Go `regexp`, Python `re2`, JS `RegExp`)

Coloque o padrão entre aspas quando ele contiver espaços ou caracteres especiais:

<FlyqlCode code={`message ~ "connection (refused|reset)"`} />
<FlyqlCode code={`host ~ "^prod-[0-9]+"`} />

### Flags de Regex

Você pode modificar o comportamento do regex usando flags inline no início do padrão com a sintaxe `(?...)`. O FlyQL suporta três flags que funcionam em todos os backends:

#### `(?i)` — Insensível a Maiúsculas/Minúsculas

Faz o padrão corresponder independentemente do caso das letras:

<FlyqlCode code={`message ~ "(?i)error"`} />
<FlyqlCode code={`level ~ "(?i)^(warn|error|fatal)$"`} />

`(?i)error` corresponde a "Error", "ERROR", "error", etc.

#### `(?m)` — Multilinha

Altera `^` e `$` para corresponder ao início e fim de cada **linha** em vez da string inteira:

<FlyqlCode code={`log ~ "(?m)^ERROR"`} />

Sem `(?m)`, `^ERROR` só corresponde se a string inteira começar com "ERROR". Com `(?m)`, também corresponde a "INFO first\nERROR second" porque "ERROR" aparece no início de uma linha.

#### `(?s)` — Dotall

Faz `.` corresponder a **qualquer** caractere incluindo quebras de linha. Por padrão, `.` não corresponde a `\n`:

<FlyqlCode code={`log ~ "(?s)start.*end"`} />

Sem `(?s)`, `start.*end` não corresponde a "start\nend" porque `.` não atravessa a quebra de linha. Com `(?s)`, corresponde.

#### Combinando Flags

As flags podem ser combinadas em um único grupo:

<FlyqlCode code={`log ~ "(?im)^ERROR"`} />
<FlyqlCode code={`log ~ "(?ims)^START.*END$"`} />

#### Suporte por Backend

Todas as três flags funcionam em todos os backends do FlyQL:

| Backend | `(?i)` | `(?m)` | `(?s)` |
|---------|--------|--------|--------|
| **ClickHouse** | Sim | Sim | Sim |
| **PostgreSQL** | Sim | Sim | Sim |
| **StarRocks** | Sim | Sim | Sim |
| **Go matcher** | Sim | Sim | Sim |
| **Python matcher** | Sim | Sim | Sim |
| **JavaScript matcher** | Sim | Sim | Sim |

Combine correspondência insensível a maiúsculas/minúsculas com negação para excluir padrões independentemente do caso:

<FlyqlCode code={`message ~ "(?i)error" and message !~ "(?i)warning"`} />

## Correspondência LIKE

Use <FlyqlToken type="operator">like</FlyqlToken> e <FlyqlToken type="operator">ilike</FlyqlToken> para correspondência de padrões estilo SQL com curingas:

<FlyqlCode code={`host like 'prod%'`} />
<FlyqlCode code={`service like 'api-%'`} />
<FlyqlCode code={`message ilike '%Error%'`} />

| Operador | Significado |
|----------|-------------|
| <FlyqlToken type="operator">like</FlyqlToken> | O valor corresponde ao padrão (sensível a maiúsculas) |
| <FlyqlToken type="operator">not like</FlyqlToken> | O valor não corresponde ao padrão |
| <FlyqlToken type="operator">ilike</FlyqlToken> | O valor corresponde ao padrão (insensível a maiúsculas) |
| <FlyqlToken type="operator">not ilike</FlyqlToken> | O valor não corresponde ao padrão (insensível a maiúsculas) |

### Curingas

Dois caracteres curinga estão disponíveis nos padrões <FlyqlToken type="operator">like</FlyqlToken> / <FlyqlToken type="operator">ilike</FlyqlToken>:

| Curinga | Significado | Exemplo | Corresponde a |
|---------|-------------|---------|---------------|
| `%` | Qualquer sequência de caracteres (incluindo vazio) | <FlyqlCode code={`host like 'prod%'`} inline /> | `prod`, `prod-1`, `production` |
| `_` | Exatamente um caractere | <FlyqlCode code={`code like 'E__'`} inline /> | `E01`, `Exx`, mas não `E1` ou `Error` |

Os curingas podem aparecer em qualquer posição do padrão — início, meio, final ou combinados:

<FlyqlCode code={`message like '%timeout%'`} />
<FlyqlCode code={`host like 'web-_-prod'`} />
<FlyqlCode code={`path like '/api/%/status'`} />

### Escapando Curingas

Para corresponder a um `%` ou `_` literal, escape com uma barra invertida:

<FlyqlCode code={`value like '100\\%'`} />
<FlyqlCode code={`col like 'foo\\_bar'`} />

`100\%` corresponde à string literal "100%", não a "100" seguido de qualquer coisa.

### Correspondência Insensível a Maiúsculas

Use <FlyqlToken type="operator">ilike</FlyqlToken> para correspondência de padrões insensível a maiúsculas:

<FlyqlCode code={`message ilike '%error%'`} />
<FlyqlCode code={`host not ilike 'TEST%'`} />

### LIKE vs. Regex

| Caso de uso | Sintaxe | Exemplo |
|-------------|---------|---------|
| Correspondência por prefixo | <FlyqlCode code={`key like 'prefix%'`} inline /> | <FlyqlCode code={`host like 'prod%'`} inline /> |
| Contém | <FlyqlCode code={`key like '%word%'`} inline /> | <FlyqlCode code={`msg like '%error%'`} inline /> |
| Padrão complexo | <FlyqlCode code={`key ~ "pattern"`} inline /> | <FlyqlCode code={`host ~ "^prod-[0-9]+"`} inline /> |

Use <FlyqlToken type="operator">like</FlyqlToken> para correspondência simples com curingas. Use regex (<FlyqlToken type="operator">{'~'}</FlyqlToken>) quando precisar de classes de caracteres, alternação ou outros padrões avançados.

### Suporte por Backend

Todos os backends SQL (ClickHouse, PostgreSQL, StarRocks) emitem operadores SQL nativos `LIKE` / `ILIKE` / `NOT LIKE` / `NOT ILIKE`. O matcher em memória converte o padrão em regex internamente.

:::note
StarRocks `ILIKE` requer versão 2.5 ou posterior.
:::

## Exemplos

<FlyqlCode code={`message ~ "error|fail"`} />
<FlyqlCode code={`path ~ "^/api/v2"`} />
<FlyqlCode code={`path !~ "^/health"`} />
<FlyqlCode code={`host like 'prod%'`} />
<FlyqlCode code={`host like 'prod%' and message ~ "timeout.*" and path !~ "^/health"`} />
