## Operadores booleanos

FlyQL usa tres operadores booleanos para combinar condiciones:

| Operador | Significado | Ejemplo |
|----------|-------------|---------|
| <FlyqlToken type="operator">and</FlyqlToken> | Ambas condiciones deben ser true | <FlyqlCode code={`status = 200 and active`} inline /> |
| <FlyqlToken type="operator">or</FlyqlToken> | Cualquiera de las dos condiciones puede ser true | <FlyqlCode code={`level = 'error' or level = 'critical'`} inline /> |
| <FlyqlToken type="operator">not</FlyqlToken> | Niega una expresión | <FlyqlCode code={`not archived`} inline /> |

### Negación combinada

<FlyqlToken type="operator">not</FlyqlToken> puede combinarse con <FlyqlToken type="operator">and</FlyqlToken> y <FlyqlToken type="operator">or</FlyqlToken>:

<FlyqlCode code={`active and not archived`} />
<FlyqlCode code={`level = 'error' or not healthy`} />

## Precedencia de operadores

<FlyqlToken type="operator">and</FlyqlToken> tiene mayor precedencia que <FlyqlToken type="operator">or</FlyqlToken>. Esto significa que:

<FlyqlCode code={`a = 1 or b = 2 and c = 3`} />

se parsea como:

<FlyqlCode code={`a = 1 or (b = 2 and c = 3)`} />

Para cambiar la precedencia, usa paréntesis de forma explícita.

## Agrupación con paréntesis

Usa paréntesis para controlar el orden de evaluación:

<FlyqlCode code={`(status >= 400 or level = 'error') and service = 'api'`} />

Sin paréntesis, <FlyqlToken type="operator">and</FlyqlToken> tendría mayor precedencia:

<FlyqlCode code={`status >= 400 or level = 'error' and service = 'api'`} />

es equivalente a:

<FlyqlCode code={`status >= 400 or (level = 'error' and service = 'api')`} />

### Agrupación anidada

Los paréntesis pueden anidarse:

<FlyqlCode code={`(a = 1 or b = 2) and not (c = 3 and d = 4)`} />

### Negar grupos

<FlyqlToken type="operator">not</FlyqlToken> se aplica a todo el grupo:

<FlyqlCode code={`not (status = 200 and healthy)`} />

Esto niega la condición combinada — solo es true cuando <FlyqlToken type="key">status</FlyqlToken> no es 200 o <FlyqlToken type="key">healthy</FlyqlToken> es false.

## Negación

<FlyqlToken type="operator">not</FlyqlToken> niega cualquier expresión — una comprobación truthy, una comparación o un grupo:

<FlyqlCode code={`not active`} />
<FlyqlCode code={`not status = 200`} />
<FlyqlCode code={`not (a = 1 and b = 2)`} />
<FlyqlCode code={`active and not archived`} />

La doble negación se cancela: <FlyqlCode code={`not not active`} inline /> es equivalente a <FlyqlCode code={`active`} inline />.

## Ejemplos

<FlyqlCode code={`status = 200 and service = 'api'`} />
<FlyqlCode code={`level = 'error' or level = 'critical'`} />
<FlyqlCode code={`status >= 400 or level = 'error' and service = 'api'`} />
<FlyqlCode code={`(status >= 400 or level = 'error') and service = 'api'`} />
<FlyqlCode code={`(status >= 500 and host = 'prod') or (level = 'error' and not service = 'healthcheck')`} />
