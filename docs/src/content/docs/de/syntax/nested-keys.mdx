## Punktnotation

Verwende punktgetrennte Pfade, um auf verschachtelte Felder in JSON, Map oder strukturierten Daten zuzugreifen:

<FlyqlCode code={`request.url = "/api/users"`} />
<FlyqlCode code={`response.headers.content_type = "application/json"`} />
<FlyqlCode code={`metadata.labels.env = 'prod'`} />

Jedes punktgetrennte Segment traversiert eine Ebene in der verschachtelten Struktur. Die SQL-Generatoren übersetzen diese Pfade in die korrekte Syntax für jeden Datenbankdialekt (JSON-Pfad-Zugriff, Map-Lookups usw.).

## Schlüssel-Segmente in Anführungszeichen

Wenn ein Schlüssel-Segment selbst einen Punkt enthält, setze es in Anführungszeichen, damit der Punkt nicht als Trenner behandelt wird:

<FlyqlCode code={`"foo.bar".baz = value`} />

Hier gilt:
- `"foo.bar"` ist ein einzelnes Schlüssel-Segment (der Punkt ist wörtlich)
- <FlyqlToken type="key">baz</FlyqlToken> ist ein Kind von `"foo.bar"`

Ohne Anführungszeichen wäre <FlyqlToken type="key">foo.bar.baz</FlyqlToken> in drei Segmente aufgeteilt: <FlyqlToken type="key">foo</FlyqlToken> → <FlyqlToken type="key">bar</FlyqlToken> → <FlyqlToken type="key">baz</FlyqlToken>.

### Regeln für Anführungszeichen

Für Schlüssel-Segmente funktionieren sowohl einfache als auch doppelte Anführungszeichen:

<FlyqlCode code={`"foo.bar".baz = value`} />
<FlyqlCode code={`'foo.bar'.baz = value`} />

Nur die Segmente mit Punkten müssen in Anführungszeichen stehen:

<FlyqlCode code={`metadata."dotted.key".value = 123`} />

Bezeichner in Anführungszeichen ermöglichen es auch, [reservierte Wörter](/de/syntax/reserved-words/) als Spaltennamen zu verwenden — zum Beispiel <FlyqlCode code={`"not" = 1`} inline />.

## Pfadtiefe

Die Verschachtelungstiefe ist unbegrenzt:

<FlyqlCode code={`a.b.c.d.e = value`} />

Jedes Segment wird in der angegebenen Reihenfolge traversiert, wenn SQL generiert oder im Speicher ausgewertet wird.

## Kombination mit Operatoren

Verschachtelte Schlüssel funktionieren mit allen Operatoren:

<FlyqlCode code={`request.status >= 400`} />
<FlyqlCode code={`response.body ~ "error.*"`} />
<FlyqlCode code={`metadata.tags in ['prod', 'staging']`} />
<FlyqlCode code={`request.headers.content_type != "text/html"`} />
<FlyqlCode code={`not request.authenticated`} />

## Zugriff auf Array-Elemente

Für Spalten vom Typ Array verwenden Sie nullbasierte numerische Pfadsegmente, um auf Elemente zuzugreifen:

<FlyqlCode code={`tags.0 = 'important'`} />
<FlyqlCode code={`tags.1 = 'second'`} />

FlyQL-Array-Indizes sind **0-basiert in allen SQL-Dialekten und im In-Memory-Matcher**. Die Generatoren übersetzen den flyql-Index in die native SQL-Konvention jedes Ziels — Arrays in ClickHouse, StarRocks und PostgreSQL sind auf der SQL-Ebene 1-basiert, sodass flyql `tags.0` in jedem Dialekt als `tags[1]` ausgegeben wird (mit dialektspezifischer Bezeichner-Quotierung und Funktionsform). Das bedeutet, dass derselbe flyql-Ausdruck pro Dialekt unterschiedliche SQL-Strings erzeugt, jedoch gegen dieselben zugrunde liegenden Daten zum **gleichen Ergebnis** ausgewertet wird.

## Beispiele
