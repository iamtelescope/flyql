## JSON Columns

ClickHouse는 두 가지 JSON column 타입을 지원하며, 생성되는 SQL도 서로 다릅니다.

### Native JSON Type

`JSON`으로 선언된 column은 세그먼트를 backtick으로 감싼 dot 표기를 사용합니다.

<SqlExamplesTable rows={jsonNative} />

### JSON String Type

문자열로 JSON을 저장하는 column(예: JSON 내용을 담는 `String` 타입)은 타입 검사를 위해 `multiIf`로 감싼 `JSONExtract` 함수를 사용합니다.

<SqlExamplesTable rows={jsonString} />

FlyQL의 배열 인덱스는 0 기반이며, ClickHouse 배열은 SQL 계층에서 1 기반이므로 ClickHouse 생성기는 SQL을 생성할 때 인덱스에 1을 더합니다.

### Column Setup

{/* TODO: native-speaker review */}
JSON 내용을 가진 text/varchar 컬럼은 합성 원시 타입 `jsonstring`으로 선언합니다.

```go
// Go
columns := map[string]*clickhouse.Column{
    "message": clickhouse.NewColumn(clickhouse.ColumnDef{Name: "message", Type: "String"}),
    "data":    clickhouse.NewColumn(clickhouse.ColumnDef{Name: "data", Type: "JSON"}),      // native JSON
    "log":     clickhouse.NewColumn(clickhouse.ColumnDef{Name: "log", Type: "jsonstring"}),       // JSON string
}
```

```javascript
// JavaScript
const columns = {
    message: newColumn({ name: "message", type: "String" }),
    data:    newColumn({ name: "data", type: "JSON" }),      // native JSON
    log:     newColumn({ name: "log", type: "jsonstring" }),       // JSON string
}
```
