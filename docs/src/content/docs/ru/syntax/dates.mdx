В FlyQL есть два способа сравнивать даты и значения дат-времени: **строковые литералы** для абсолютных дат и **временные функции** для относительных временных окон. Выбирайте тот, что соответствует смыслу запроса.

## Дата как строковый литерал

Сравните колонку с датой со строковым литералом в формате ISO-8601. ClickHouse, PostgreSQL и StarRocks приводят строки ISO-8601 к значениям `DATE` / `DATETIME` на стороне базы данных, поэтому никакой специальный конструктор даты на уровне FlyQL не нужен.

<FlyqlCode code={`due_date < '2023-12-31'`} />
<FlyqlCode code={`created_at >= '2023-12-31T00:00:00Z'`} />
<FlyqlCode code={`event_time >= '2023-01-01' and event_time <= '2023-12-31'`} />

| Тип колонки | Принимает | Сгенерированное сравнение |
|-------------|-----------|---------------------------|
| ClickHouse `Date` / `DateTime` / `DateTime64` | Строка ISO-8601 | Прямое сравнение — ClickHouse разбирает литерал во время выполнения |
| PostgreSQL `date` / `timestamp` / `timestamptz` | Строка ISO-8601 | Прямое сравнение — PostgreSQL приводит литерал во время выполнения |
| StarRocks `DATE` / `DATETIME` | Строка ISO-8601 | Прямое сравнение — StarRocks разбирает литерал во время выполнения |

Время суток, смещение часового пояса и форма «только дата» — всё поддерживается, потому что литерал разбирает сама база данных.

:::note[Строки должны быть в кавычках]
Литералы дат — это строки, и их **обязательно** заключать в кавычки. Без кавычек FlyQL интерпретирует значение как ссылку на колонку. <FlyqlCode code={`due_date < 2023-12-31`} inline /> разобрался бы как арифметика над колонкой с именем `2023`, а не как литерал даты. См. [Значения и типы — String](/ru/syntax/values/#string).
:::

## Временные функции

Для относительных запросов о времени — «за последний час», «с начала недели», «с сегодня» — FlyQL предоставляет встроенные временные функции: <FlyqlCode code={`ago(...)`} inline />, <FlyqlCode code={`now()`} inline />, <FlyqlCode code={`today()`} inline /> и <FlyqlCode code={`startOf(...)`} inline />.

<FlyqlCode code={`created_at > ago(1h)`} />
<FlyqlCode code={`updated_at >= ago(7d)`} />
<FlyqlCode code={`expires_at < now()`} />
<FlyqlCode code={`event_time > startOf('day')`} />
<FlyqlCode code={`scheduled_for >= today('Europe/Berlin')`} />

Полная грамматика (единицы, правила порядка, аргументы часового пояса) описана в разделе [Значения и типы — Временные функции](/ru/syntax/values/#временные-функции).

## Когда что использовать

| Намерение | Используйте | Пример |
|-----------|-------------|--------|
| Сравнить с фиксированной календарной датой или меткой времени | Строковый литерал | <FlyqlCode code={`due_date < '2023-12-31'`} inline /> |
| Сравнить со скользящим временным окном | <FlyqlCode code={`ago()`} inline /> | <FlyqlCode code={`created_at > ago(1h)`} inline /> |
| Сравнить с текущим моментом | <FlyqlCode code={`now()`} inline /> | <FlyqlCode code={`expires_at < now()`} inline /> |
| Сравнить с сегодняшним днём (с часовым поясом) | <FlyqlCode code={`today()`} inline /> | <FlyqlCode code={`scheduled_for = today('UTC')`} inline /> |
| Сравнить с началом дня / недели / месяца | <FlyqlCode code={`startOf()`} inline /> | <FlyqlCode code={`event_time > startOf('week')`} inline /> |

Строковые литералы остаются неизменными — одна и та же строка запроса каждый день возвращает один и тот же SQL. Временные функции вычисляются во время выполнения, поэтому один и тот же запрос FlyQL каждый раз даёт другое временное окно.
