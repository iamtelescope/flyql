## Correspondencia Regex

Use <FlyqlToken type="operator">{'~'}</FlyqlToken> para verificar se um valor corresponde a uma expressao regular, e <FlyqlToken type="operator">{'!~'}</FlyqlToken> para negar a correspondencia:

<FlyqlCode code={`message ~ "error.*timeout"`} />
<FlyqlCode code={`path !~ "^/health"`} />

| Operador | Significado |
|----------|-------------|
| <FlyqlToken type="operator">{'~'}</FlyqlToken> | O valor corresponde ao padrao regex |
| <FlyqlToken type="operator">{'!~'}</FlyqlToken> | O valor nao corresponde ao padrao regex |

### Sintaxe de Padroes Regex

A sintaxe regex depende do backend de destino:

- **ClickHouse** — utiliza a funcao `match()` (sintaxe RE2)
- **PostgreSQL** — utiliza os operadores <FlyqlToken type="operator">{'~'}</FlyqlToken> e <FlyqlToken type="operator">{'!~'}</FlyqlToken> (regex POSIX)
- **StarRocks** — utiliza a funcao `regexp`
- **Matcher em memoria** — utiliza o motor de regex da linguagem anfitria (Go `regexp`, Python `re2`, JS `RegExp`)

Coloque o padrao entre aspas quando contiver espacos ou caracteres especiais:

<FlyqlCode code={`message ~ "connection (refused|reset)"`} />
<FlyqlCode code={`host ~ "^prod-[0-9]+"`} />

### Flags Regex

Pode modificar o comportamento da regex utilizando flags inline no inicio do padrao com a sintaxe `(?...)`. O FlyQL suporta tres flags que funcionam em todos os backends:

#### `(?i)` — Insensivel a Maiusculas/Minusculas

Faz com que o padrao corresponda independentemente das maiusculas e minusculas:

<FlyqlCode code={`message ~ "(?i)error"`} />
<FlyqlCode code={`level ~ "(?i)^(warn|error|fatal)$"`} />

`(?i)error` corresponde a "Error", "ERROR", "error", etc.

#### `(?m)` — Multilinha

Altera `^` e `$` para corresponder ao inicio e fim de cada **linha** em vez da string inteira:

<FlyqlCode code={`log ~ "(?m)^ERROR"`} />

Sem `(?m)`, `^ERROR` so corresponde se a string inteira comecar com "ERROR". Com `(?m)`, tambem corresponde a "INFO first\nERROR second" porque "ERROR" aparece no inicio de uma linha.

#### `(?s)` — Dotall

Faz com que `.` corresponda a **qualquer** caractere, incluindo quebras de linha. Por defeito, `.` nao corresponde a `\n`:

<FlyqlCode code={`log ~ "(?s)start.*end"`} />

Sem `(?s)`, `start.*end` nao corresponde a "start\nend" porque `.` nao atravessa a quebra de linha. Com `(?s)`, corresponde.

#### Combinar Flags

As flags podem ser combinadas num unico grupo:

<FlyqlCode code={`log ~ "(?im)^ERROR"`} />
<FlyqlCode code={`log ~ "(?ims)^START.*END$"`} />

#### Suporte por Backend

As tres flags funcionam em todos os backends do FlyQL:

| Backend | `(?i)` | `(?m)` | `(?s)` |
|---------|--------|--------|--------|
| **ClickHouse** | Sim | Sim | Sim |
| **PostgreSQL** | Sim | Sim | Sim |
| **StarRocks** | Sim | Sim | Sim |
| **Go matcher** | Sim | Sim | Sim |
| **Python matcher** | Sim | Sim | Sim |
| **JavaScript matcher** | Sim | Sim | Sim |

Combine correspondencia insensivel a maiusculas/minusculas com negacao para excluir padroes independentemente das maiusculas:

<FlyqlCode code={`message ~ "(?i)error" and message !~ "(?i)warning"`} />

## Correspondencia LIKE

Use <FlyqlToken type="operator">like</FlyqlToken> e <FlyqlToken type="operator">ilike</FlyqlToken> para correspondencia de padroes estilo SQL com wildcards:

<FlyqlCode code={`host like 'prod%'`} />
<FlyqlCode code={`service like 'api-%'`} />
<FlyqlCode code={`message ilike '%Error%'`} />

| Operador | Significado |
|----------|-------------|
| <FlyqlToken type="operator">like</FlyqlToken> | O valor corresponde ao padrao (sensivel a maiusculas) |
| <FlyqlToken type="operator">not like</FlyqlToken> | O valor nao corresponde ao padrao |
| <FlyqlToken type="operator">ilike</FlyqlToken> | O valor corresponde ao padrao (insensivel a maiusculas) |
| <FlyqlToken type="operator">not ilike</FlyqlToken> | O valor nao corresponde ao padrao (insensivel a maiusculas) |

### Wildcards

Dois caracteres wildcard estao disponiveis nos padroes <FlyqlToken type="operator">like</FlyqlToken> / <FlyqlToken type="operator">ilike</FlyqlToken>:

| Wildcard | Significado | Exemplo | Corresponde a |
|----------|-------------|---------|---------------|
| `%` | Qualquer sequencia de caracteres (incluindo vazio) | <FlyqlCode code={`host like 'prod%'`} inline /> | `prod`, `prod-1`, `production` |
| `_` | Exactamente um caractere | <FlyqlCode code={`code like 'E__'`} inline /> | `E01`, `Exx`, mas nao `E1` ou `Error` |

Os wildcards podem aparecer em qualquer posicao do padrao — inicio, meio, fim ou combinados:

<FlyqlCode code={`message like '%timeout%'`} />
<FlyqlCode code={`host like 'web-_-prod'`} />
<FlyqlCode code={`path like '/api/%/status'`} />

### Escapar Wildcards

Para corresponder a um `%` ou `_` literal, escape com uma barra invertida:

<FlyqlCode code={`value like '100\\%'`} />
<FlyqlCode code={`col like 'foo\\_bar'`} />

`100\%` corresponde a cadeia literal "100%", nao a "100" seguido de qualquer coisa.

### Correspondencia Insensivel a Maiusculas

Use <FlyqlToken type="operator">ilike</FlyqlToken> para correspondencia de padroes insensivel a maiusculas:

<FlyqlCode code={`message ilike '%error%'`} />
<FlyqlCode code={`host not ilike 'TEST%'`} />

### LIKE vs. Regex

| Caso de uso | Sintaxe | Exemplo |
|-------------|---------|---------|
| Correspondencia por prefixo | <FlyqlCode code={`key like 'prefix%'`} inline /> | <FlyqlCode code={`host like 'prod%'`} inline /> |
| Contem | <FlyqlCode code={`key like '%word%'`} inline /> | <FlyqlCode code={`msg like '%error%'`} inline /> |
| Padrao complexo | <FlyqlCode code={`key ~ "pattern"`} inline /> | <FlyqlCode code={`host ~ "^prod-[0-9]+"`} inline /> |

Use <FlyqlToken type="operator">like</FlyqlToken> para correspondencia simples com wildcards. Use regex (<FlyqlToken type="operator">{'~'}</FlyqlToken>) quando necessitar de classes de caracteres, alternacao ou outros padroes avancados.

### Suporte por Backend

Todos os backends SQL (ClickHouse, PostgreSQL, StarRocks) emitem operadores SQL nativos `LIKE` / `ILIKE` / `NOT LIKE` / `NOT ILIKE`. O matcher em memoria converte o padrao em regex internamente.

:::note
StarRocks `ILIKE` requer a versao 2.5 ou posterior.
:::

## Exemplos

<FlyqlCode code={`message ~ "error|fail"`} />
<FlyqlCode code={`path ~ "^/api/v2"`} />
<FlyqlCode code={`path !~ "^/health"`} />
<FlyqlCode code={`host like 'prod%'`} />
<FlyqlCode code={`host like 'prod%' and message ~ "timeout.*" and path !~ "^/health"`} />
