O FlyQL suporta 7 tipos de valores: **Integer**, **BigInt**, **Float**, **String**, **Boolean**, **Null** e **Array**. Cada valor no AST carrega um campo `value_type` explícito.

## Tipos de Valores

### Integer

Números inteiros dentro do intervalo int64:

<FlyqlCode code={`count = 42`} />
<FlyqlCode code={`status != 200`} />
<FlyqlCode code={`age >= 18`} />

AST: `value_type: "integer"`

### BigInt

Inteiros que excedem o intervalo int64 (Go/Python) ou `MAX_SAFE_INTEGER` (JavaScript):

<FlyqlCode code={`id = 18446744073709551615`} />

AST: `value_type: "bigint"`

### Float

Números com ponto decimal:

<FlyqlCode code={`price > 9.99`} />
<FlyqlCode code={`ratio = 3.14`} />
<FlyqlCode code={`temperature<=-0.5`} />

AST: `value_type: "float"`

### String

Valores de texto, com ou sem aspas:

<FlyqlCode code={`level = error`} />
<FlyqlCode code={`user = "John Doe"`} />
<FlyqlCode code={`message = 'hello world'`} />
<FlyqlCode code={`count = '42'`} />

AST: `value_type: "string"`

Escape de aspas com barra invertida:

```
user = 'John
message="said \"hello\""
```

### Boolean

As palavras-chave <FlyqlToken type="boolean">true</FlyqlToken> e <FlyqlToken type="boolean">false</FlyqlToken> (minúsculas, sensível a maiúsculas/minúsculas):

<FlyqlCode code={`active = true`} />
<FlyqlCode code={`debug = false`} />
<FlyqlCode code={`active != true`} />

AST: `value_type: "boolean"`, o valor é um booleano nativo.

Variantes com inicial maiúscula (`True`, `FALSE`) são tratadas como strings.

**Geração de SQL:**
| Dialeto | FlyQL | SQL Gerado |
|---------|-------|------------|
| ClickHouse | <FlyqlCode code={`field = true`} inline /> | `field = true` |
| ClickHouse | <FlyqlCode code={`field = false`} inline /> | `field = false` |
| PostgreSQL | <FlyqlCode code={`field = true`} inline /> | `field = TRUE` |
| PostgreSQL | <FlyqlCode code={`field = false`} inline /> | `field = FALSE` |
| StarRocks | <FlyqlCode code={`field = true`} inline /> | `field = true` |
| StarRocks | <FlyqlCode code={`field = false`} inline /> | `field = false` |

### Null

A palavra-chave <FlyqlToken type="null">null</FlyqlToken> (minúscula, sensível a maiúsculas/minúsculas):

<FlyqlCode code={`field = null`} />
<FlyqlCode code={`field != null`} />

AST: `value_type: "null"`, o valor é null nativo.

Variantes com inicial maiúscula (`Null`, `NULL`) são tratadas como strings.

**Restrições:** Apenas os operadores <FlyqlToken type="operator">=</FlyqlToken> e <FlyqlToken type="operator">!=</FlyqlToken> são válidos com null. Usar <FlyqlToken type="operator">{'>'}</FlyqlToken>, <FlyqlToken type="operator">{'<'}</FlyqlToken>, <FlyqlToken type="operator">{'>='}</FlyqlToken>, <FlyqlToken type="operator">{'<='}</FlyqlToken>, <FlyqlToken type="operator">{'~'}</FlyqlToken> ou <FlyqlToken type="operator">{'!~'}</FlyqlToken> com null produz um erro de análise.

**Geração de SQL:** <FlyqlCode code={`field = null`} inline /> gera `field IS NULL`, <FlyqlCode code={`field != null`} inline /> gera `field IS NOT NULL` (todos os dialetos).

### Array

Arrays são usados com os operadores <FlyqlToken type="operator">in</FlyqlToken> e <FlyqlToken type="operator">not in</FlyqlToken>. Os elementos podem ser de tipos mistos (heterogêneos):

<FlyqlCode code={`status in [200, 201, 202]`} />
<FlyqlCode code={`name in ['alice', 'bob']`} />
<FlyqlCode code={`field in [1, "hello", true, null]`} />

Cada elemento carrega seu próprio tipo no array `values_types`.

## Expressões Truthy/Falsy

Uma chave isolada sem operador verifica se o campo tem um valor truthy:

<FlyqlCode code={`active`} />
<FlyqlCode code={`message and status`} />

Um valor é considerado **falsy** se for:
- `null` / `None` / ausente
- String vazia `""`
- Zero `0`
- Booleano <FlyqlToken type="boolean">false</FlyqlToken>

Todo o resto é **truthy**.

Use <FlyqlToken type="operator">not</FlyqlToken> para verificar valores falsy:

<FlyqlCode code={`not archived`} />
<FlyqlCode code={`active and not debug`} />

Verificações truthy/falsy são distintas de comparações com null: <FlyqlCode code={`not field`} inline /> corresponde quando o campo é null, vazio ou zero. <FlyqlCode code={`field = null`} inline /> corresponde apenas quando o campo é especificamente null.

## Espaços em Branco

O FlyQL é tolerante a espaços em branco. Espaços ao redor dos operadores são opcionais:

<FlyqlCode code={`status = 200`} />
<FlyqlCode code={`status =200`} />

Todos os exemplos acima são equivalentes.

## Módulo de Constantes de Tipo

Todas as constantes de tipo estão disponíveis no módulo `flyql.types`:

**Python:**
```python
from flyql.types import ValueType
# ValueType.INTEGER, ValueType.BIGINT, ValueType.FLOAT,
# ValueType.STRING, ValueType.BOOLEAN, ValueType.NULL, ValueType.ARRAY
```

**Go:**
```go
import "github.com/iamtelescope/flyql/golang/types"
// types.Integer, types.BigInt, types.Float,
// types.String, types.Boolean, types.Null, types.Array
```

**JavaScript:**
```javascript
import { ValueType } from 'flyql/types'
// ValueType.INTEGER, ValueType.BIGINT, ValueType.FLOAT,
// ValueType.STRING, ValueType.BOOLEAN, ValueType.NULL, ValueType.ARRAY
```

## Funções Temporais

O FlyQL suporta chamadas de funções temporais como valores no lado direito de expressões de comparação. Elas permitem filtros relativos ao tempo sem a necessidade de codificar timestamps.

AST: `value_type: "function"`, `value` é um objeto `FunctionCall`.

### `ago(duration)`

Retorna um timestamp relativo ao momento atual. Suporta durações compostas.

<FlyqlCode code={`timestamp > ago(1h)`} />
<FlyqlCode code={`created_at >= ago(7d)`} />
<FlyqlCode code={`updated_at > ago(1h30m)`} />

**Unidades de duração:** `s` (segundos), `m` (minutos), `h` (horas), `d` (dias), `w` (semanas)

### `now()`

Retorna o timestamp atual.

<FlyqlCode code={`expires_at < now()`} />

### `today()`

Retorna a data de hoje. Aceita um argumento opcional de fuso horário IANA.

<FlyqlCode code={`date = today()`} />
<FlyqlCode code={`date = today('Europe/Berlin')`} />

### `startOf(unit)`

Retorna o início do dia, semana (segunda-feira) ou mês atual. Aceita um fuso horário opcional.

<FlyqlCode code={`created_at > startOf('day')`} />
<FlyqlCode code={`created_at > startOf('week')`} />
<FlyqlCode code={`created_at > startOf('month', 'Asia/Tokyo')`} />

**Unidades válidas:** `'day'`, `'week'`, `'month'`

### Restrições de Operadores

Funções temporais só podem ser usadas com operadores de comparação: `=`, `!=`, `>`, `>=`, `<`, `<=`.

Usar `~` ou `!~` com uma função temporal produz um erro de análise. Operadores como `in`, `has`, `like` e `ilike` utilizam caminhos de análise separados e não são compatíveis com valores de função.

### Referência de Coluna Sem Parênteses

Nomes de função sem parênteses são tratados como referências de coluna, não como chamadas de função:

<FlyqlCode code={`field = ago`} />

Isso produz `value_type: "column"`, não uma chamada de função.

## Notas de Migração

Se estiver atualizando de uma versão anterior do FlyQL:

- **Quebra de compatibilidade:** `ValueTypeString`/`ValueTypeNumber` do Go foram removidos, substituídos pelo `types.ValueType` de 7 valores
- **Quebra de compatibilidade:** O campo `value_type` do AST mudou de `"string"`/`"number"` para `"integer"`/`"float"`/`"bigint"`/`"string"`/`"boolean"`/`"null"`
- **Quebra de compatibilidade:** A imposição de tipo uniforme em listas foi removida — arrays heterogêneos agora são válidos
- **Novo:** <FlyqlToken type="boolean">true</FlyqlToken>, <FlyqlToken type="boolean">false</FlyqlToken>, <FlyqlToken type="null">null</FlyqlToken> são analisados como literais tipados (não strings)
- **Novo:** O array `values_types` em expressões de lista IN fornece tipos por elemento
- **Novo:** O módulo `flyql.types` exporta constantes de tipo canônicas
