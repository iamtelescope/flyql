---
title: Schema-Konfiguration
description: Struktur des Spaltenschemas für den FlyQL-Editor
---

Die Prop `columns` definiert das Datenschema, das die Vorschläge im Editor steuert. Schlüssel sind Spaltennamen, Werte sind Spaltendefinitionen.

## Grundschema

```javascript
import { ColumnSchema } from 'flyql-vue'

const columns = ColumnSchema.fromPlainObject({
    level: { type: 'enum', suggest: true, autocomplete: true, values: ['debug', 'info', 'error'] },
    service: { type: 'string', suggest: true, autocomplete: true },
    status_code: { type: 'number', suggest: true },
    host: { type: 'string', suggest: true },
})
```

## Felder der Spaltendefinition

| Feld | Typ | Default | Beschreibung |
|------|-----|---------|--------------|
| `type` | `string` | — | Spaltentyp: `'string'`, `'number'`, `'enum'`, `'object'` |
| `suggest` | `boolean` | `true` | Zeigt diese Spalte in Vorschlägen an |
| `autocomplete` | `boolean` | `false` | Aktiviert Wert-Autovervollständigung für diese Spalte |
| `values` | `string[]` oder `number[]` | — | Statische Werte für Autovervollständigung (werden vor `onAutocomplete` verwendet) |
| `children` | `object` | — | Verschachtelte Spaltendefinitionen für Objektspalten |
| `display_name` | `string` | `''` | Optionaler benutzerfreundlicher Anzeigename. Bei geparsten Spalten überschreibt der Alias diesen Wert |

## Spaltentypen

### `string`

Allgemeine Textspalte. Alle Vergleichsoperatoren inklusive Regex (`~`, `!~`) sind verfügbar.

```javascript
{ type: 'string', suggest: true }
```

### `number`

Numerische Spalte. Regex-Operatoren (`~`, `!~`) werden aus den Operatorvorschlägen ausgeschlossen.

```javascript
{ type: 'number', suggest: true }
```

### `enum`

Enumerierte Spalte mit festem Wertesatz. Wie bei `number` werden Regex-Operatoren ausgeschlossen. Verwende `values`, um den Wertesatz bereitzustellen:

```javascript
{ type: 'enum', suggest: true, autocomplete: true, values: ['debug', 'info', 'warning', 'error', 'critical'] }
```

### `object`

Verschachtelte Spalte mit untergeordneten Feldern. Wenn der Benutzer den Spaltennamen gefolgt von `.` eingibt, werden untergeordnete Spalten vorgeschlagen:

```javascript
{
    type: 'object',
    suggest: true,
    children: {
        labels: {
            type: 'object',
            suggest: true,
            children: {
                tier: { type: 'string', suggest: true },
                env: { type: 'string', suggest: true },
            },
        },
        version: { type: 'string', suggest: true },
    },
}
```

## Wert-Autovervollständigung

### Statische Werte

Gib für Spalten mit bekannten Optionen ein `values`-Array an:

```javascript
{
    level: {
        type: 'enum',
        suggest: true,
        autocomplete: true,
        values: ['debug', 'info', 'warning', 'error', 'critical'],
    },
    status_code: {
        type: 'number',
        suggest: true,
        autocomplete: true,
        values: [200, 201, 204, 400, 404, 500],
    },
}
```

### Asynchrone Werte

Setze `autocomplete: true` ohne `values` und stelle im Editor eine `onAutocomplete`-Callback-Funktion bereit. Der Callback wird aufgerufen, wenn der Benutzer die Wertposition für diese Spalte erreicht.

```javascript
// Spaltendefinition — keine statischen Werte
{ type: 'string', suggest: true, autocomplete: true }

// Editor-Callback lädt Werte aus der API
async function onAutocomplete(key, value) {
    const resp = await fetch(`/api/values?column=${key}`)
    const data = await resp.json()
    return { items: data.values }
}
```

## Verschachtelte Spalten (JSONSchema)

Definiere verschachtelte Strukturen mit `children` bei Objektspalten:

```javascript
const columns = ColumnSchema.fromPlainObject({
    metadata: {
        type: 'object',
        suggest: true,
        children: {
            labels: {
                type: 'object',
                suggest: true,
                children: {
                    tier: { type: 'string', suggest: true, autocomplete: true, values: ['dev', 'staging', 'prod'] },
                    env: { type: 'string', suggest: true, autocomplete: true },
                },
            },
            version: { type: 'string', suggest: true },
        },
    },
})
```

Bei `metadata.` werden `labels` und `version` vorgeschlagen. Bei `metadata.labels.` werden `tier` und `env` vorgeschlagen. Die Tiefe ist unbegrenzt.

## Remote-Key-Discovery (schemafrei)

Bei Objektspalten, deren verschachtelte Schlüssel zur Schema-Definition noch nicht bekannt sind, lasse `children` weg und gib eine `onKeyDiscovery`-Callback-Funktion an:

```javascript
const columns = ColumnSchema.fromPlainObject({
    request: { type: 'object', suggest: true }, // no children — triggers discovery
}

async function onKeyDiscovery(columnName, segments) {
    const resp = await fetch(`/api/keys?column=${columnName}&path=${segments.join(',')}`)
    const data = await resp.json()
    return data.keys // Array<{ name: string, type?: string, hasChildren?: boolean }>
})
```

Wenn der Benutzer `request.` eingibt, wird der Callback mit `('request', ['request'])` aufgerufen. Entdeckte Schlüssel werden pro Aktivierungssitzung zwischengespeichert.

Wenn eine Spalte sowohl `children` als auch `onKeyDiscovery` hat, haben die statischen `children` Vorrang.

## Vollständiges Beispiel

```javascript
const columns = ColumnSchema.fromPlainObject({
    // Enum mit statischen Werten
    level: { type: 'enum', suggest: true, autocomplete: true, values: ['debug', 'info', 'error'] },

    // String mit asynchroner Autovervollständigung
    service: { type: 'string', suggest: true, autocomplete: true },

    // Number — Regex-Operatoren ausgeschlossen
    status_code: { type: 'number', suggest: true, autocomplete: true, values: [200, 400, 500] },

    // Einfacher String
    host: { type: 'string', suggest: true },

    // Verschachtelt mit bekannten Kindern
    metadata: {
        type: 'object',
        suggest: true,
        children: {
            labels: {
                type: 'object',
                suggest: true,
                children: {
                    tier: { type: 'string', suggest: true, autocomplete: true, values: ['dev', 'staging', 'prod'] },
                    env: { type: 'string', suggest: true, autocomplete: true },
                },
            },
            version: { type: 'string', suggest: true },
        },
    },

    // Schemalos — verschachtelte Schlüssel werden zur Laufzeit erkannt
    request: { type: 'object', suggest: true },
})
```
