## Регулярные выражения

Используйте `~` для проверки соответствия регулярному выражению и `!~` для отрицания:

```
message~"error.*timeout"
path!~"^/health"
```

| Оператор | Значение |
|----------|----------|
| `~` | значение соответствует регулярному выражению |
| `!~` | значение не соответствует регулярному выражению |

### Синтаксис регулярных выражений

Синтаксис зависит от целевого бэкенда:

- **ClickHouse** — `match()` (RE2)
- **PostgreSQL** — `~` и `!~` (POSIX-регулярные выражения)
- **StarRocks** — `regexp`
- **Сопоставление в памяти** — движок регулярных выражений языка выполнения

### Регистронезависимое сопоставление

Используйте `(?i)` в начале шаблона:

```
message~"(?i)error"
level~"(?i)^(warn|error|fatal)$"
```

## Подстановочный шаблон

Используйте `*` в конце некавыченного значения для сопоставления по префиксу:

```
host=prod*
service=api-*
```

## Примеры

```
message~"error|fail"
path~"^/api/v2"
path!~"^/health"
host=prod*
host=prod* and message~"timeout.*" and path!~"^/health"
```
