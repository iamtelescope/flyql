PostgreSQL の SQL では、識別子をダブルクォートし、regex には <FlyqlToken type="operator">{'~'}</FlyqlToken>、ブール演算子には大文字の <FlyqlToken type="operator">and</FlyqlToken>/<FlyqlToken type="operator">or</FlyqlToken> を使い、ネストアクセスには JSONB 演算子を使います。

## 基本例

| FlyQL | 生成される SQL |
|-------|-----------------|
| <FlyqlCode code={`message = 'hello'`} inline /> | `"message" = 'hello'` |
| <FlyqlCode code={`message != 'hello'`} inline /> | `"message" != 'hello'` |
| <FlyqlCode code={`count > 10`} inline /> | `"count" > 10` |
| <FlyqlCode code={`count <= 100`} inline /> | `"count" <= 100` |
| <FlyqlCode code={`message ~ 'error.*'`} inline /> | `"message" ~ 'error.*'` |
| <FlyqlCode code={`message !~ 'test.*'`} inline /> | `"message" !~ 'test.*'` |
| <FlyqlCode code={`host = 'prod'`} inline /> | `"host" = 'prod'` |

## ブール演算子

PostgreSQL では、ClickHouse や StarRocks と異なり <FlyqlToken type="operator">and</FlyqlToken>/<FlyqlToken type="operator">or</FlyqlToken> を大文字で出力します。

| FlyQL | 生成される SQL |
|-------|-----------------|
| <FlyqlCode code={`a = '1' and b > 10`} inline /> | `("a" = '1' AND "b" > 10)` |
| <FlyqlCode code={`a = '1' or b = '2'`} inline /> | `("a" = '1' OR "b" = '2')` |
| <FlyqlCode code={`not a = '1'`} inline /> | `NOT ("a" = '1')` |

## リスト所属

| FlyQL | 生成される SQL |
|-------|-----------------|
| <FlyqlCode code={`status in [200, 201]`} inline /> | `"status" IN (200, 201)` |
| <FlyqlCode code={`env not in ['dev', 'test']`} inline /> | `"env" NOT IN ('dev', 'test')` |
| <FlyqlCode code={`status in []`} inline /> | `FALSE` |
| <FlyqlCode code={`status not in []`} inline /> | `TRUE` |

## 包含（`has` / `not has`）

<FlyqlToken type="operator">has</FlyqlToken> 演算子は、カラム型によって異なる SQL を生成します。

| カラム型 | FlyQL | 生成される SQL |
|----------|-------|-----------------|
| String | <FlyqlCode code={`message has 'error'`} inline /> | `position('error' in "message") > 0` |
| String（否定） | <FlyqlCode code={`message not has 'error'`} inline /> | `("message" IS NULL OR position('error' in "message") = 0)` |
| Array | <FlyqlCode code={`tags has 'web'`} inline /> | `'web' = ANY("tags")` |
| Array（否定） | <FlyqlCode code={`tags not has 'web'`} inline /> | `NOT ('web' = ANY("tags"))` |
| JSONB | <FlyqlCode code={`data has 'key'`} inline /> | `"data" ? 'key'` |
| JSONB（否定） | <FlyqlCode code={`data not has 'key'`} inline /> | `NOT ("data" ? 'key')` |
| Hstore | <FlyqlCode code={`metadata has 'key'`} inline /> | `"metadata" ? 'key'` |

## JSONB カラム

PostgreSQL は `jsonb_typeof()` による型チェック付きで JSONB 演算子を使います。

### 文字列比較

| FlyQL | 生成される SQL |
|-------|-----------------|
| <FlyqlCode code={`data.name = 'test'`} inline /> | `(jsonb_typeof("data"->'name') = 'string' AND "data"->>'name' = 'test')` |
| <FlyqlCode code={`data.user.name = 'john'`} inline /> | `(jsonb_typeof("data"->'user'->'name') = 'string' AND "data"->'user'->>'name' = 'john')` |

### 数値比較

数値比較では、抽出した値をキャストします。

| FlyQL | 生成される SQL |
|-------|-----------------|
| <FlyqlCode code={`data.age > 25`} inline /> | `(jsonb_typeof("data"->'age') = 'number' AND ("data"->>'age')::numeric > 25)` |

### JSONB 演算子

- `->` は JSONB を返し、パスの中間走査に使います
- `->>` はテキストを返し、最終値の抽出に使います
- `jsonb_typeof()` は比較前に JSON 型を確認します

## Hstore（Map）カラム

PostgreSQL では、hstore 型に対して `->` 演算子でキーアクセスを行います。

| FlyQL | 生成される SQL |
|-------|-----------------|
| <FlyqlCode code={`metadata.key1 = 'value1'`} inline /> | `"metadata"->'key1' = 'value1'` |
| <FlyqlCode code={`metadata.key1 != 'value1'`} inline /> | `"metadata"->'key1' != 'value1'` |
| <FlyqlCode code={`metadata.pattern ~ 'test.*'`} inline /> | `"metadata"->'pattern' ~ 'test.*'` |

## Array カラム

PostgreSQL の array は 1 始まりです。FlyQL は 0 始まりのインデックスを自動変換します。

| FlyQL | 生成される SQL |
|-------|-----------------|
| <FlyqlCode code={`tags.0 = 'first'`} inline /> | `"tags"[1] = 'first'` |
| <FlyqlCode code={`tags.1 = 'second'`} inline /> | `"tags"[2] = 'second'` |
| <FlyqlCode code={`tags.0 ~ 'tag.*'`} inline /> | `"tags"[1] ~ 'tag.*'` |

### カラム設定

PostgreSQL のカラムには `jsonString` パラメータがありません。JSONB 型かどうかはカラム型から判定されます。

```go
// Go
columns := map[string]*postgresql.Column{
    "message": postgresql.NewColumn(postgresql.ColumnDef{Name: "message", Type: "varchar"}),
    "data":    postgresql.NewColumn(postgresql.ColumnDef{Name: "data", Type: "jsonb"}),
}
```

```javascript
// JavaScript
const columns = {
    message: newColumn("message", "varchar", null),
    data:    newColumn("data", "jsonb", null),
}
```

## Transformers

transformer は PostgreSQL 標準の関数呼び出しを生成します。

| FlyQL | 生成される SQL |
|-------|-----------------|
| <FlyqlCode code={'status\|upper = "ERROR"'} inline /> | `UPPER(status) = 'ERROR'` |
| <FlyqlCode code={'status\|lower = "info"'} inline /> | `LOWER(status) = 'info'` |
| <FlyqlCode code={'message\|len > 100'} inline /> | `LENGTH(message) > 100` |
| <FlyqlCode code={'message\|lower\|len > 100'} inline /> | `LENGTH(LOWER(message)) > 100` |
| <FlyqlCode code={'tags\|split(",") has "urgent"'} inline /> | `'urgent' = ANY(STRING_TO_ARRAY(tags, ','))` |
