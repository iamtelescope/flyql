FlyQL поддерживает 10 типов значений: **Integer**, **BigInt**, **Float**, **String**, **Boolean**, **Null**, **Array**, **Column**, **Function** и **Parameter**. Каждое значение в AST содержит явное поле `value_type`.

## Типы значений

### Integer

Целые числа в диапазоне int64:

<FlyqlCode code={`count = 42`} />
<FlyqlCode code={`status != 200`} />
<FlyqlCode code={`age >= 18`} />

AST: `value_type: "integer"`

### BigInt

Целые числа, превышающие диапазон int64 (Go/Python) или `MAX_SAFE_INTEGER` (JavaScript):

<FlyqlCode code={`id = 18446744073709551615`} />

AST: `value_type: "bigint"`

### Float

Числа с десятичной точкой:

<FlyqlCode code={`price > 9.99`} />
<FlyqlCode code={`ratio = 3.14`} />
<FlyqlCode code={`temperature<=-0.5`} />

AST: `value_type: "float"`

### String

Текстовые значения, в кавычках или без:

<FlyqlCode code={`level = 'error'`} />
<FlyqlCode code={`user = "John Doe"`} />
<FlyqlCode code={`message = 'hello world'`} />
<FlyqlCode code={`count = '42'`} />

AST: `value_type: "string"`

Экранирование кавычек обратным слешем:

```
user = 'John's account'
message="said \"hello\""
```

### Boolean

Ключевые слова <FlyqlToken type="boolean">true</FlyqlToken> и <FlyqlToken type="boolean">false</FlyqlToken> (в нижнем регистре, с учётом регистра):

<FlyqlCode code={`active = true`} />
<FlyqlCode code={`debug = false`} />
<FlyqlCode code={`active != true`} />

AST: `value_type: "boolean"`, значение — нативный boolean.

Варианты с заглавной буквы (`True`, `FALSE`) интерпретируются как строки.

**Генерация SQL:**
| Диалект | FlyQL | Сгенерированный SQL |
|---------|-------|--------------|
| ClickHouse | <FlyqlCode code={`field = true`} inline /> | `field = true` |
| ClickHouse | <FlyqlCode code={`field = false`} inline /> | `field = false` |
| PostgreSQL | <FlyqlCode code={`field = true`} inline /> | `field = TRUE` |
| PostgreSQL | <FlyqlCode code={`field = false`} inline /> | `field = FALSE` |
| StarRocks | <FlyqlCode code={`field = true`} inline /> | `field = true` |
| StarRocks | <FlyqlCode code={`field = false`} inline /> | `field = false` |

### Null

Ключевое слово <FlyqlToken type="null">null</FlyqlToken> (в нижнем регистре, с учётом регистра):

<FlyqlCode code={`field = null`} />
<FlyqlCode code={`field != null`} />

AST: `value_type: "null"`, значение — нативный null.

Варианты с заглавной буквы (`Null`, `NULL`) интерпретируются как строки.

**Ограничения:** С null допустимы только операторы <FlyqlToken type="operator">=</FlyqlToken> и <FlyqlToken type="operator">!=</FlyqlToken>. Использование <FlyqlToken type="operator">{'>'}</FlyqlToken>, <FlyqlToken type="operator">{'<'}</FlyqlToken>, <FlyqlToken type="operator">{'>='}</FlyqlToken>, <FlyqlToken type="operator">{'<='}</FlyqlToken>, <FlyqlToken type="operator">{'~'}</FlyqlToken> или <FlyqlToken type="operator">{'!~'}</FlyqlToken> с null вызывает ошибку парсинга.

**Генерация SQL:** <FlyqlCode code={`field = null`} inline /> генерирует `field IS NULL`, <FlyqlCode code={`field != null`} inline /> генерирует `field IS NOT NULL` (во всех диалектах).

### Array

Array используется с операторами <FlyqlToken type="operator">in</FlyqlToken> и <FlyqlToken type="operator">not in</FlyqlToken>. Элементы могут быть разных типов (гетерогенные):

<FlyqlCode code={`status in [200, 201, 202]`} />
<FlyqlCode code={`name in ['alice', 'bob']`} />
<FlyqlCode code={`field in [1, "hello", true, null]`} />

Каждый элемент имеет свой тип в массиве `values_types`.

## Проверки на истинность и ложность

Ключ без оператора интерпретируется как проверка на истинность:

<FlyqlCode code={`active`} />
<FlyqlCode code={`message and status`} />

Значение считается **ложным** (falsy), если оно:
- `null` / `None` / отсутствует
- Пустая строка `""`
- Ноль `0`
- Boolean <FlyqlToken type="boolean">false</FlyqlToken>

Всё остальное считается **истинным** (truthy).

Используйте <FlyqlToken type="operator">not</FlyqlToken> для проверки на ложность:

<FlyqlCode code={`not archived`} />
<FlyqlCode code={`active and not debug`} />

Проверки на истинность/ложность отличаются от сравнения с null: <FlyqlCode code={`not field`} inline /> срабатывает, когда поле равно null, пусто или равно нулю. <FlyqlCode code={`field = null`} inline /> срабатывает только когда поле конкретно равно null.

## Пробелы

FlyQL допускает свободное использование пробелов. Пробелы вокруг операторов необязательны:

<FlyqlCode code={`status = 200`} />
<FlyqlCode code={`status =200`} />

Все варианты выше эквивалентны.

## Константы LiteralKind

Разобранные значения выражений снабжены меткой `LiteralKind`. Константы экспортируются из корневого пакета в каждой реализации.

**Python:**
```python
from flyql import LiteralKind
# LiteralKind.INTEGER, LiteralKind.BIGINT, LiteralKind.FLOAT,
# LiteralKind.STRING, LiteralKind.BOOLEAN, LiteralKind.NULL, LiteralKind.ARRAY
```

**Go:**
```go
import "github.com/iamtelescope/flyql/golang/literal"
// literal.Integer, literal.BigInt, literal.Float,
// literal.String, literal.Boolean, literal.Null, literal.Array
```

**JavaScript:**
```javascript
import { LiteralKind } from 'flyql'
// LiteralKind.INTEGER, LiteralKind.BIGINT, LiteralKind.FLOAT,
// LiteralKind.STRING, LiteralKind.BOOLEAN, LiteralKind.NULL, LiteralKind.ARRAY
```

## Временные функции

FlyQL поддерживает вызовы временных функций в качестве значений в правой части выражений сравнения. Они позволяют создавать фильтры относительно текущего времени без жёсткого указания временных меток.

AST: `value_type: "function"`, `value` — объект `FunctionCall`.

### `ago(duration)`

Возвращает временную метку относительно текущего времени. Поддерживает составные продолжительности.

<FlyqlCode code={`timestamp > ago(1h)`} />
<FlyqlCode code={`created_at >= ago(7d)`} />
<FlyqlCode code={`updated_at > ago(1h30m)`} />

**Единицы продолжительности:** `s` (секунды), `m` (минуты), `h` (часы), `d` (дни), `w` (недели)

### `now()`

Возвращает текущую временную метку.

<FlyqlCode code={`expires_at < now()`} />

### `today()`

Возвращает сегодняшнюю дату. Принимает необязательный аргумент часового пояса IANA.

<FlyqlCode code={`date = today()`} />
<FlyqlCode code={`date = today('Europe/Berlin')`} />

### `startOf(unit)`

Возвращает начало текущего дня, недели (понедельник) или месяца. Принимает необязательный часовой пояс.

<FlyqlCode code={`created_at > startOf('day')`} />
<FlyqlCode code={`created_at > startOf('week')`} />
<FlyqlCode code={`created_at > startOf('month', 'Asia/Tokyo')`} />

**Допустимые единицы:** `'day'`, `'week'`, `'month'`

### Ограничения операторов

Временные функции можно использовать только с операторами сравнения: `=`, `!=`, `>`, `>=`, `<`, `<=`.

Использование `~` или `!~` с временной функцией вызывает ошибку парсинга. Операторы `in`, `has`, `like` и `ilike` используют отдельные пути разбора и несовместимы с функциональными значениями.

### Ссылка на столбец без скобок

Имена функций без скобок интерпретируются как ссылки на столбцы, а не как вызовы функций:

<FlyqlCode code={`field = ago`} />

Это генерирует `value_type: "column"`, а не вызов функции.

## Parameter

Плейсхолдеры параметров (`$name` для именованных, `$1` для позиционных) заменяют значения, которые будут разрешены во время выполнения функцией `bindParams()`. Они могут появляться везде, где допустимо литеральное значение — включая IN-списки, аргументы временных функций и аргументы трансформеров.

<FlyqlCode code={`status = $code`} />
<FlyqlCode code={`level in [$primary, 'warn']`} />
<FlyqlCode code={`created > ago($duration)`} />

AST: `value_type: "parameter"`, значение — объект `Parameter` с полями `name` и `positional`.

Полный справочник см. в разделе [Параметры](/ru/syntax/parameters/).
