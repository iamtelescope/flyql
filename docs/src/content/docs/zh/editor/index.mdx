FlyQL 编辑器是一个 Vue 3 组件，提供基于列定义的查询输入、自动补全、语法高亮和键盘导航能力。

## 快速接入

```vue
<script setup>
import { ref } from 'vue'
import { FlyqlEditor } from 'flyql/editor'

const query = ref('')
const columns = {
    status: { type: 'number', suggest: true },
    level: { type: 'enum', suggest: true, autocomplete: true, values: ['debug', 'info', 'error'] },
    service: { type: 'string', suggest: true, autocomplete: true },
    host: { type: 'string', suggest: true },
}

function onSubmit() {
    console.log('Query submitted:', query.value)
}

function onParseError(error) {
    console.log('Parse error:', error)
}
</script>

<template>
    <FlyqlEditor
        v-model="query"
        :columns="columns"
        placeholder="输入 FlyQL 查询..."
        @submit="onSubmit"
        @parse-error="onParseError"
    />
</template>
```

该组件会渲染一个文本输入框，并基于 `columns` 列定义提供字段提示、类型感知运算符以及布尔运算符提示。

## 安装

```bash
npm install flyql
```

编辑器依赖 Vue 3：

```bash
npm install vue
```

从 `flyql/editor` 子路径导入：

```javascript
import { FlyqlEditor, FlyqlColumns } from 'flyql/editor'
```

两个组件都会自动导入 `flyql.css`，其中包含主题变量、建议面板样式和 token 高亮。大多数标准打包器（如 Vite、webpack）都可以直接处理。

如果你的构建环境不会处理 JS 中的 CSS 导入，请手动引入：

```javascript
import 'flyql/editor/flyql.css'
```

## Props

| Prop | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `modelValue` | `String` | `''` | 查询文本（`v-model`） |
| `columns` | `Object` | `{}` | [列定义](/zh/editor/schema/) |
| `onAutocomplete` | `Function` | `null` | 异步值补全函数 |
| `onKeyDiscovery` | `Function` | `null` | 远程发现字段的函数 |
| `placeholder` | `String` | `''` | 占位文本 |
| `autofocus` | `Boolean` | `false` | 挂载后自动聚焦 |
| `debug` | `Boolean` | `false` | 调试日志 |
| `debounceMs` | `Number` | `300` | 异步补全延迟时间（毫秒） |
| `dark` | `Boolean` | `false` | 启用暗色主题（参见[主题](/zh/editor/theming/)） |

**函数签名：**

- `onAutocomplete(key, value)`：返回 `Promise<{ items: string[], incomplete?: boolean }>`
- `onKeyDiscovery(columnName, segments)`：返回 `Promise<Array<{ name, type?, hasChildren? }>>`

当返回结果中的 `incomplete` 为 `true` 时，表示当前列表是部分结果，编辑器会在下一次输入时继续请求更新。异步调用会自动防抖，并在发起新请求时取消旧请求。

## 事件

| 事件 | 载荷 | 说明 |
|------|------|------|
| `update:modelValue` | `string` | 查询文本变化 |
| `submit` | — | 用户按下 `Shift+Enter` |
| `parse-error` | `string` | 查询无效时返回错误消息 |
| `focus` | — | 编辑器获得焦点 |
| `blur` | — | 编辑器失去焦点 |

## 暴露方法

可以通过模板引用访问组件方法：

```vue
<script setup>
import { ref } from 'vue'
import { FlyqlEditor } from 'flyql/editor'

const editorRef = ref(null)

function checkStatus() {
    const status = editorRef.value.getQueryStatus()
    console.log(status.valid, status.message)
}
</script>

<template>
    <FlyqlEditor ref="editorRef" v-model="query" :columns="columns" />
    <button @click="checkStatus">检查</button>
</template>
```

| 方法 | 返回值 | 说明 |
|------|--------|------|
| `focus()` | — | 聚焦编辑器输入框 |
| `blur()` | — | 取消焦点 |
| `getQueryStatus()` | `{ valid: boolean, message: string }` | 校验当前查询 |

## 异步值补全

通过 `onAutocomplete` 从 API 获取值建议：

```vue
<script setup>
import { ref } from 'vue'
import { FlyqlEditor } from 'flyql/editor'

const query = ref('')
const columns = {
    service: { type: 'string', suggest: true, autocomplete: true },
    level: { type: 'enum', suggest: true, autocomplete: true, values: ['info', 'error'] },
}

async function onAutocomplete(key, value) {
    const resp = await fetch(`/api/autocomplete?key=${encodeURIComponent(key)}`)
    const data = await resp.json()
    return { items: data.values }
}
</script>

<template>
    <FlyqlEditor v-model="query" :columns="columns" :on-autocomplete="onAutocomplete" />
</template>
```

该回调接收列名 `key` 和当前输入前缀 `value`，返回结构为 `{ items: string[], incomplete?: boolean }`。若响应时间超过 200ms，编辑器会自动显示加载状态。

### 有状态补全流程

- **首次进入值输入上下文**：例如输入 `service=` 时立即请求，不进行防抖
- **完整结果集**：若 `incomplete: false`，编辑器将结果缓存到本地，后续输入直接前端过滤
- **部分结果集**：若 `incomplete: true`，编辑器会先显示当前结果，再在后台继续按输入防抖请求更新

如果服务端返回 0 个结果，编辑器会显示 “No matching values”。若是前端对完整结果进行本地过滤后为空，则只是不显示候选项，用户回删后可重新看到更多结果。

## 键盘快捷键

| 按键 | 动作 |
|------|------|
| 方向上 / 下 | 在建议中移动 |
| Enter / Tab | 选择当前建议 |
| Escape | 关闭建议列表 |
| Shift+Enter | 提交查询并触发 `submit` 事件 |
