FlyQL unterstützt 7 Werttypen: **Integer**, **BigInt**, **Float**, **String**, **Boolean**, **Null** und **Array**. Jeder Wert im AST trägt ein explizites Feld `value_type`.

## Werttypen

### Integer

Ganzzahlen innerhalb des int64-Bereichs:

<FlyqlCode code={`count = 42`} />
<FlyqlCode code={`status != 200`} />
<FlyqlCode code={`age >= 18`} />

AST: `value_type: "integer"`

### BigInt

Ganzzahlen, die den int64-Bereich (Go/Python) oder `MAX_SAFE_INTEGER` (JavaScript) überschreiten:

<FlyqlCode code={`id = 18446744073709551615`} />

AST: `value_type: "bigint"`

### Float

Zahlen mit Dezimalpunkt:

<FlyqlCode code={`price > 9.99`} />
<FlyqlCode code={`ratio = 3.14`} />
<FlyqlCode code={`temperature<=-0.5`} />

AST: `value_type: "float"`

### String

Textwerte, mit oder ohne Anführungszeichen:

<FlyqlCode code={`level = error`} />
<FlyqlCode code={`user = "John Doe"`} />
<FlyqlCode code={`message = 'hello world'`} />
<FlyqlCode code={`count = '42'`} />

AST: `value_type: "string"`

Anführungszeichen lassen sich mit Backslash escapen:

```
user = 'John
message="said \"hello\""
```

### Boolean

Die Schlüsselwörter <FlyqlToken type="boolean">true</FlyqlToken> und <FlyqlToken type="boolean">false</FlyqlToken> (kleingeschrieben, case-sensitive):

<FlyqlCode code={`active = true`} />
<FlyqlCode code={`debug = false`} />
<FlyqlCode code={`active != true`} />

AST: `value_type: "boolean"`, der Wert ist ein nativer Boolean.

Großgeschriebene Varianten (`True`, `FALSE`) werden als Zeichenketten behandelt.

**SQL-Generierung:**
| Dialekt | FlyQL | Generiertes SQL |
|---------|-------|---------------|
| ClickHouse | <FlyqlCode code={`field = true`} inline /> | `field = true` |
| ClickHouse | <FlyqlCode code={`field = false`} inline /> | `field = false` |
| PostgreSQL | <FlyqlCode code={`field = true`} inline /> | `field = TRUE` |
| PostgreSQL | <FlyqlCode code={`field = false`} inline /> | `field = FALSE` |
| StarRocks | <FlyqlCode code={`field = true`} inline /> | `field = true` |
| StarRocks | <FlyqlCode code={`field = false`} inline /> | `field = false` |

### Null

Das Schlüsselwort <FlyqlToken type="null">null</FlyqlToken> (kleingeschrieben, case-sensitive):

<FlyqlCode code={`field = null`} />
<FlyqlCode code={`field != null`} />

AST: `value_type: "null"`, der Wert ist natives null.

Großgeschriebene Varianten (`Null`, `NULL`) werden als Zeichenketten behandelt.

**Einschränkungen:** Mit null sind nur die Operatoren <FlyqlToken type="operator">=</FlyqlToken> und <FlyqlToken type="operator">!=</FlyqlToken> gültig. Die Verwendung von <FlyqlToken type="operator">{'>'}</FlyqlToken>, <FlyqlToken type="operator">{'<'}</FlyqlToken>, <FlyqlToken type="operator">{'>='}</FlyqlToken>, <FlyqlToken type="operator">{'<='}</FlyqlToken>, <FlyqlToken type="operator">{'~'}</FlyqlToken> oder <FlyqlToken type="operator">{'!~'}</FlyqlToken> mit null erzeugt einen Parse-Fehler.

**SQL-Generierung:** <FlyqlCode code={`field = null`} inline /> erzeugt `field IS NULL`, <FlyqlCode code={`field != null`} inline /> erzeugt `field IS NOT NULL` (in allen Dialekten).

### Array

Arrays werden mit den Operatoren <FlyqlToken type="operator">in</FlyqlToken> und <FlyqlToken type="operator">not in</FlyqlToken> verwendet. Elemente dürfen gemischte Typen haben (heterogen):

<FlyqlCode code={`status in [200, 201, 202]`} />
<FlyqlCode code={`name in ['alice', 'bob']`} />
<FlyqlCode code={`field in [1, "hello", true, null]`} />

Jedes Element trägt seinen eigenen Typ im Array `values_types`.

## Truthy/Falsy-Ausdrücke

Ein einzelner Schlüssel ohne Operator prüft, ob das Feld einen truthy Wert hat:

<FlyqlCode code={`active`} />
<FlyqlCode code={`message and status`} />

Ein Wert gilt als **falsy**, wenn er:
- `null` / `None` / nicht vorhanden ist
- eine leere Zeichenkette `""` ist
- Null `0` ist
- der Boolean <FlyqlToken type="boolean">false</FlyqlToken> ist

Alles andere ist **truthy**.

Verwende <FlyqlToken type="operator">not</FlyqlToken>, um auf falsy Werte zu prüfen:

<FlyqlCode code={`not archived`} />
<FlyqlCode code={`active and not debug`} />

Truthy/Falsy-Prüfungen unterscheiden sich von Null-Vergleichen: <FlyqlCode code={`not field`} inline /> trifft zu, wenn das Feld null, leer oder Null ist. <FlyqlCode code={`field = null`} inline /> trifft nur dann zu, wenn das Feld explizit null ist.

## Leerzeichen

FlyQL ist tolerant gegenüber Leerzeichen. Leerzeichen um Operatoren sind optional:

<FlyqlCode code={`status = 200`} />
<FlyqlCode code={`status =200`} />

Alle Varianten sind äquivalent.

## Typkonstanten-Modul

Alle Typkonstanten sind über das Modul `flyql.types` verfügbar:

**Python:**
```python
from flyql.types import ValueType
# ValueType.INTEGER, ValueType.BIGINT, ValueType.FLOAT,
# ValueType.STRING, ValueType.BOOLEAN, ValueType.NULL, ValueType.ARRAY
```

**Go:**
```go
import "github.com/iamtelescope/flyql/golang/types"
// types.Integer, types.BigInt, types.Float,
// types.String, types.Boolean, types.Null, types.Array
```

**JavaScript:**
```javascript
import { ValueType } from 'flyql/types'
// ValueType.INTEGER, ValueType.BIGINT, ValueType.FLOAT,
// ValueType.STRING, ValueType.BOOLEAN, ValueType.NULL, ValueType.ARRAY
```

## Hinweise zur Migration

Beim Upgrade von einer früheren FlyQL-Version:

- **Breaking:** Go `ValueTypeString` / `ValueTypeNumber` wurden entfernt und durch `types.ValueType` mit 7 Werten ersetzt
- **Breaking:** Das AST-Feld `value_type` wurde von `"string"` / `"number"` auf `"integer"` / `"float"` / `"bigint"` / `"string"` / `"boolean"` / `"null"` geändert
- **Breaking:** Die Erzwingung homogener Listen wurde entfernt -- heterogene Arrays sind jetzt gültig
- **Neu:** <FlyqlToken type="boolean">true</FlyqlToken>, <FlyqlToken type="boolean">false</FlyqlToken>, <FlyqlToken type="null">null</FlyqlToken> werden als typisierte Literale geparst (nicht als Zeichenketten)
- **Neu:** Das Array `values_types` bei IN-Listen-Ausdrücken stellt Typen pro Element bereit
- **Neu:** Das Modul `flyql.types` exportiert die kanonischen Typkonstanten
