Os transformadores modificam os valores das colunas antes de serem comparados em consultas ou emitidos em SQL. Eles usam a sintaxe de pipe (`|`), aplicada da esquerda para a direita.

## Sintaxe de Pipe

Aplique um transformador a uma coluna com `|`:

<FlyqlCode code={`message|upper = "ERROR"`} />

Isso transforma o valor de <FlyqlToken type="key">message</FlyqlToken> usando <FlyqlToken type="transformer">upper</FlyqlToken> antes de comparar com `"ERROR"`.

## Encadeamento

Encadeie múltiplos transformadores com pipes adicionais:

<FlyqlCode code={`message|lower|len > 100`} />

Os transformadores são aplicados da esquerda para a direita: <FlyqlToken type="transformer">lower</FlyqlToken> executa primeiro (string para string), depois <FlyqlToken type="transformer">len</FlyqlToken> (string para int). O resultado é comparado com `> 100`.

## Sistema de Tipos

Cada transformador declara um tipo de entrada e um tipo de saída. Transformadores encadeados devem ser compatíveis em tipos — o tipo de saída de um deve corresponder ao tipo de entrada do próximo.

| Tipo | Descrição |
|------|-----------|
| `string` | Valores de texto |
| `int` | Números inteiros |
| `float` | Números de ponto flutuante |
| `array` | Array de valores (use o operador <FlyqlToken type="operator">has</FlyqlToken> para verificar pertencimento) |

## Transformadores Integrados

| Transformador | Entrada | Saída | Descrição |
|---------------|---------|-------|-----------|
| <FlyqlToken type="transformer">upper</FlyqlToken> | string | string | Converter para maiúsculas |
| <FlyqlToken type="transformer">lower</FlyqlToken> | string | string | Converter para minúsculas |
| <FlyqlToken type="transformer">len</FlyqlToken> | string | int | Obter comprimento da string |
| <FlyqlToken type="transformer">split</FlyqlToken> | string | array | Dividir por delimitador: `split(".")` |

## Saída SQL por Dialeto

Os transformadores geram chamadas de função SQL específicas para cada dialeto. Transformadores encadeados produzem chamadas aninhadas (o mais interno primeiro):

**Transformador único:** <FlyqlCode code={`status|upper = "ERROR"`} inline />

| Dialeto | SQL Gerado |
|---------|------------|
| ClickHouse | `equals(upper(status), 'ERROR')` |
| PostgreSQL | `UPPER(status) = 'ERROR'` |
| StarRocks | `UPPER(\`status\`) = 'ERROR'` |

**Transformadores encadeados:** <FlyqlCode code={`message|lower|len > 100`} inline />

| Dialeto | SQL Gerado |
|---------|------------|
| ClickHouse | `length(lower(message)) > 100` |
| PostgreSQL | `LENGTH(LOWER(message)) > 100` |
| StarRocks | `LENGTH(LOWER(\`message\`)) > 100` |

**Split com has:** <FlyqlCode code={`message|split(" ") has "hello"`} inline />

| Dialeto | SQL Gerado |
|---------|------------|
| ClickHouse | `has(splitByChar(' ', message), 'hello')` |
| PostgreSQL | `'hello' = ANY(STRING_TO_ARRAY(message, ' '))` |
| StarRocks | `` array_contains(SPLIT(`message`, ' '), 'hello') `` |

## Erros de Tipo

Se uma cadeia tiver tipos incompatíveis, o editor exibe um erro:

<FlyqlCode code={`message|len|upper`} />

Isso falha porque <FlyqlToken type="transformer">len</FlyqlToken> produz `int`, mas <FlyqlToken type="transformer">upper</FlyqlToken> requer entrada `string`. O editor exibe: "len outputs int, upper requires string input".

## Exemplos

<FlyqlCode code={`status|upper = "ERROR"`} />
<FlyqlCode code={`message|lower|len > 100`} />
<FlyqlCode code={`host|lower != "localhost"`} />
<FlyqlCode code={`path|len > 0`} />
<FlyqlCode code={`tags|split(",") has "urgent"`} />
