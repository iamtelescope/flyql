FlyQL admite los siguientes operadores de comparación:

## Igualdad

| Operador | Nombre | Ejemplo |
|----------|--------|---------|
| <FlyqlToken type="operator">=</FlyqlToken> | Igual | <FlyqlCode code={`status = 200`} inline /> |
| <FlyqlToken type="operator">!=</FlyqlToken> | Distinto | <FlyqlCode code={`env != prod`} inline /> |

<FlyqlCode code={`status = 200`} />
<FlyqlCode code={`service != "api-gateway"`} />

Ambos lados del operador son obligatorios — una <FlyqlToken type="key">clave</FlyqlToken> y un <FlyqlToken type="value">valor</FlyqlToken>.

## Comparación numérica

| Operador | Nombre | Ejemplo |
|----------|--------|---------|
| <FlyqlToken type="operator">{'>'}</FlyqlToken> | Mayor que | <FlyqlCode code={`status > 399`} inline /> |
| <FlyqlToken type="operator">{'>='}</FlyqlToken> | Mayor o igual que | <FlyqlCode code={`status >= 400`} inline /> |
| <FlyqlToken type="operator">{'<'}</FlyqlToken> | Menor que | <FlyqlCode code={`duration < 1000`} inline /> |
| <FlyqlToken type="operator">{'<='}</FlyqlToken> | Menor o igual que | <FlyqlCode code={`duration <= 500`} inline /> |

<FlyqlCode code={`status >= 400 and duration < 5000`} />
<FlyqlCode code={`response_time > 100 and response_time <= 2000`} />

Los operadores de comparación numérica funcionan sobre valores numéricos. Si se usan con cadenas, se aplica comparación lexicográfica.

## Coincidencia regex

| Operador | Nombre | Ejemplo |
|----------|--------|---------|
| <FlyqlToken type="operator">{'~'}</FlyqlToken> | Coincide con regex | <FlyqlCode code={`message ~ "error.*"`} inline /> |
| <FlyqlToken type="operator">{'!~'}</FlyqlToken> | No coincide con regex | <FlyqlCode code={`path !~ "^/health"`} inline /> |

<FlyqlCode code={`message ~ "timeout|connection refused"`} />
<FlyqlCode code={`host !~ "^test-.*"`} />

Los patrones regex siguen la sintaxis del runtime o la base de datos de destino. Consulta [Coincidencia de patrones](/es/syntax/pattern-matching/) para flags regex y detalles.

## Pertenencia a listas

| Operador | Nombre | Ejemplo |
|----------|--------|---------|
| <FlyqlToken type="operator">in</FlyqlToken> | Está en la lista | <FlyqlCode code={`status in [200, 201]`} inline /> |
| <FlyqlToken type="operator">not in</FlyqlToken> | No está en la lista | <FlyqlCode code={`env not in ['dev', 'test']`} inline /> |

Consulta la página de [Listas](/es/syntax/lists/) para ver las reglas detalladas de sintaxis de listas.

## Comprobación de contención

| Operador | Nombre | Ejemplo |
|----------|--------|---------|
| <FlyqlToken type="operator">has</FlyqlToken> | Contiene | <FlyqlCode code={`message has 'error'`} inline /> |
| <FlyqlToken type="operator">not has</FlyqlToken> | No contiene | <FlyqlCode code={`message not has 'test'`} inline /> |

<FlyqlCode code={`message has 'timeout'`} />
<FlyqlCode code={`tags not has 'deprecated'`} />

El operador <FlyqlToken type="operator">has</FlyqlToken> realiza una comprobación de contención dependiente del tipo:

- **String** — coincidencia por subcadena: <FlyqlCode code={`message has 'error'`} inline /> es true si <FlyqlToken type="key">message</FlyqlToken> contiene <FlyqlToken type="string">"error"</FlyqlToken>
- **Map** — existencia de clave: <FlyqlCode code={`metadata has 'region'`} inline /> es true si el mapa tiene una clave <FlyqlToken type="string">"region"</FlyqlToken>
- **Array/List** — pertenencia de elemento: <FlyqlCode code={`tags has 'web'`} inline /> es true si el array contiene <FlyqlToken type="string">"web"</FlyqlToken>
- **JSON** — existencia de clave: <FlyqlCode code={`data has 'name'`} inline /> es true si el objeto JSON tiene una clave <FlyqlToken type="string">"name"</FlyqlToken>
- **Null/vacío** — siempre false para <FlyqlToken type="operator">has</FlyqlToken>, siempre true para <FlyqlToken type="operator">not has</FlyqlToken>

<FlyqlToken type="operator">has</FlyqlToken> recibe un único valor, no una lista. Usa <FlyqlToken type="operator">in</FlyqlToken> para comprobar pertenencia dentro de un conjunto de valores.

## Resumen de operadores

| Operador | Nombre | Ejemplo |
|----------|--------|---------|
| <FlyqlToken type="operator">=</FlyqlToken> | Igual | <FlyqlCode code={`status = 200`} inline /> |
| <FlyqlToken type="operator">!=</FlyqlToken> | Distinto | <FlyqlCode code={`env != prod`} inline /> |
| <FlyqlToken type="operator">{'>'}</FlyqlToken> | Mayor que | <FlyqlCode code={`count > 10`} inline /> |
| <FlyqlToken type="operator">{'>='}</FlyqlToken> | Mayor o igual que | <FlyqlCode code={`count >= 10`} inline /> |
| <FlyqlToken type="operator">{'<'}</FlyqlToken> | Menor que | <FlyqlCode code={`count < 100`} inline /> |
| <FlyqlToken type="operator">{'<='}</FlyqlToken> | Menor o igual que | <FlyqlCode code={`count <= 100`} inline /> |
| <FlyqlToken type="operator">{'~'}</FlyqlToken> | Coincide con regex | <FlyqlCode code={`msg ~ "err.*"`} inline /> |
| <FlyqlToken type="operator">{'!~'}</FlyqlToken> | No coincide con regex | <FlyqlCode code={`msg !~ "^ok"`} inline /> |
| <FlyqlToken type="operator">in</FlyqlToken> | Está en la lista | <FlyqlCode code={`s in [1, 2]`} inline /> |
| <FlyqlToken type="operator">not in</FlyqlToken> | No está en la lista | <FlyqlCode code={`s not in [3]`} inline /> |
| <FlyqlToken type="operator">has</FlyqlToken> | Contiene | <FlyqlCode code={`msg has 'err'`} inline /> |
| <FlyqlToken type="operator">not has</FlyqlToken> | No contiene | <FlyqlCode code={`msg not has 'ok'`} inline /> |
