FlyQL은 10가지 값 타입을 지원합니다: **Integer**, **BigInt**, **Float**, **String**, **Boolean**, **Null**, **Array**, **Column**, **Function**, **Parameter**. AST의 각 값에는 명시적인 `value_type` 필드가 포함됩니다.

## 값 타입

### Integer

int64 범위 안의 정수:

<FlyqlCode code={`count = 42`} />
<FlyqlCode code={`status != 200`} />
<FlyqlCode code={`age >= 18`} />

AST: `value_type: "integer"`

### BigInt

int64 범위(Go/Python) 또는 `MAX_SAFE_INTEGER`(JavaScript)를 초과하는 정수:

<FlyqlCode code={`id = 18446744073709551615`} />

AST: `value_type: "bigint"`

### Float

소수점을 포함한 숫자:

<FlyqlCode code={`price > 9.99`} />
<FlyqlCode code={`ratio = 3.14`} />
<FlyqlCode code={`temperature<=-0.5`} />

AST: `value_type: "float"`

### String

텍스트 값은 작은따옴표나 큰따옴표로 **반드시 감싸야 합니다**. 따옴표 없는 값은 문자열이 아니라 컬럼 참조로 해석됩니다.

<FlyqlCode code={`level = 'error'`} />
<FlyqlCode code={`user = "John Doe"`} />
<FlyqlCode code={`message = 'hello world'`} />
<FlyqlCode code={`count = '42'`} />

AST: `value_type: "string"`

따옴표는 백슬래시로 이스케이프할 수 있습니다.

```
user = 'John\'s account'
message="said \"hello\""
```

### Boolean

소문자이며 대소문자를 구분하는 키워드 <FlyqlToken type="boolean">true</FlyqlToken>, <FlyqlToken type="boolean">false</FlyqlToken>:

<FlyqlCode code={`active = true`} />
<FlyqlCode code={`debug = false`} />
<FlyqlCode code={`active != true`} />

AST: `value_type: "boolean"`, 값은 네이티브 boolean입니다.

대문자가 섞인 변형(`True`, `FALSE`)은 boolean이 아니라 컬럼 참조로 처리됩니다.

**SQL 생성:**
| 방언 | FlyQL | 생성된 SQL |
|------|-------|---------------|
| ClickHouse | <FlyqlCode code={`field = true`} inline /> | `field = true` |
| ClickHouse | <FlyqlCode code={`field = false`} inline /> | `field = false` |
| PostgreSQL | <FlyqlCode code={`field = true`} inline /> | `field = TRUE` |
| PostgreSQL | <FlyqlCode code={`field = false`} inline /> | `field = FALSE` |
| StarRocks | <FlyqlCode code={`field = true`} inline /> | `field = true` |
| StarRocks | <FlyqlCode code={`field = false`} inline /> | `field = false` |

### Null

소문자이며 대소문자를 구분하는 키워드 <FlyqlToken type="null">null</FlyqlToken>:

<FlyqlCode code={`field = null`} />
<FlyqlCode code={`field != null`} />

AST: `value_type: "null"`, 값은 네이티브 null입니다.

대문자가 섞인 변형(`Null`, `NULL`)은 null이 아니라 컬럼 참조로 처리됩니다.

**제약 사항:** null과 함께 유효한 연산자는 <FlyqlToken type="operator">=</FlyqlToken>와 <FlyqlToken type="operator">!=</FlyqlToken>뿐입니다. <FlyqlToken type="operator">{'>'}</FlyqlToken>, <FlyqlToken type="operator">{'<'}</FlyqlToken>, <FlyqlToken type="operator">{'>='}</FlyqlToken>, <FlyqlToken type="operator">{'<='}</FlyqlToken>, <FlyqlToken type="operator">{'~'}</FlyqlToken>, <FlyqlToken type="operator">{'!~'}</FlyqlToken>를 null과 함께 사용하면 parse error가 발생합니다.

**SQL 생성:** <FlyqlCode code={`field = null`} inline />은 `field IS NULL`, <FlyqlCode code={`field != null`} inline />은 `field IS NOT NULL`을 생성합니다 (모든 방언 공통).

### 컬럼 참조

숫자, boolean, null, 시간 함수가 아닌 따옴표 없는 값은 모두 **컬럼 참조**로 처리됩니다. 이는 컬럼과 컬럼을 비교할 때 사용하는, 다른 컬럼을 가리키는 포인터입니다.

<FlyqlCode code={`price > min_price`} />
<FlyqlCode code={`updated_at > created_at`} />
<FlyqlCode code={`field = otherfield`} />

AST: `value_type: "column"`

문자열 값과 매칭하려면 **반드시** 따옴표를 사용해야 합니다.

<FlyqlCode code={`level = 'error'`} />

따옴표가 없으면 <FlyqlCode code={`level = error`} inline />에서 `error`는 `error`라는 이름의 컬럼 참조로 처리됩니다.

### Array

Array는 <FlyqlToken type="operator">in</FlyqlToken>과 <FlyqlToken type="operator">not in</FlyqlToken>에서 사용합니다. 요소 타입은 서로 달라도 됩니다(heterogeneous).

<FlyqlCode code={`status in [200, 201, 202]`} />
<FlyqlCode code={`name in ['alice', 'bob']`} />
<FlyqlCode code={`field in [1, "hello", true, null]`} />

각 요소는 `values_types` 배열에 자신의 타입을 가집니다.

## Truthy/Falsy 표현식

연산자 없이 단독으로 쓰인 키는 해당 필드가 truthy 값인지 검사합니다.

<FlyqlCode code={`active`} />
<FlyqlCode code={`message and status`} />

다음 값은 **falsy**로 간주됩니다:
- `null` / `None` / 누락된 값
- 빈 문자열 `""`
- 0
- boolean <FlyqlToken type="boolean">false</FlyqlToken>

그 외는 모두 **truthy**입니다.

falsy 값을 확인하려면 <FlyqlToken type="operator">not</FlyqlToken>을 사용하세요.

<FlyqlCode code={`not archived`} />
<FlyqlCode code={`active and not debug`} />

Truthy/Falsy 검사는 null 비교와 다릅니다. <FlyqlCode code={`not field`} inline />는 필드가 null, 빈 값 또는 0일 때 일치합니다. <FlyqlCode code={`field = null`} inline />은 필드가 정확히 null일 때만 일치합니다.

## 공백

FlyQL은 공백에 관대합니다. 연산자 주변 공백은 선택 사항입니다.

<FlyqlCode code={`status = 200`} />
<FlyqlCode code={`status =200`} />

위 표현은 모두 동일합니다.

## 타입 상수 모듈

모든 타입 상수는 `flyql.types` 모듈에서 사용할 수 있습니다.

**Python:**
```python
from flyql.types import ValueType
# ValueType.INTEGER, ValueType.BIGINT, ValueType.FLOAT,
# ValueType.STRING, ValueType.BOOLEAN, ValueType.NULL, ValueType.ARRAY
```

**Go:**
```go
import "github.com/iamtelescope/flyql/golang/types"
// types.Integer, types.BigInt, types.Float,
// types.String, types.Boolean, types.Null, types.Array
```

**JavaScript:**
```javascript
import { ValueType } from 'flyql/types'
// ValueType.INTEGER, ValueType.BIGINT, ValueType.FLOAT,
// ValueType.STRING, ValueType.BOOLEAN, ValueType.NULL, ValueType.ARRAY
```

## 시간 함수

FlyQL은 비교 표현식의 오른쪽 값으로 시간 함수 호출을 지원합니다. 타임스탬프를 하드코딩하지 않고도 시간 기준 필터를 사용할 수 있습니다.

AST: `value_type: "function"`, `value`는 `FunctionCall` 객체입니다.

### `ago(duration)`

현재 시간 기준 상대적인 타임스탬프를 반환합니다. 복합 기간을 지원합니다.

<FlyqlCode code={`timestamp > ago(1h)`} />
<FlyqlCode code={`created_at >= ago(7d)`} />
<FlyqlCode code={`updated_at > ago(1h30m)`} />

**기간 단위:** `s` (초), `m` (분), `h` (시간), `d` (일), `w` (주)

### `now()`

현재 타임스탬프를 반환합니다.

<FlyqlCode code={`expires_at < now()`} />

### `today()`

오늘 날짜를 반환합니다. 선택적으로 IANA 타임존 인수를 받습니다.

<FlyqlCode code={`date = today()`} />
<FlyqlCode code={`date = today('Europe/Berlin')`} />

### `startOf(unit)`

현재 일, 주(월요일), 또는 월의 시작 시점을 반환합니다. 선택적으로 타임존을 받습니다.

<FlyqlCode code={`created_at > startOf('day')`} />
<FlyqlCode code={`created_at > startOf('week')`} />
<FlyqlCode code={`created_at > startOf('month', 'Asia/Tokyo')`} />

**유효한 단위:** `'day'`, `'week'`, `'month'`

### 연산자 제한

시간 함수는 비교 연산자에서만 사용할 수 있습니다: `=`, `!=`, `>`, `>=`, `<`, `<=`.

시간 함수와 함께 `~` 또는 `!~`를 사용하면 parse error가 발생합니다. `in`, `has`, `like`, `ilike` 같은 연산자는 별도의 파싱 경로를 사용하며 함수 값과 호환되지 않습니다.

### 괄호 없는 컬럼 참조

괄호 없는 함수 이름은 함수 호출이 아닌 컬럼 참조로 처리됩니다.

<FlyqlCode code={`field = ago`} />

이는 함수 호출이 아닌 `value_type: "column"`을 생성합니다.

## Parameter

파라미터 placeholder(named의 경우 `$name`, positional의 경우 `$1`)는 `bindParams()`가 런타임에 해석할 값을 대체합니다. 리터럴 값이 유효한 모든 위치에 사용할 수 있으며, IN 리스트, 시간 함수 인수, transformer 인수 자리 모두에서 사용할 수 있습니다.

<FlyqlCode code={`status = $code`} />
<FlyqlCode code={`level in [$primary, 'warn']`} />
<FlyqlCode code={`created > ago($duration)`} />

AST: `value_type: "parameter"`, 값은 `name`과 `positional` 필드를 가진 `Parameter` 객체입니다.

전체 레퍼런스는 [파라미터](/ko/syntax/parameters/)를 참고하세요.

## 마이그레이션 참고

이전 FlyQL 버전에서 업그레이드하는 경우:

- **Breaking:** Go의 `ValueTypeString` / `ValueTypeNumber`는 제거되었고, 7개 값을 갖는 `types.ValueType`으로 대체되었습니다
- **Breaking:** AST `value_type` 필드는 `"string"` / `"number"`에서 `"integer"` / `"float"` / `"bigint"` / `"string"` / `"boolean"` / `"null"`로 변경되었습니다
- **Breaking:** 혼합 타입 리스트 강제 규칙이 제거되어, 이제 heterogeneous array가 유효합니다
- **New:** <FlyqlToken type="boolean">true</FlyqlToken>, <FlyqlToken type="boolean">false</FlyqlToken>, <FlyqlToken type="null">null</FlyqlToken>은 문자열이 아니라 타입 지정 리터럴로 parse됩니다
- **New:** IN 리스트 표현식의 `values_types` 배열이 요소별 타입 정보를 제공합니다
- **New:** `flyql.types` 모듈이 정식 타입 상수를 export합니다
- **Breaking:** 따옴표 없는 비숫자 값은 이제 `"string"`이 아닌 `value_type: "column"`(컬럼 참조)로 parse됩니다. 문자열과 매칭하려면 값을 따옴표로 감싸야 합니다: `field = 'value'`
- **New:** `value_type: "function"`을 가진 시간 함수 값(`ago()`, `now()`, `today()`, `startOf()`)
- **New:** `value_type: "parameter"`를 가진 파라미터 placeholder(`$name`, `$1`), 런타임에 `bindParams()`로 해석됩니다
