---
title: Valores y tipos
description: Tipos de valores, literales tipados y expresiones en FlyQL
---

FlyQL admite 7 tipos de valor: **Integer**, **BigInt**, **Float**, **String**, **Boolean**, **Null** y **Array**. Cada valor del AST incluye un campo `value_type` explícito.

## Tipos de valor

### Integer

Números enteros dentro del rango de int64:

```
count=42
status!=200
age>=18
```

AST: `value_type: "integer"`

### BigInt

Enteros que superan el rango de int64 (Go/Python) o `MAX_SAFE_INTEGER` (JavaScript):

```
id=18446744073709551615
```

AST: `value_type: "bigint"`

### Float

Números con punto decimal:

```
price>9.99
ratio=3.14
temperature<=-0.5
```

AST: `value_type: "float"`

### String

Valores de texto, con o sin comillas:

```
level=error              # unquoted string
user="John Doe"          # double-quoted (spaces)
message='hello world'    # single-quoted (spaces)
count='42'               # quoted number is a string
```

AST: `value_type: "string"`

Escapa las comillas con barra invertida:

```
user='John\'s account'
message="said \"hello\""
```

### Boolean

Las palabras clave `true` y `false` en minúsculas y sensibles a mayúsculas:

```
active=true
debug=false
active!=true
```

AST: `value_type: "boolean"`, el valor es un boolean nativo.

Las variantes con mayúsculas (`True`, `FALSE`) se tratan como strings.

**Generación SQL:**
| Dialecto | `field=true` | `field=false` |
|---------|-------------|--------------|
| ClickHouse | `field = true` | `field = false` |
| PostgreSQL | `field = TRUE` | `field = FALSE` |
| StarRocks | `field = true` | `field = false` |

### Null

La palabra clave `null` en minúsculas y sensible a mayúsculas:

```
field=null
field!=null
```

AST: `value_type: "null"`, el valor es un null nativo.

Las variantes con mayúsculas (`Null`, `NULL`) se tratan como strings.

**Restricciones:** solo los operadores `=` y `!=` son válidos con null. Usar `>`, `<`, `>=`, `<=`, `~` o `!~` con null produce un error de parseo.

**Generación SQL:** `field=null` genera `field IS NULL` y `field!=null` genera `field IS NOT NULL` en todos los dialectos.

### Array

Los arrays se usan con `in` y `not in`. Sus elementos pueden mezclar tipos:

```
status in [200, 201, 202]
name in ['alice', 'bob']
field in [1, "hello", true, null]
```

Cada elemento lleva su propio tipo en el array `values_types`.

## Expresiones truthy/falsy

Una clave independiente sin operador comprueba si el campo tiene un valor truthy:

```
active
message and status
```

Un valor se considera **falsy** si es:
- `null` / `None` / inexistente
- string vacío `""`
- cero `0`
- boolean `false`

Todo lo demás es **truthy**.

Usa `not` para comprobar valores falsy:

```
not archived
active and not debug
```

Las comprobaciones truthy/falsy son distintas de las comparaciones con null: `not field` coincide cuando el campo es null, vacío o cero. `field=null` coincide solo cuando el campo es específicamente null.

## Espacios en blanco

FlyQL tolera espacios en blanco. Los espacios alrededor de los operadores son opcionales:

```
status=200
status = 200
status =200
```

Todas las variantes anteriores son equivalentes.

## Módulo de constantes de tipos

Todas las constantes de tipo están disponibles en el módulo `flyql.types`:

**Python:**
```python
from flyql.types import ValueType
# ValueType.INTEGER, ValueType.BIGINT, ValueType.FLOAT,
# ValueType.STRING, ValueType.BOOLEAN, ValueType.NULL, ValueType.ARRAY
```

**Go:**
```go
import "github.com/iamtelescope/flyql/golang/types"
// types.Integer, types.BigInt, types.Float,
// types.String, types.Boolean, types.Null, types.Array
```

**JavaScript:**
```javascript
import { ValueType } from 'flyql/types'
// ValueType.INTEGER, ValueType.BIGINT, ValueType.FLOAT,
// ValueType.STRING, ValueType.BOOLEAN, ValueType.NULL, ValueType.ARRAY
```

## Notas de migración

Si actualizas desde una versión anterior de FlyQL:

- **Breaking:** en Go se eliminaron `ValueTypeString` y `ValueTypeNumber`, sustituidos por `types.ValueType` con 7 valores
- **Breaking:** el campo `value_type` del AST cambió de `"string"`/`"number"` a `"integer"`/`"float"`/`"bigint"`/`"string"`/`"boolean"`/`"null"`
- **Breaking:** se eliminó la restricción de listas de tipo mixto; ahora los arrays heterogéneos son válidos
- **Nuevo:** `true`, `false` y `null` se parsean como literales tipados, no como strings
- **Nuevo:** el array `values_types` en expresiones de listas `IN` proporciona el tipo de cada elemento
- **Nuevo:** el módulo `flyql.types` exporta las constantes de tipo canónicas
