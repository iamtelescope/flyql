Система трансформеров FlyQL расширяема. Вы можете регистрировать собственные трансформеры, которые будут работать так же, как встроенные (`upper`, `lower`, `len`) в парсерах, SQL-генераторах, механизме сопоставления в памяти и автодополнении редактора.

## Интерфейс трансформера

Каждый трансформер реализует 6 элементов:

| Элемент | Тип | Описание |
|---------|-----|----------|
| `name` | string | Уникальный идентификатор в нижнем регистре (например, `"firstoctet"`) |
| `inputType` | Type | Ожидаемый входной тип (`string`, `int`, `float`, `bool`, `array`), либо `any` для универсального входа, принимающего любой вышестоящий тип -- см. [Трансформеры со входом `Any`](#трансформеры-со-входом-any) |
| `outputType` | Type | Выходной тип после преобразования |
| `argSchema` | ArgSpec[] | Объявленные типы аргументов и арность (по умолчанию: пусто = без аргументов) |
| `sql(dialect, columnRef, args)` | string | Генерация SQL-выражения, специфичного для диалекта, оборачивающего ссылку на столбец |
| `apply(value, args)` | any | Преобразование значения во время выполнения в памяти |

Метод `sql()` получает название SQL-диалекта (`"clickhouse"`, `"postgresql"`, `"starrocks"`), строку-ссылку на столбец и список разобранных аргументов. Он возвращает SQL-выражение, оборачивающее эту ссылку. Метод `apply()` получает значение во время выполнения и список аргументов, и возвращает преобразованное значение.

### ArgSpec

Каждая запись в `argSchema` объявляет один ожидаемый аргумент:

| Поле | Тип | По умолчанию | Описание |
|------|-----|-------------|----------|
| `type` | Type | — | Ожидаемый тип аргумента (`string`, `int`, `float`, `bool`) |
| `required` | boolean | `true` | Должен ли аргумент быть предоставлен |

Арность выводится из длины схемы: обязательные аргументы должны присутствовать, завершающие необязательные аргументы могут быть опущены. Трансформеры без аргументов (такие как `upper`, `lower`, `len`) наследуют пустую схему по умолчанию.

## API реестра

| Метод | Описание |
|-------|----------|
| `defaultRegistry()` | Возвращает новый реестр, предварительно загруженный встроенными трансформерами |
| `registry.register(transformer)` | Добавляет пользовательский трансформер (выбрасывает ошибку, если имя уже зарегистрировано) |
| `registry.get(name)` | Ищет трансформер по имени (возвращает null/None/nil, если не найден) |
| `registry.names()` | Возвращает список имён всех зарегистрированных трансформеров |

Схема использования: получите реестр по умолчанию, зарегистрируйте свои трансформеры, передайте реестр генераторам и механизму сопоставления.

## Python

Определите пользовательский трансформер, наследуясь от `Transformer`:

```python
from flyql.transformers import Transformer, Type, default_registry

class FirstOctetTransformer(Transformer):
    @property
    def name(self): return "firstoctet"
    @property
    def input_type(self): return Type.STRING
    @property
    def output_type(self): return Type.INT
    def sql(self, dialect, column_ref, args=None):
        if dialect == "clickhouse":
            return f"toUInt8(splitByChar('.', {column_ref})[1])"
        return f"CAST(SPLIT_PART({column_ref}, '.', 1) AS INTEGER)"
    def apply(self, value, args=None):
        return int(str(value).split('.')[0])
```

Атрибут класса `arg_schema` по умолчанию равен `()` (без аргументов). Чтобы объявить аргументы, переопределите его кортежем `ArgSpec`:

```python
from typing import ClassVar, Tuple
from flyql.transformers.base import ArgSpec

class SubstringTransformer(Transformer):
    arg_schema: ClassVar[Tuple[ArgSpec, ...]] = (
        ArgSpec(type=Type.INT),                    # start (required)
        ArgSpec(type=Type.INT, required=False),    # length (optional)
    )
    # ... name, input_type, output_type, sql, apply ...
```

Регистрация и использование:

```python
from flyql import parse
from flyql.generators.postgresql.generator import to_sql_where, Column

registry = default_registry()
registry.register(FirstOctetTransformer())

result = parse("src_ip|firstoctet > 192")
columns = {"src_ip": Column("src_ip", "text")}
sql = to_sql_where(result.root, columns, registry=registry)
# CAST(SPLIT_PART(src_ip, '.', 1) AS INTEGER) > 192
```

Для сопоставления в памяти:

```python
from flyql.matcher.evaluator import Evaluator
from flyql.matcher.record import Record

evaluator = Evaluator(registry=registry)
record = Record({"src_ip": "193.0.0.1"})
evaluator.evaluate(result.root, record)  # True (193 > 192)
```

## JavaScript

Определите пользовательский трансформер, расширяя `Transformer`:

```javascript
import { Transformer, Type, ArgSpec, defaultRegistry } from 'flyql/transformers'

class FirstOctetTransformer extends Transformer {
    get name() { return 'firstoctet' }
    get inputType() { return Type.STRING }
    get outputType() { return Type.INT }
    sql(dialect, columnRef, args = []) {
        if (dialect === 'clickhouse') return `toUInt8(splitByChar('.', ${columnRef})[1])`
        return `CAST(SPLIT_PART(${columnRef}, '.', 1) AS INTEGER)`
    }
    apply(value, args = []) { return parseInt(String(value).split('.')[0], 10) }
}
```

Геттер `argSchema` по умолчанию возвращает `[]` (без аргументов). Чтобы объявить аргументы, переопределите его:

```javascript
class SubstringTransformer extends Transformer {
    get argSchema() {
        return [
            new ArgSpec(Type.INT, true),     // start (required)
            new ArgSpec(Type.INT, false),    // length (optional)
        ]
    }
    // ... name, inputType, outputType, sql, apply ...
}
```

Регистрация и использование:

```javascript
import { parse } from 'flyql'
import { generateWhere, newColumn } from 'flyql/generators/postgresql'

const registry = defaultRegistry()
registry.register(new FirstOctetTransformer())

const result = parse("src_ip|firstoctet > 192")
const columns = { src_ip: newColumn({ name: 'src_ip', type: 'text' }) }
const sql = generateWhere(result.root, columns, registry)
// CAST(SPLIT_PART(src_ip, '.', 1) AS INTEGER) > 192
```

Для сопоставления в памяти:

```javascript
import { match } from 'flyql/matcher'

match("src_ip|firstoctet > 192", { src_ip: "193.0.0.1" }, registry) // true
```

Для автодополнения в редакторе — пользовательские трансформеры отображаются наравне со встроенными:

```javascript
import { EditorEngine } from 'flyql-vue'

const engine = new EditorEngine(columns, { registry })
// After typing "src_ip|", suggestions include "firstoctet", "upper", "lower", "len"
```

## Go

Определите пользовательский трансформер, реализовав интерфейс `Transformer`:

```go
package main

import (
    "fmt"
    "strconv"
    "strings"
    "github.com/iamtelescope/flyql/golang/transformers"
)

type FirstOctet struct{}

func (f FirstOctet) Name() string               { return "firstoctet" }
func (f FirstOctet) InputType() transformers.Type  { return transformers.TransformerTypeString }
func (f FirstOctet) OutputType() transformers.Type { return transformers.TransformerTypeInt }
func (f FirstOctet) ArgSchema() []transformers.ArgSpec        { return []transformers.ArgSpec{} }
func (f FirstOctet) SQL(dialect, colRef string, args []any) string {
    if dialect == "clickhouse" {
        return fmt.Sprintf("toUInt8(splitByChar('.', %s)[1])", colRef)
    }
    return fmt.Sprintf("CAST(SPLIT_PART(%s, '.', 1) AS INTEGER)", colRef)
}
func (f FirstOctet) Apply(value interface{}, args []any) interface{} {
    parts := strings.SplitN(fmt.Sprint(value), ".", 2)
    n, _ := strconv.Atoi(parts[0])
    return n
}
```

Чтобы объявить аргументы, верните записи `ArgSpec` из `ArgSchema()`:

```go
func (s Substring) ArgSchema() []transformers.ArgSpec {
    return []transformers.ArgSpec{
        {Type: transformers.TransformerTypeInt, Required: true},   // start (required)
        {Type: transformers.TransformerTypeInt, Required: false},  // length (optional)
    }
}
```

Регистрация и использование:

```go
import (
    flyql "github.com/iamtelescope/flyql/golang"
    postgresqlgen "github.com/iamtelescope/flyql/golang/generators/postgresql"
)

registry := transformers.DefaultRegistry()
registry.Register(FirstOctet{})

result, _ := flyql.Parse("src_ip|firstoctet > 192")
columns := map[string]*postgresqlgen.Column{
    "src_ip": postgresqlgen.NewColumn("src_ip", "text", nil),
}
sql, _ := postgresqlgen.ToSQLWhere(result.Root, columns, registry)
// CAST(SPLIT_PART(src_ip, '.', 1) AS INTEGER) > 192
```

Для сопоставления в памяти:

```go
import "github.com/iamtelescope/flyql/golang/matcher"

matched, _ := matcher.Match("src_ip|firstoctet > 192", map[string]any{"src_ip": "193.0.0.1"}, registry)
// matched == true
```

## Трансформеры со входом `Any`

Трансформер может объявить `inputType = Type.Any` (строковое значение `"any"`), чтобы принимать любой вышестоящий тип -- тип колонки или выходной тип предыдущего трансформера в цепочке. Мотивирующий случай -- трансформер `type`, возвращающий имя типа колонки.

Правила:

- `Any` допустим только на `inputType`. Объявление `Any` в качестве `outputType` или на любом `ArgSpec.type` отклоняется во время `register()`. Каждый трансформер обязан объявить конкретный выходной тип.
- Когда трансформер с входом `Any` используется в цепочке, проверка равенства входного типа пропускается. Строгость нижестоящих звеньев сохраняется -- например, `col|accepts_any_returning_array|upper` всё равно вызовет ошибку на `upper`, потому что `array` не является допустимым входом для `upper`.
- `parseFlyQLType("any")` выбрасывает ошибку: `any` не является допустимым типом колонки или значения.

```python
from flyql.flyql_type import Type
from flyql.transformers import Transformer

class TypeTransformer(Transformer):
    @property
    def name(self): return "type"
    @property
    def input_type(self): return Type.Any      # принимает любой вышестоящий тип
    @property
    def output_type(self): return Type.String  # должен быть конкретным
    def sql(self, dialect, column_ref, args=None):
        if dialect == "clickhouse":
            return f"toTypeName({column_ref})"
        return f"pg_typeof({column_ref})::text"
    def apply(self, value, args=None):
        return type(value).__name__
```

type TypeTransformer struct{}

func (TypeTransformer) Description() string               { return "" }
func (TypeTransformer) InputType() flyqltype.Type         { return flyqltype.Any }     // wildcard
func (TypeTransformer) OutputType() flyqltype.Type        { return flyqltype.String }  // конкретный
func (TypeTransformer) ArgSchema() []transformers.ArgSpec { return nil }
func (TypeTransformer) SQL(dialect, columnRef string, args []any) string {
    if dialect == "clickhouse" {
        return fmt.Sprintf("toTypeName(%s)", columnRef)
    }
    return fmt.Sprintf("pg_typeof(%s)::text", columnRef)
}
func (TypeTransformer) Apply(value any, args []any) any {
    if value == nil {
        return "nil"
    }
    return reflect.TypeOf(value).String()
}

## Совместимость типов

Пользовательские трансформеры участвуют в той же цепочке валидации типов, что и встроенные. Если вы объедините в цепочку `field|firstoctet|upper`, система обнаружит, что `firstoctet` возвращает `int`, а `upper` требует `string`.

Доступные типы: см. [`flyql.Type`](/advanced/column-types/). Единственное исключение -- `Any`, который является универсальным входом для трансформеров и не является допустимым типом колонки или значения.

## Валидация с помощью `diagnose()`

Функция `diagnose()` обходит разобранное AST по схеме колонок и реестру трансформеров, возвращая позиционированные записи `Diagnostic` без выбрасывания исключений. Она проверяет:

- **Неизвестная колонка** -- базовый ключ отсутствует в списке колонок
- **Неизвестный трансформер** -- имя трансформера отсутствует в реестре
- **Неверная арность** -- слишком мало или слишком много аргументов относительно `argSchema`
- **Неверный тип аргумента** -- тип значения аргумента не совпадает с объявленным типом `ArgSpec` (расширение int до float допускается)
- **Несовместимость типов в цепочке** -- выходной тип одного трансформера не совпадает со входным типом следующего, или нормализованный тип колонки не совпадает со входным типом первого трансформера

Каждая диагностика содержит `range` (позиция в исходном коде), `message`, `severity` (`"error"`) и `code` (`"unknown_column"`, `"unknown_transformer"`, `"arg_count"`, `"arg_type"`, `"chain_type"`).

```python
# Python
from flyql import parse, Column, ColumnSchema, Type, diagnose

ast = parse("hstt|upper=X").root
columns = ColumnSchema.from_columns([Column(name="host", column_type=Type.String)])
diags = diagnose(ast, columns)  # columns -- это ColumnSchema
# [Diagnostic(range=Range(0, 4), message="column 'hstt' is not defined", severity='error', code='unknown_column')]
```

```javascript
// JavaScript
import { parse, Column, ColumnSchema, Type, diagnose } from 'flyql'

const ast = parse('hstt|upper=X').root
const columns = ColumnSchema.fromColumns([new Column('host', false, Type.String)])
const diags = diagnose(ast, columns)  // columns -- это ColumnSchema
// [Diagnostic { range: Range(0, 4), message: "column 'hstt' is not defined", severity: 'error', code: 'unknown_column' }]
```

```go
// Go
import flyql "github.com/iamtelescope/flyql/golang"

result, _ := flyql.Parse("hstt|upper=X")
schema := flyql.FromColumns([]flyql.Column{flyql.NewColumn("host", flyql.TypeString)})
diags := flyql.Diagnose(result.Root, schema, nil) // nil = реестр по умолчанию
// []Diagnostic{{Range: {0, 4}, Message: "column 'hstt' is not defined", Severity: "error", Code: "unknown_column"}}
```

Передайте пользовательский реестр третьим аргументом, если у вас зарегистрированы пользовательские трансформеры. Передайте `nil`/`None`/`null` для использования реестра по умолчанию.

## Безопасность

Метод `sql()` пользовательских трансформеров получает только ссылки на столбцы (из схемы), а не пользовательские значения. Пользовательские значения экранируются/параметризуются генератором. Это сохраняет гарантию FlyQL от инъекций -- пользовательский SQL оборачивает имена столбцов, а не пользовательский ввод.
