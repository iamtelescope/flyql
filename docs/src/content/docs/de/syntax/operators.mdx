FlyQL unterstützt die folgenden Vergleichsoperatoren:

## Gleichheit

| Operator | Name | Beispiel |
|----------|------|----------|
| <FlyqlToken type="operator">=</FlyqlToken> | Gleich | <FlyqlCode code={`status = 200`} inline /> |
| <FlyqlToken type="operator">!=</FlyqlToken> | Ungleich | <FlyqlCode code={`env != prod`} inline /> |

<FlyqlCode code={`status = 200`} />
<FlyqlCode code={`service != "api-gateway"`} />

Beide Seiten des Operators sind erforderlich — ein <FlyqlToken type="key">Schlüssel</FlyqlToken> und ein <FlyqlToken type="value">Wert</FlyqlToken>.

## Numerische Vergleiche

| Operator | Name | Beispiel |
|----------|------|----------|
| <FlyqlToken type="operator">{'>'}</FlyqlToken> | Größer als | <FlyqlCode code={`status > 399`} inline /> |
| <FlyqlToken type="operator">{'>='}</FlyqlToken> | Größer oder gleich | <FlyqlCode code={`status >= 400`} inline /> |
| <FlyqlToken type="operator">{'<'}</FlyqlToken> | Kleiner als | <FlyqlCode code={`duration < 1000`} inline /> |
| <FlyqlToken type="operator">{'<='}</FlyqlToken> | Kleiner oder gleich | <FlyqlCode code={`duration <= 500`} inline /> |

<FlyqlCode code={`status >= 400 and duration < 5000`} />
<FlyqlCode code={`response_time > 100 and response_time <= 2000`} />

Numerische Vergleichsoperatoren arbeiten mit numerischen Werten. Bei Strings wird lexikografisch verglichen.

## Regex-Matching

| Operator | Name | Beispiel |
|----------|------|----------|
| <FlyqlToken type="operator">{'~'}</FlyqlToken> | Regex-Match | <FlyqlCode code={`message ~ "error.*"`} inline /> |
| <FlyqlToken type="operator">{'!~'}</FlyqlToken> | Kein Regex-Match | <FlyqlCode code={`path !~ "^/health"`} inline /> |

<FlyqlCode code={`message ~ "timeout|connection refused"`} />
<FlyqlCode code={`host !~ "^test-.*"`} />

Regex-Muster folgen der Syntax der Zieldatenbank oder der Laufzeitumgebung. Siehe [Musterabgleich](/de/syntax/pattern-matching/) für Regex-Flags und Details.

## Listenzugehörigkeit

| Operator | Name | Beispiel |
|----------|------|----------|
| <FlyqlToken type="operator">in</FlyqlToken> | In Liste | <FlyqlCode code={`status in [200, 201]`} inline /> |
| <FlyqlToken type="operator">not in</FlyqlToken> | Nicht in Liste | <FlyqlCode code={`env not in ['dev', 'test']`} inline /> |

Siehe die Seite [Listen](/de/syntax/lists/) für detaillierte Regeln zur Listensyntax.

## Musterabgleich (LIKE)

| Operator | Name | Beispiel |
|----------|------|----------|
| <FlyqlToken type="operator">like</FlyqlToken> | Like-Muster | <FlyqlCode code={`message like 'error%'`} inline /> |
| <FlyqlToken type="operator">not like</FlyqlToken> | Kein Like-Muster | <FlyqlCode code={`host not like 'test%'`} inline /> |
| <FlyqlToken type="operator">ilike</FlyqlToken> | Groß-/Kleinschreibung ignorierend | <FlyqlCode code={`message ilike 'Error%'`} inline /> |
| <FlyqlToken type="operator">not ilike</FlyqlToken> | Groß-/Kleinschreibung ignorierend, negiert | <FlyqlCode code={`host not ilike 'TEST%'`} inline /> |

<FlyqlCode code={`message like '%timeout%'`} />
<FlyqlCode code={`host not like '%-staging'`} />
<FlyqlCode code={`path ilike '/API/%'`} />

SQL-artiger Musterabgleich mit zwei Wildcards:

- <FlyqlToken type="operator">%</FlyqlToken> — entspricht einer beliebigen Zeichenfolge (auch leer)
- <FlyqlToken type="operator">_</FlyqlToken> — entspricht genau einem Zeichen

Wildcards können mit einem Backslash escaped werden, um sie wörtlich zu verwenden: `\%` entspricht einem literalen `%`, `\_` entspricht einem literalen `_`.

Siehe [Musterabgleich](/de/syntax/pattern-matching/) für weitere Details und Beispiele.

## Enthalten-Prüfung

| Operator | Name | Beispiel |
|----------|------|----------|
| <FlyqlToken type="operator">has</FlyqlToken> | Enthält | <FlyqlCode code={`message has 'error'`} inline /> |
| <FlyqlToken type="operator">not has</FlyqlToken> | Enthält nicht | <FlyqlCode code={`message not has 'test'`} inline /> |

<FlyqlCode code={`message has 'timeout'`} />
<FlyqlCode code={`tags not has 'deprecated'`} />

Der <FlyqlToken type="operator">has</FlyqlToken> Operator führt eine typabhängige Enthalten-Prüfung aus:

- **String** — Teilstring-Match: <FlyqlCode code={`message has 'error'`} inline /> ist true, wenn <FlyqlToken type="key">message</FlyqlToken> <FlyqlToken type="string">"error"</FlyqlToken> enthält
- **Map** — Schlüsselerkennung: <FlyqlCode code={`metadata has 'region'`} inline /> ist true, wenn die Map einen Schlüssel <FlyqlToken type="string">"region"</FlyqlToken> hat
- **Array/List** — Elementzugehörigkeit: <FlyqlCode code={`tags has 'web'`} inline /> ist true, wenn das Array <FlyqlToken type="string">"web"</FlyqlToken> enthält
- **JSON** — Schlüsselerkennung: <FlyqlCode code={`data has 'name'`} inline /> ist true, wenn das JSON-Objekt einen Schlüssel <FlyqlToken type="string">"name"</FlyqlToken> hat
- **Null/leer** — immer false für <FlyqlToken type="operator">has</FlyqlToken>, immer true für <FlyqlToken type="operator">not has</FlyqlToken>

<FlyqlToken type="operator">has</FlyqlToken> akzeptiert einen einzelnen Wert, keine Liste. Verwende <FlyqlToken type="operator">in</FlyqlToken>, um die Zugehörigkeit zu einer Wertemenge zu prüfen.
