## Installation

```bash
npm install flyql
```

**Anforderungen:** Node.js 16+. Alle Imports verwenden ES-Module.

## Eine Abfrage parsen

```javascript
import { parse } from 'flyql'

const result = parse("status = 200 and active")
console.log(result.root)
```

`parse()` gibt ein `ParseResult`-Objekt mit einem `root`-Knoten zurück, der den AST repräsentiert. Bei ungültiger Eingabe wird ein `ParserError` geworfen:

```javascript
import { parse, ParserError } from 'flyql'

try {
    const result = parse("status = 200 and active")
    console.log(result.root)
} catch (err) {
    if (err instanceof ParserError) {
        console.error(`Parse error: ${err.message}`)
    }
}
```

## SQL generieren

Verwende einen Generator, um den AST in eine parametrisierte WHERE-Klausel umzuwandeln. Jeder Datenbankdialekt wird über einen eigenen Unterpfad importiert.

### ClickHouse

```javascript
import { parse } from 'flyql'
import { generateWhere, newColumn } from 'flyql/generators/clickhouse'

const result = parse("status >= 400 and host like 'prod%'")

const columns = {
    status: newColumn({ name: "status", type: "UInt32" }),
    host: newColumn({ name: "host", type: "String" }),
}

const sql = generateWhere(result.root, columns)
console.log(sql)
```

### Andere Dialekte

Die Generatoren für PostgreSQL und StarRocks folgen demselben Muster:

```javascript
import { generateWhere, newColumn } from 'flyql/generators/postgresql'
import { generateWhere, newColumn } from 'flyql/generators/starrocks'
```

Hinweis: Alle drei `newColumn`-Factorys haben dieselbe Form – `newColumn({ name, type, values? })`.

## Im Speicher matchen

Werte eine Abfrage gegen einen Datensatz aus, ohne SQL zu generieren:

```javascript
import { match } from 'flyql/matcher'

const data = {
    status: 200,
    active: true,
    host: "prod-api-01",
}

const matches = match("status = 200 and active", data)
console.log(`Matches: ${matches}`) // true
```

## Spaltenausdrücke parsen

Parse einen Spaltenausdruck in strukturierte Daten mit Segmenten, Transformern und Aliasen:

```javascript
import { parse, parseToJson } from 'flyql/columns'

// Einfache Spalten parsen (Transformers sind standardmäßig deaktiviert)
const parsed = parse("message, status")
for (const col of parsed) {
    console.log(`${col.name} (display: ${JSON.stringify(col.displayName)}, segments: ${col.segments})`)
}

// Transformers über Capabilities aktivieren
const withTransforms = parse("message|chars(25) as msg, status", { transformers: true })

// Oder direkt als JSON für API-Antworten serialisieren
const json = parseToJson("message, status|upper", { transformers: true })
console.log(json)
```

## Parametrisierte Abfragen

Verwende `$name`- und `$1`-Platzhalter und löse sie zur Laufzeit mit `bindParams()` auf:

```javascript
import { parse, bindParams } from 'flyql'
import { generateWhere, newColumn } from 'flyql/generators/clickhouse'

// Eine Abfrage mit Parameter-Platzhaltern parsen
const result = parse("status = $code and env in [$env, 'staging']")

// Konkrete Werte an die Parameter binden
bindParams(result.root, { code: 200, env: 'prod' })

// SQL generieren
const columns = {
    status: newColumn({ name: "status", type: "Int32" }),
    env: newColumn({ name: "env", type: "String" }),
}
const sql = generateWhere(result.root, columns)
console.log(sql)
```

Siehe [Parameter](/de/syntax/parameters/) für die vollständige Referenz.

## Transformers in Abfragen

Parse eine Abfrage mit Transformern und generiere SQL:

```javascript
import { parse } from 'flyql'
import { generateWhere, newColumn } from 'flyql/generators/clickhouse'

// Eine Abfrage mit Transformern parsen
const result = parse("message|upper = 'ERROR'")

// SQL für ClickHouse generieren
const columns = { message: newColumn({ name: 'message', type: 'String' }) }
const sql = generateWhere(result.root, columns)
console.log(sql) // equals(upper(message), 'ERROR')
```

Siehe [Transformers](/de/syntax/transformers/) für die vollständige Syntaxreferenz.

## Verfügbare Unterpfad-Imports

| Import-Pfad | Inhalt |
|-------------|--------|
| `flyql` | Core-Parser (`parse`, `ParserError`, Operatoren, Konstanten) |
| `flyql/core` | Direktes Core-Parser-Modul |
| `flyql/columns` | Parser für Spaltenausdrücke |
| `flyql/generators/clickhouse` | ClickHouse-SQL-Generator |
| `flyql/generators/postgresql` | PostgreSQL-SQL-Generator |
| `flyql/generators/starrocks` | StarRocks-SQL-Generator |
| `flyql/matcher` | In-Memory-Record-Matcher |
| `flyql/tokenize` | Reines [Tokenizer](/de/advanced/tokenize/)-Primitiv — flache typisierte Tokens für eigene Formatter |
