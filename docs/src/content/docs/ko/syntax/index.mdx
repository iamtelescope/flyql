FlyQL 쿼리는 불리언 연산자(<FlyqlToken type="operator">and</FlyqlToken>, <FlyqlToken type="operator">or</FlyqlToken>, <FlyqlToken type="operator">not</FlyqlToken>)로 연결된 하나 이상의 조건으로 구성됩니다. 각 조건은 비교식이거나 truthy 검사입니다.

## 기본 쿼리 구조

<FlyqlCode code={`status = 200 and active and not archived`} />

- <FlyqlCode code={`status = 200`} inline /> — <FlyqlToken type="key">status</FlyqlToken> 필드가 <FlyqlToken type="value">200</FlyqlToken>과 같음
- <FlyqlCode code={`active`} inline /> — <FlyqlToken type="key">active</FlyqlToken> 필드가 truthy 값을 가짐
- <FlyqlCode code={`not archived`} inline /> — <FlyqlToken type="key">archived</FlyqlToken> 필드가 falsy 임 (null, 빈 값, 0 또는 false)

## 추가 예제

<FlyqlCode code={`service != 'api' or user = "john doe"`} />

<FlyqlCode code={`message ~ "error.*" and not debug`} />

<FlyqlCode code={`(a = 1 or b = 2) and not (c = 3 and d = 4)`} />

<FlyqlCode code={`status in [200, 201] and method not in ['DELETE', 'PUT']`} />

<FlyqlCode code={`message has 'error' and tags not has 'debug'`} />

## 일반 규칙

- **단독 키** — 연산자 없는 키는 truthy 검사로 처리됩니다: <FlyqlCode code={`active`} inline />
- **비교식** — 연산자가 있는 키에는 대응하는 값이 있어야 합니다: <FlyqlCode code={`status = 200`} inline />
- **공백** — 연산자 주변 공백은 선택 사항입니다: <FlyqlCode code={`status=200`} inline /> 과 <FlyqlCode code={`status = 200`} inline /> 은 동일합니다
- **연산자 우선순위** — <FlyqlToken type="operator">and</FlyqlToken>는 <FlyqlToken type="operator">or</FlyqlToken>보다 더 강하게 결합합니다: <FlyqlCode code={`a = 1 or b = 2 and c = 3`} inline /> 은 <FlyqlCode code={`a = 1 or (b = 2 and c = 3)`} inline /> 으로 parse됩니다
- **괄호** — <FlyqlToken type="operator">( )</FlyqlToken>로 우선순위를 덮어씁니다: <FlyqlCode code={`(a = 1 or b = 2) and c = 3`} inline />

## 구문 섹션

- [연산자](/ko/syntax/operators/) — 비교 연산자 (<FlyqlToken type="operator">=</FlyqlToken>, <FlyqlToken type="operator">!=</FlyqlToken>, <FlyqlToken type="operator">{'>'}</FlyqlToken>, <FlyqlToken type="operator">{'>='}</FlyqlToken>, <FlyqlToken type="operator">{'<'}</FlyqlToken>, <FlyqlToken type="operator">{'<='}</FlyqlToken>)
- [불리언 로직](/ko/syntax/boolean-logic/) — <FlyqlToken type="operator">and</FlyqlToken>, <FlyqlToken type="operator">or</FlyqlToken>, <FlyqlToken type="operator">not</FlyqlToken>, 우선순위, 그룹화
- [패턴 매칭](/ko/syntax/pattern-matching/) — regex (<FlyqlToken type="operator">{'~'}</FlyqlToken>, <FlyqlToken type="operator">{'!~'}</FlyqlToken>) 및 와일드카드(<FlyqlToken type="operator">{'*'}</FlyqlToken>) 매칭
- [리스트](/ko/syntax/lists/) — 리스트 값에 대한 <FlyqlToken type="operator">in</FlyqlToken>, <FlyqlToken type="operator">not in</FlyqlToken>
- [포함 검사](/ko/syntax/operators/#포함-검사) — 부분 문자열, 키 존재, 항목 포함 여부를 위한 <FlyqlToken type="operator">has</FlyqlToken>, <FlyqlToken type="operator">not has</FlyqlToken>
- [값과 표현식](/ko/syntax/values/) — truthy/falsy, 따옴표 문자열, 숫자
- [중첩 키](/ko/syntax/nested-keys/) — 점으로 구분된 경로와 따옴표로 감싼 키 세그먼트
- [파라미터](/ko/syntax/parameters/) — 런타임에 `bindParams()`로 해석되는 `$name`과 `$1` placeholder
