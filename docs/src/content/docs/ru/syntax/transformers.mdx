Преобразователи изменяют значения колонок перед сравнением в запросах или генерацией SQL. Они используют синтаксис пайпа (`|`) и применяются слева направо.

## Синтаксис пайпа

Примените преобразователь к колонке с помощью `|`:

<FlyqlCode code={`message|upper = "ERROR"`} />

Это преобразует значение <FlyqlToken type="key">message</FlyqlToken> с помощью <FlyqlToken type="transformer">upper</FlyqlToken> перед сравнением с `"ERROR"`.

## Цепочки

Объединяйте несколько преобразователей дополнительными пайпами:

<FlyqlCode code={`message|lower|len > 100`} />

Преобразователи применяются слева направо: сначала выполняется <FlyqlToken type="transformer">lower</FlyqlToken> (строка в строку), затем <FlyqlToken type="transformer">len</FlyqlToken> (строка в int). Результат сравнивается с `> 100`.

## Система типов

Каждый преобразователь объявляет входной и выходной тип. Цепочки преобразователей должны быть типово совместимыми — выходной тип одного должен совпадать со входным типом следующего.

| Тип | Описание |
|-----|----------|
| `string` | Текстовые значения |
| `int` | Целые числа |
| `float` | Числа с плавающей точкой |
| `array` | Массив значений (используйте оператор <FlyqlToken type="operator">has</FlyqlToken> для проверки принадлежности) |

## Встроенные преобразователи

| Преобразователь | Вход | Выход | Описание |
|-----------------|------|-------|----------|
| <FlyqlToken type="transformer">upper</FlyqlToken> | string | string | Перевод в верхний регистр |
| <FlyqlToken type="transformer">lower</FlyqlToken> | string | string | Перевод в нижний регистр |
| <FlyqlToken type="transformer">len</FlyqlToken> | string | int | Длина строки |
| <FlyqlToken type="transformer">split</FlyqlToken> | string | array | Разделение по разделителю: `split(".")` |

## SQL-вывод по диалектам

Преобразователи генерируют вызовы функций, специфичные для каждого диалекта SQL. Цепочки преобразователей создают вложенные вызовы (самый внутренний выполняется первым):

**Один преобразователь:** <FlyqlCode code={`status|upper = "ERROR"`} inline />

| Диалект | Сгенерированный SQL |
|---------|---------------------|
| ClickHouse | `equals(upper(status), 'ERROR')` |
| PostgreSQL | `UPPER(status) = 'ERROR'` |
| StarRocks | `UPPER(\`status\`) = 'ERROR'` |

**Цепочка преобразователей:** <FlyqlCode code={`message|lower|len > 100`} inline />

| Диалект | Сгенерированный SQL |
|---------|---------------------|
| ClickHouse | `length(lower(message)) > 100` |
| PostgreSQL | `LENGTH(LOWER(message)) > 100` |
| StarRocks | `LENGTH(LOWER(\`message\`)) > 100` |

**Split с has:** <FlyqlCode code={`message|split(" ") has "hello"`} inline />

| Диалект | Сгенерированный SQL |
|---------|---------------------|
| ClickHouse | `has(splitByChar(' ', message), 'hello')` |
| PostgreSQL | `'hello' = ANY(STRING_TO_ARRAY(message, ' '))` |
| StarRocks | `` array_contains(SPLIT(`message`, ' '), 'hello') `` |

## Ошибки типов

Если цепочка содержит несовместимые типы, редактор выводит ошибку:

<FlyqlCode code={`message|len|upper`} />

Это вызовет ошибку, потому что <FlyqlToken type="transformer">len</FlyqlToken> возвращает `int`, а <FlyqlToken type="transformer">upper</FlyqlToken> требует `string` на входе. Редактор покажет: "len outputs int, upper requires string input".

## Примеры

<FlyqlCode code={`status|upper = "ERROR"`} />
<FlyqlCode code={`message|lower|len > 100`} />
<FlyqlCode code={`host|lower != "localhost"`} />
<FlyqlCode code={`path|len > 0`} />
<FlyqlCode code={`tags|split(",") has "urgent"`} />
