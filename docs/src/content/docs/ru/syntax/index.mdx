Запросы FlyQL состоят из одного или нескольких условий, объединённых булевыми операторами `and`, `or` и `not`. Каждое условие — это либо сравнение, либо проверка на истинность.

## Базовая структура запроса

```
status=200 and active and not archived
```

- `status=200` — поле `status` равно `200`
- `active` — поле `active` имеет истинное значение
- `not archived` — поле `archived` имеет ложное значение (`null`, пустое, `0` или `false`)

## Дополнительные примеры

```
service!=api or user="john doe"
```

```
message~"error.*" and not debug
```

```
(a=1 or b=2) and not (c=3 and d=4)
```

```
status in [200, 201] and method not in ['DELETE', 'PUT']
```

```
message has 'error' and tags not has 'debug'
```

## Общие правила

- **Отдельный ключ** — ключ без оператора трактуется как проверка на истинность
- **Сравнения** — при использовании оператора обязательно должен быть указан соответствующий value
- **Пробелы** — пробелы вокруг операторов допустимы: `status=200` и `status = 200` эквивалентны
- **Приоритет операторов** — `and` связывается сильнее, чем `or`
- **Скобки** — используйте `(` и `)` для явной группировки
