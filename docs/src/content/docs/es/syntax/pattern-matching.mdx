## Coincidencia regex

Usa <FlyqlToken type="operator">{'~'}</FlyqlToken> para comparar un valor con una expresión regular y <FlyqlToken type="operator">{'!~'}</FlyqlToken> para negar la coincidencia:

<FlyqlCode code={`message ~ "error.*timeout"`} />
<FlyqlCode code={`path !~ "^/health"`} />

| Operador | Significado |
|----------|-------------|
| <FlyqlToken type="operator">{'~'}</FlyqlToken> | El valor coincide con el patrón regex |
| <FlyqlToken type="operator">{'!~'}</FlyqlToken> | El valor no coincide con el patrón regex |

### Sintaxis de patrones regex

La sintaxis regex depende del backend de destino:

- **ClickHouse**: usa la función `match()` (sintaxis RE2)
- **PostgreSQL**: usa los operadores <FlyqlToken type="operator">{'~'}</FlyqlToken> y <FlyqlToken type="operator">{'!~'}</FlyqlToken> (regex POSIX)
- **StarRocks**: usa la función `regexp`
- **Matcher en memoria**: usa el motor regex del lenguaje host (Go `regexp`, Python `re2`, JS `RegExp`)

Pon el patrón entre comillas cuando contenga espacios o caracteres especiales:

<FlyqlCode code={`message ~ "connection (refused|reset)"`} />
<FlyqlCode code={`host ~ "^prod-[0-9]+"`} />

### Flags de regex

Puedes modificar el comportamiento de la regex usando flags inline al inicio del patrón con la sintaxis `(?...)`. FlyQL soporta tres flags que funcionan en todos los backends:

#### `(?i)` — Sin distinción de mayúsculas

Hace que el patrón coincida sin importar las mayúsculas o minúsculas:

<FlyqlCode code={`message ~ "(?i)error"`} />
<FlyqlCode code={`level ~ "(?i)^(warn|error|fatal)$"`} />

`(?i)error` coincide con "Error", "ERROR", "error", etc.

#### `(?m)` — Multilínea

Cambia `^` y `$` para que coincidan con el inicio y el final de cada **línea** en lugar de toda la cadena:

<FlyqlCode code={`log ~ "(?m)^ERROR"`} />

Sin `(?m)`, `^ERROR` solo coincide si la cadena completa empieza con "ERROR". Con `(?m)`, también coincide con "INFO first\nERROR second" porque "ERROR" aparece al inicio de una línea.

#### `(?s)` — Dotall

Hace que `.` coincida con **cualquier** carácter, incluidos los saltos de línea. Por defecto, `.` no coincide con `\n`:

<FlyqlCode code={`log ~ "(?s)start.*end"`} />

Sin `(?s)`, `start.*end` no coincide con "start\nend" porque `.` no cruza el salto de línea. Con `(?s)`, sí coincide.

#### Combinar flags

Los flags se pueden combinar en un solo grupo:

<FlyqlCode code={`log ~ "(?im)^ERROR"`} />
<FlyqlCode code={`log ~ "(?ims)^START.*END$"`} />

#### Compatibilidad con backends

Los tres flags funcionan en todos los backends de FlyQL:

| Backend | `(?i)` | `(?m)` | `(?s)` |
|---------|--------|--------|--------|
| **ClickHouse** | Yes | Yes | Yes |
| **PostgreSQL** | Yes | Yes | Yes |
| **StarRocks** | Yes | Yes | Yes |
| **Go matcher** | Yes | Yes | Yes |
| **Python matcher** | Yes | Yes | Yes |
| **JavaScript matcher** | Yes | Yes | Yes |

Combina la coincidencia sin distinción de mayúsculas con negación para excluir patrones sin importar el caso:

<FlyqlCode code={`message ~ "(?i)error" and message !~ "(?i)warning"`} />

## Coincidencia con comodines

Usa <FlyqlToken type="operator">{'*'}</FlyqlToken> al final de un valor sin comillas para hacer coincidencia por prefijo:

<FlyqlCode code={`host = prod*`} />
<FlyqlCode code={`service = api-*`} />

Esto coincide con cualquier valor que empiece por el prefijo dado. Los comodines solo se admiten al final del valor; `*prod` y `pr*od` no están soportados.

### Comodines frente a regex

| Patrón | Sintaxis | Ejemplo |
|--------|----------|---------|
| Comodín por prefijo | <FlyqlCode code={`key = value*`} inline /> | <FlyqlCode code={`host = prod*`} inline /> |
| Regex completa | <FlyqlCode code={`key ~ "pattern"`} inline /> | <FlyqlCode code={`host ~ "^prod-[0-9]+"`} inline /> |

Usa comodines para coincidencias simples por prefijo. Usa regex cuando necesites patrones más complejos.

## Ejemplos

<FlyqlCode code={`message ~ "error|fail"`} />
<FlyqlCode code={`path ~ "^/api/v2"`} />
<FlyqlCode code={`path !~ "^/health"`} />
<FlyqlCode code={`host = prod*`} />
<FlyqlCode code={`host = prod* and message ~ "timeout.*" and path !~ "^/health"`} />
