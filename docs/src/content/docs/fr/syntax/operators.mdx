FlyQL prend en charge les operateurs de comparaison suivants :

## Egalite

| Operateur | Nom | Exemple |
|-----------|-----|---------|
| <FlyqlToken type="operator">=</FlyqlToken> | Egal | <FlyqlCode code={`status = 200`} inline /> |
| <FlyqlToken type="operator">!=</FlyqlToken> | Different | <FlyqlCode code={`env != 'prod'`} inline /> |

<FlyqlCode code={`status = 200`} />
<FlyqlCode code={`service != "api-gateway"`} />

Les deux cotes de l'operateur sont requis — une <FlyqlToken type="key">cle</FlyqlToken> et une <FlyqlToken type="value">valeur</FlyqlToken>.

## Comparaison numerique

| Operateur | Nom | Exemple |
|-----------|-----|---------|
| <FlyqlToken type="operator">{'>'}</FlyqlToken> | Superieur a | <FlyqlCode code={`status > 399`} inline /> |
| <FlyqlToken type="operator">{'>='}</FlyqlToken> | Superieur ou egal | <FlyqlCode code={`status >= 400`} inline /> |
| <FlyqlToken type="operator">{'<'}</FlyqlToken> | Inferieur a | <FlyqlCode code={`duration < 1000`} inline /> |
| <FlyqlToken type="operator">{'<='}</FlyqlToken> | Inferieur ou egal | <FlyqlCode code={`duration <= 500`} inline /> |

<FlyqlCode code={`status >= 400 and duration < 5000`} />
<FlyqlCode code={`response_time > 100 and response_time <= 2000`} />

Les operateurs de comparaison numerique fonctionnent sur les valeurs numeriques. Lorsqu'ils sont utilises avec des valeurs de type chaine, la comparaison lexicographique s'applique.

## Correspondance par regex

| Operateur | Nom | Exemple |
|-----------|-----|---------|
| <FlyqlToken type="operator">{'~'}</FlyqlToken> | Correspondance regex | <FlyqlCode code={`message ~ "error.*"`} inline /> |
| <FlyqlToken type="operator">{'!~'}</FlyqlToken> | Non-correspondance regex | <FlyqlCode code={`path !~ "^/health"`} inline /> |

<FlyqlCode code={`message ~ "timeout|connection refused"`} />
<FlyqlCode code={`host !~ "^test-.*"`} />

Les motifs regex suivent la syntaxe de la base de donnees ou du runtime cible. Consultez [Correspondance de motifs](/fr/syntax/pattern-matching/) pour les drapeaux regex et les details.

:::note
Les operateurs `~` et `!~` ne peuvent pas etre utilises avec des valeurs de fonctions temporelles (par ex., `ago(1h)`, `now()`). Cela produit une erreur d'analyse. De meme, les operateurs `in`, `has`, `like` et `ilike` ne sont pas compatibles avec les valeurs de fonction.
:::

## Appartenance a une liste

| Operateur | Nom | Exemple |
|-----------|-----|---------|
| <FlyqlToken type="operator">in</FlyqlToken> | Dans la liste | <FlyqlCode code={`status in [200, 201]`} inline /> |
| <FlyqlToken type="operator">not in</FlyqlToken> | Pas dans la liste | <FlyqlCode code={`env not in ['dev', 'test']`} inline /> |

Consultez la page [Listes](/fr/syntax/lists/) pour les regles detaillees de syntaxe des listes.

## Correspondance de motifs (LIKE)

| Operateur | Nom | Exemple |
|-----------|-----|---------|
| <FlyqlToken type="operator">like</FlyqlToken> | Motif like | <FlyqlCode code={`message like 'error%'`} inline /> |
| <FlyqlToken type="operator">not like</FlyqlToken> | Non-correspondance like | <FlyqlCode code={`host not like 'test%'`} inline /> |
| <FlyqlToken type="operator">ilike</FlyqlToken> | Like insensible a la casse | <FlyqlCode code={`message ilike 'Error%'`} inline /> |
| <FlyqlToken type="operator">not ilike</FlyqlToken> | Not like insensible a la casse | <FlyqlCode code={`host not ilike 'TEST%'`} inline /> |

<FlyqlCode code={`message like '%timeout%'`} />
<FlyqlCode code={`host not like '%-staging'`} />
<FlyqlCode code={`path ilike '/API/%'`} />

Correspondance de motifs de type SQL avec deux jokers :

- <FlyqlToken type="operator">%</FlyqlToken> — correspond a toute sequence de caracteres (y compris vide)
- <FlyqlToken type="operator">_</FlyqlToken> — correspond a exactement un caractere

Echappez les jokers avec un antislash pour les prendre au sens litteral : `\%` correspond a un `%` litteral, `\_` correspond a un `_` litteral.

Voir [Correspondance de motifs](/fr/syntax/pattern-matching/) pour plus de details et d'exemples.

## Verification d'inclusion

| Operateur | Nom | Exemple |
|-----------|-----|---------|
| <FlyqlToken type="operator">has</FlyqlToken> | Contient | <FlyqlCode code={`message has 'error'`} inline /> |
| <FlyqlToken type="operator">not has</FlyqlToken> | Ne contient pas | <FlyqlCode code={`message not has 'test'`} inline /> |

<FlyqlCode code={`message has 'timeout'`} />
<FlyqlCode code={`tags not has 'deprecated'`} />

L'operateur <FlyqlToken type="operator">has</FlyqlToken> effectue une verification d'inclusion dependante du type :

- **Chaine** — correspondance de sous-chaine : <FlyqlCode code={`message has 'error'`} inline /> est vrai si <FlyqlToken type="key">message</FlyqlToken> contient <FlyqlToken type="string">"error"</FlyqlToken>
- **Map** — existence de cle : <FlyqlCode code={`metadata has 'region'`} inline /> est vrai si le map possede une cle <FlyqlToken type="string">"region"</FlyqlToken>
- **Tableau/Liste** — appartenance d'element : <FlyqlCode code={`tags has 'web'`} inline /> est vrai si le tableau contient <FlyqlToken type="string">"web"</FlyqlToken>
- **JSON** — existence de cle : <FlyqlCode code={`data has 'name'`} inline /> est vrai si l'objet JSON possede une cle <FlyqlToken type="string">"name"</FlyqlToken>
- **Null/vide** — toujours faux pour <FlyqlToken type="operator">has</FlyqlToken>, toujours vrai pour <FlyqlToken type="operator">not has</FlyqlToken>

<FlyqlToken type="operator">has</FlyqlToken> prend une seule valeur (pas une liste). Utilisez <FlyqlToken type="operator">in</FlyqlToken> pour verifier l'appartenance a un ensemble de valeurs.
