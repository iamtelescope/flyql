ClickHouse SQL uses unquoted column names, `match()` for regex, and `LIKE` for wildcards.

## Basic Examples

| FlyQL | Generated SQL |
|-------|--------------|
| <FlyqlCode code={`message = 'hello'`} inline /> | `message = 'hello'` |
| <FlyqlCode code={`message != 'hello'`} inline /> | `message != 'hello'` |
| <FlyqlCode code={`count > 10`} inline /> | `count > 10` |
| <FlyqlCode code={`count <= 100`} inline /> | `count <= 100` |
| <FlyqlCode code={`message ~ 'error.*'`} inline /> | `match(message, 'error.*')` |
| <FlyqlCode code={`message !~ 'test.*'`} inline /> | `not match(message, 'test.*')` |
| <FlyqlCode code={`host = 'prod'`} inline /> | `host = 'prod'` |

## Truthy Checks

A bare column reference (`active`, `not active`) generates type-aware SQL — there is no single "is truthy" expression, because what counts as falsy differs by column type.

| Column Type | FlyQL | Generated SQL |
|-------------|-------|--------------|
| String | <FlyqlCode code={`message`} inline /> | `(message IS NOT NULL AND message != '')` |
| Int / Float | <FlyqlCode code={`count`} inline /> | `(count IS NOT NULL AND count != 0)` |
| Bool | <FlyqlCode code={`active`} inline /> | `active` |
| Date | <FlyqlCode code={`created_at`} inline /> | `(created_at IS NOT NULL)` |
| Negated String | <FlyqlCode code={`not message`} inline /> | `(message IS NULL OR message = '')` |

## Containment (`has` / `not has`)

The <FlyqlToken type="operator">has</FlyqlToken> operator generates different SQL depending on the column type:

| Column Type | FlyQL | Generated SQL |
|-------------|-------|--------------|
| String | <FlyqlCode code={`message has 'error'`} inline /> | `position(message, 'error') > 0` |
| String (negated) | <FlyqlCode code={`message not has 'error'`} inline /> | `(message IS NULL OR position(message, 'error') = 0)` |
| Array | <FlyqlCode code={`tags has 'web'`} inline /> | `has(tags, 'web')` |
| Array (negated) | <FlyqlCode code={`tags not has 'web'`} inline /> | `NOT has(tags, 'web')` |
| Map | <FlyqlCode code={`metadata has 'key'`} inline /> | `mapContains(metadata, 'key')` |
| Map (negated) | <FlyqlCode code={`metadata not has 'key'`} inline /> | `NOT mapContains(metadata, 'key')` |
| Native JSON | <FlyqlCode code={`data has 'key'`} inline /> | `JSON_EXISTS(data, concat('$.', 'key'))` |
| JSON String | <FlyqlCode code={`data has 'key'`} inline /> | `JSONHas(data, 'key')` |

## JSON Columns

ClickHouse supports two JSON column types with different SQL output.

### Native JSON Type

Columns declared as `JSON` use dot notation with backtick-quoted segments:

| FlyQL | Generated SQL |
|-------|--------------|
| <FlyqlCode code={`data.name = 'test'`} inline /> | `` data.`name` = 'test' `` |
| <FlyqlCode code={`data.user.name = 'john'`} inline /> | `` data.`user`.`name` = 'john' `` |
| <FlyqlCode code={`data.age = 25`} inline /> | `` data.`age` = 25 `` |

### JSON String Type

Columns storing JSON as a string (e.g., `String` type with JSON content) use `JSONExtract` functions wrapped in `multiIf` for type checking:

| FlyQL | Generated SQL |
|-------|--------------|
| <FlyqlCode code={`data.name = 'test'`} inline /> | `multiIf(JSONType(data, 'name') = 'String', equals(JSONExtractString(data, 'name'), 'test'), 0)` |
| <FlyqlCode code={`data.user.name = 'john'`} inline /> | `multiIf(JSONType(data, 'user', 'name') = 'String', equals(JSONExtractString(data, 'user', 'name'), 'john'), 0)` |

## Map Columns

Map columns use bracket notation with `equals()`/`notEquals()` functions:

| FlyQL | Generated SQL |
|-------|--------------|
| <FlyqlCode code={`metadata.key1 = 'value1'`} inline /> | `equals(metadata['key1'], 'value1')` |
| <FlyqlCode code={`metadata.key1 != 'value1'`} inline /> | `notEquals(metadata['key1'], 'value1')` |
| <FlyqlCode code={`metadata.pattern ~ 'test.*'`} inline /> | `match(metadata['pattern'], 'test.*')` |

## Array Columns

FlyQL array indices are 0-based; the ClickHouse generator adds 1 when emitting SQL because ClickHouse arrays are 1-indexed at the SQL layer:

| FlyQL | Generated SQL |
|-------|--------------|
| <FlyqlCode code={`tags.0 = 'first'`} inline /> | `equals(tags[1], 'first')` |
| <FlyqlCode code={`tags.1 = 'second'`} inline /> | `equals(tags[2], 'second')` |
| <FlyqlCode code={`tags.0 ~ 'tag.*'`} inline /> | `match(tags[1], 'tag.*')` |

### Column Setup

Text/varchar columns whose contents are valid JSON are declared with the synthetic `jsonstring` raw type:

```go
// Go
columns := map[string]*clickhouse.Column{
    "message": clickhouse.NewColumn(clickhouse.ColumnDef{Name: "message", Type: "String"}),
    "data":    clickhouse.NewColumn(clickhouse.ColumnDef{Name: "data", Type: "JSON"}),       // native JSON
    "log":     clickhouse.NewColumn(clickhouse.ColumnDef{Name: "log", Type: "jsonstring"}),  // JSON-in-text
}
```

```javascript
// JavaScript
const columns = {
    message: newColumn("message", "String"),
    data:    newColumn("data", "JSON"),        // native JSON
    log:     newColumn("log", "jsonstring"),    // JSON-in-text
}
```

## Transformers

Transformers generate ClickHouse-specific function calls:

| FlyQL | Generated SQL |
|-------|--------------|
| <FlyqlCode code={'status\|upper = "ERROR"'} inline /> | `equals(upper(status), 'ERROR')` |
| <FlyqlCode code={'status\|lower = "info"'} inline /> | `equals(lower(status), 'info')` |
| <FlyqlCode code={'message\|len > 100'} inline /> | `length(message) > 100` |
| <FlyqlCode code={'message\|lower\|len > 100'} inline /> | `length(lower(message)) > 100` |
| <FlyqlCode code={'tags\|split(",") has "urgent"'} inline /> | `has(splitByChar(',', tags), 'urgent')` |
