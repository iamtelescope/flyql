Le SQL PostgreSQL utilise des identifiants entre guillemets doubles, <FlyqlToken type="operator">{'~'}</FlyqlToken> pour les regex, <FlyqlToken type="operator">and</FlyqlToken>/<FlyqlToken type="operator">or</FlyqlToken> en majuscules, et des operateurs JSONB pour l'acces imbrique.

## Exemples de base

| FlyQL | SQL genere |
|-------|--------------|
| <FlyqlCode code={`message = 'hello'`} inline /> | `"message" = 'hello'` |
| <FlyqlCode code={`message != 'hello'`} inline /> | `"message" != 'hello'` |
| <FlyqlCode code={`count > 10`} inline /> | `"count" > 10` |
| <FlyqlCode code={`count <= 100`} inline /> | `"count" <= 100` |
| <FlyqlCode code={`message ~ 'error.*'`} inline /> | `"message" ~ 'error.*'` |
| <FlyqlCode code={`message !~ 'test.*'`} inline /> | `"message" !~ 'test.*'` |
| <FlyqlCode code={`host = 'prod'`} inline /> | `"host" = 'prod'` |

## Operateurs booleens

PostgreSQL utilise <FlyqlToken type="operator">and</FlyqlToken>/<FlyqlToken type="operator">or</FlyqlToken> en majuscules (contrairement a ClickHouse et StarRocks) :

| FlyQL | SQL genere |
|-------|--------------|
| <FlyqlCode code={`a = '1' and b > 10`} inline /> | `("a" = '1' AND "b" > 10)` |
| <FlyqlCode code={`a = '1' or b = '2'`} inline /> | `("a" = '1' OR "b" = '2')` |
| <FlyqlCode code={`not a = '1'`} inline /> | `NOT ("a" = '1')` |

## Appartenance a une liste

| FlyQL | SQL genere |
|-------|--------------|
| <FlyqlCode code={`status in [200, 201]`} inline /> | `"status" IN (200, 201)` |
| <FlyqlCode code={`env not in ['dev', 'test']`} inline /> | `"env" NOT IN ('dev', 'test')` |
| <FlyqlCode code={`status in []`} inline /> | `FALSE` |
| <FlyqlCode code={`status not in []`} inline /> | `TRUE` |

## Inclusion (`has` / `not has`)

L'operateur <FlyqlToken type="operator">has</FlyqlToken> genere un SQL different selon le type de colonne :

| Type de colonne | FlyQL | SQL genere |
|-------------|-------|--------------|
| String | <FlyqlCode code={`message has 'error'`} inline /> | `position('error' in "message") > 0` |
| String (inverse) | <FlyqlCode code={`message not has 'error'`} inline /> | `("message" IS NULL OR position('error' in "message") = 0)` |
| Array | <FlyqlCode code={`tags has 'web'`} inline /> | `'web' = ANY("tags")` |
| Array (inverse) | <FlyqlCode code={`tags not has 'web'`} inline /> | `NOT ('web' = ANY("tags"))` |
| JSONB | <FlyqlCode code={`data has 'key'`} inline /> | `"data" ? 'key'` |
| JSONB (inverse) | <FlyqlCode code={`data not has 'key'`} inline /> | `NOT ("data" ? 'key')` |
| Hstore | <FlyqlCode code={`metadata has 'key'`} inline /> | `"metadata" ? 'key'` |

## Colonnes JSONB

PostgreSQL utilise des operateurs JSONB avec verification de type via `jsonb_typeof()` :

### Comparaison de chaines

| FlyQL | SQL genere |
|-------|--------------|
| <FlyqlCode code={`data.name = 'test'`} inline /> | `(jsonb_typeof("data"->'name') = 'string' AND "data"->>'name' = 'test')` |
| <FlyqlCode code={`data.user.name = 'john'`} inline /> | `(jsonb_typeof("data"->'user'->'name') = 'string' AND "data"->'user'->>'name' = 'john')` |

### Comparaison numerique

Les comparaisons numeriques effectuent un cast de la valeur extraite :

| FlyQL | SQL genere |
|-------|--------------|
| <FlyqlCode code={`data.age > 25`} inline /> | `(jsonb_typeof("data"->'age') = 'number' AND ("data"->>'age')::numeric > 25)` |

### Operateurs JSONB

- `->` retourne du JSONB (utilise pour la traversee de chemin intermediaire)
- `->>` retourne du texte (utilise pour l'extraction de la valeur finale)
- `jsonb_typeof()` verifie le type JSON avant la comparaison

## Colonnes Hstore (Map)

PostgreSQL utilise le type hstore avec l'operateur `->` pour l'acces aux cles :

| FlyQL | SQL genere |
|-------|--------------|
| <FlyqlCode code={`metadata.key1 = 'value1'`} inline /> | `"metadata"->'key1' = 'value1'` |
| <FlyqlCode code={`metadata.key1 != 'value1'`} inline /> | `"metadata"->'key1' != 'value1'` |
| <FlyqlCode code={`metadata.pattern ~ 'test.*'`} inline /> | `"metadata"->'pattern' ~ 'test.*'` |

## Colonnes Array

Les tableaux PostgreSQL sont indexes a partir de 1 — FlyQL convertit automatiquement l'index base 0 :

| FlyQL | SQL genere |
|-------|--------------|
| <FlyqlCode code={`tags.0 = 'first'`} inline /> | `"tags"[1] = 'first'` |
| <FlyqlCode code={`tags.1 = 'second'`} inline /> | `"tags"[2] = 'second'` |
| <FlyqlCode code={`tags.0 ~ 'tag.*'`} inline /> | `"tags"[1] ~ 'tag.*'` |

### Configuration des colonnes

Les colonnes PostgreSQL n'ont pas de parametre `jsonString` — le type JSONB est detecte a partir du type de colonne :

```go
// Go
columns := map[string]*postgresql.Column{
    "message": postgresql.NewColumn("message", "varchar", nil),
    "data":    postgresql.NewColumn("data", "jsonb", nil),
}
```

```javascript
// JavaScript
const columns = {
    message: newColumn("message", "varchar", null),
    data:    newColumn("data", "jsonb", null),
}
```

## Transformateurs

Les transformateurs generent des appels de fonctions standard PostgreSQL :

| FlyQL | SQL genere |
|-------|--------------|
| <FlyqlCode code={'status\|upper = "ERROR"'} inline /> | `UPPER(status) = 'ERROR'` |
| <FlyqlCode code={'status\|lower = "info"'} inline /> | `LOWER(status) = 'info'` |
| <FlyqlCode code={'message\|len > 100'} inline /> | `LENGTH(message) > 100` |
| <FlyqlCode code={'message\|lower\|len > 100'} inline /> | `LENGTH(LOWER(message)) > 100` |
| <FlyqlCode code={'tags\|split(",") has "urgent"'} inline /> | `'urgent' = ANY(STRING_TO_ARRAY(tags, ','))` |
