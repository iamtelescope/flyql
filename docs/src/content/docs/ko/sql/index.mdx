FlyQL generator는 모든 값을 포함한(escape된) 일반 SQL 문자열 형태의 WHERE 절 fragment를 생성합니다. 동일한 FlyQL 쿼리라도 대상 dialect에 따라 서로 다른 SQL이 생성됩니다.

## 지원하는 방언

- [ClickHouse](/ko/sql/clickhouse/) — column quoting 없음, regex에는 `match()`, JSON 문자열에는 `JSONExtract` 함수 사용
- [PostgreSQL](/ko/sql/postgresql/) — 큰따옴표 identifier, regex에는 `~`, JSONB operator 사용
- [StarRocks](/ko/sql/starrocks/) — 백틱 identifier, regex에는 `regexp()`, JSON 문자열에는 `parse_json()` 사용

## 주요 차이점

| 기능 | ClickHouse | PostgreSQL | StarRocks |
|---------|-----------|-----------|-----------|
| Column quoting | 없음 | `"name"` | `` `name` `` |
| Regex | `match(col, 'pat')` | `"col" ~ 'pat'` | `` regexp(`col`, 'pat') `` |
| Negated regex | `NOT match(col, 'pat')` | `"col" !~ 'pat'` | `` NOT regexp(`col`, 'pat') `` |
| Wildcard | `col LIKE 'val%'` | `"col" LIKE 'val%'` | `` `col` LIKE 'val%' `` |
| Boolean ops | `AND` / `OR` | `AND` / `OR` | `AND` / `OR` |
| Empty `IN []` | `0` | `FALSE` | `0` |
| Empty `NOT IN []` | `1` | `TRUE` | `1` |
| String <FlyqlToken type="operator">has</FlyqlToken> | `position(col, 'v') > 0` | `position('v' in "col") > 0` | `` INSTR(`col`, 'v') > 0 `` |
| Array <FlyqlToken type="operator">has</FlyqlToken> | `has(col, 'v')` | `'v' = ANY("col")` | `` array_contains(`col`, 'v') `` |
| Map <FlyqlToken type="operator">has</FlyqlToken> | `mapContains(col, 'k')` | `"col" ? 'k'` | `` array_contains(map_keys(`col`), 'k') `` |
| JSON <FlyqlToken type="operator">has</FlyqlToken> | `JSONHas(toJSONString(col), 'k')` | `"col" ? 'k'` | `json_exists(...)` |

## JSON 접근: 가장 큰 차이점

각 dialect는 중첩 JSON column을 매우 다르게 처리합니다.

**FlyQL 입력:** <FlyqlCode code={`data.user.name = 'john'`} inline />

**ClickHouse** (native JSON column):
```sql
data.`user`.`name` = 'john'
```

**ClickHouse** (문자열로 저장된 JSON):
```sql
multiIf(JSONType(data, 'user', 'name') = 'String',
  equals(JSONExtractString(data, 'user', 'name'), 'john'), 0)
```

**PostgreSQL** (JSONB):
```sql
(jsonb_typeof("data"->'user'->'name') = 'string'
  AND "data"->'user'->>'name' = 'john')
```

**StarRocks** (native JSON):
```sql
`data`->'\"user\"'->'\"name\"' = 'john'
```

전체 예제는 각 dialect 페이지를 참고하세요.

FlyQL의 배열 인덱스는 모든 방언에서 일관되게 0 기반입니다. 세 방언 모두 SQL 계층에서 1 기반 배열을 사용하므로, 생성기는 출력하기 전에 flyql 인덱스에 1을 더합니다. 동일한 FlyQL 표현식은 방언마다 서로 다른 SQL 문자열을 생성하지만(식별자 인용 및 함수 구문의 차이), 동일한 기반 데이터에 대해 동일한 결과로 평가됩니다.

## 출력 형식

Generator는 WHERE 절 fragment를 반환합니다. `WHERE` 키워드도 없고, parameterized placeholder도 없습니다. 모든 값은 escape되어 직접 포함됩니다.

```
message = 'hello' and count > 10
```

다음과 같은 형태가 아닙니다:

```
WHERE message = ? AND count > ?
```
