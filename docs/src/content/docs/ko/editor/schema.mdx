---
title: Schema 설정
description: FlyQL editor를 위한 컬럼 schema 구조
---

`columns` prop은 editor 제안을 구동하는 데이터 schema를 정의합니다. 키는 컬럼 이름이고, 값은 컬럼 정의입니다.

## 기본 schema

```javascript
import { ColumnSchema } from 'flyql-vue'

const columns = ColumnSchema.fromPlainObject({
    level: { type: 'enum', suggest: true, autocomplete: true, values: ['debug', 'info', 'error'] },
    service: { type: 'string', suggest: true, autocomplete: true },
    status_code: { type: 'number', suggest: true },
    host: { type: 'string', suggest: true },
})
```

## 컬럼 정의 필드

| 필드 | 타입 | 기본값 | 설명 |
|------|------|--------|------|
| `type` | `string` | — | 컬럼 타입: `'string'`, `'number'`, `'enum'`, `'object'` |
| `suggest` | `boolean` | `true` | 이 컬럼을 제안 목록에 표시 |
| `autocomplete` | `boolean` | `false` | 이 컬럼의 값 autocomplete 활성화 |
| `values` | `string[]` 또는 `number[]` | — | autocomplete용 정적 값 (`onAutocomplete` callback보다 먼저 사용됨) |
| `children` | `object` | — | object 타입 컬럼의 중첩 컬럼 정의 |
| `display_name` | `string` | `''` | 선택적 사용자 친화적 표시 이름. parse된 컬럼에서는 alias가 이 값을 덮어씀 |

## 컬럼 타입

### `string`

일반 텍스트 컬럼입니다. regex(`~`, `!~`)를 포함한 모든 비교 연산자를 사용할 수 있습니다.

```javascript
{ type: 'string', suggest: true }
```

### `number`

숫자 컬럼입니다. regex 연산자(`~`, `!~`)는 연산자 제안에서 제외됩니다.

```javascript
{ type: 'number', suggest: true }
```

### `enum`

고정된 값 집합을 가지는 enum 컬럼입니다. `number`와 마찬가지로 regex 연산자는 제외됩니다. 값 집합은 `values`로 제공합니다.

```javascript
{ type: 'enum', suggest: true, autocomplete: true, values: ['debug', 'info', 'warning', 'error', 'critical'] }
```

### `object`

자식 컬럼을 가진 중첩 컬럼입니다. 사용자가 컬럼 이름 뒤에 `.`를 입력하면 자식 컬럼이 제안됩니다.

```javascript
{
    type: 'object',
    suggest: true,
    children: {
        labels: {
            type: 'object',
            suggest: true,
            children: {
                tier: { type: 'string', suggest: true },
                env: { type: 'string', suggest: true },
            },
        },
        version: { type: 'string', suggest: true },
    },
}
```

## 값 autocomplete

### 정적 값

선택지가 이미 정해져 있는 컬럼에는 `values` 배열을 제공합니다.

```javascript
{
    level: {
        type: 'enum',
        suggest: true,
        autocomplete: true,
        values: ['debug', 'info', 'warning', 'error', 'critical'],
    },
    status_code: {
        type: 'number',
        suggest: true,
        autocomplete: true,
        values: [200, 201, 204, 400, 404, 500],
    },
}
```

### 비동기 값

`values` 없이 `autocomplete: true`를 설정하고 editor 컴포넌트에 `onAutocomplete` callback을 제공합니다. 사용자가 해당 컬럼의 값 위치까지 입력하면 callback이 호출됩니다.

```javascript
// 컬럼 정의 — 정적 값 없음
{ type: 'string', suggest: true, autocomplete: true }

// Editor callback이 API에서 값을 가져옴
async function onAutocomplete(key, value) {
    const resp = await fetch(`/api/values?column=${key}`)
    const data = await resp.json()
    return { items: data.values }
}
```

## 중첩 컬럼 (JSONSchema)

object 타입 컬럼의 `children`을 사용해 중첩 구조를 정의합니다.

```javascript
const columns = ColumnSchema.fromPlainObject({
    metadata: {
        type: 'object',
        suggest: true,
        children: {
            labels: {
                type: 'object',
                suggest: true,
                children: {
                    tier: { type: 'string', suggest: true, autocomplete: true, values: ['dev', 'staging', 'prod'] },
                    env: { type: 'string', suggest: true, autocomplete: true },
                },
            },
            version: { type: 'string', suggest: true },
        },
    },
})
```

`metadata.`를 입력하면 `labels`와 `version`이 제안되고, `metadata.labels.`를 입력하면 `tier`와 `env`가 제안됩니다. 깊이 제한은 없습니다.

## 원격 키 탐색 (Schema-less)

Schema 정의 시점에 중첩 키를 알 수 없는 object 컬럼의 경우 `children`을 생략하고 `onKeyDiscovery` callback을 제공합니다.

```javascript
const columns = ColumnSchema.fromPlainObject({
    request: { type: 'object', suggest: true }, // 자식 없음 — discovery 트리거
}

async function onKeyDiscovery(columnName, segments) {
    const resp = await fetch(`/api/keys?column=${columnName}&path=${segments.join(',')}`)
    const data = await resp.json()
    return data.keys // Array<{ name: string, type?: string, hasChildren?: boolean }>
})
```

사용자가 `request.`를 입력하면 callback은 `('request', ['request'])`로 호출됩니다. 탐색된 키는 활성 세션 단위로 캐시됩니다.

한 컬럼에 `children`과 `onKeyDiscovery`가 모두 있으면 정적 `children`이 우선합니다.

## 전체 예제

```javascript
const columns = ColumnSchema.fromPlainObject({
    // 정적 값을 가진 enum
    level: { type: 'enum', suggest: true, autocomplete: true, values: ['debug', 'info', 'error'] },

    // 비동기 autocomplete를 사용하는 string
    service: { type: 'string', suggest: true, autocomplete: true },

    // number — regex 연산자 제외
    status_code: { type: 'number', suggest: true, autocomplete: true, values: [200, 400, 500] },

    // 단순 string
    host: { type: 'string', suggest: true },

    // 알려진 자식을 가진 중첩 컬럼
    metadata: {
        type: 'object',
        suggest: true,
        children: {
            labels: {
                type: 'object',
                suggest: true,
                children: {
                    tier: { type: 'string', suggest: true, autocomplete: true, values: ['dev', 'staging', 'prod'] },
                    env: { type: 'string', suggest: true, autocomplete: true },
                },
            },
            version: { type: 'string', suggest: true },
        },
    },

    // Schema-less — 중첩 키를 런타임에 탐색
    request: { type: 'object', suggest: true },
})
```
