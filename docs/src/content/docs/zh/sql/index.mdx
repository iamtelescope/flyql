FlyQL 生成器会输出一个纯 SQL 的 WHERE 条件片段，所有值都会在输出前被正确转义并直接嵌入。相同的 FlyQL 查询，在不同数据库方言下会生成不同的 SQL。

## 支持的方言

- [ClickHouse](/zh/sql/clickhouse/)：列名通常不加引号，正则使用 `match()`
- [PostgreSQL](/zh/sql/postgresql/)：使用双引号标识符，正则使用 `~`
- [StarRocks](/zh/sql/starrocks/)：使用反引号标识符，正则使用 `regexp()`

## 主要差异

| 特性 | ClickHouse | PostgreSQL | StarRocks |
|------|-----------|-----------|-----------|
| 列名引用 | 无 | `"name"` | `` `name` `` |
| 正则 | `match(col, 'pat')` | `"col" ~ 'pat'` | `` regexp(`col`, 'pat') `` |
| 取反正则 | `NOT match(col, 'pat')` | `"col" !~ 'pat'` | `` NOT regexp(`col`, 'pat') `` |
| 通配前缀 | `col LIKE 'val%'` | `"col" LIKE 'val%'` | `` `col` LIKE 'val%' `` |
| 布尔操作符 | `AND` / `OR` | `AND` / `OR` | `AND` / `OR` |
| 空 `IN []` | `0` | `FALSE` | `0` |
| 空 `NOT IN []` | `1` | `TRUE` | `1` |
| 字符串 <FlyqlToken type="operator">has</FlyqlToken> | `position(col, 'v') > 0` | `position('v' in "col") > 0` | `` INSTR(`col`, 'v') > 0 `` |
| 数组 <FlyqlToken type="operator">has</FlyqlToken> | `has(col, 'v')` | `'v' = ANY("col")` | `` array_contains(`col`, 'v') `` |
| Map <FlyqlToken type="operator">has</FlyqlToken> | `mapContains(col, 'k')` | `"col" ? 'k'` | `` array_contains(map_keys(`col`), 'k') `` |
| JSON <FlyqlToken type="operator">has</FlyqlToken> | `JSONHas(toJSONString(col), 'k')` | `"col" ? 'k'` | `json_exists(...)` |

## JSON 访问

不同方言在处理嵌套 JSON 时差异最大。

**FlyQL 输入：** <FlyqlCode code={`data.user.name = 'john'`} inline />

**ClickHouse（原生 JSON）：**

```sql
data.`user`.`name` = 'john'
```

**ClickHouse（字符串形式 JSON）：**

```sql
multiIf(JSONType(data, 'user', 'name') = 'String',
  equals(JSONExtractString(data, 'user', 'name'), 'john'), 0)
```

**PostgreSQL（JSONB）：**

```sql
(jsonb_typeof("data"->'user'->'name') = 'string'
  AND "data"->'user'->>'name' = 'john')
```

**StarRocks（原生 JSON）：**

```sql
`data`->'\"user\"'->'\"name\"' = 'john'
```

FlyQL 的数组索引在所有方言中均统一为 0 起始。三种方言在 SQL 层都使用 1 起始的数组，因此生成器在输出前会将 flyql 索引加 1。同一个 FlyQL 表达式在不同方言下生成不同的 SQL 字符串（由于标识符引用和函数语法的差异），但对相同底层数据评估为相同的结果。

## 输出格式

生成器返回的是 WHERE 条件片段，而不是完整 SQL：

```
message = 'hello' and count > 10
```

而不是：

```
WHERE message = ? AND count > ?
```
