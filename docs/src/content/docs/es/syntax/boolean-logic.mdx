## Operadores booleanos

FlyQL usa tres operadores booleanos para combinar condiciones:

| Operador | Significado | Ejemplo |
|----------|-------------|---------|
| `and` | Ambas condiciones deben ser true | `status = 200 and active` |
| `or` | Cualquiera de las dos condiciones puede ser true | `level = error or level = critical` |
| `not` | Niega una expresión | `not archived` |

### Negación combinada

`not` puede combinarse con `and` y `or`:

```
active and not archived
level = error or not healthy
```

## Precedencia de operadores

`and` tiene mayor precedencia que `or`. Esto significa que:

```
a = 1 or b = 2 and c = 3
```

se parsea como:

```
a = 1 or (b = 2 and c = 3)
```

Para cambiar la precedencia, usa paréntesis de forma explícita.

## Agrupación con paréntesis

Usa paréntesis para controlar el orden de evaluación:

```
(status >= 400 or level = error) and service = api
```

Sin paréntesis, `and` tendría mayor precedencia:

```
status >= 400 or level = error and service = api
→ status >= 400 or (level = error and service = api)
```

### Agrupación anidada

Los paréntesis pueden anidarse:

```
(a = 1 or b = 2) and not (c = 3 and d = 4)
```

### Negar grupos

`not` se aplica a todo el grupo:

```
not (status = 200 and healthy)
```

Esto niega la condición combinada: solo es true cuando `status` no es 200 o `healthy` es false.

## Negación

`not` niega cualquier expresión: una comprobación truthy, una comparación o un grupo:

```
not active                  # field is falsy
not status = 200              # status is not 200
not (a = 1 and b = 2)           # negates the group
active and not archived     # combined with other conditions
```

La doble negación se cancela: `not not active` es equivalente a `active`.

## Ejemplos

```
# AND simple
status = 200 and service = api

# OR simple
level = error or level = critical

# Mezclado con precedencia
status >= 400 or level = error and service = api

# Agrupación explícita
(status >= 400 or level = error) and service = api

# Consulta compleja con negación y agrupación
(status >= 500 and host = prod*) or (level = error and not service = healthcheck)
```
