## 불리언 연산자

FlyQL은 조건을 결합하기 위해 세 가지 불리언 연산자를 사용합니다.

| 연산자 | 의미 | 예제 |
|--------|------|------|
| <FlyqlToken type="operator">and</FlyqlToken> | 두 조건이 모두 true여야 함 | <FlyqlCode code={`status = 200 and active`} inline /> |
| <FlyqlToken type="operator">or</FlyqlToken> | 두 조건 중 하나만 true여도 됨 | <FlyqlCode code={`level = error or level = critical`} inline /> |
| <FlyqlToken type="operator">not</FlyqlToken> | 표현식을 부정함 | <FlyqlCode code={`not archived`} inline /> |

### 결합된 부정

<FlyqlToken type="operator">not</FlyqlToken>은 <FlyqlToken type="operator">and</FlyqlToken>, <FlyqlToken type="operator">or</FlyqlToken>와 함께 사용할 수 있습니다.

<FlyqlCode code={`active and not archived`} />
<FlyqlCode code={`level = error or not healthy`} />

## 연산자 우선순위

<FlyqlToken type="operator">and</FlyqlToken>는 <FlyqlToken type="operator">or</FlyqlToken>보다 더 강하게 결합합니다. 즉,

<FlyqlCode code={`a = 1 or b = 2 and c = 3`} />

은 다음과 같이 parse됩니다.

<FlyqlCode code={`a = 1 or (b = 2 and c = 3)`} />

우선순위를 바꾸려면 괄호를 명시적으로 사용하세요.

## 괄호를 이용한 그룹화

괄호를 사용해 평가 순서를 제어할 수 있습니다.

<FlyqlCode code={`(status >= 400 or level = error) and service = api`} />

괄호가 없으면 <FlyqlToken type="operator">and</FlyqlToken>가 더 강하게 결합합니다.

<FlyqlCode code={`status >= 400 or level = error and service = api`} />

은 다음과 동일합니다.

<FlyqlCode code={`status >= 400 or (level = error and service = api)`} />

### 중첩 그룹화

괄호는 중첩해서 사용할 수 있습니다.

<FlyqlCode code={`(a = 1 or b = 2) and not (c = 3 and d = 4)`} />

### 그룹 부정

<FlyqlToken type="operator">not</FlyqlToken>은 그룹 전체에 적용됩니다.

<FlyqlCode code={`not (status = 200 and healthy)`} />

이는 결합된 조건 전체를 부정합니다 — <FlyqlToken type="key">status</FlyqlToken>가 200이 아니거나 <FlyqlToken type="key">healthy</FlyqlToken>가 false일 때만 true가 됩니다.

## 부정

<FlyqlToken type="operator">not</FlyqlToken>은 truthy 검사, 비교식, 그룹 등 모든 표현식을 부정할 수 있습니다.

<FlyqlCode code={`not active`} />
<FlyqlCode code={`not status = 200`} />
<FlyqlCode code={`not (a = 1 and b = 2)`} />
<FlyqlCode code={`active and not archived`} />

이중 부정은 상쇄됩니다: <FlyqlCode code={`not not active`} inline />는 <FlyqlCode code={`active`} inline />와 같습니다.

## 예제

<FlyqlCode code={`status = 200 and service = api`} />
<FlyqlCode code={`level = error or level = critical`} />
<FlyqlCode code={`status >= 400 or level = error and service = api`} />
<FlyqlCode code={`(status >= 400 or level = error) and service = api`} />
<FlyqlCode code={`(status >= 500 and host = prod*) or (level = error and not service = healthcheck)`} />
