O sistema de transformadores do FlyQL é extensível. Você pode registrar transformadores customizados que funcionam de forma idêntica aos integrados (`upper`, `lower`, `len`) em parsers, geradores SQL, matcher em memória e autocomplete do editor.

## Interface do Transformador

Todo transformador implementa 6 membros:

| Membro | Tipo | Descrição |
|--------|------|-------------|
| `name` | string | Identificador único em minúsculas (ex.: `"firstoctet"`) |
| `inputType` | Type | Tipo de entrada esperado (`string`, `int`, `float`, `bool`, `array`, ou `any` para entrada coringa que aceita qualquer tipo upstream — veja [Transformers com entrada Any](#transformers-com-entrada-any)) |
| `outputType` | Type | Tipo de saída após a transformação |
| `argSchema` | ArgSpec[] | Tipos de argumento declarados e aridade (padrão: vazio = sem argumentos) |
| `sql(dialect, columnRef, args)` | string | Gera SQL específico do dialeto envolvendo a referência da coluna |
| `apply(value, args)` | any | Transforma um valor em tempo de execução na memória |

O método `sql()` recebe o nome do dialeto SQL (`"clickhouse"`, `"postgresql"`, `"starrocks"`), uma string de referência de coluna e a lista de argumentos analisados. Ele retorna uma expressão SQL envolvendo essa referência. O método `apply()` recebe um valor em tempo de execução e a lista de argumentos, e retorna o valor transformado.

### ArgSpec

Cada entrada em `argSchema` declara um argumento esperado:

| Campo | Tipo | Padrão | Descrição |
|-------|------|--------|-------------|
| `type` | Type | — | Tipo de argumento esperado (`string`, `int`, `float`, `bool`) |
| `required` | boolean | `true` | Se o argumento deve ser fornecido |

A aridade é derivada do comprimento do esquema: argumentos obrigatórios devem estar presentes, argumentos opcionais finais podem ser omitidos. Transformadores sem argumentos (como `upper`, `lower`, `len`) herdam o esquema vazio padrão.

## API do Registro

| Método | Descrição |
|--------|-------------|
| `defaultRegistry()` | Retorna um registro novo pré-carregado com os transformadores integrados |
| `registry.register(transformer)` | Adiciona um transformador customizado (gera erro se o nome já estiver registrado) |
| `registry.get(name)` | Busca um transformador pelo nome (retorna null/None/nil se não encontrado) |
| `registry.names()` | Lista todos os nomes de transformadores registrados |

O padrão: obtenha um registro padrão, registre seus transformadores customizados, passe o registro para geradores e matcher.

## Python

Defina um transformador customizado estendendo `Transformer`:

```python
from flyql.transformers import Transformer, Type, default_registry

class FirstOctetTransformer(Transformer):
    @property
    def name(self): return "firstoctet"
    @property
    def input_type(self): return Type.STRING
    @property
    def output_type(self): return Type.INT
    def sql(self, dialect, column_ref, args=None):
        if dialect == "clickhouse":
            return f"toUInt8(splitByChar('.', {column_ref})[1])"
        return f"CAST(SPLIT_PART({column_ref}, '.', 1) AS INTEGER)"
    def apply(self, value, args=None):
        return int(str(value).split('.')[0])
```

O atributo de classe `arg_schema` tem como valor padrão `()` (sem argumentos). Para declarar argumentos, substitua-o por uma tupla de `ArgSpec`:

```python
from typing import ClassVar, Tuple
from flyql.transformers.base import ArgSpec

class SubstringTransformer(Transformer):
    arg_schema: ClassVar[Tuple[ArgSpec, ...]] = (
        ArgSpec(type=Type.INT),                    # start (required)
        ArgSpec(type=Type.INT, required=False),    # length (optional)
    )
    # ... name, input_type, output_type, sql, apply ...
```

Registre e use:

```python
from flyql import parse
from flyql.generators.postgresql.generator import to_sql_where, Column

registry = default_registry()
registry.register(FirstOctetTransformer())

result = parse("src_ip|firstoctet > 192")
columns = {"src_ip": Column("src_ip", "text")}
sql = to_sql_where(result.root, columns, registry=registry)
# CAST(SPLIT_PART(src_ip, '.', 1) AS INTEGER) > 192
```

Para matching em memória:

```python
from flyql.matcher.evaluator import Evaluator
from flyql.matcher.record import Record

evaluator = Evaluator(registry=registry)
record = Record({"src_ip": "193.0.0.1"})
evaluator.evaluate(result.root, record)  # True (193 > 192)
```

## JavaScript

Defina um transformador customizado estendendo `Transformer`:

```javascript
import { Transformer, Type, ArgSpec, defaultRegistry } from 'flyql/transformers'

class FirstOctetTransformer extends Transformer {
    get name() { return 'firstoctet' }
    get inputType() { return Type.STRING }
    get outputType() { return Type.INT }
    sql(dialect, columnRef, args = []) {
        if (dialect === 'clickhouse') return `toUInt8(splitByChar('.', ${columnRef})[1])`
        return `CAST(SPLIT_PART(${columnRef}, '.', 1) AS INTEGER)`
    }
    apply(value, args = []) { return parseInt(String(value).split('.')[0], 10) }
}
```

O getter `argSchema` tem como valor padrão `[]` (sem argumentos). Para declarar argumentos, substitua-o:

```javascript
class SubstringTransformer extends Transformer {
    get argSchema() {
        return [
            new ArgSpec(Type.INT, true),     // start (required)
            new ArgSpec(Type.INT, false),    // length (optional)
        ]
    }
    // ... name, inputType, outputType, sql, apply ...
}
```

Registre e use:

```javascript
import { parse } from 'flyql'
import { generateWhere, newColumn } from 'flyql/generators/postgresql'

const registry = defaultRegistry()
registry.register(new FirstOctetTransformer())

const result = parse("src_ip|firstoctet > 192")
const columns = { src_ip: newColumn({ name: 'src_ip', type: 'text' }) }
const sql = generateWhere(result.root, columns, registry)
// CAST(SPLIT_PART(src_ip, '.', 1) AS INTEGER) > 192
```

Para matching em memória:

```javascript
import { match } from 'flyql/matcher'

match("src_ip|firstoctet > 192", { src_ip: "193.0.0.1" }, registry) // true
```

Para autocomplete do editor — transformadores customizados aparecem junto com os integrados:

```javascript
import { EditorEngine } from 'flyql-vue'

const engine = new EditorEngine(columns, { registry })
// Após digitar "src_ip|", as sugestões incluem "firstoctet", "upper", "lower", "len"
```

## Go

Defina um transformador customizado implementando a interface `Transformer`:

```go
package main

import (
    "fmt"
    "strconv"
    "strings"
    "github.com/iamtelescope/flyql/golang/transformers"
)

type FirstOctet struct{}

func (f FirstOctet) Name() string               { return "firstoctet" }
func (f FirstOctet) InputType() transformers.Type  { return transformers.TransformerTypeString }
func (f FirstOctet) OutputType() transformers.Type { return transformers.TransformerTypeInt }
func (f FirstOctet) ArgSchema() []transformers.ArgSpec        { return []transformers.ArgSpec{} }
func (f FirstOctet) SQL(dialect, colRef string, args []any) string {
    if dialect == "clickhouse" {
        return fmt.Sprintf("toUInt8(splitByChar('.', %s)[1])", colRef)
    }
    return fmt.Sprintf("CAST(SPLIT_PART(%s, '.', 1) AS INTEGER)", colRef)
}
func (f FirstOctet) Apply(value interface{}, args []any) interface{} {
    parts := strings.SplitN(fmt.Sprint(value), ".", 2)
    n, _ := strconv.Atoi(parts[0])
    return n
}
```

Para declarar argumentos, retorne entradas `ArgSpec` a partir de `ArgSchema()`:

```go
func (s Substring) ArgSchema() []transformers.ArgSpec {
    return []transformers.ArgSpec{
        {Type: transformers.TransformerTypeInt, Required: true},   // start (required)
        {Type: transformers.TransformerTypeInt, Required: false},  // length (optional)
    }
}
```

Registre e use:

```go
import (
    flyql "github.com/iamtelescope/flyql/golang"
    postgresqlgen "github.com/iamtelescope/flyql/golang/generators/postgresql"
)

registry := transformers.DefaultRegistry()
registry.Register(FirstOctet{})

result, _ := flyql.Parse("src_ip|firstoctet > 192")
columns := map[string]*postgresqlgen.Column{
    "src_ip": postgresqlgen.NewColumn("src_ip", "text", nil),
}
sql, _ := postgresqlgen.ToSQLWhere(result.Root, columns, registry)
// CAST(SPLIT_PART(src_ip, '.', 1) AS INTEGER) > 192
```

Para matching em memória:

```go
import "github.com/iamtelescope/flyql/golang/matcher"

matched, _ := matcher.Match("src_ip|firstoctet > 192", map[string]any{"src_ip": "193.0.0.1"}, registry)
// matched == true
```

## Transformers com entrada `Any`

Um transformador pode declarar `inputType = Type.Any` (valor string `"any"`) para aceitar qualquer tipo de coluna ou de saída de transformador upstream. O caso de uso motivador são operações agnósticas a tipo — por exemplo, um transformador `type` que retorna o nome do tipo da coluna independentemente de qual seja esse tipo.

Regras:

- `Any` é válido **apenas** em `inputType`. Declarar `Any` como `outputType` ou em qualquer `ArgSpec.type` é rejeitado em tempo de `register()` com um erro direcionado. Todo transformador deve declarar um tipo de saída concreto.
- Quando um transformador com entrada `Any` é usado em uma cadeia, a verificação de igualdade de tipo de entrada é ignorada para essa etapa. A rigidez downstream é preservada — o próximo transformador na cadeia ainda é verificado contra o `outputType` concreto do transformador de entrada `Any`. Por exemplo, `col|accepts_any_returning_array|upper` ainda gera erro em `upper` porque `array` não é uma entrada válida para `upper`.
- `parseFlyQLType("any")` gera erro: `any` não é um tipo válido de coluna ou valor. É um sentinela de entrada de transformador apenas.

```python
from flyql.flyql_type import Type
from flyql.transformers import Transformer

class TypeTransformer(Transformer):
    @property
    def name(self): return "type"
    @property
    def input_type(self): return Type.Any      # aceita qualquer tipo upstream
    @property
    def output_type(self): return Type.String  # deve ser concreto
    def sql(self, dialect, column_ref, args=None):
        if dialect == "clickhouse":
            return f"toTypeName({column_ref})"
        return f"pg_typeof({column_ref})::text"
    def apply(self, value, args=None):
        return type(value).__name__
```

## Compatibilidade de Tipos

Transformadores customizados participam da mesma validação de cadeia de tipos que os integrados. Se você encadear `field|firstoctet|upper`, o sistema detecta que `firstoctet` gera `int` mas `upper` requer `string`.

Tipos disponíveis: consulte [`flyql.Type`](/advanced/column-types/). A única exceção é `Any`, que é um coringa de entrada de transformer e não é um tipo de coluna ou valor válido.

## Validação com `diagnose()`

A função `diagnose()` percorre uma AST parseada contra um schema de colunas e um registro de transformadores, retornando registros `Diagnostic` posicionados sem lançar exceções. Ela verifica:

- **Coluna desconhecida** — chave base não está na lista de colunas
- **Transformador desconhecido** — nome do transformador não está no registro
- **Aridade incorreta** — argumentos demais ou de menos em relação ao `argSchema`
- **Tipo de argumento incorreto** — o tipo do valor do argumento não corresponde ao tipo declarado em `ArgSpec` (alargamento de int para float é permitido)
- **Incompatibilidade de tipo na cadeia** — o tipo de saída de um transformador não corresponde ao tipo de entrada do próximo, ou o tipo normalizado da coluna não corresponde ao tipo de entrada do primeiro transformador

Cada diagnóstico carrega um `range` (posição no código-fonte), `message`, `severity` (`"error"`) e `code` (`"unknown_column"`, `"unknown_transformer"`, `"arg_count"`, `"arg_type"`, `"chain_type"`).

```python
# Python
from flyql import parse, Column, ColumnSchema, Type, diagnose

ast = parse("hstt|upper=X").root
columns = ColumnSchema.from_columns([Column(name="host", column_type=Type.String)])
diags = diagnose(ast, columns)  # columns é um ColumnSchema
# [Diagnostic(range=Range(0, 4), message="column 'hstt' is not defined", severity='error', code='unknown_column')]
```

```javascript
// JavaScript
import { parse, Column, ColumnSchema, Type, diagnose } from 'flyql'

const ast = parse('hstt|upper=X').root
const columns = ColumnSchema.fromColumns([new Column('host', false, Type.String)])
const diags = diagnose(ast, columns)  // columns é um ColumnSchema
// [Diagnostic { range: Range(0, 4), message: "column 'hstt' is not defined", severity: 'error', code: 'unknown_column' }]
```

```go
// Go
import flyql "github.com/iamtelescope/flyql/golang"

result, _ := flyql.Parse("hstt|upper=X")
schema := flyql.FromColumns([]flyql.Column{flyql.NewColumn("host", flyql.TypeString)})
diags := flyql.Diagnose(result.Root, schema, nil) // nil = registro padrão
// []Diagnostic{{Range: {0, 4}, Message: "column 'hstt' is not defined", Severity: "error", Code: "unknown_column"}}
```

Passe um registro customizado como terceiro argumento se você tiver transformadores customizados registrados. Passe `nil`/`None`/`null` para usar o registro padrão.

## Segurança

Os métodos `sql()` dos transformadores customizados recebem apenas referências de colunas (do schema), nunca valores fornecidos pelo usuário. Valores do usuário são escapados/parametrizados pelo gerador. Isso preserva a garantia de proteção contra injeção do FlyQL — SQL customizado envolve nomes de colunas, não entrada do usuário.
