Transformers は、クエリで比較する前、または SQL として出力する前にカラム値を変換します。パイプ構文（`|`）を使い、左から右へ適用されます。

## パイプ構文

カラムに transformer を適用するには `|` を使います。

<FlyqlCode code={`message|upper = "ERROR"`} />

これは <FlyqlToken type="key">message</FlyqlToken> の値に <FlyqlToken type="transformer">upper</FlyqlToken> を適用してから `"ERROR"` と比較します。

## チェーン

複数の transformer はパイプを追加して連結できます。

<FlyqlCode code={`message|lower|len > 100`} />

transformer は左から右へ適用されます。まず <FlyqlToken type="transformer">lower</FlyqlToken>（string → string）、次に <FlyqlToken type="transformer">len</FlyqlToken>（string → int）が実行され、その結果を `> 100` で比較します。

## 型システム

各 transformer は入力型と出力型を宣言します。チェーンされた transformer 同士は型互換である必要があります。前の transformer の出力型は、次の transformer の入力型と一致しなければなりません。

| 型 | 説明 |
|----|------|
| `string` | テキスト値 |
| `int` | 整数 |
| `float` | 浮動小数点数 |
| `array` | 値の配列。所属チェックには <FlyqlToken type="operator">has</FlyqlToken> を使います |

## 組み込み Transformers

| Transformer | 入力 | 出力 | 説明 |
|-------------|------|------|------|
| <FlyqlToken type="transformer">upper</FlyqlToken> | string | string | 大文字に変換 |
| <FlyqlToken type="transformer">lower</FlyqlToken> | string | string | 小文字に変換 |
| <FlyqlToken type="transformer">len</FlyqlToken> | string | int | 文字列長を取得 |
| <FlyqlToken type="transformer">split</FlyqlToken> | string | array | 区切り文字で分割: `split(".")` |

## 方言ごとの SQL 出力

transformer は方言ごとの SQL 関数呼び出しを生成します。チェーンされた transformer はネストした呼び出しになり、最も内側から評価されます。

**単一 transformer:** <FlyqlCode code={`status|upper = "ERROR"`} inline />

| 方言 | 生成される SQL |
|------|-----------------|
| ClickHouse | `equals(upper(status), 'ERROR')` |
| PostgreSQL | `UPPER(status) = 'ERROR'` |
| StarRocks | `UPPER(\`status\`) = 'ERROR'` |

**チェーンされた transformer:** <FlyqlCode code={`message|lower|len > 100`} inline />

| 方言 | 生成される SQL |
|------|-----------------|
| ClickHouse | `length(lower(message)) > 100` |
| PostgreSQL | `LENGTH(LOWER(message)) > 100` |
| StarRocks | `LENGTH(LOWER(\`message\`)) > 100` |

**Split と has:** <FlyqlCode code={`message|split(" ") has "hello"`} inline />

| 方言 | 生成される SQL |
|------|-----------------|
| ClickHouse | `has(splitByChar(' ', message), 'hello')` |
| PostgreSQL | `'hello' = ANY(STRING_TO_ARRAY(message, ' '))` |
| StarRocks | `` array_contains(SPLIT(`message`, ' '), 'hello') `` |

## 型エラー

チェーン内に型互換のない transformer がある場合、editor はエラーを表示します。

<FlyqlCode code={`message|len|upper`} />

これは <FlyqlToken type="transformer">len</FlyqlToken> が `int` を返す一方、<FlyqlToken type="transformer">upper</FlyqlToken> が `string` 入力を必要とするため失敗します。editor には "len outputs int, upper requires string input" と表示されます。

## 例

<FlyqlCode code={`status|upper = "ERROR"`} />
<FlyqlCode code={`message|lower|len > 100`} />
<FlyqlCode code={`host|lower != "localhost"`} />
<FlyqlCode code={`path|len > 0`} />
<FlyqlCode code={`tags|split(",") has "urgent"`} />
