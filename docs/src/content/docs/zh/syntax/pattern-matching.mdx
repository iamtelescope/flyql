## 正则匹配

使用 <FlyqlToken type="operator">{'~'}</FlyqlToken> 将值与正则表达式匹配，使用 <FlyqlToken type="operator">{'!~'}</FlyqlToken> 表示不匹配：

<FlyqlCode code={`message ~ "error.*timeout"`} />
<FlyqlCode code={`path !~ "^/health"`} />

| 操作符 | 含义 |
|--------|------|
| <FlyqlToken type="operator">{'~'}</FlyqlToken> | 值匹配正则 |
| <FlyqlToken type="operator">{'!~'}</FlyqlToken> | 值不匹配正则 |

### 正则语法

正则语法依赖具体后端：

- **ClickHouse**：使用 `match()`，采用 RE2 语法
- **PostgreSQL**：使用 <FlyqlToken type="operator">{'~'}</FlyqlToken> 和 <FlyqlToken type="operator">{'!~'}</FlyqlToken>，采用 POSIX 正则
- **StarRocks**：使用 `regexp` 函数
- **内存匹配器**：使用宿主语言的正则引擎（Go `regexp`、Python `re2`、JS `RegExp`）

当正则中包含空格或特殊字符时，应使用引号：

<FlyqlCode code={`message ~ "connection (refused|reset)"`} />
<FlyqlCode code={`host ~ "^prod-[0-9]+"`} />

### 正则标志

你可以在模式开头使用 `(?...)` 语法添加内联标志来改变正则行为。FlyQL 支持三个在所有后端都通用的标志：

#### `(?i)` — 忽略大小写

让模式匹配时不区分大小写：

<FlyqlCode code={`message ~ "(?i)error"`} />
<FlyqlCode code={`level ~ "(?i)^(warn|error|fatal)$"`} />

`(?i)error` 会匹配 "Error"、"ERROR"、"error" 等。

#### `(?m)` — 多行模式

使 `^` 和 `$` 匹配每一**行**的开头和结尾，而不是整个字符串的开头和结尾：

<FlyqlCode code={`log ~ "(?m)^ERROR"`} />

不使用 `(?m)` 时，`^ERROR` 仅在整个字符串以 "ERROR" 开头时匹配。使用 `(?m)` 后，"INFO first\nERROR second" 也会匹配，因为 "ERROR" 出现在某一行的开头。

#### `(?s)` — Dotall

使 `.` 匹配**任意**字符，包括换行符。默认情况下 `.` 不匹配 `\n`：

<FlyqlCode code={`log ~ "(?s)start.*end"`} />

不使用 `(?s)` 时，`start.*end` 不会匹配 "start\nend"（因为 `.` 无法跨越换行）。使用 `(?s)` 后则可以匹配。

#### 组合标志

标志可以在一个分组中组合使用：

<FlyqlCode code={`log ~ "(?im)^ERROR"`} />
<FlyqlCode code={`log ~ "(?ims)^START.*END$"`} />

#### 后端支持情况

所有三个标志在每个 FlyQL 后端中都受支持：

| 后端 | `(?i)` | `(?m)` | `(?s)` |
|------|--------|--------|--------|
| **ClickHouse** | Yes | Yes | Yes |
| **PostgreSQL** | Yes | Yes | Yes |
| **StarRocks** | Yes | Yes | Yes |
| **Go matcher** | Yes | Yes | Yes |
| **Python matcher** | Yes | Yes | Yes |
| **JavaScript matcher** | Yes | Yes | Yes |

可以将忽略大小写与否定结合使用，以不区分大小写地排除特定模式：

<FlyqlCode code={`message ~ "(?i)error" and message !~ "(?i)warning"`} />

## LIKE 匹配

使用 <FlyqlToken type="operator">like</FlyqlToken> 和 <FlyqlToken type="operator">ilike</FlyqlToken> 进行 SQL 风格的通配符模式匹配：

<FlyqlCode code={`host like 'prod%'`} />
<FlyqlCode code={`service like 'api-%'`} />
<FlyqlCode code={`message ilike '%Error%'`} />

| 操作符 | 含义 |
|--------|------|
| <FlyqlToken type="operator">like</FlyqlToken> | 值匹配模式（区分大小写） |
| <FlyqlToken type="operator">not like</FlyqlToken> | 值不匹配模式 |
| <FlyqlToken type="operator">ilike</FlyqlToken> | 值匹配模式（不区分大小写） |
| <FlyqlToken type="operator">not ilike</FlyqlToken> | 值不匹配模式（不区分大小写） |

### 通配符

<FlyqlToken type="operator">like</FlyqlToken> / <FlyqlToken type="operator">ilike</FlyqlToken> 模式中有两个通配符可用：

| 通配符 | 含义 | 示例 | 匹配 |
|--------|------|------|------|
| `%` | 任意字符序列（包括空序列） | <FlyqlCode code={`host like 'prod%'`} inline /> | `prod`、`prod-1`、`production` |
| `_` | 精确匹配一个字符 | <FlyqlCode code={`code like 'E__'`} inline /> | `E01`、`Exx`，但不匹配 `E1` 或 `Error` |

通配符可以出现在模式的任何位置——开头、中间、末尾或组合使用：

<FlyqlCode code={`message like '%timeout%'`} />
<FlyqlCode code={`host like 'web-_-prod'`} />
<FlyqlCode code={`path like '/api/%/status'`} />

### 转义通配符

要匹配字面的 `%` 或 `_`，用反斜杠转义：

<FlyqlCode code={`value like '100\\%'`} />
<FlyqlCode code={`col like 'foo\\_bar'`} />

`100\%` 匹配字面字符串 "100%"，而不是 "100" 后面跟任意内容。

### 不区分大小写匹配

使用 <FlyqlToken type="operator">ilike</FlyqlToken> 进行不区分大小写的模式匹配：

<FlyqlCode code={`message ilike '%error%'`} />
<FlyqlCode code={`host not ilike 'TEST%'`} />

### LIKE 与正则的区别

| 用途 | 语法 | 示例 |
|------|------|------|
| 前缀匹配 | <FlyqlCode code={`key like 'prefix%'`} inline /> | <FlyqlCode code={`host like 'prod%'`} inline /> |
| 包含 | <FlyqlCode code={`key like '%word%'`} inline /> | <FlyqlCode code={`msg like '%error%'`} inline /> |
| 复杂模式 | <FlyqlCode code={`key ~ "pattern"`} inline /> | <FlyqlCode code={`host ~ "^prod-[0-9]+"`} inline /> |

简单通配符匹配用 <FlyqlToken type="operator">like</FlyqlToken>。需要字符类、交替或其他高级模式时，使用正则 (<FlyqlToken type="operator">{'~'}</FlyqlToken>)。

### 后端支持

所有 SQL 后端（ClickHouse、PostgreSQL、StarRocks）均生成原生的 `LIKE` / `ILIKE` / `NOT LIKE` / `NOT ILIKE` SQL 操作符。内存匹配器会在内部将模式转换为正则表达式。

:::note
StarRocks `ILIKE` 需要 2.5 或更高版本。
:::

## 示例

<FlyqlCode code={`message ~ "error|fail"`} />
<FlyqlCode code={`path ~ "^/api/v2"`} />
<FlyqlCode code={`path !~ "^/health"`} />
<FlyqlCode code={`host like 'prod%'`} />
<FlyqlCode code={`host like 'prod%' and message ~ "timeout.*" and path !~ "^/health"`} />
