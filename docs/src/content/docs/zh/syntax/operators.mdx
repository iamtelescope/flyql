---
title: 比较操作符
description: FlyQL 中用于数据过滤的比较操作符
---

FlyQL 支持以下比较操作符：

## 相等比较

| 操作符 | 名称 | 示例 |
|--------|------|------|
| `=` | 等于 | `status=200` |
| `!=` | 不等于 | `env!=prod` |

```
status=200
service!="api-gateway"
```

操作符两侧都必须存在内容，也就是字段和值都必须提供。

## 数值比较

| 操作符 | 名称 | 示例 |
|--------|------|------|
| `>` | 大于 | `status>399` |
| `>=` | 大于等于 | `status>=400` |
| `<` | 小于 | `duration<1000` |
| `<=` | 小于等于 | `duration<=500` |

```
status>=400 and duration<5000
response_time>100 and response_time<=2000
```

数值比较通常用于数字值。如果用于字符串，则会执行字典序比较。

## 正则匹配

| 操作符 | 名称 | 示例 |
|--------|------|------|
| `~` | 正则匹配 | `message~"error.*"` |
| `!~` | 不匹配正则 | `path!~"^/health"` |

```
message~"timeout|connection refused"
host!~"^test-.*"
```

正则语法依赖目标数据库或运行时环境。

## 列表成员检查

| 操作符 | 名称 | 示例 |
|--------|------|------|
| `in` | 在列表中 | `status in [200, 201]` |
| `not in` | 不在列表中 | `env not in ['dev', 'test']` |

列表语法的详细规则请查看[列表](/zh/syntax/lists/)。

## 包含检查

| 操作符 | 名称 | 示例 |
|--------|------|------|
| `has` | 包含 | `message has 'error'` |
| `not has` | 不包含 | `message not has 'test'` |

```
message has 'timeout'
tags not has 'deprecated'
```

`has` 会根据字段类型执行不同的检查：

- **字符串**：子串匹配，例如 `message has 'error'`
- **Map**：键是否存在，例如 `metadata has 'region'`
- **数组/列表**：元素是否存在，例如 `tags has 'web'`
- **JSON**：对象是否包含某个键，例如 `data has 'name'`
- **空值/空内容**：对 `has` 来说恒为假，对 `not has` 来说恒为真

`has` 接收的是单个值，而不是列表。若需要判断某值是否属于多个候选值，请使用 `in`。

## 操作符汇总

| 操作符 | 名称 | 示例 |
|--------|------|------|
| `=` | 等于 | `status=200` |
| `!=` | 不等于 | `env!=prod` |
| `>` | 大于 | `count>10` |
| `>=` | 大于等于 | `count>=10` |
| `<` | 小于 | `count<100` |
| `<=` | 小于等于 | `count<=100` |
| `~` | 正则匹配 | `msg~"err.*"` |
| `!~` | 非正则匹配 | `msg!~"^ok"` |
| `in` | 在列表中 | `s in [1, 2]` |
| `not in` | 不在列表中 | `s not in [3]` |
| `has` | 包含 | `msg has 'err'` |
| `not has` | 不包含 | `msg not has 'ok'` |
