## IN 列表的类型一致性

解析器**始终**接受异构列表，例如 <FlyqlCode code={`[200, 'ok', 404]`} inline />。SQL 生成器**仅**在以下三个条件全部满足时才进行类型一致性检查：(a) 列声明了类型，(b) 键不是分段的，(c) 列表是同质的（`len(set(values_types)) == 1`）。异构列表、分段键以及无类型列都会在调用点直接绕过检查。

本配方在带类型的列上演示三种情况：同质匹配、异构绕过，以及同质类型不匹配——后者会抛出标准错误 `type mismatch in IN list: ...`。Python、Go 与 JavaScript 中的错误文本完全相同。
