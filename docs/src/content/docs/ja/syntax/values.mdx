FlyQL は 7 つの値型をサポートします。**Integer**、**BigInt**、**Float**、**String**、**Boolean**、**Null**、**Array** です。AST 内の各値には明示的な `value_type` フィールドが付きます。

## 値型

### Integer

int64 の範囲内に収まる整数です。

<FlyqlCode code={`count = 42`} />
<FlyqlCode code={`status != 200`} />
<FlyqlCode code={`age >= 18`} />

AST: `value_type: "integer"`

### BigInt

int64 の範囲（Go/Python）または `MAX_SAFE_INTEGER`（JavaScript）を超える整数です。

<FlyqlCode code={`id = 18446744073709551615`} />

AST: `value_type: "bigint"`

### Float

小数点を含む数値です。

<FlyqlCode code={`price > 9.99`} />
<FlyqlCode code={`ratio = 3.14`} />
<FlyqlCode code={`temperature<=-0.5`} />

AST: `value_type: "float"`

### String

クォート付きまたはクォートなしのテキスト値です。

<FlyqlCode code={`level = error`} />
<FlyqlCode code={`user = "John Doe"`} />
<FlyqlCode code={`message = 'hello world'`} />
<FlyqlCode code={`count = '42'`} />

AST: `value_type: "string"`

クォートはバックスラッシュでエスケープできます。

```
user = 'John
message="said \"hello\""
```

### Boolean

<FlyqlToken type="boolean">true</FlyqlToken> と <FlyqlToken type="boolean">false</FlyqlToken> という小文字のキーワードです。大文字小文字は区別されます。

<FlyqlCode code={`active = true`} />
<FlyqlCode code={`debug = false`} />
<FlyqlCode code={`active != true`} />

AST: `value_type: "boolean"`。値はネイティブ boolean です。

先頭が大文字のバリエーション（`True`、`FALSE`）は文字列として扱われます。

**SQL 生成:**
| 方言 | FlyQL | 生成される SQL |
|------|-------|---------------|
| ClickHouse | <FlyqlCode code={`field = true`} inline /> | `field = true` |
| ClickHouse | <FlyqlCode code={`field = false`} inline /> | `field = false` |
| PostgreSQL | <FlyqlCode code={`field = true`} inline /> | `field = TRUE` |
| PostgreSQL | <FlyqlCode code={`field = false`} inline /> | `field = FALSE` |
| StarRocks | <FlyqlCode code={`field = true`} inline /> | `field = true` |
| StarRocks | <FlyqlCode code={`field = false`} inline /> | `field = false` |

### Null

<FlyqlToken type="null">null</FlyqlToken> という小文字のキーワードです。大文字小文字は区別されます。

<FlyqlCode code={`field = null`} />
<FlyqlCode code={`field != null`} />

AST: `value_type: "null"`。値はネイティブ null です。

先頭が大文字のバリエーション（`Null`、`NULL`）は文字列として扱われます。

**制約:** null と組み合わせて使えるのは <FlyqlToken type="operator">=</FlyqlToken> と <FlyqlToken type="operator">!=</FlyqlToken> のみです。<FlyqlToken type="operator">{'>'}</FlyqlToken>、<FlyqlToken type="operator">{'<'}</FlyqlToken>、<FlyqlToken type="operator">{'>='}</FlyqlToken>、<FlyqlToken type="operator">{'<='}</FlyqlToken>、<FlyqlToken type="operator">{'~'}</FlyqlToken>、<FlyqlToken type="operator">{'!~'}</FlyqlToken> を null と使うと parse error になります。

**SQL 生成:** <FlyqlCode code={`field = null`} inline /> は `field IS NULL`、<FlyqlCode code={`field != null`} inline /> は `field IS NOT NULL` を生成します。全方言で共通です。

### Array

Array は <FlyqlToken type="operator">in</FlyqlToken> と <FlyqlToken type="operator">not in</FlyqlToken> で使います。要素には異なる型を混在できます。

<FlyqlCode code={`status in [200, 201, 202]`} />
<FlyqlCode code={`name in ['alice', 'bob']`} />
<FlyqlCode code={`field in [1, "hello", true, null]`} />

各要素は `values_types` 配列にそれぞれの型を持ちます。

## Truthy/Falsy 式

演算子を持たない単独のキーは、そのフィールドが truthy な値を持つかどうかをチェックします。

<FlyqlCode code={`active`} />
<FlyqlCode code={`message and status`} />

次の値は **falsy** とみなされます。
- `null` / `None` / 未存在
- 空文字列 `""`
- 0
- boolean <FlyqlToken type="boolean">false</FlyqlToken>

それ以外は **truthy** です。

falsy をチェックするには <FlyqlToken type="operator">not</FlyqlToken> を使います。

<FlyqlCode code={`not archived`} />
<FlyqlCode code={`active and not debug`} />

truthy/falsy チェックは null 比較とは異なります。<FlyqlCode code={`not field`} inline /> は、フィールドが null、空、または 0 の場合に一致します。<FlyqlCode code={`field = null`} inline /> は、フィールドが明示的に null の場合にのみ一致します。

## 空白

FlyQL は空白に寛容です。演算子の前後のスペースは省略できます。

<FlyqlCode code={`status = 200`} />
<FlyqlCode code={`status =200`} />

上記はすべて同じ意味です。

## 型定数モジュール

型定数はすべて `flyql.types` モジュールから利用できます。

**Python:**
```python
from flyql.types import ValueType
# ValueType.INTEGER, ValueType.BIGINT, ValueType.FLOAT,
# ValueType.STRING, ValueType.BOOLEAN, ValueType.NULL, ValueType.ARRAY
```

**Go:**
```go
import "github.com/iamtelescope/flyql/golang/types"
// types.Integer, types.BigInt, types.Float,
// types.String, types.Boolean, types.Null, types.Array
```

**JavaScript:**
```javascript
import { ValueType } from 'flyql/types'
// ValueType.INTEGER, ValueType.BIGINT, ValueType.FLOAT,
// ValueType.STRING, ValueType.BOOLEAN, ValueType.NULL, ValueType.ARRAY
```

## 移行時の注意

以前の FlyQL バージョンからアップグレードする場合:

- **Breaking:** Go の `ValueTypeString` / `ValueTypeNumber` は削除され、7 値の `types.ValueType` に置き換えられました
- **Breaking:** AST の `value_type` フィールドは `"string"` / `"number"` から `"integer"` / `"float"` / `"bigint"` / `"string"` / `"boolean"` / `"null"` に変更されました
- **Breaking:** 混在型リストの制約が削除され、異種型の array が有効になりました
- **New:** <FlyqlToken type="boolean">true</FlyqlToken>、<FlyqlToken type="boolean">false</FlyqlToken>、<FlyqlToken type="null">null</FlyqlToken> は文字列ではなく型付きリテラルとして parse されます
- **New:** IN リスト式の `values_types` 配列で、要素ごとの型を参照できます
- **New:** `flyql.types` モジュールから標準の型定数を export します
