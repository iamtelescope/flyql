---
title: StarRocks
description: Ejemplos de SQL generado para StarRocks
---

El SQL de StarRocks usa identificadores entre backticks, `regexp()` para regex y `->` con claves serializadas como JSON para el acceso a JSON.

## Ejemplos básicos

| FlyQL | SQL generado |
|-------|--------------|
| `message='hello'` | `` `message` = 'hello' `` |
| `message!='hello'` | `` `message` != 'hello' `` |
| `count>10` | `` `count` > 10 `` |
| `count<=100` | `` `count` <= 100 `` |
| `message~'error.*'` | `` regexp(`message`, 'error.*') `` |
| `message!~'test.*'` | `` not regexp(`message`, 'test.*') `` |
| `host=prod*` | `` `host` LIKE 'prod%' `` |

## Operadores booleanos

| FlyQL | SQL generado |
|-------|--------------|
| `a='1' and b>10` | `` (`a` = '1' and `b` > 10) `` |
| `a='1' or b='2'` | `` (`a` = '1' or `b` = '2') `` |
| `not a='1'` | `` not (`a` = '1') `` |

## Pertenencia a listas

| FlyQL | SQL generado |
|-------|--------------|
| `status in [200, 201]` | `` `status` IN (200, 201) `` |
| `env not in ['dev', 'test']` | `` `env` NOT IN ('dev', 'test') `` |
| `status in []` | `0` |
| `status not in []` | `1` |

## Contención (`has` / `not has`)

El operador `has` genera SQL distinto según el tipo de columna:

| Tipo de columna | FlyQL | SQL generado |
|-----------------|-------|--------------|
| String | `message has 'error'` | `` INSTR(`message`, 'error') > 0 `` |
| String con negación | `message not has 'error'` | `` (`message` IS NULL OR INSTR(`message`, 'error') = 0) `` |
| Array | `tags has 'web'` | `` array_contains(`tags`, 'web') `` |
| Array con negación | `tags not has 'web'` | `` NOT array_contains(`tags`, 'web') `` |
| Map | `metadata has 'key'` | `` array_contains(map_keys(`metadata`), 'key') `` |
| Map con negación | `metadata not has 'key'` | `` NOT array_contains(map_keys(`metadata`), 'key') `` |
| JSON nativo | `data has 'key'` | `` json_exists(`data`, concat('$.', 'key')) `` |

## Columnas JSON

StarRocks admite dos tipos de columnas JSON con salidas SQL distintas.

### Tipo JSON nativo

Las columnas JSON nativas usan el operador `->` con claves serializadas como JSON:

| FlyQL | SQL generado |
|-------|--------------|
| `data.name='test'` | `` `data`->'\"name\"' = 'test' `` |
| `data.user.name='john'` | `` `data`->'\"user\"'->'\"name\"' = 'john' `` |

### Tipo JSON string

Las columnas que almacenan JSON como string requieren `parse_json()` antes del recorrido:

| FlyQL | SQL generado |
|-------|--------------|
| `data.name='test'` | `` parse_json(`data`)->'\"name\"' = 'test' `` |
| `data.user.name='john'` | `` parse_json(`data`)->'\"user\"'->'\"name\"' = 'john' `` |

## Columnas Map

Las columnas Map usan notación con corchetes y comparación directa:

| FlyQL | SQL generado |
|-------|--------------|
| `metadata.key1='value1'` | `` `metadata`['key1'] = 'value1' `` |
| `metadata.key1!='value1'` | `` `metadata`['key1'] != 'value1' `` |
| `metadata.nested.key='value'` | `` `metadata`['nested']['key'] = 'value' `` |
| `metadata.pattern~'test.*'` | `` `metadata`['pattern'] regexp 'test.*' `` |

## Columnas Array

Las columnas Array usan indexación base 0 con corchetes:

| FlyQL | SQL generado |
|-------|--------------|
| `tags.0='first'` | `` `tags`[0] = 'first' `` |
| `tags.1='second'` | `` `tags`[1] = 'second' `` |

### Configuración de columnas

El parámetro `jsonString` controla qué modo se usa, igual que en ClickHouse:

```go
// Go
columns := map[string]*starrocks.Column{
    "message": starrocks.NewColumn("message", false, "VARCHAR(255)", nil),
    "data":    starrocks.NewColumn("data", false, "JSON", nil),      // native JSON
    "log":     starrocks.NewColumn("log", true, "STRING", nil),       // JSON string
}
```

```javascript
// JavaScript
const columns = {
    message: newColumn("message", false, "VARCHAR(255)", null),
    data:    newColumn("data", false, "JSON", null),      // native JSON
    log:     newColumn("log", true, "STRING", null),       // JSON string
}
```

## Transformers

Los transformers generan llamadas a funciones estándar de StarRocks con nombres de columnas entre backticks:

| FlyQL | SQL generado |
|-------|--------------|
| `status\|upper = "ERROR"` | `` UPPER(`status`) = 'ERROR' `` |
| `status\|lower = "info"` | `` LOWER(`status`) = 'info' `` |
| `message\|len > 100` | `` LENGTH(`message`) > 100 `` |
| `message\|lower\|len > 100` | `` LENGTH(LOWER(`message`)) > 100 `` |
| `tags\|split(",") has "urgent"` | `` array_contains(SPLIT(`tags`, ','), 'urgent') `` |
