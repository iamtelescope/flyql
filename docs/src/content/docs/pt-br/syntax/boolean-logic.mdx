## Operadores Booleanos

O FlyQL usa três operadores booleanos para combinar condições:

| Operador | Significado | Exemplo |
|----------|-------------|---------|
| <FlyqlToken type="operator">and</FlyqlToken> | Ambas as condições devem ser verdadeiras | <FlyqlCode code={`status = 200 and active`} inline /> |
| <FlyqlToken type="operator">or</FlyqlToken> | Qualquer uma das condições pode ser verdadeira | <FlyqlCode code={`level = error or level = critical`} inline /> |
| <FlyqlToken type="operator">not</FlyqlToken> | Nega uma expressão | <FlyqlCode code={`not archived`} inline /> |

### Negação Combinada

<FlyqlToken type="operator">not</FlyqlToken> pode ser combinado com <FlyqlToken type="operator">and</FlyqlToken> e <FlyqlToken type="operator">or</FlyqlToken>:

<FlyqlCode code={`active and not archived`} />
<FlyqlCode code={`level = error or not healthy`} />

## Precedência de Operadores

<FlyqlToken type="operator">and</FlyqlToken> tem maior precedência que <FlyqlToken type="operator">or</FlyqlToken>. Isso significa que:

<FlyqlCode code={`a = 1 or b = 2 and c = 3`} />

é interpretado como:

<FlyqlCode code={`a = 1 or (b = 2 and c = 3)`} />

Para alterar a precedência, use parênteses explicitamente.

## Agrupamento com Parênteses

Use parênteses para controlar a ordem de avaliação:

<FlyqlCode code={`(status >= 400 or level = error) and service = api`} />

Sem parênteses, <FlyqlToken type="operator">and</FlyqlToken> teria maior precedência:

<FlyqlCode code={`status >= 400 or level = error and service = api`} />

é equivalente a:

<FlyqlCode code={`status >= 400 or (level = error and service = api)`} />

### Agrupamento Aninhado

Parênteses podem ser aninhados:

<FlyqlCode code={`(a = 1 or b = 2) and not (c = 3 and d = 4)`} />

### Negando Grupos

<FlyqlToken type="operator">not</FlyqlToken> se aplica ao grupo inteiro:

<FlyqlCode code={`not (status = 200 and healthy)`} />

Isso nega a condição combinada — é verdadeiro apenas quando <FlyqlToken type="key">status</FlyqlToken> não é 200 ou <FlyqlToken type="key">healthy</FlyqlToken> é falso.

## Negação

<FlyqlToken type="operator">not</FlyqlToken> nega qualquer expressão — uma verificação truthy, uma comparação ou um grupo:

<FlyqlCode code={`not active`} />
<FlyqlCode code={`not status = 200`} />
<FlyqlCode code={`not (a = 1 and b = 2)`} />
<FlyqlCode code={`active and not archived`} />

A dupla negação se cancela: <FlyqlCode code={`not not active`} inline /> é equivalente a <FlyqlCode code={`active`} inline />.

## Exemplos

<FlyqlCode code={`status = 200 and service = api`} />
<FlyqlCode code={`level = error or level = critical`} />
<FlyqlCode code={`status >= 400 or level = error and service = api`} />
<FlyqlCode code={`(status >= 400 or level = error) and service = api`} />
<FlyqlCode code={`(status >= 500 and host = prod*) or (level = error and not service = healthcheck)`} />
