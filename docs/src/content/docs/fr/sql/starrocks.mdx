## Colonnes JSON

### Type JSON natif

<SqlExamplesTable rows={jsonString} />

## Types complexes imbriques

Le generateur StarRocks traite un chemin segmente comme appartenant a un seul type conteneur — il ne descend actuellement pas a travers des imbrications mixtes comme `Map(K, Struct)`, `Array(Map)` ou des champs `Struct` qui sont eux-memes des `Map`/`Array`. Pour ces colonnes, aplatissez le chemin ou exposez la valeur interne comme une colonne distincte.

### Configuration des colonnes

{/* TODO: native-speaker review */}
Les colonnes text/varchar dont le contenu est du JSON valide sont declarees avec le type brut synthetique `jsonstring` (identique a ClickHouse) :

```go
// Go
columns := map[string]*starrocks.Column{
    "message": starrocks.NewColumn(starrocks.ColumnDef{Name: "message", Type: "VARCHAR(255)"}),
    "data":    starrocks.NewColumn(starrocks.ColumnDef{Name: "data", Type: "JSON"}),      // native JSON
    "log":     starrocks.NewColumn(starrocks.ColumnDef{Name: "log", Type: "jsonstring"}),       // JSON string
}
```

```javascript
// JavaScript
const columns = {
    message: newColumn({ name: "message", type: "VARCHAR(255)" }),
    data:    newColumn({ name: "data", type: "JSON" }),      // native JSON
    log:     newColumn({ name: "log", type: "jsonstring" }),       // JSON string
}
```
