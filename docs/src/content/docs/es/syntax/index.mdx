Las consultas de FlyQL constan de una o más condiciones conectadas por operadores booleanos (`and`, `or`, `not`). Cada condición es una comparación o una comprobación truthy.

## Estructura básica de una consulta

<FlyqlCode code={`status = 200 and active and not archived`} />

- `status = 200`: el campo `status` es igual a `200`
- `active`: el campo `active` tiene un valor truthy
- `not archived`: el campo `archived` es falsy (`null`, vacío, cero o `false`)

## Más ejemplos

<FlyqlCode code={`message ~ "error.*" and not debug`} />

<FlyqlCode code={`(a = 1 or b = 2) and not (c = 3 and d = 4)`} />

<FlyqlCode code={`status in [200, 201] and method not in ['DELETE', 'PUT']`} />

<FlyqlCode code={`message has 'error' and tags not has 'debug'`} />

## Reglas generales

- **Claves independientes**: una clave sin operador se interpreta como una comprobación truthy
- **Comparaciones**: una clave con operador debe tener un valor correspondiente
- **Espacios en blanco**: se permiten espacios alrededor de los operadores; `status = 200` y `status = 200` son equivalentes
- **Precedencia de operadores**: `and` tiene mayor precedencia que `or`, así que `a = 1 or b = 2 and c = 3` se parsea como `a = 1 or (b = 2 and c = 3)`
- **Paréntesis**: usa `(` y `)` para cambiar la precedencia y agrupar condiciones de forma explícita

## Secciones de sintaxis

- [Operadores](/es/syntax/operators/): operadores de comparación (`=`, `!=`, `>`, `>=`, `<`, `<=`)
- [Lógica booleana](/es/syntax/boolean-logic/): `and`, `or`, `not`, precedencia y agrupación
- [Coincidencia de patrones](/es/syntax/pattern-matching/): regex (`~`, `!~`) y coincidencia con comodines (`*`)
- [Listas](/es/syntax/lists/): `in` y `not in` con valores de lista
- [Comprobación de contención](/es/syntax/operators/#comprobación-de-contención): `has` y `not has` para subcadenas, existencia de claves y pertenencia de elementos
- [Valores y expresiones](/es/syntax/values/): truthy/falsy, cadenas entre comillas, números
- [Claves anidadas](/es/syntax/nested-keys/): rutas separadas por puntos y segmentos de clave entre comillas
