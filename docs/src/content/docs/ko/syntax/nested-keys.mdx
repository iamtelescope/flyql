## 점 표기법

JSON, Map, 또는 구조화된 데이터의 중첩 필드에 접근하려면 점으로 구분된 경로를 사용합니다.

<FlyqlCode code={`request.url = "/api/users"`} />
<FlyqlCode code={`response.headers.content_type = "application/json"`} />
<FlyqlCode code={`metadata.labels.env = 'prod'`} />

점으로 구분된 각 세그먼트는 중첩 구조를 한 단계씩 내려갑니다. SQL generator는 이러한 경로를 각 데이터베이스 방언에 맞는 문법(JSON 경로 접근, Map 조회 등)으로 변환합니다.

## 따옴표로 감싼 키 세그먼트

키 세그먼트 자체에 점이 포함되어 있다면, 그 점이 구분자로 처리되지 않도록 세그먼트를 따옴표로 감싸세요.

<FlyqlCode code={`"foo.bar".baz = value`} />

여기서:
- `"foo.bar"` 는 하나의 키 세그먼트이며 점은 리터럴입니다
- <FlyqlToken type="key">baz</FlyqlToken> 는 `"foo.bar"` 의 하위 키입니다

따옴표가 없으면 <FlyqlToken type="key">foo.bar.baz</FlyqlToken>는 <FlyqlToken type="key">foo</FlyqlToken> → <FlyqlToken type="key">bar</FlyqlToken> → <FlyqlToken type="key">baz</FlyqlToken>의 세 세그먼트로 해석됩니다.

### 따옴표 규칙

키 세그먼트에는 작은따옴표와 큰따옴표를 모두 사용할 수 있습니다.

<FlyqlCode code={`"foo.bar".baz = value`} />
<FlyqlCode code={`'foo.bar'.baz = value`} />

점이 포함된 세그먼트에만 따옴표가 필요합니다.

<FlyqlCode code={`metadata."dotted.key".value = 123`} />

따옴표로 감싼 식별자를 사용하면 [예약어](/ko/syntax/reserved-words/)를 컬럼 이름으로 사용할 수도 있습니다 — 예: <FlyqlCode code={`"not" = 1`} inline />.

## 경로 깊이

중첩 깊이에 제한은 없습니다.

<FlyqlCode code={`a.b.c.d.e = value`} />

SQL을 생성하거나 메모리에서 평가할 때 각 세그먼트가 순서대로 순회됩니다.

## 연산자와 함께 사용

중첩 키는 모든 연산자와 함께 사용할 수 있습니다.

<FlyqlCode code={`request.status >= 400`} />
<FlyqlCode code={`response.body ~ "error.*"`} />
<FlyqlCode code={`metadata.tags in ['prod', 'staging']`} />
<FlyqlCode code={`request.headers.content_type != "text/html"`} />
<FlyqlCode code={`not request.authenticated`} />

## 배열 요소 접근

배열 타입 컬럼에서는 0부터 시작하는 숫자 경로 세그먼트를 사용해 요소에 접근합니다.

<FlyqlCode code={`tags.0 = 'important'`} />
<FlyqlCode code={`tags.1 = 'second'`} />

FlyQL의 배열 인덱싱은 **모든 SQL 방언과 인메모리 매처에서 0 기반** 입니다. 생성기는 flyql 인덱스를 각 대상의 네이티브 SQL 규칙으로 변환합니다. ClickHouse, StarRocks, PostgreSQL의 배열은 모두 SQL 계층에서 1 기반이므로, flyql의 `tags.0`은 모든 방언에서 `tags[1]`로 출력됩니다(방언별 식별자 인용 및 함수 형식 적용). 즉, 동일한 flyql 표현식이 방언마다 서로 다른 SQL 문자열을 생성하지만, 동일한 기반 데이터에 대해 **동일한 결과로 평가됩니다**.

## 예제

<FlyqlCode code={`request.method = 'GET'`} />
<FlyqlCode code={`response.headers.content_type = "application/json"`} />
<FlyqlCode code={`"app.config".debug = true`} />
<FlyqlCode code={`request.status >= 400 and response.body ~ "error" and not request.internal`} />
