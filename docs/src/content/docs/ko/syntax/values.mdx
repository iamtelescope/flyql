---
title: 값과 타입
description: FlyQL의 값 타입, 타입 지정 리터럴, 표현식
---

FlyQL은 7가지 값 타입을 지원합니다: **Integer**, **BigInt**, **Float**, **String**, **Boolean**, **Null**, **Array**. AST의 각 값에는 명시적인 `value_type` 필드가 포함됩니다.

## 값 타입

### Integer

int64 범위 안의 정수:

```
count = 42
status != 200
age >= 18
```

AST: `value_type: "integer"`

### BigInt

int64 범위(Go/Python) 또는 `MAX_SAFE_INTEGER`(JavaScript)를 초과하는 정수:

```
id = 18446744073709551615
```

AST: `value_type: "bigint"`

### Float

소수점을 포함한 숫자:

```
price > 9.99
ratio = 3.14
temperature<=-0.5
```

AST: `value_type: "float"`

### String

따옴표가 있거나 없는 텍스트 값:

```
level = error              # unquoted string
user = "John Doe"          # double-quoted (spaces)
message = 'hello world'    # single-quoted (spaces)
count = '42'               # quoted number is a string
```

AST: `value_type: "string"`

따옴표는 백슬래시로 이스케이프할 수 있습니다.

```
user = 'John
message="said \"hello\""
```

### Boolean

소문자이며 대소문자를 구분하는 키워드 `true`, `false`:

```
active = true
debug = false
active != true
```

AST: `value_type: "boolean"`, 값은 네이티브 boolean입니다.

대문자가 섞인 변형(`True`, `FALSE`)은 문자열로 처리됩니다.

**SQL 생성:**
| 방언 | `field=true` | `field=false` |
|------|--------------|---------------|
| ClickHouse | `field = true` | `field = false` |
| PostgreSQL | `field = TRUE` | `field = FALSE` |
| StarRocks | `field = true` | `field = false` |

### Null

소문자이며 대소문자를 구분하는 키워드 `null`:

```
field = null
field != null
```

AST: `value_type: "null"`, 값은 네이티브 null입니다.

대문자가 섞인 변형(`Null`, `NULL`)은 문자열로 처리됩니다.

**제약 사항:** null과 함께 유효한 연산자는 `=`와 `!=`뿐입니다. `>`, `<`, `>=`, `<=`, `~`, `!~`를 null과 함께 사용하면 parse error가 발생합니다.

**SQL 생성:** `field = null`은 `field IS NULL`, `field != null`은 `field IS NOT NULL`을 생성합니다 (모든 방언 공통).

### Array

Array는 `in`과 `not in`에서 사용합니다. 요소 타입은 서로 달라도 됩니다(heterogeneous).

```
status in [200, 201, 202]
name in ['alice', 'bob']
field in [1, "hello", true, null]
```

각 요소는 `values_types` 배열에 자신의 타입을 가집니다.

## Truthy/Falsy 표현식

연산자 없이 단독으로 쓰인 키는 해당 필드가 truthy 값인지 검사합니다.

```
active
message and status
```

다음 값은 **falsy**로 간주됩니다:
- `null` / `None` / 누락된 값
- 빈 문자열 `""`
- 0
- boolean `false`

그 외는 모두 **truthy**입니다.

falsy 값을 확인하려면 `not`을 사용하세요.

```
not archived
active and not debug
```

Truthy/Falsy 검사는 null 비교와 다릅니다. `not field`는 필드가 null, 빈 값 또는 0일 때 일치합니다. `field = null`은 필드가 정확히 null일 때만 일치합니다.

## 공백

FlyQL은 공백에 관대합니다. 연산자 주변 공백은 선택 사항입니다.

```
status = 200
status = 200
status =200
```

위 세 표현은 모두 동일합니다.

## 타입 상수 모듈

모든 타입 상수는 `flyql.types` 모듈에서 사용할 수 있습니다.

**Python:**
```python
from flyql.types import ValueType
# ValueType.INTEGER, ValueType.BIGINT, ValueType.FLOAT,
# ValueType.STRING, ValueType.BOOLEAN, ValueType.NULL, ValueType.ARRAY
```

**Go:**
```go
import "github.com/iamtelescope/flyql/golang/types"
// types.Integer, types.BigInt, types.Float,
// types.String, types.Boolean, types.Null, types.Array
```

**JavaScript:**
```javascript
import { ValueType } from 'flyql/types'
// ValueType.INTEGER, ValueType.BIGINT, ValueType.FLOAT,
// ValueType.STRING, ValueType.BOOLEAN, ValueType.NULL, ValueType.ARRAY
```

## 마이그레이션 참고

이전 FlyQL 버전에서 업그레이드하는 경우:

- **Breaking:** Go의 `ValueTypeString` / `ValueTypeNumber`는 제거되었고, 7개 값을 갖는 `types.ValueType`으로 대체되었습니다
- **Breaking:** AST `value_type` 필드는 `"string"` / `"number"`에서 `"integer"` / `"float"` / `"bigint"` / `"string"` / `"boolean"` / `"null"`로 변경되었습니다
- **Breaking:** 혼합 타입 리스트 강제 규칙이 제거되어, 이제 heterogeneous array가 유효합니다
- **New:** `true`, `false`, `null`은 문자열이 아니라 타입 지정 리터럴로 parse됩니다
- **New:** IN 리스트 표현식의 `values_types` 배열이 요소별 타입 정보를 제공합니다
- **New:** `flyql.types` 모듈이 정식 타입 상수를 export합니다
