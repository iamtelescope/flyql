## इंस्टॉलेशन

```bash
npm install flyql
```

**आवश्यकताएँ:** Node.js 16+। सभी इम्पोर्ट ES मॉड्यूल का उपयोग करते हैं।

## क्वेरी पार्स करें

```javascript
import { parse } from 'flyql'

const result = parse("status = 200 and active")
console.log(result.root)
```

`parse()` AST का प्रतिनिधित्व करने वाले `root` नोड के साथ एक `Parser` ऑब्जेक्ट लौटाता है। अमान्य इनपुट पर यह `ParserError` फेंकता है:

```javascript
import { parse, ParserError } from 'flyql'

try {
    const result = parse("status = 200 and active")
    console.log(result.root)
} catch (err) {
    if (err instanceof ParserError) {
        console.error(`Parse error: ${err.message}`)
    }
}
```

## SQL उत्पन्न करें

AST को पैरामीटराइज़्ड WHERE क्लॉज़ में बदलने के लिए जनरेटर का उपयोग करें। प्रत्येक डेटाबेस बोली एक अलग सबपाथ इम्पोर्ट है।

### ClickHouse

```javascript
import { parse } from 'flyql'
import { generateWhere, newColumn } from 'flyql/generators/clickhouse'

const result = parse("status >= 400 and host = 'prod*'")

const columns = {
    status: newColumn("status", false, "UInt32", null),
    host: newColumn("host", false, "String", null),
}

const sql = generateWhere(result.root, columns)
console.log(sql)
```

### अन्य बोलियाँ

PostgreSQL और StarRocks जनरेटर समान पैटर्न का पालन करते हैं:

```javascript
import { generateWhere, newColumn } from 'flyql/generators/postgresql'
import { generateWhere, newColumn } from 'flyql/generators/starrocks'
```

नोट: PostgreSQL के `newColumn` का अलग हस्ताक्षर है — `newColumn(name, type, values)` (कोई `jsonString` पैरामीटर नहीं)।

## इन-मेमोरी मैच

SQL उत्पन्न किए बिना डेटा रिकॉर्ड के विरुद्ध क्वेरी का मूल्यांकन करें:

```javascript
import { match } from 'flyql/matcher'

const data = {
    status: 200,
    active: true,
    host: "prod-api-01",
}

const matches = match("status = 200 and active", data)
console.log(`Matches: ${matches}`) // true
```

## कॉलम एक्सप्रेशन पार्स करें

कॉलम एक्सप्रेशन को सेगमेंट, ट्रांसफ़ॉर्मर, और उपनामों के साथ संरचित डेटा में पार्स करें:

```javascript
import { parse, parseToJson } from 'flyql/columns'

// Parse basic columns (transformers disabled by default)
const parsed = parse("message, status")
for (const col of parsed) {
    console.log(`${col.name} (display: ${JSON.stringify(col.displayName)}, segments: ${col.segments})`)
}

// Enable transformers via capabilities
const withTransforms = parse("message|chars(25) as msg, status", { transformers: true })

// Or serialize directly to JSON for API responses
const json = parseToJson("message, status|upper", { transformers: true })
console.log(json)
```

## क्वेरी में ट्रांसफ़ॉर्मर

ट्रांसफ़ॉर्मर के साथ क्वेरी पार्स करें और SQL उत्पन्न करें:

```javascript
import { parse } from 'flyql'
import { generateWhere, newColumn } from 'flyql/generators/clickhouse'

// Parse a transformer query
const result = parse("message|upper = 'ERROR'")

// Generate SQL for ClickHouse
const columns = { message: newColumn('message', false, 'String') }
const sql = generateWhere(result.root, columns)
console.log(sql) // equals(upper(message), 'ERROR')
```

पूर्ण सिंटैक्स संदर्भ के लिए [ट्रांसफ़ॉर्मर](/hi/syntax/transformers/) देखें।

## उपलब्ध सबपाथ इम्पोर्ट

| इम्पोर्ट पथ | सामग्री |
|-------------|----------|
| `flyql` | कोर पार्सर (`parse`, `ParserError`, ऑपरेटर, स्थिरांक) |
| `flyql/core` | कोर पार्सर मॉड्यूल सीधे |
| `flyql/columns` | कॉलम एक्सप्रेशन पार्सर |
| `flyql/generators/clickhouse` | ClickHouse SQL जनरेटर |
| `flyql/generators/postgresql` | PostgreSQL SQL जनरेटर |
| `flyql/generators/starrocks` | StarRocks SQL जनरेटर |
| `flyql/matcher` | इन-मेमोरी रिकॉर्ड मैचर |
| `flyql/editor` | वेब एडिटर कम्पोनेंट (Vue 3 आवश्यक) |
