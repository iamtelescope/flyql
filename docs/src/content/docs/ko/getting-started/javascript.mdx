## 설치

```bash
npm install flyql
```

**요구 사항:** Node.js 16+. 모든 import는 ES modules를 사용합니다.

## 쿼리 parse하기

```javascript
import { parse } from 'flyql'

const result = parse("status = 200 and active")
console.log(result.root)
```

`parse()`는 AST를 나타내는 `root` 노드를 가진 `Parser` 객체를 반환합니다. 입력이 잘못되면 `ParserError`를 throw합니다.

```javascript
import { parse, ParserError } from 'flyql'

try {
    const result = parse("status = 200 and active")
    console.log(result.root)
} catch (err) {
    if (err instanceof ParserError) {
        console.error(`Parse error: ${err.message}`)
    }
}
```

## SQL 생성

Generator를 사용하면 AST를 파라미터화된 WHERE 절로 바꿀 수 있습니다. 각 데이터베이스 방언은 별도의 하위 경로에서 import합니다.

### ClickHouse

```javascript
import { parse } from 'flyql'
import { generateWhere, newColumn } from 'flyql/generators/clickhouse'

const result = parse("status >= 400 and host = 'prod*'")

const columns = {
    status: newColumn("status", false, "UInt32", null),
    host: newColumn("host", false, "String", null),
}

const sql = generateWhere(result.root, columns)
console.log(sql)
```

### 다른 방언

PostgreSQL과 StarRocks generator도 같은 패턴을 따릅니다.

```javascript
import { generateWhere, newColumn } from 'flyql/generators/postgresql'
import { generateWhere, newColumn } from 'flyql/generators/starrocks'
```

참고: PostgreSQL의 `newColumn`은 시그니처가 다릅니다. `newColumn(name, type, values)`이며 `jsonString` 매개변수가 없습니다.

## 메모리에서 매칭하기

SQL을 생성하지 않고 데이터 레코드에 대해 쿼리를 평가할 수 있습니다.

```javascript
import { match } from 'flyql/matcher'

const data = {
    status: 200,
    active: true,
    host: "prod-api-01",
}

const matches = match("status = 200 and active", data)
console.log(`Matches: ${matches}`) // true
```

## 컬럼 표현식 parse하기

컬럼 표현식을 segments, transformers, aliases를 포함하는 구조화된 데이터로 parse할 수 있습니다.

```javascript
import { parse, parseToJson } from 'flyql/columns'

// 기본 컬럼 parse (transformers는 기본적으로 비활성화)
const parsed = parse("message, status")
for (const col of parsed) {
    console.log(`${col.name} (display: ${JSON.stringify(col.displayName)}, segments: ${col.segments})`)
}

// capabilities로 transformers 활성화
const withTransforms = parse("message|chars(25) as msg, status", { transformers: true })

// 또는 API 응답용으로 직접 JSON으로 직렬화
const json = parseToJson("message, status|upper", { transformers: true })
console.log(json)
```

## 파라미터화된 쿼리

`$name` 및 `$1` placeholder를 사용하고, 런타임에 `bindParams()`로 해석할 수 있습니다.

```javascript
import { parse, bindParams } from 'flyql'

const ast = parse('status = $code and env = $env')
bindParams(ast.root, { code: 200, env: 'prod' })
```

전체 레퍼런스는 [파라미터](/ko/syntax/parameters/)를 참고하세요.

## 쿼리에서 Transformers 사용하기

transformer가 포함된 쿼리를 parse하고 SQL을 생성할 수 있습니다.

```javascript
import { parse } from 'flyql'
import { generateWhere, newColumn } from 'flyql/generators/clickhouse'

// transformer가 포함된 쿼리 parse
const result = parse("message|upper = 'ERROR'")

// ClickHouse용 SQL 생성
const columns = { message: newColumn('message', false, 'String') }
const sql = generateWhere(result.root, columns)
console.log(sql) // equals(upper(message), 'ERROR')
```

전체 문법은 [Transformers](/ko/syntax/transformers/)를 참고하세요.

## 사용 가능한 하위 경로 import

| Import 경로 | 내용 |
|-------------|------|
| `flyql` | Core parser (`parse`, `ParserError`, 연산자, 상수) |
| `flyql/core` | Core parser 모듈 직접 사용 |
| `flyql/columns` | 컬럼 표현식 parser |
| `flyql/generators/clickhouse` | ClickHouse SQL generator |
| `flyql/generators/postgresql` | PostgreSQL SQL generator |
| `flyql/generators/starrocks` | StarRocks SQL generator |
| `flyql/matcher` | In-memory record matcher |
| `flyql/editor` | 웹 editor 컴포넌트 (Vue 3 필요) |
