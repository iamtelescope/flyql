## Filtrar registros donde `event_type` sea `'login'` o `'logout'`

## Consistencia de tipos en listas IN

El parser **siempre** acepta listas heterogéneas como <FlyqlCode code={`[200, 'ok', 404]`} inline />. Los generadores SQL ejecutan la verificación de consistencia de tipos **únicamente** cuando (a) la columna tiene un tipo declarado, (b) la clave no es segmentada y (c) la lista es homogénea (`len(set(values_types)) == 1`). Las listas heterogéneas, las claves segmentadas y las columnas sin tipo evitan la verificación en el sitio de llamada.

La receta demuestra tres casos contra columnas tipadas: una coincidencia homogénea, un bypass heterogéneo y un caso homogéneo de tipo incorrecto que lanza el error canónico `type mismatch in IN list: ...`. El texto del error es idéntico en Python, Go y JavaScript.
