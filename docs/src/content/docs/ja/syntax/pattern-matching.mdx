## Regex マッチ

値を正規表現に一致させるには <FlyqlToken type="operator">{'~'}</FlyqlToken>、否定するには <FlyqlToken type="operator">{'!~'}</FlyqlToken> を使います。

<FlyqlCode code={`message ~ "error.*timeout"`} />
<FlyqlCode code={`path !~ "^/health"`} />

| 演算子 | 意味 |
|--------|------|
| <FlyqlToken type="operator">{'~'}</FlyqlToken> | 値が regex パターンに一致する |
| <FlyqlToken type="operator">{'!~'}</FlyqlToken> | 値が regex パターンに一致しない |

### Regex パターン構文

regex の構文はターゲットのバックエンドによって異なります。

- **ClickHouse**: `match()` 関数を使用（RE2 構文）
- **PostgreSQL**: <FlyqlToken type="operator">{'~'}</FlyqlToken> と <FlyqlToken type="operator">{'!~'}</FlyqlToken> 演算子を使用（POSIX regex）
- **StarRocks**: `regexp` 関数を使用
- **メモリ上の matcher**: ホスト言語の regex engine を使用（Go `regexp`、Python `re2`、JS `RegExp`）

スペースや特殊文字を含むパターンはクォートしてください。

<FlyqlCode code={`message ~ "connection (refused|reset)"`} />
<FlyqlCode code={`host ~ "^prod-[0-9]+"`} />

### Regex フラグ

パターンの先頭に `(?...)` 構文でインラインフラグを指定すると、regex の動作を変更できます。FlyQL はすべてのバックエンドで動作する 3 つのフラグをサポートしています。

#### `(?i)` — 大文字小文字を区別しない

大文字小文字に関係なくパターンをマッチさせます。

<FlyqlCode code={`message ~ "(?i)error"`} />
<FlyqlCode code={`level ~ "(?i)^(warn|error|fatal)$"`} />

`(?i)error` は「Error」「ERROR」「error」などにマッチします。

#### `(?m)` — マルチライン

`^` と `$` が文字列全体ではなく、各**行**の先頭と末尾にマッチするよう変更します。

<FlyqlCode code={`log ~ "(?m)^ERROR"`} />

`(?m)` なしの場合、`^ERROR` は文字列全体が「ERROR」で始まるときのみマッチします。`(?m)` ありの場合、「INFO first\nERROR second」にもマッチします。なぜなら「ERROR」が行頭に出現するためです。

#### `(?s)` — Dotall

`.` が改行を含む**すべての**文字にマッチするようにします。デフォルトでは `.` は `\n` にマッチしません。

<FlyqlCode code={`log ~ "(?s)start.*end"`} />

`(?s)` なしの場合、`start.*end` は「start\nend」にマッチしません（`.` が改行を越えないため）。`(?s)` ありならマッチします。

#### フラグの組み合わせ

フラグは 1 つのグループ内で組み合わせ可能です。

<FlyqlCode code={`log ~ "(?im)^ERROR"`} />
<FlyqlCode code={`log ~ "(?ims)^START.*END$"`} />

#### バックエンドのサポート状況

3 つのフラグはすべての FlyQL バックエンドで動作します。

| バックエンド | `(?i)` | `(?m)` | `(?s)` |
|--------------|--------|--------|--------|
| **ClickHouse** | Yes | Yes | Yes |
| **PostgreSQL** | Yes | Yes | Yes |
| **StarRocks** | Yes | Yes | Yes |
| **Go matcher** | Yes | Yes | Yes |
| **Python matcher** | Yes | Yes | Yes |
| **JavaScript matcher** | Yes | Yes | Yes |

大文字小文字を区別しないマッチと否定を組み合わせることで、大小文字に関係なく特定パターンを除外できます。

<FlyqlCode code={`message ~ "(?i)error" and message !~ "(?i)warning"`} />

## LIKE マッチ

SQL スタイルのワイルドカードパターンマッチには <FlyqlToken type="operator">like</FlyqlToken> と <FlyqlToken type="operator">ilike</FlyqlToken> を使います。

<FlyqlCode code={`host like 'prod%'`} />
<FlyqlCode code={`service like 'api-%'`} />
<FlyqlCode code={`message ilike '%Error%'`} />

| 演算子 | 意味 |
|--------|------|
| <FlyqlToken type="operator">like</FlyqlToken> | 値がパターンに一致する（大文字小文字を区別） |
| <FlyqlToken type="operator">not like</FlyqlToken> | 値がパターンに一致しない |
| <FlyqlToken type="operator">ilike</FlyqlToken> | 値がパターンに一致する（大文字小文字を区別しない） |
| <FlyqlToken type="operator">not ilike</FlyqlToken> | 値がパターンに一致しない（大文字小文字を区別しない） |

### ワイルドカード

<FlyqlToken type="operator">like</FlyqlToken> / <FlyqlToken type="operator">ilike</FlyqlToken> パターンでは 2 種類のワイルドカードが使えます。

| ワイルドカード | 意味 | 例 | 一致する値 |
|---------------|------|-----|-----------|
| `%` | 任意の文字列（空文字を含む） | <FlyqlCode code={`host like 'prod%'`} inline /> | `prod`, `prod-1`, `production` |
| `_` | ちょうど 1 文字 | <FlyqlCode code={`code like 'E__'`} inline /> | `E01`, `Exx`。`E1` や `Error` には一致しない |

ワイルドカードはパターン内のどこにでも配置できます — 先頭、中間、末尾、組み合わせ。

<FlyqlCode code={`message like '%timeout%'`} />
<FlyqlCode code={`host like 'web-_-prod'`} />
<FlyqlCode code={`path like '/api/%/status'`} />

### ワイルドカードのエスケープ

リテラルの `%` や `_` に一致させるにはバックスラッシュでエスケープします。

<FlyqlCode code={`value like '100\\%'`} />
<FlyqlCode code={`col like 'foo\\_bar'`} />

`100\%` はリテラル文字列 "100%" に一致し、"100" に任意の文字列が続くものには一致しません。

### 大文字小文字を区別しないマッチ

大文字小文字を区別しないパターンマッチには <FlyqlToken type="operator">ilike</FlyqlToken> を使います。

<FlyqlCode code={`message ilike '%error%'`} />
<FlyqlCode code={`host not ilike 'TEST%'`} />

### LIKE と regex の違い

| ユースケース | 構文 | 例 |
|-------------|------|----|
| プレフィックス一致 | <FlyqlCode code={`key like 'prefix%'`} inline /> | <FlyqlCode code={`host like 'prod%'`} inline /> |
| 部分文字列を含む | <FlyqlCode code={`key like '%word%'`} inline /> | <FlyqlCode code={`msg like '%error%'`} inline /> |
| 複雑なパターン | <FlyqlCode code={`key ~ "pattern"`} inline /> | <FlyqlCode code={`host ~ "^prod-[0-9]+"`} inline /> |

単純なワイルドカードマッチには <FlyqlToken type="operator">like</FlyqlToken> を使い、文字クラス、選択、その他の高度なパターンが必要な場合は regex (<FlyqlToken type="operator">{'~'}</FlyqlToken>) を使ってください。

### バックエンドのサポート状況

すべての SQL バックエンド（ClickHouse、PostgreSQL、StarRocks）はネイティブの `LIKE` / `ILIKE` / `NOT LIKE` / `NOT ILIKE` SQL 演算子を出力します。メモリ上の matcher はパターンを内部で regex に変換します。

:::note
StarRocks の `ILIKE` はバージョン 2.5 以降が必要です。
:::

## 例

<FlyqlCode code={`message ~ "error|fail"`} />
<FlyqlCode code={`path ~ "^/api/v2"`} />
<FlyqlCode code={`path !~ "^/health"`} />
<FlyqlCode code={`host like 'prod%'`} />
<FlyqlCode code={`host like 'prod%' and message ~ "timeout.*" and path !~ "^/health"`} />
