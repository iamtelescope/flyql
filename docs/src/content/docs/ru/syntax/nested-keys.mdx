## Точечная нотация

Используйте пути через точку для доступа к вложенным полям в JSON, Map или структурированных данных:

<FlyqlCode code={`request.url = "/api/users"`} />
<FlyqlCode code={`response.headers.content_type = "application/json"`} />
<FlyqlCode code={`metadata.labels.env = 'prod'`} />

Каждый сегмент после точки означает переход на один уровень глубже во вложенную структуру. SQL-генераторы преобразуют эти пути в корректный синтаксис для каждого диалекта базы данных (доступ по JSON-пути, поиск в Map и т. д.).

## Сегменты ключа в кавычках

Если сам сегмент ключа содержит точку, оберните его в кавычки, чтобы точка не интерпретировалась как разделитель:

<FlyqlCode code={`"foo.bar".baz = value`} />

Здесь:
- `"foo.bar"` — это один сегмент ключа (точка является литералом)
- <FlyqlToken type="key">baz</FlyqlToken> — дочерний элемент `"foo.bar"`

Без кавычек <FlyqlToken type="key">foo.bar.baz</FlyqlToken> будет интерпретировано как три сегмента: <FlyqlToken type="key">foo</FlyqlToken> → <FlyqlToken type="key">bar</FlyqlToken> → <FlyqlToken type="key">baz</FlyqlToken>.

### Правила использования кавычек

Для сегментов ключей можно использовать как одинарные, так и двойные кавычки:

<FlyqlCode code={`"foo.bar".baz = value`} />
<FlyqlCode code={`'foo.bar'.baz = value`} />

Оборачивать в кавычки нужно только сегменты, содержащие точки:

<FlyqlCode code={`metadata."dotted.key".value = 123`} />

## Глубина пути

Ограничений на глубину вложенности нет:

<FlyqlCode code={`a.b.c.d.e = value`} />

Каждый сегмент обрабатывается последовательно при генерации SQL или вычислении в памяти.

## Комбинация с операторами

Вложенные ключи работают со всеми операторами:

<FlyqlCode code={`request.status >= 400`} />
<FlyqlCode code={`response.body ~ "error.*"`} />
<FlyqlCode code={`metadata.tags in ['prod', 'staging']`} />
<FlyqlCode code={`request.headers.content_type != "text/html"`} />
<FlyqlCode code={`not request.authenticated`} />

## Доступ к элементам массива

Для колонок типа массив используйте числовые сегменты пути с **нуля** для доступа к элементам:

<FlyqlCode code={`tags.0 = 'important'`} />
<FlyqlCode code={`tags.1 = 'second'`} />

Индексация массивов в FlyQL **является 0-индексной во всех диалектах SQL и во встроенном matcher**. Генераторы переводят индекс flyql в нативное SQL-представление каждого целевого диалекта: массивы в ClickHouse, StarRocks и PostgreSQL на уровне SQL индексируются с единицы, поэтому flyql-выражение `tags.0` во всех диалектах транслируется в `tags[1]` (с диалект-специфичным экранированием идентификаторов и формой функций). То есть одно и то же flyql-выражение порождает разные SQL-строки в разных диалектах, но **возвращает одинаковый результат** для одних и тех же исходных данных.

## Примеры

<FlyqlCode code={`"app.config".debug = true`} />
