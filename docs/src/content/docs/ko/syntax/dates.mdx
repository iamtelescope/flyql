FlyQL은 날짜와 날짜시간을 비교하는 두 가지 방법을 제공합니다: 절대 날짜를 위한 **문자열 리터럴**과 상대 시간 윈도우를 위한 **시간 함수**. 쿼리의 의도에 맞는 것을 선택하세요.

## 문자열 리터럴 형태의 날짜

날짜 컬럼을 ISO-8601 형식의 문자열 리터럴과 비교합니다. ClickHouse, PostgreSQL, StarRocks 모두 ISO-8601 문자열을 데이터베이스 경계에서 `DATE` / `DATETIME` 값으로 변환하므로, FlyQL 레벨에서는 특별한 날짜 생성자가 필요 없습니다.

<FlyqlCode code={`due_date < '2023-12-31'`} />
<FlyqlCode code={`created_at >= '2023-12-31T00:00:00Z'`} />
<FlyqlCode code={`event_time >= '2023-01-01' and event_time <= '2023-12-31'`} />

| 컬럼 타입 | 받는 값 | 생성된 비교 |
|----------|--------|-------------|
| ClickHouse `Date` / `DateTime` / `DateTime64` | ISO-8601 문자열 | 직접 비교 — ClickHouse가 실행 시 리터럴 파싱 |
| PostgreSQL `date` / `timestamp` / `timestamptz` | ISO-8601 문자열 | 직접 비교 — PostgreSQL이 실행 시 리터럴 캐스트 |
| StarRocks `DATE` / `DATETIME` | ISO-8601 문자열 | 직접 비교 — StarRocks가 실행 시 리터럴 파싱 |

리터럴은 데이터베이스 자체가 파싱하기 때문에 시간, 시간대 오프셋, 날짜만의 형식 모두 지원됩니다.

:::note[문자열은 따옴표가 필수]
날짜 리터럴은 문자열이며 **반드시** 따옴표로 감싸야 합니다. 따옴표가 없으면 FlyQL은 값을 컬럼 참조로 해석합니다. <FlyqlCode code={`due_date < 2023-12-31`} inline />는 `2023`이라는 이름의 컬럼에 대한 산술로 파싱되며, 날짜 리터럴이 되지 않습니다. [값과 타입 — String](/ko/syntax/values/#string)을 참조하세요.
:::

## 시간 함수

상대 시간 쿼리 — "지난 한 시간 내", "이번 주 시작부터", "오늘부터" — 의 경우 FlyQL은 내장 시간 함수를 제공합니다: <FlyqlCode code={`ago(...)`} inline />, <FlyqlCode code={`now()`} inline />, <FlyqlCode code={`today()`} inline />, <FlyqlCode code={`startOf(...)`} inline />.

<FlyqlCode code={`created_at > ago(1h)`} />
<FlyqlCode code={`updated_at >= ago(7d)`} />
<FlyqlCode code={`expires_at < now()`} />
<FlyqlCode code={`event_time > startOf('day')`} />
<FlyqlCode code={`scheduled_for >= today('Europe/Berlin')`} />

전체 문법(단위, 순서 규칙, 시간대 인수)은 [값과 타입 — 시간 함수](/ko/syntax/values/)에서 다룹니다.

## 언제 무엇을 사용할까

| 의도 | 사용 | 예시 |
|------|------|------|
| 고정된 달력 날짜 또는 타임스탬프와 비교 | 문자열 리터럴 | <FlyqlCode code={`due_date < '2023-12-31'`} inline /> |
| 슬라이딩 시간 윈도우와 비교 | <FlyqlCode code={`ago()`} inline /> | <FlyqlCode code={`created_at > ago(1h)`} inline /> |
| 현재 순간과 비교 | <FlyqlCode code={`now()`} inline /> | <FlyqlCode code={`expires_at < now()`} inline /> |
| 오늘과 비교(시간대 인식) | <FlyqlCode code={`today()`} inline /> | <FlyqlCode code={`scheduled_for = today('UTC')`} inline /> |
| 일/주/월의 시작과 비교 | <FlyqlCode code={`startOf()`} inline /> | <FlyqlCode code={`event_time > startOf('week')`} inline /> |

문자열 리터럴은 일정합니다 — 같은 쿼리 문자열은 매일 같은 SQL을 반환합니다. 시간 함수는 실행 시 평가되므로, 같은 FlyQL 쿼리가 실행될 때마다 다른 시간 윈도우를 생성합니다.
