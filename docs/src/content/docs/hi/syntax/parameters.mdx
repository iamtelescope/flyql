FlyQL **पैरामीटर प्लेसहोल्डर** का समर्थन करता है — `$name` (नामित) और `$1` (स्थितीय) — जो किसी भी स्थान पर लिटरल की जगह खड़े हो सकते हैं जहाँ सामान्यतः लिटरल आता है। पैरामीटर रनटाइम पर `bindParams()` कॉल करके हल किए जाते हैं, SQL उत्पन्न करने या क्वेरी का इन-मेमोरी मूल्यांकन करने से पहले।

यह क्वेरी टेम्पलेट करने का सुरक्षित तरीका है: आप क्वेरी टेक्स्ट को स्थिर रखते हैं और बाद में ठोस मानों को बाइंड करते हैं। कोई स्ट्रिंग कॉन्कैटनेशन नहीं, कोई एस्केपिंग नहीं, कोई इंजेक्शन जोखिम नहीं।

## पैरामीटर क्यों

एक सामान्य उपयोग केस एक टूल या डैशबोर्ड है जो विभिन्न इनपुट के साथ समान क्वेरी आकार चलाता है:

<FlyqlCode code={`status = $code and env = $env`} />

```python
from flyql import parse, bind_params

ast = parse("status = $code and env = $env")
bind_params(ast.root, {"code": 200, "env": "prod"})
```

पैरामीटर मान फ़ॉर्म फ़ील्ड, API अनुरोधों, या किसी अन्य रनटाइम स्रोत से आपूर्ति किए जा सकते हैं — बिना कभी क्वेरी स्ट्रिंग को पुनर्निर्माण किए।

## सिंटैक्स

### नामित पैरामीटर

एक नामित पैरामीटर `$` के बाद एक अक्षर या अंडरस्कोर, फिर अक्षरों, अंकों, और अंडरस्कोर के किसी भी संयोजन से बनता है:

<FlyqlCode code={`status = $code`} />
<FlyqlCode code={`env = $my_env`} />
<FlyqlCode code={`user = $_internal`} />

पैरामीटर नाम में `$` उपसर्ग **शामिल नहीं** होता — `$code` `code` नामक पैरामीटर को संदर्भित करता है।

### स्थितीय पैरामीटर

एक स्थितीय पैरामीटर `$` के बाद केवल अंकों से बनता है। स्थितीय पैरामीटर **1-अनुक्रमित** हैं (SQL `$1` की तरह):

<FlyqlCode code={`a = $1 and b = $2`} />

`$0` की अनुमति नहीं है। `$1abc` (अंकों के बाद अक्षर) की अनुमति नहीं है।

## पैरामीटर कहाँ दिखाई दे सकते हैं

पैरामीटर हर उस स्थान पर काम करते हैं जहाँ एक लिटरल मान मान्य है:

### तुलना मान

<FlyqlCode code={`status = $code`} />
<FlyqlCode code={`age >= $min_age`} />
<FlyqlCode code={`name != $banned`} />
<FlyqlCode code={`score > $1`} />

### `has`, `like`, `ilike`

<FlyqlCode code={`message has $needle`} />
<FlyqlCode code={`path like $pattern`} />
<FlyqlCode code={`title ilike $search`} />
<FlyqlCode code={`tags not has $excluded`} />

### IN-सूचियाँ

पैरामीटर को `in` और `not in` सूचियों के अंदर लिटरल मानों के साथ मिलाया जा सकता है:

<FlyqlCode code={`status in [$x, $y]`} />
<FlyqlCode code={`env in [$current, 'staging', 'dev']`} />
<FlyqlCode code={`code not in [$banned, 401, 403]`} />

### टेम्पोरल फ़ंक्शन तर्क

पैरामीटर `ago()` के अंदर अवधि मानों, और `today()` तथा `startOf()` के अंदर टाइमज़ोन तर्कों को प्रतिस्थापित कर सकते हैं:

<FlyqlCode code={`created > ago($duration)`} />
<FlyqlCode code={`date = today($tz)`} />
<FlyqlCode code={`created > startOf('day', $tz)`} />

`ago()` पैरामीटर के लिए बाउंड मान एक अवधि स्ट्रिंग होनी चाहिए जैसे `"5m"`, `"1h"`, या `"7d"`।

### ट्रांसफ़ॉर्मर तर्क

पैरामीटर ट्रांसफ़ॉर्मर तर्क स्थितियों के अंदर दिखाई दे सकते हैं:

<FlyqlCode code={`tags|split($sep) has $needle`} />

## bindParams()

पार्स करने के बाद, पैरामीटर प्लेसहोल्डरों को ठोस मानों से प्रतिस्थापित करने के लिए `bindParams()` कॉल करें। फ़ंक्शन AST को यथास्थान वॉक करता है और मिलने वाले हर `Parameter` नोड को प्रतिस्थापित करता है।

### Python

```python
from flyql import parse, bind_params

ast = parse("status = $code and env = $env")
bind_params(ast.root, {"code": 200, "env": "prod"})
```

### JavaScript

```javascript
import { parse, bindParams } from 'flyql'

const ast = parse('status = $code and env = $env')
bindParams(ast.root, { code: 200, env: 'prod' })
```

### Go

```go
import flyql "github.com/iamtelescope/flyql/golang"

ast, _ := flyql.Parse(`status = $code and env = $env`)
err := flyql.BindParams(ast.Root, map[string]any{
    "code": 200,
    "env":  "prod",
})
```

### API

| भाषा | हस्ताक्षर |
|----------|-----------|
| Python | `bind_params(node, params: dict) -> Node` |
| JavaScript | `bindParams(node, params: object): Node` |
| Go | `BindParams(node *Node, params map[string]any) error` |

- **`node`** — `parse()` द्वारा लौटाया गया रूट `Node`।
- **`params`** — पैरामीटर नामों को ठोस मानों से मैप करने वाला dict/object/map। नामित पैरामीटर कुंजी के रूप में नाम का उपयोग करते हैं (जैसे `"code"`)। स्थितीय पैरामीटर अंकों की स्ट्रिंग कुंजी का उपयोग करते हैं (जैसे `"1"`, `"2"`)। मिश्रित dict की अनुमति है।
- **रिटर्न मान** — Python और JS चेनिंग के लिए वही `Node` लौटाते हैं (यथास्थान बदला गया)। Go एक `error` लौटाता है।

### प्रकार मैपिंग

बाउंड मान का नेटिव प्रकार AST में परिणामी `value_type` निर्धारित करता है:

| नेटिव प्रकार | परिणामी `value_type` |
|-------------|------------------------|
| `string` | `"string"` |
| `int` (int64 रेंज के भीतर) | `"integer"` |
| `int` (int64 रेंज से बाहर) | `"bigint"` |
| `float` | `"float"` |
| `bool` | `"boolean"` |
| `null` / `None` / `nil` | `"null"` |

प्रकार **बाउंड मान के रनटाइम प्रकार** से निर्धारित होता है, मान को स्ट्रिंग के रूप में पार्स करके नहीं। `"200"` को बाइंड करना एक स्ट्रिंग उत्पन्न करता है, पूर्णांक नहीं:

```python
bind_params(parse("a = $x").root, {"x": "200"})
# expression.value      → "200"
# expression.value_type → "string"

bind_params(parse("a = $x").root, {"x": 200})
# expression.value      → 200
# expression.value_type → "integer"
```

असमर्थित प्रकार (जैसे `Decimal`, `BigInt`, कस्टम ऑब्जेक्ट, `undefined`) त्रुटि उत्पन्न करते हैं:

```
unsupported parameter value type: Decimal
```

## सख्त मान्यकरण

`bindParams()` अनुपस्थित और अतिरिक्त दोनों पैरामीटर के बारे में **सख्त** है।

### अनुपस्थित पैरामीटर

यदि क्वेरी किसी ऐसे पैरामीटर का संदर्भ देती है जो प्रदत्त dict में नहीं है, तो एक त्रुटि उत्पन्न होती है:

```python
bind_params(parse("a = $x").root, {})
# FlyqlError: unbound parameter: $x
```

### अतिरिक्त पैरामीटर

यदि dict में ऐसा पैरामीटर है जो क्वेरी में संदर्भित नहीं है, तो एक त्रुटि उत्पन्न होती है:

```python
bind_params(parse("a = $x").root, {"x": 1, "y": 2})
# FlyqlError: unused parameter: y
```

यह सामान्य SQL ड्राइवरों से अधिक सख्त है, जो अक्सर पैरामीटर मान्यकरण को क्वेरी निष्पादन समय तक टाल देते हैं। बाइंड समय पर इन त्रुटियों को पकड़ना गलतियों को दृश्यमान रखता है।

### स्थितीय पैरामीटर

स्थितीय पैरामीटर के लिए, `1` से अधिकतम संदर्भित सूचकांक तक के प्रत्येक सूचकांक को प्रदान किया जाना चाहिए। अंतरालों की अनुमति नहीं है:

```python
bind_params(parse("a = $1 and b = $3").root, {"1": 1, "3": 3})
# FlyqlError: unbound parameter: $2
```

## इन-मेमोरी मैचिंग

पैरामीटर मैचर के साथ उसी तरह काम करते हैं। पहले बाइंड करें, फिर मूल्यांकन करें:

```python
from flyql import parse, bind_params
from flyql.matcher.evaluator import Evaluator

ast = parse("status = $code")
bind_params(ast.root, {"code": 200})

result = Evaluator().evaluate(ast.root, {"status": 200})
```

यदि आप मूल्यांकन से पहले `bind_params()` कॉल करना भूल जाते हैं, तो मैचर एक स्पष्ट त्रुटि उत्पन्न करता है:

```
unbound parameter '$code' — call bind_params() before evaluating
```

वही सुरक्षा प्रत्येक SQL जनरेटर में मौजूद है, इसलिए आप गलती से अनसुलझे पैरामीटर वाला SQL उत्पन्न नहीं कर सकते।

## AST प्रतिनिधित्व

एक पैरामीटर AST में `value_type: "parameter"` और एक `Parameter` मान ऑब्जेक्ट के साथ एक `Expression` के रूप में दर्शाया जाता है:

क्वेरी: <FlyqlCode code="status = $code" inline />

```
Node (leaf)
  expression:
    key: { segments: ["status"], raw: "status" }
    operator: "="
    value: { name: "code", positional: false }
    value_type: "parameter"
```

IN-सूचियों में पैरामीटर के लिए, `Parameter` ऑब्जेक्ट `expression.values` के अंदर दिखाई देता है:

क्वेरी: <FlyqlCode code="status in [$x, 'ok']" inline />

```
Node (leaf)
  expression:
    key: { segments: ["status"], raw: "status" }
    operator: "in"
    values: [{ name: "x", positional: false }, "ok"]
    values_types: ["parameter", "string"]
```

टेम्पोरल फ़ंक्शन में पैरामीटर के लिए, वे `FunctionCall.parameter_args` में दिखाई देते हैं:

क्वेरी: <FlyqlCode code="created > ago($d)" inline />

```
Node (leaf)
  expression:
    key: { segments: ["created"], raw: "created" }
    operator: ">"
    value:
      name: "ago"
      duration_args: []
      parameter_args: [{ name: "d", positional: false }]
    value_type: "function"
```

`bindParams()` द्वारा उन्हें हल करने के बाद, ये `Parameter` प्लेसहोल्डर ठोस मानों से प्रतिस्थापित हो जाते हैं और AST एक सामान्य पार्स की गई क्वेरी की तरह दिखता है।

## त्रुटियाँ

| त्रुटि | Errno | कब |
|-------|-------|------|
| `empty parameter name` | 74 | क्वेरी में बिना नाम के `$` है (जैसे, `status = $`) |
| `invalid parameter name` | 75 | नाम अंक से शुरू होता है लेकिन गैर-अंक शामिल हैं (जैसे, `$1abc`) |
| `positional parameters are 1-indexed` | 76 | स्थितीय `$0` (जैसे, `status = $0`) |

`bindParams()` `FlyqlError` उत्पन्न करता है (या Go में त्रुटि लौटाता है) इन के लिए:

- `unbound parameter: $name` — क्वेरी में संदर्भित, dict से अनुपस्थित
- `unused parameter: name` — dict में प्रदान किया गया, क्वेरी में संदर्भित नहीं
- `unsupported parameter value type: <type>` — बाउंड मान समर्थित प्रकारों में से एक नहीं है
