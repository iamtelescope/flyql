## Согласованность типов в IN-списке

Парсер **всегда** принимает гетерогенные списки вроде <FlyqlCode code={`[200, 'ok', 404]`} inline />. SQL-генераторы запускают проверку согласованности типов **только**, когда (а) у колонки объявлен тип, (б) ключ не сегментированный и (в) список однородный (`len(set(values_types)) == 1`). Гетерогенные списки, сегментированные ключи и колонки без типа обходят проверку на стороне вызова.

Рецепт демонстрирует три случая на колонках с типом: однородное соответствие, гетерогенный пропуск и однородный неверный тип, который вызывает каноническую ошибку `type mismatch in IN list: ...`. Текст ошибки идентичен в Python, Go и JavaScript.
