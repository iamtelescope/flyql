## Filtrer les enregistrements où `event_type` est `'login'` ou `'logout'`

Deux vérifications d'égalité contre la même colonne reliées par <FlyqlToken type="operator">or</FlyqlToken>. ## Cohérence des types dans les listes IN

Le parser accepte **toujours** les listes hétérogènes comme <FlyqlCode code={`[200, 'ok', 404]`} inline />. Les générateurs SQL exécutent la vérification de cohérence des types **uniquement** lorsque (a) la colonne a un type déclaré, (b) la clé n'est pas segmentée et (c) la liste est homogène (`len(set(values_types)) == 1`). Les listes hétérogènes, les clés segmentées et les colonnes sans type contournent toutes la vérification au site d'appel.

La recette démontre trois cas contre des colonnes typées : une correspondance homogène, un contournement hétérogène et un cas homogène avec un mauvais type qui lance l'erreur canonique `type mismatch in IN list: ...`. Le texte de l'erreur est identique en Python, Go et JavaScript.
