O FlyQL suporta 10 tipos de valores: **Integer**, **BigInt**, **Float**, **String**, **Boolean**, **Null**, **Array**, **Column**, **Function** e **Parameter**. Cada valor no AST possui um campo `value_type` explicito.

## Tipos de Valores

### Integer

Numeros inteiros dentro do intervalo int64:

<FlyqlCode code={`count = 42`} />
<FlyqlCode code={`status != 200`} />
<FlyqlCode code={`age >= 18`} />

AST: `value_type: "integer"`

### BigInt

Inteiros que excedem o intervalo int64 (Go/Python) ou `MAX_SAFE_INTEGER` (JavaScript):

<FlyqlCode code={`id = 18446744073709551615`} />

AST: `value_type: "bigint"`

### Float

Numeros com ponto decimal:

<FlyqlCode code={`price > 9.99`} />
<FlyqlCode code={`ratio = 3.14`} />
<FlyqlCode code={`temperature<=-0.5`} />

AST: `value_type: "float"`

### String

Valores de texto, com ou sem aspas:

<FlyqlCode code={`level = 'error'`} />
<FlyqlCode code={`user = "John Doe"`} />
<FlyqlCode code={`message = 'hello world'`} />
<FlyqlCode code={`count = '42'`} />

AST: `value_type: "string"`

Escape de aspas com barra invertida:

<FlyqlCode code={`user = 'John\\'s'`} />
<FlyqlCode code={`message = "said \\"hello\\""`} />

:::tip
As aspas simples e duplas sao intercambiaveis, por isso pode escolher o delimitador que evita o escape. Use aspas duplas quando o valor contem apostrofos e aspas simples quando contem aspas duplas:

<FlyqlCode code={`user = "John's"`} />
<FlyqlCode code={`message = 'said "hello"'`} />
:::

### Boolean

As palavras-chave <FlyqlToken type="boolean">true</FlyqlToken> e <FlyqlToken type="boolean">false</FlyqlToken> (minusculas, sensivel a maiusculas/minusculas):

<FlyqlCode code={`active = true`} />
<FlyqlCode code={`debug = false`} />
<FlyqlCode code={`active != true`} />

AST: `value_type: "boolean"`, o valor e um booleano nativo.

Variantes em maiusculas (`True`, `FALSE`) sao tratadas como strings.

**Geracao de SQL:**
| Dialecto | FlyQL | SQL Gerado |
|----------|-------|------------|
| ClickHouse | <FlyqlCode code={`field = true`} inline /> | `field = true` |
| ClickHouse | <FlyqlCode code={`field = false`} inline /> | `field = false` |
| PostgreSQL | <FlyqlCode code={`field = true`} inline /> | `field = TRUE` |
| PostgreSQL | <FlyqlCode code={`field = false`} inline /> | `field = FALSE` |
| StarRocks | <FlyqlCode code={`field = true`} inline /> | `field = true` |
| StarRocks | <FlyqlCode code={`field = false`} inline /> | `field = false` |

### Null

A palavra-chave <FlyqlToken type="null">null</FlyqlToken> (minusculas, sensivel a maiusculas/minusculas):

<FlyqlCode code={`field = null`} />
<FlyqlCode code={`field != null`} />

AST: `value_type: "null"`, o valor e null nativo.

Variantes em maiusculas (`Null`, `NULL`) sao tratadas como strings.

**Restricoes:** Apenas os operadores <FlyqlToken type="operator">=</FlyqlToken> e <FlyqlToken type="operator">!=</FlyqlToken> sao validos com null. Utilizar <FlyqlToken type="operator">{'>'}</FlyqlToken>, <FlyqlToken type="operator">{'<'}</FlyqlToken>, <FlyqlToken type="operator">{'>='}</FlyqlToken>, <FlyqlToken type="operator">{'<='}</FlyqlToken>, <FlyqlToken type="operator">{'~'}</FlyqlToken> ou <FlyqlToken type="operator">{'!~'}</FlyqlToken> com null produz um erro de analise.

**Geracao de SQL:** <FlyqlCode code={`field = null`} inline /> gera `field IS NULL`, <FlyqlCode code={`field != null`} inline /> gera `field IS NOT NULL` (todos os dialectos).

### Array

Os arrays sao utilizados com os operadores <FlyqlToken type="operator">in</FlyqlToken> e <FlyqlToken type="operator">not in</FlyqlToken>. Os elementos podem ser de tipos mistos (heterogeneos):

<FlyqlCode code={`status in [200, 201, 202]`} />
<FlyqlCode code={`name in ['alice', 'bob']`} />
<FlyqlCode code={`field in [1, "hello", true, null]`} />

Cada elemento possui o seu proprio tipo no array `values_types`.

## Expressoes Truthy/Falsy

Uma chave isolada sem operador verifica se o campo tem um valor truthy:

<FlyqlCode code={`active`} />
<FlyqlCode code={`message and status`} />

Um valor e considerado **falsy** se for:
- `null` / `None` / ausente
- String vazia `""`
- Zero `0`
- Booleano <FlyqlToken type="boolean">false</FlyqlToken>

Tudo o resto e **truthy**.

Use <FlyqlToken type="operator">not</FlyqlToken> para verificar valores falsy:

<FlyqlCode code={`not archived`} />
<FlyqlCode code={`active and not debug`} />

As verificacoes truthy/falsy sao distintas das comparacoes com null: <FlyqlCode code={`not field`} inline /> corresponde quando o campo e null, vazio ou zero. <FlyqlCode code={`field = null`} inline /> corresponde apenas quando o campo e especificamente null.

## Espacos em Branco

O FlyQL e tolerante a espacos em branco. Espacos a volta dos operadores sao opcionais:

<FlyqlCode code={`status = 200`} />
<FlyqlCode code={`status =200`} />

Todas as formas acima sao equivalentes.

## Constantes LiteralKind

Os valores de expressoes analisadas carregam uma etiqueta `LiteralKind`. As constantes sao exportadas pelo pacote raiz em cada implementacao.

**Python:**
```python
from flyql import LiteralKind
# LiteralKind.INTEGER, LiteralKind.BIGINT, LiteralKind.FLOAT,
# LiteralKind.STRING, LiteralKind.BOOLEAN, LiteralKind.NULL, LiteralKind.ARRAY
```

**Go:**
```go
import "github.com/iamtelescope/flyql/golang/literal"
// literal.Integer, literal.BigInt, literal.Float,
// literal.String, literal.Boolean, literal.Null, literal.Array
```

**JavaScript:**
```javascript
import { LiteralKind } from 'flyql'
// LiteralKind.INTEGER, LiteralKind.BIGINT, LiteralKind.FLOAT,
// LiteralKind.STRING, LiteralKind.BOOLEAN, LiteralKind.NULL, LiteralKind.ARRAY
```

## Funcoes Temporais

O FlyQL suporta chamadas de funcoes temporais como valores no lado direito de expressoes de comparacao. Estas permitem filtros relativos ao tempo sem codificar carimbos de data/hora.

AST: `value_type: "function"`, `value` e um objecto `FunctionCall`.

### `ago(duration)`

Devolve um carimbo de data/hora relativo a hora actual. Suporta duracoes compostas.

<FlyqlCode code={`timestamp > ago(1h)`} />
<FlyqlCode code={`created_at >= ago(7d)`} />
<FlyqlCode code={`updated_at > ago(1h30m)`} />

**Unidades de duracao:** `s` (segundos), `m` (minutos), `h` (horas), `d` (dias), `w` (semanas)

### `now()`

Devolve o carimbo de data/hora actual.

<FlyqlCode code={`expires_at < now()`} />

### `today()`

Devolve a data de hoje. Aceita um argumento opcional de fuso horario IANA.

<FlyqlCode code={`date = today()`} />
<FlyqlCode code={`date = today('Europe/Berlin')`} />

### `startOf(unit)`

Devolve o inicio do dia, semana (segunda-feira) ou mes actual. Aceita um fuso horario opcional.

<FlyqlCode code={`created_at > startOf('day')`} />
<FlyqlCode code={`created_at > startOf('week')`} />
<FlyqlCode code={`created_at > startOf('month', 'Asia/Tokyo')`} />

**Unidades validas:** `'day'`, `'week'`, `'month'`

### Restricoes de Operadores

As funcoes temporais so podem ser utilizadas com operadores de comparacao: `=`, `!=`, `>`, `>=`, `<`, `<=`.

Utilizar `~` ou `!~` com uma funcao temporal produz um erro de analise. Operadores como `in`, `has`, `like` e `ilike` utilizam caminhos de analise separados e nao sao compativeis com valores de funcao.

### Referencia de Coluna Sem Parenteses

Nomes de funcao sem parenteses sao tratados como referencias de coluna, nao como chamadas de funcao:

<FlyqlCode code={`field = ago`} />

Isto produz `value_type: "column"`, nao uma chamada de funcao.

## Parameter

Os marcadores de parâmetros (`$name` para nomeados, `$1` para posicionais) substituem valores que serão resolvidos em tempo de execução por `bindParams()`. Podem aparecer em qualquer posição onde um valor literal seja válido — incluindo listas IN, argumentos de funções temporais e argumentos de transformadores.

<FlyqlCode code={`status = $code`} />
<FlyqlCode code={`level in [$primary, 'warn']`} />
<FlyqlCode code={`created > ago($duration)`} />

AST: `value_type: "parameter"`, o valor é um objeto `Parameter` com os campos `name` e `positional`.

Consulte [Parâmetros](/pt/syntax/parameters/) para a referência completa.
