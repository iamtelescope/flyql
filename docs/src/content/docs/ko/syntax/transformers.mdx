Transformers는 쿼리에서 비교되거나 SQL로 출력되기 전에 컬럼 값을 변경합니다. 파이프(`|`) 문법을 사용하며, 왼쪽에서 오른쪽 순서로 적용됩니다.

## 파이프 문법

컬럼에 transformer를 적용하려면 `|`를 사용합니다.

<FlyqlCode code={`message|upper = "ERROR"`} />

이렇게 하면 <FlyqlToken type="key">message</FlyqlToken> 값에 먼저 <FlyqlToken type="transformer">upper</FlyqlToken>를 적용한 뒤 `"ERROR"`와 비교합니다.

## 체이닝

추가 파이프를 사용해 여러 transformer를 연결할 수 있습니다.

<FlyqlCode code={`message|lower|len > 100`} />

Transformer는 왼쪽에서 오른쪽으로 적용됩니다. 먼저 <FlyqlToken type="transformer">lower</FlyqlToken>(string → string), 다음으로 <FlyqlToken type="transformer">len</FlyqlToken>(string → int)이 실행됩니다. 그 결과를 `> 100`과 비교합니다.

## 타입 시스템

각 transformer는 입력 타입과 출력 타입을 선언합니다. 체인으로 연결된 transformer는 타입 호환이 되어야 합니다. 이전 transformer의 출력 타입은 다음 transformer의 입력 타입과 일치해야 합니다.

| 타입 | 설명 |
|------|------|
| `string` | 텍스트 값 |
| `int` | 정수 |
| `float` | 부동소수점 수 |
| `bool` | 불리언 값 |
| `array` | 값 배열 (포함 여부는 <FlyqlToken type="operator">has</FlyqlToken> 연산자로 확인) |

## 내장 Transformers

| Transformer | 입력 | 출력 | 설명 |
|-------------|------|------|------|
| <FlyqlToken type="transformer">upper</FlyqlToken> | string | string | 대문자로 변환 |
| <FlyqlToken type="transformer">lower</FlyqlToken> | string | string | 소문자로 변환 |
| <FlyqlToken type="transformer">len</FlyqlToken> | string | int | 문자열 길이 반환 |
| <FlyqlToken type="transformer">split</FlyqlToken> | string | array | 구분자로 분할: `split(".")` |

## 방언별 SQL 출력

Transformer는 방언별 SQL 함수 호출을 생성합니다. 체인된 transformer는 중첩 함수 호출로 생성되며, 가장 안쪽부터 평가됩니다.

**단일 transformer:** <FlyqlCode code={`status|upper = "ERROR"`} inline />

| 방언 | 생성되는 SQL |
|------|---------------|
| ClickHouse | `equals(upper(status), 'ERROR')` |
| PostgreSQL | `UPPER(status) = 'ERROR'` |
| StarRocks | `UPPER(\`status\`) = 'ERROR'` |

**체인된 transformer:** <FlyqlCode code={`message|lower|len > 100`} inline />

| 방언 | 생성되는 SQL |
|------|---------------|
| ClickHouse | `length(lower(message)) > 100` |
| PostgreSQL | `LENGTH(LOWER(message)) > 100` |
| StarRocks | `LENGTH(LOWER(\`message\`)) > 100` |

**Split와 has 조합:** <FlyqlCode code={`message|split(" ") has "hello"`} inline />

| 방언 | 생성되는 SQL |
|------|---------------|
| ClickHouse | `has(splitByChar(' ', message), 'hello')` |
| PostgreSQL | `'hello' = ANY(STRING_TO_ARRAY(message, ' '))` |
| StarRocks | `` array_contains(SPLIT(`message`, ' '), 'hello') `` |

## 타입 오류

체인에 타입이 맞지 않는 transformer가 있으면 editor가 오류를 표시합니다.

<FlyqlCode code={`message|len|upper`} />

이는 <FlyqlToken type="transformer">len</FlyqlToken>이 `int`를 출력하지만, <FlyqlToken type="transformer">upper</FlyqlToken>는 `string` 입력을 요구하므로 실패합니다. Editor에는 "len outputs int, upper requires string input" 이 표시됩니다.

## 예제

<FlyqlCode code={`status|upper = "ERROR"`} />
<FlyqlCode code={`message|lower|len > 100`} />
<FlyqlCode code={`host|lower != "localhost"`} />
<FlyqlCode code={`path|len > 0`} />
<FlyqlCode code={`tags|split(",") has "urgent"`} />

## Transformers와 Renderers의 차이

Transformers는 별칭 **전에** 실행되며, flyql이 SQL로 출력하거나 매칭 대상으로 사용하는 값을 변경합니다. [Renderers](/syntax/renderers/)는 별칭 **후에** 실행되며, 생성되는 SQL이나 matcher 결과에 영향을 주지 않고 표시 메타데이터를 첨부합니다. "이 값을 대문자로 반환하고 싶다"면 transformer를 사용합니다. "이 컬럼을 앱에서 클릭 가능한 링크로 렌더링하고 싶다"면 renderer를 사용합니다.

| 관점                      | Transformer          | Renderer             |
| ------------------------- | -------------------- | -------------------- |
| 구문상 위치               | 별칭 전              | 별칭 **후**          |
| 생성되는 SQL에 영향       | 예                   | **아니요**           |
| matcher 출력에 영향       | 예                   | **아니요**           |
| 기본 레지스트리           | 4개 내장             | 비어 있음            |

별칭 뒤 파이프 문법은 [Renderers](/syntax/renderers/)를, 등록 API는 [Custom Renderers](/advanced/custom-renderers/)를 참조하세요.
