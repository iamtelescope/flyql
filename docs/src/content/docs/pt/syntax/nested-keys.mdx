## Notacao por Pontos

Utilize caminhos separados por pontos para aceder a campos aninhados em JSON, Map ou dados estruturados:

<FlyqlCode code={`request.url = "/api/users"`} />
<FlyqlCode code={`response.headers.content_type = "application/json"`} />
<FlyqlCode code={`metadata.labels.env = 'prod'`} />

Cada segmento separado por ponto percorre um nivel na estrutura aninhada. Os geradores SQL traduzem estes caminhos para a sintaxe correcta de cada dialecto de base de dados (acesso a caminhos JSON, consultas em Map, etc.).

## Segmentos de Chave entre Aspas

Quando um segmento de chave contem um ponto, coloque-o entre aspas para que o ponto nao seja tratado como separador:

<FlyqlCode code={`"foo.bar".baz = value`} />

Aqui:
- `"foo.bar"` e um unico segmento de chave (o ponto e literal)
- <FlyqlToken type="key">baz</FlyqlToken> e filho de `"foo.bar"`

Sem aspas, <FlyqlToken type="key">foo.bar.baz</FlyqlToken> seria tres segmentos: <FlyqlToken type="key">foo</FlyqlToken> → <FlyqlToken type="key">bar</FlyqlToken> → <FlyqlToken type="key">baz</FlyqlToken>.

### Regras de Aspas

Tanto aspas simples como aspas duplas funcionam para segmentos de chave:

<FlyqlCode code={`"foo.bar".baz = value`} />
<FlyqlCode code={`'foo.bar'.baz = value`} />

Apenas os segmentos que contem pontos precisam de estar entre aspas:

<FlyqlCode code={`metadata."dotted.key".value = 123`} />

## Profundidade do Caminho

Nao ha limite de profundidade de aninhamento:

<FlyqlCode code={`a.b.c.d.e = value`} />

Cada segmento e percorrido por ordem ao gerar SQL ou ao avaliar em memoria.

## Combinacao com Operadores

As chaves aninhadas funcionam com todos os operadores:

<FlyqlCode code={`request.status >= 400`} />
<FlyqlCode code={`response.body ~ "error.*"`} />
<FlyqlCode code={`metadata.tags in ['prod', 'staging']`} />
<FlyqlCode code={`request.headers.content_type != "text/html"`} />
<FlyqlCode code={`not request.authenticated`} />

## Acesso a Elementos de Array

Para colunas tipadas como arrays, utilize segmentos de caminho numericos baseados em zero para aceder aos elementos:

<FlyqlCode code={`tags.0 = 'important'`} />
<FlyqlCode code={`tags.1 = 'second'`} />

A indexacao de arrays no FlyQL e **baseada em 0 em todos os dialectos SQL e no matcher em memoria**. Os geradores traduzem o indice flyql para a convencao SQL nativa de cada destino: os arrays no ClickHouse, StarRocks e PostgreSQL sao indexados a partir de 1 ao nivel de SQL, pelo que `tags.0` em flyql e emitido como `tags[1]` em cada dialecto (com aspas de identificadores e forma de funcao especificas do dialecto). Isto significa que a mesma expressao flyql produz strings SQL diferentes por dialecto mas **avalia para o mesmo resultado** sobre os mesmos dados subjacentes.
