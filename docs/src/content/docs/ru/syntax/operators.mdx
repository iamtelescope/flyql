FlyQL поддерживает следующие операторы сравнения:

## Равенство

| Оператор | Название | Пример |
|----------|----------|--------|
| <FlyqlToken type="operator">=</FlyqlToken> | равно | <FlyqlCode code={`status = 200`} inline /> |
| <FlyqlToken type="operator">!=</FlyqlToken> | не равно | <FlyqlCode code={`env != 'prod'`} inline /> |

<FlyqlCode code={`status = 200`} />
<FlyqlCode code={`service != "api-gateway"`} />

Обе стороны оператора обязательны — <FlyqlToken type="key">ключ</FlyqlToken> и <FlyqlToken type="value">значение</FlyqlToken>.

## Числовые сравнения

| Оператор | Название | Пример |
|----------|----------|--------|
| <FlyqlToken type="operator">{'>'}</FlyqlToken> | больше | <FlyqlCode code={`status > 399`} inline /> |
| <FlyqlToken type="operator">{'>='}</FlyqlToken> | больше или равно | <FlyqlCode code={`status >= 400`} inline /> |
| <FlyqlToken type="operator">{'<'}</FlyqlToken> | меньше | <FlyqlCode code={`duration < 1000`} inline /> |
| <FlyqlToken type="operator">{'<='}</FlyqlToken> | меньше или равно | <FlyqlCode code={`duration <= 500`} inline /> |

<FlyqlCode code={`status >= 400 and duration < 5000`} />
<FlyqlCode code={`response_time > 100 and response_time <= 2000`} />

Операторы числового сравнения работают с числовыми значениями. При использовании со строками применяется лексикографическое сравнение.

## Регулярные выражения

| Оператор | Название | Пример |
|----------|----------|--------|
| <FlyqlToken type="operator">{'~'}</FlyqlToken> | совпадение по регулярному выражению | <FlyqlCode code={`message ~ "error.*"`} inline /> |
| <FlyqlToken type="operator">{'!~'}</FlyqlToken> | несовпадение по регулярному выражению | <FlyqlCode code={`path !~ "^/health"`} inline /> |

<FlyqlCode code={`message ~ "timeout|connection refused"`} />
<FlyqlCode code={`host !~ "^test-.*"`} />

Синтаксис регулярных выражений определяется целевой базой данных или средой выполнения. Подробности — в разделе [Сопоставление с образцом](/ru/syntax/pattern-matching/).

:::note
Операторы `~` и `!~` нельзя использовать с временными функциями (например, `ago(1h)`, `now()`). Это вызывает ошибку парсинга. Аналогично, операторы `in`, `has`, `like` и `ilike` несовместимы с функциональными значениями.
:::

## Проверка принадлежности списку

| Оператор | Название | Пример |
|----------|----------|--------|
| <FlyqlToken type="operator">in</FlyqlToken> | входит в список | <FlyqlCode code={`status in [200, 201]`} inline /> |
| <FlyqlToken type="operator">not in</FlyqlToken> | не входит в список | <FlyqlCode code={`env not in ['dev', 'test']`} inline /> |

Подробности — в разделе [Списки](/ru/syntax/lists/).

## Сопоставление по шаблону (LIKE)

| Оператор | Название | Пример |
|----------|----------|--------|
| <FlyqlToken type="operator">like</FlyqlToken> | Шаблон like | <FlyqlCode code={`message like 'error%'`} inline /> |
| <FlyqlToken type="operator">not like</FlyqlToken> | Не соответствует like | <FlyqlCode code={`host not like 'test%'`} inline /> |
| <FlyqlToken type="operator">ilike</FlyqlToken> | Like без учёта регистра | <FlyqlCode code={`message ilike 'Error%'`} inline /> |
| <FlyqlToken type="operator">not ilike</FlyqlToken> | Not like без учёта регистра | <FlyqlCode code={`host not ilike 'TEST%'`} inline /> |

<FlyqlCode code={`message like '%timeout%'`} />
<FlyqlCode code={`host not like '%-staging'`} />
<FlyqlCode code={`path ilike '/API/%'`} />

SQL-подобное сопоставление по шаблону с двумя подстановочными знаками:

- <FlyqlToken type="operator">%</FlyqlToken> — соответствует любой последовательности символов (включая пустую)
- <FlyqlToken type="operator">_</FlyqlToken> — соответствует ровно одному символу

Экранируйте подстановочные знаки обратной косой чертой для литерального совпадения: `\%` — литеральный `%`, `\_` — литеральный `_`.

Подробности — в разделе [Сопоставление с образцом](/ru/syntax/pattern-matching/).

## Проверка на вхождение

| Оператор | Название | Пример |
|----------|----------|--------|
| <FlyqlToken type="operator">has</FlyqlToken> | содержит | <FlyqlCode code={`message has 'error'`} inline /> |
| <FlyqlToken type="operator">not has</FlyqlToken> | не содержит | <FlyqlCode code={`message not has 'test'`} inline /> |

<FlyqlCode code={`message has 'timeout'`} />
<FlyqlCode code={`tags not has 'deprecated'`} />

Оператор <FlyqlToken type="operator">has</FlyqlToken> выполняет проверку вхождения в зависимости от типа:

- **String** — поиск подстроки: <FlyqlCode code={`message has 'error'`} inline /> истинно, если <FlyqlToken type="key">message</FlyqlToken> содержит <FlyqlToken type="string">"error"</FlyqlToken>
- **Map** — наличие ключа: <FlyqlCode code={`metadata has 'region'`} inline /> истинно, если в карте есть ключ <FlyqlToken type="string">"region"</FlyqlToken>
- **Array/List** — наличие элемента: <FlyqlCode code={`tags has 'web'`} inline /> истинно, если массив содержит <FlyqlToken type="string">"web"</FlyqlToken>
- **JSON** — наличие ключа: <FlyqlCode code={`data has 'name'`} inline /> истинно, если JSON-объект имеет ключ <FlyqlToken type="string">"name"</FlyqlToken>
- **Null/пусто** — всегда false для <FlyqlToken type="operator">has</FlyqlToken>, всегда true для <FlyqlToken type="operator">not has</FlyqlToken>

<FlyqlToken type="operator">has</FlyqlToken> принимает одно значение (не список). Для проверки принадлежности к множеству значений используйте <FlyqlToken type="operator">in</FlyqlToken>.

|----------|----------|--------|

| <FlyqlToken type="operator">!=</FlyqlToken> | не равно | <FlyqlCode code={`env != 'prod'`} inline /> |
| <FlyqlToken type="operator">{'>'}</FlyqlToken> | больше | <FlyqlCode code={`count > 10`} inline /> |
| <FlyqlToken type="operator">{'>='}</FlyqlToken> | больше или равно | <FlyqlCode code={`count >= 10`} inline /> |
| <FlyqlToken type="operator">{'<'}</FlyqlToken> | меньше | <FlyqlCode code={`count < 100`} inline /> |
| <FlyqlToken type="operator">{'<='}</FlyqlToken> | меньше или равно | <FlyqlCode code={`count <= 100`} inline /> |
