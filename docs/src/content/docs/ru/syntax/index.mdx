Запросы FlyQL состоят из одного или нескольких условий, объединённых булевыми операторами (<FlyqlToken type="operator">and</FlyqlToken>, <FlyqlToken type="operator">or</FlyqlToken>, <FlyqlToken type="operator">not</FlyqlToken>). Каждое условие — это либо сравнение, либо проверка на истинность.

## Базовая структура запроса

<FlyqlCode code={`status = 200 and active and not archived`} />

- <FlyqlCode code={`status = 200`} inline /> — поле <FlyqlToken type="key">status</FlyqlToken> равно <FlyqlToken type="value">200</FlyqlToken>
- <FlyqlCode code={`active`} inline /> — поле <FlyqlToken type="key">active</FlyqlToken> имеет истинное значение
- <FlyqlCode code={`not archived`} inline /> — поле <FlyqlToken type="key">archived</FlyqlToken> имеет ложное значение (null, пустое, 0 или false)

## Дополнительные примеры

<FlyqlCode code={`service != 'api' or user = "john doe"`} />

<FlyqlCode code={`message ~ "error.*" and not debug`} />

<FlyqlCode code={`(a = 1 or b = 2) and not (c = 3 and d = 4)`} />

<FlyqlCode code={`status in [200, 201] and method not in ['DELETE', 'PUT']`} />

<FlyqlCode code={`message has 'error' and tags not has 'debug'`} />

## Общие правила

- **Отдельный ключ** — ключ без оператора трактуется как проверка на истинность: <FlyqlCode code={`active`} inline />
- **Сравнения** — при использовании оператора обязательно должно быть указано значение: <FlyqlCode code={`status = 200`} inline />
- **Пробелы** — пробелы вокруг операторов необязательны: <FlyqlCode code={`status=200`} inline /> и <FlyqlCode code={`status = 200`} inline /> эквивалентны
- **Приоритет операторов** — <FlyqlToken type="operator">and</FlyqlToken> связывается сильнее, чем <FlyqlToken type="operator">or</FlyqlToken>: <FlyqlCode code={`a = 1 or b = 2 and c = 3`} inline /> парсится как <FlyqlCode code={`a = 1 or (b = 2 and c = 3)`} inline />
- **Скобки** — используйте <FlyqlToken type="operator">( )</FlyqlToken> для изменения приоритета: <FlyqlCode code={`(a = 1 or b = 2) and c = 3`} inline />
