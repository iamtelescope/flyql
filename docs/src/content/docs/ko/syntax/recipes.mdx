## `event_type`이 `'login'` 또는 `'logout'`인 레코드

같은 컬럼에 대한 두 개의 동등 검사를 <FlyqlToken type="operator">or</FlyqlToken>로 결합합니다. <FlyqlCode code={`'login'`} inline />과 <FlyqlCode code={`'logout'`} inline /> 주변의 **작은따옴표는 필수**입니다: 따옴표가 없으면 FlyQL은 <FlyqlCode code={`login`} inline />을 컬럼 참조로 해석하며, 문자열 리터럴로 해석하지 않습니다.

<FlyqlCode code={`event_type = 'login' or event_type = 'logout'`} />

값 목록의 경우 <FlyqlToken type="operator">in</FlyqlToken> 형식이 더 좋습니다: <FlyqlCode code={`event_type in ['login', 'logout']`} inline />.

## IN 리스트 타입 일관성

파서는 <FlyqlCode code={`[200, 'ok', 404]`} inline />와 같은 이종 리스트를 **항상** 받아들입니다. SQL 생성기는 (a) 컬럼이 선언된 타입을 가지고, (b) 키가 분할되지 않았으며, (c) 리스트가 동질일 때(`len(set(values_types)) == 1`)에만 타입 일관성 검사를 실행합니다. 이종 리스트, 분할된 키, 타입이 없는 컬럼은 모두 호출 지점에서 검사를 우회합니다.

이 레시피는 타입이 지정된 컬럼에 대해 세 가지 경우를 보여줍니다: 동질 일치, 이종 우회, 그리고 표준 오류 `type mismatch in IN list: ...`를 발생시키는 동질 잘못된 타입 경우. 오류 텍스트는 Python, Go, JavaScript에서 동일합니다.
