FlyQL 查询由一个或多个条件组成，条件之间通过 <FlyqlToken type="operator">and</FlyqlToken> / <FlyqlToken type="operator">or</FlyqlToken> 连接，并可使用 <FlyqlToken type="operator">not</FlyqlToken> 进行取反。每个条件要么是比较表达式，要么是真值检查。

## 基础查询结构

<FlyqlCode code={`status = 200 and active and not archived`} />

- <FlyqlCode code={`status = 200`} inline /> — <FlyqlToken type="key">status</FlyqlToken> 字段等于 <FlyqlToken type="value">200</FlyqlToken>
- <FlyqlCode code={`active`} inline /> — <FlyqlToken type="key">active</FlyqlToken> 字段为真值
- <FlyqlCode code={`not archived`} inline /> — <FlyqlToken type="key">archived</FlyqlToken> 字段为假值（null、空、0 或 false）

## 更多示例

<FlyqlCode code={`service != 'api' or user = "john doe"`} />

<FlyqlCode code={`message ~ "error.*" and not debug`} />

<FlyqlCode code={`(a = 1 or b = 2) and not (c = 3 and d = 4)`} />

<FlyqlCode code={`status in [200, 201] and method not in ['DELETE', 'PUT']`} />

<FlyqlCode code={`message has 'error' and tags not has 'debug'`} />

## 通用规则

- **独立键名** — 没有操作符的字段会被当作真值检查: <FlyqlCode code={`active`} inline />
- **比较表达式** — 带操作符的字段必须提供对应的值: <FlyqlCode code={`status = 200`} inline />
- **空白字符** — 操作符两侧的空格是可选的: <FlyqlCode code={`status=200`} inline /> 与 <FlyqlCode code={`status = 200`} inline /> 等价
- **标准操作符优先级** — <FlyqlToken type="operator">not</FlyqlToken> 结合最紧，其次是 <FlyqlToken type="operator">and</FlyqlToken>，最后是 <FlyqlToken type="operator">or</FlyqlToken>（与 SQL、Python、JavaScript、Go 相同）: <FlyqlCode code={`a = 1 or b = 2 and c = 3`} inline /> 会被解析为 <FlyqlCode code={`a = 1 or (b = 2 and c = 3)`} inline />
- **括号** — 使用 <FlyqlToken type="operator">( )</FlyqlToken> 覆盖默认优先级: <FlyqlCode code={`(a = 1 or b = 2) and c = 3`} inline />
