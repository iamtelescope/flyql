## Instalación

FlyQL para Python está disponible en PyPI:

```bash
pip install flyql
```

Esto instala el paquete `flyql` junto con su dependencia `google-re2`.

**Requisitos:** Python 3.10+

## Parsear una consulta

```python
from flyql import parse

result = parse("status = 200 and active")
print(result.root)
```

`parse()` devuelve un objeto `Parser` con un nodo `root` que representa el AST. Lanza `ParserError` si la entrada no es válida.

## Generar SQL

Usa un generador para convertir el AST en una cláusula WHERE parametrizada. Cada dialecto de base de datos tiene su propio generador.

### ClickHouse

```python
from flyql import parse
from flyql.generators.clickhouse.generator import to_sql_where
from flyql.generators.clickhouse.column import Column

result = parse("status >= 400 and host like 'prod%'")

columns = {
    "status": Column("status", False, "UInt32"),
    "host": Column("host", False, "String"),
}

sql = to_sql_where(result.root, columns)
print(sql)
```

Los generadores de PostgreSQL y StarRocks siguen el mismo patrón; solo cambia la importación a `flyql.generators.postgresql` o `flyql.generators.starrocks`.

## Evaluar en memoria

Evalúa una consulta sobre un registro de datos sin generar SQL:

```python
from flyql import parse
from flyql.matcher import Evaluator, Record


result = parse("status = 200 and active")

data = {
    "status": 200,
    "active": True,
    "host": "prod-api-01",
}

evaluator = Evaluator()
matches = evaluator.evaluate(result.root, Record(data))
print(f"Matches: {matches}")  # True
```

## Parsear expresiones de columnas

Parsea una expresión de columnas en datos estructurados con segmentos, transformers y alias:

```python
from flyql.columns import parse, parse_to_json

# Parsea columnas básicas (los transformers están desactivados por defecto)
parsed = parse("message, status")
for col in parsed:
    print(f"{col.name} (display: {col.display_name!r}, segments: {col.segments})")

# Activa transformers mediante capabilities
with_transforms = parse("message|chars(25) as msg, status", capabilities={"transformers": True})

# O serializa directamente a JSON para respuestas de API
json_str = parse_to_json("message, status|upper", capabilities={"transformers": True})
print(json_str)
```

## Consultas parametrizadas

Usa marcadores `$name` y `$1`, y resuélvelos en tiempo de ejecución con `bind_params()`:

```python
from flyql import parse, bind_params

ast = parse("status = $code and env = $env")
bind_params(ast.root, {"code": 200, "env": "prod"})
```

Consulta [Parámetros](/es/syntax/parameters/) para ver la referencia completa.

## Transformers en consultas

Parsea una consulta con transformers y genera SQL:

```python
from flyql import parse
from flyql.generators.clickhouse.generator import to_sql_where, Column

# Parsea una consulta con transformers
result = parse("message|upper = 'ERROR'")

# Genera SQL para ClickHouse
columns = {"message": Column("message", False, "String")}
sql = to_sql_where(result.root, columns)
print(sql)  # equals(upper(message), 'ERROR')
```
