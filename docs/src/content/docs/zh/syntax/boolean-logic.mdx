## 布尔操作符

FlyQL 使用三个布尔操作符组合条件：

| 操作符 | 含义 | 示例 |
|--------|------|------|
| `and` | 两个条件都必须为真 | `status = 200 and active` |
| `or` | 任意一个条件为真即可 | `level = error or level = critical` |
| `not` | 取反一个表达式 | `not archived` |

### 与取反组合

`not` 可以与 `and`、`or` 组合使用：

```
active and not archived
level = error or not healthy
```

## 操作符优先级

`and` 的优先级高于 `or`。例如：

```
a = 1 or b = 2 and c = 3
```

会被解析为：

```
a = 1 or (b = 2 and c = 3)
```

如果要覆盖默认优先级，请显式使用括号。

## 括号分组

括号可以控制表达式的计算顺序：

```
(status >= 400 or level = error) and service = api
```

如果不加括号，`and` 会先绑定：

```
status >= 400 or level = error and service = api
→ status >= 400 or (level = error and service = api)
```

### 嵌套分组

括号可以继续嵌套：

```
(a = 1 or b = 2) and not (c = 3 and d = 4)
```

### 对整组取反

`not` 会作用于整个分组：

```
not (status = 200 and healthy)
```

这表示对组合条件整体取反，只有当 `status` 不等于 `200` 或 `healthy` 为假时才为真。

## 取反

`not` 可以作用于任何表达式：真值检查、比较表达式或整个分组。

```
not active
not status = 200
not (a = 1 and b = 2)
active and not archived
```

双重取反会互相抵消：`not not active` 等价于 `active`。

## 示例

```
# 简单 AND
status = 200 and service = api

# 简单 OR
level = error or level = critical

# 混合优先级
status >= 400 or level = error and service = api

# 显式分组
(status >= 400 or level = error) and service = api

# 带取反和分组的复杂查询
(status >= 500 and host = prod*) or (level = error and not service = healthcheck)
```
