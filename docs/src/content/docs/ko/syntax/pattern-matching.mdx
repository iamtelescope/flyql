## Regex 매칭

값을 정규식과 비교하려면 <FlyqlToken type="operator">{'~'}</FlyqlToken>를, 매칭을 부정하려면 <FlyqlToken type="operator">{'!~'}</FlyqlToken>를 사용합니다.

<FlyqlCode code={`message ~ "error.*timeout"`} />
<FlyqlCode code={`path !~ "^/health"`} />

| 연산자 | 의미 |
|--------|------|
| <FlyqlToken type="operator">{'~'}</FlyqlToken> | 값이 regex 패턴과 일치함 |
| <FlyqlToken type="operator">{'!~'}</FlyqlToken> | 값이 regex 패턴과 일치하지 않음 |

### Regex 패턴 문법

Regex 문법은 대상 백엔드에 따라 달라집니다.

- **ClickHouse** — `match()` 함수 사용 (RE2 문법)
- **PostgreSQL** — <FlyqlToken type="operator">{'~'}</FlyqlToken>, <FlyqlToken type="operator">{'!~'}</FlyqlToken> 연산자 사용 (POSIX regex)
- **StarRocks** — `regexp` 함수 사용
- **In-memory matcher** — 호스트 언어의 regex 엔진 사용 (Go `regexp`, Python `re2`, JS `RegExp`)

패턴에 공백이나 특수 문자가 포함되면 따옴표로 감싸세요.

<FlyqlCode code={`message ~ "connection (refused|reset)"`} />
<FlyqlCode code={`host ~ "^prod-[0-9]+"`} />

### Regex 플래그

패턴 시작 부분에 `(?...)` 구문으로 인라인 플래그를 지정하여 regex 동작을 변경할 수 있습니다. FlyQL은 모든 백엔드에서 동작하는 세 가지 플래그를 지원합니다.

#### `(?i)` — 대소문자 구분 없음

대소문자에 관계없이 패턴을 매칭합니다:

<FlyqlCode code={`message ~ "(?i)error"`} />
<FlyqlCode code={`level ~ "(?i)^(warn|error|fatal)$"`} />

`(?i)error`는 "Error", "ERROR", "error" 등과 일치합니다.

#### `(?m)` — 멀티라인

`^`와 `$`가 전체 문자열이 아닌 각 **줄**의 시작과 끝에 일치하도록 변경합니다:

<FlyqlCode code={`log ~ "(?m)^ERROR"`} />

`(?m)` 없이 `^ERROR`는 전체 문자열이 "ERROR"로 시작할 때만 일치합니다. `(?m)`을 사용하면 "INFO first\nERROR second"에서도 일치합니다. "ERROR"가 줄의 시작에 나타나기 때문입니다.

#### `(?s)` — Dotall

`.`이 줄바꿈을 포함한 **모든** 문자와 일치하도록 합니다. 기본적으로 `.`은 `\n`과 일치하지 않습니다:

<FlyqlCode code={`log ~ "(?s)start.*end"`} />

`(?s)` 없이 `start.*end`는 "start\nend"와 일치하지 않습니다(`.`이 줄바꿈을 넘지 못하기 때문). `(?s)`를 사용하면 일치합니다.

#### 플래그 조합

플래그는 하나의 그룹에서 조합할 수 있습니다:

<FlyqlCode code={`log ~ "(?im)^ERROR"`} />
<FlyqlCode code={`log ~ "(?ims)^START.*END$"`} />

#### 백엔드 지원 현황

세 가지 플래그 모두 모든 FlyQL 백엔드에서 동작합니다:

| 백엔드 | `(?i)` | `(?m)` | `(?s)` |
|--------|--------|--------|--------|
| **ClickHouse** | Yes | Yes | Yes |
| **PostgreSQL** | Yes | Yes | Yes |
| **StarRocks** | Yes | Yes | Yes |
| **Go matcher** | Yes | Yes | Yes |
| **Python matcher** | Yes | Yes | Yes |
| **JavaScript matcher** | Yes | Yes | Yes |

대소문자 구분 없는 매칭과 부정을 결합하면 대소문자에 상관없이 특정 패턴을 제외할 수 있습니다:

<FlyqlCode code={`message ~ "(?i)error" and message !~ "(?i)warning"`} />

## LIKE 매칭

SQL 스타일 와일드카드 패턴 매칭을 위해 <FlyqlToken type="operator">like</FlyqlToken>와 <FlyqlToken type="operator">ilike</FlyqlToken>를 사용합니다:

<FlyqlCode code={`host like 'prod%'`} />
<FlyqlCode code={`service like 'api-%'`} />
<FlyqlCode code={`message ilike '%Error%'`} />

| 연산자 | 의미 |
|--------|------|
| <FlyqlToken type="operator">like</FlyqlToken> | 값이 패턴과 일치함 (대소문자 구분) |
| <FlyqlToken type="operator">not like</FlyqlToken> | 값이 패턴과 일치하지 않음 |
| <FlyqlToken type="operator">ilike</FlyqlToken> | 값이 패턴과 일치함 (대소문자 구분 없음) |
| <FlyqlToken type="operator">not ilike</FlyqlToken> | 값이 패턴과 일치하지 않음 (대소문자 구분 없음) |

### 와일드카드

<FlyqlToken type="operator">like</FlyqlToken> / <FlyqlToken type="operator">ilike</FlyqlToken> 패턴에서 두 가지 와일드카드 문자를 사용할 수 있습니다:

| 와일드카드 | 의미 | 예제 | 일치하는 값 |
|----------|------|------|-----------|
| `%` | 임의의 문자열 (빈 문자열 포함) | <FlyqlCode code={`host like 'prod%'`} inline /> | `prod`, `prod-1`, `production` |
| `_` | 정확히 한 문자 | <FlyqlCode code={`code like 'E__'`} inline /> | `E01`, `Exx`, 하지만 `E1`이나 `Error`는 아님 |

와일드카드는 패턴의 어디에든 올 수 있습니다 -- 시작, 중간, 끝 또는 조합:

<FlyqlCode code={`message like '%timeout%'`} />
<FlyqlCode code={`host like 'web-_-prod'`} />
<FlyqlCode code={`path like '/api/%/status'`} />

### 와일드카드 이스케이프

리터럴 `%` 또는 `_`와 매칭하려면 백슬래시로 이스케이프하세요:

<FlyqlCode code={`value like '100\\%'`} />
<FlyqlCode code={`col like 'foo\\_bar'`} />

`100\%`는 리터럴 문자열 "100%"와 일치하며, "100" 뒤에 아무거나 오는 것과는 일치하지 않습니다.

### 대소문자 구분 없는 매칭

대소문자 구분 없는 패턴 매칭에는 <FlyqlToken type="operator">ilike</FlyqlToken>를 사용하세요:

<FlyqlCode code={`message ilike '%error%'`} />
<FlyqlCode code={`host not ilike 'TEST%'`} />

### LIKE vs. Regex

| 사용 사례 | 문법 | 예제 |
|----------|------|------|
| 접두사 매칭 | <FlyqlCode code={`key like 'prefix%'`} inline /> | <FlyqlCode code={`host like 'prod%'`} inline /> |
| 포함 | <FlyqlCode code={`key like '%word%'`} inline /> | <FlyqlCode code={`msg like '%error%'`} inline /> |
| 복잡한 패턴 | <FlyqlCode code={`key ~ "pattern"`} inline /> | <FlyqlCode code={`host ~ "^prod-[0-9]+"`} inline /> |

단순 와일드카드 매칭에는 <FlyqlToken type="operator">like</FlyqlToken>를 사용하세요. 문자 클래스, 선택, 또는 기타 고급 패턴이 필요하면 regex (<FlyqlToken type="operator">{'~'}</FlyqlToken>)를 사용하세요.

### 백엔드 지원 현황

모든 SQL 백엔드 (ClickHouse, PostgreSQL, StarRocks)는 네이티브 `LIKE` / `ILIKE` / `NOT LIKE` / `NOT ILIKE` SQL 연산자를 생성합니다. In-memory matcher는 패턴을 내부적으로 regex로 변환합니다.

:::note
StarRocks `ILIKE`는 버전 2.5 이상이 필요합니다.
:::

## 예제

<FlyqlCode code={`message ~ "error|fail"`} />
<FlyqlCode code={`path ~ "^/api/v2"`} />
<FlyqlCode code={`path !~ "^/health"`} />
<FlyqlCode code={`host like 'prod%'`} />
<FlyqlCode code={`host like 'prod%' and message ~ "timeout.*" and path !~ "^/health"`} />
