ClickHouse-SQL verwendet unquotierte Spaltennamen, `match()` für Regex und `LIKE` für Wildcards.

## Grundlegende Beispiele

| FlyQL | Generiertes SQL |
|-------|-----------------|
| <FlyqlCode code={`message = 'hello'`} inline /> | `message = 'hello'` |
| <FlyqlCode code={`message != 'hello'`} inline /> | `message != 'hello'` |
| <FlyqlCode code={`count > 10`} inline /> | `count > 10` |
| <FlyqlCode code={`count <= 100`} inline /> | `count <= 100` |
| <FlyqlCode code={`message ~ 'error.*'`} inline /> | `match(message, 'error.*')` |
| <FlyqlCode code={`message !~ 'test.*'`} inline /> | `not match(message, 'test.*')` |
| <FlyqlCode code={`host = prod*`} inline /> | `host LIKE 'prod%'` |

## Boolesche Operatoren

| FlyQL | Generiertes SQL |
|-------|-----------------|
| <FlyqlCode code={`a = '1' and b > 10`} inline /> | `(a = '1' and b > 10)` |
| <FlyqlCode code={`a = '1' or b = '2'`} inline /> | `(a = '1' or b = '2')` |
| <FlyqlCode code={`not a = '1'`} inline /> | `not (a = '1')` |

## Listenzugehörigkeit

| FlyQL | Generiertes SQL |
|-------|-----------------|
| <FlyqlCode code={`status in [200, 201]`} inline /> | `status IN (200, 201)` |
| <FlyqlCode code={`env not in ['dev', 'test']`} inline /> | `env NOT IN ('dev', 'test')` |
| <FlyqlCode code={`status in []`} inline /> | `0` |
| <FlyqlCode code={`status not in []`} inline /> | `1` |

## Enthalten (`has` / `not has`)

Der <FlyqlToken type="operator">has</FlyqlToken> Operator erzeugt je nach Spaltentyp unterschiedliches SQL:

| Spaltentyp | FlyQL | Generiertes SQL |
|------------|-------|-----------------|
| String | <FlyqlCode code={`message has 'error'`} inline /> | `position(message, 'error') > 0` |
| String (negiert) | <FlyqlCode code={`message not has 'error'`} inline /> | `(message IS NULL OR position(message, 'error') = 0)` |
| Array | <FlyqlCode code={`tags has 'web'`} inline /> | `has(tags, 'web')` |
| Array (negiert) | <FlyqlCode code={`tags not has 'web'`} inline /> | `NOT has(tags, 'web')` |
| Map | <FlyqlCode code={`metadata has 'key'`} inline /> | `mapContains(metadata, 'key')` |
| Map (negiert) | <FlyqlCode code={`metadata not has 'key'`} inline /> | `NOT mapContains(metadata, 'key')` |
| Natives JSON | <FlyqlCode code={`data has 'key'`} inline /> | `JSON_EXISTS(data, concat('$.', 'key'))` |
| JSON-String | <FlyqlCode code={`data has 'key'`} inline /> | `JSONHas(data, 'key')` |

## JSON-Spalten

ClickHouse unterstützt zwei Typen von JSON-Spalten mit unterschiedlicher SQL-Ausgabe.

### Nativer JSON-Typ

Spalten des Typs `JSON` verwenden Punktnotation mit in Backticks gesetzten Segmenten:

| FlyQL | Generiertes SQL |
|-------|-----------------|
| <FlyqlCode code={`data.name = 'test'`} inline /> | `` data.`name` = 'test' `` |
| <FlyqlCode code={`data.user.name = 'john'`} inline /> | `` data.`user`.`name` = 'john' `` |
| <FlyqlCode code={`data.age = 25`} inline /> | `` data.`age` = 25 `` |

### JSON-String-Typ

Spalten, die JSON als String speichern (z. B. Typ `String` mit JSON-Inhalt), verwenden `JSONExtract`-Funktionen, die zur Typprüfung in `multiIf` eingebettet sind:

| FlyQL | Generiertes SQL |
|-------|-----------------|
| <FlyqlCode code={`data.name = 'test'`} inline /> | `multiIf(JSONType(data, 'name') = 'String', equals(JSONExtractString(data, 'name'), 'test'), 0)` |
| <FlyqlCode code={`data.user.name = 'john'`} inline /> | `multiIf(JSONType(data, 'user', 'name') = 'String', equals(JSONExtractString(data, 'user', 'name'), 'john'), 0)` |

## Map-Spalten

Map-Spalten verwenden Klammernotation mit den Funktionen `equals()` / `notEquals()`:

| FlyQL | Generiertes SQL |
|-------|-----------------|
| <FlyqlCode code={`metadata.key1 = 'value1'`} inline /> | `equals(metadata['key1'], 'value1')` |
| <FlyqlCode code={`metadata.key1 != 'value1'`} inline /> | `notEquals(metadata['key1'], 'value1')` |
| <FlyqlCode code={`metadata.pattern ~ 'test.*'`} inline /> | `match(metadata['pattern'], 'test.*')` |

## Array-Spalten

Array-Spalten verwenden 0-basierte Klammerindizes mit `equals()`:

| FlyQL | Generiertes SQL |
|-------|-----------------|
| <FlyqlCode code={`tags.0 = 'first'`} inline /> | `equals(tags[0], 'first')` |
| <FlyqlCode code={`tags.1 = 'second'`} inline /> | `equals(tags[1], 'second')` |
| <FlyqlCode code={`tags.0 ~ 'tag.*'`} inline /> | `match(tags[0], 'tag.*')` |

### Spaltenkonfiguration

Der Parameter `jsonString` steuert, welcher Modus verwendet wird:

```go
// Go
columns := map[string]*clickhouse.Column{
    "message": clickhouse.NewColumn("message", false, "String", nil),
    "data":    clickhouse.NewColumn("data", false, "JSON", nil),      // native JSON
    "log":     clickhouse.NewColumn("log", true, "String", nil),       // JSON string
}
```

```javascript
// JavaScript
const columns = {
    message: newColumn("message", false, "String", null),
    data:    newColumn("data", false, "JSON", null),      // native JSON
    log:     newColumn("log", true, "String", null),       // JSON string
}
```

## Transformers

Transformers erzeugen ClickHouse-spezifische Funktionsaufrufe:

| FlyQL | Generiertes SQL |
|-------|-----------------|
| <FlyqlCode code={'status\|upper = "ERROR"'} inline /> | `equals(upper(status), 'ERROR')` |
| <FlyqlCode code={'status\|lower = "info"'} inline /> | `equals(lower(status), 'info')` |
| <FlyqlCode code={'message\|len > 100'} inline /> | `length(message) > 100` |
| <FlyqlCode code={'message\|lower\|len > 100'} inline /> | `length(lower(message)) > 100` |
| <FlyqlCode code={'tags\|split(",") has "urgent"'} inline /> | `has(splitByChar(',', tags), 'urgent')` |
