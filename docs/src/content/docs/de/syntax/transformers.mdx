Transformers verändern Spaltenwerte, bevor sie in Abfragen verglichen oder als SQL ausgegeben werden. Sie verwenden die Pipe-Syntax (`|`) und werden von links nach rechts angewendet.

## Pipe-Syntax

Wende einen Transformer mit `|` auf eine Spalte an:

<FlyqlCode code={`message|upper = "ERROR"`} />

Dadurch wird der Wert von <FlyqlToken type="key">message</FlyqlToken> zuerst mit <FlyqlToken type="transformer">upper</FlyqlToken> transformiert und dann mit `"ERROR"` verglichen.

## Verkettung

Mehrere Transformer lassen sich mit weiteren Pipes verketten:

<FlyqlCode code={`message|lower|len > 100`} />

Transformers werden von links nach rechts angewendet: zuerst <FlyqlToken type="transformer">lower</FlyqlToken> (String zu String), dann <FlyqlToken type="transformer">len</FlyqlToken> (String zu int). Das Ergebnis wird anschließend mit `> 100` verglichen.

## Typsystem

Jeder Transformer deklariert einen Eingabetyp und einen Ausgabetyp. Verkettete Transformer müssen typkompatibel sein – der Ausgabetyp des einen muss dem Eingabetyp des nächsten entsprechen.

| Typ | Beschreibung |
|-----|--------------|
| `string` | Textwerte |
| `int` | Ganze Zahlen |
| `float` | Gleitkommazahlen |
| `array` | Werteliste (Mitgliedschaft wird typischerweise mit <FlyqlToken type="operator">has</FlyqlToken> geprüft) |

## Eingebaute Transformers

| Transformer | Eingabe | Ausgabe | Beschreibung |
|-------------|---------|---------|--------------|
| <FlyqlToken type="transformer">upper</FlyqlToken> | string | string | In Großbuchstaben umwandeln |
| <FlyqlToken type="transformer">lower</FlyqlToken> | string | string | In Kleinbuchstaben umwandeln |
| <FlyqlToken type="transformer">len</FlyqlToken> | string | int | Zeichenkettenlänge ermitteln |
| <FlyqlToken type="transformer">split</FlyqlToken> | string | array | Nach Trennzeichen aufteilen: `split(".")` |

## SQL-Ausgabe nach Dialekt

Transformers erzeugen dialektspezifische SQL-Funktionsaufrufe. Verkettete Transformer ergeben verschachtelte Aufrufe (innerster Aufruf zuerst):

**Einzelner Transformer:** <FlyqlCode code={`status|upper = "ERROR"`} inline />

| Dialekt | Generiertes SQL |
|---------|-----------------|
| ClickHouse | `equals(upper(status), 'ERROR')` |
| PostgreSQL | `UPPER(status) = 'ERROR'` |
| StarRocks | `UPPER(\`status\`) = 'ERROR'` |

**Verkettete Transformer:** <FlyqlCode code={`message|lower|len > 100`} inline />

| Dialekt | Generiertes SQL |
|---------|-----------------|
| ClickHouse | `length(lower(message)) > 100` |
| PostgreSQL | `LENGTH(LOWER(message)) > 100` |
| StarRocks | `LENGTH(LOWER(\`message\`)) > 100` |

**Split mit has:** <FlyqlCode code={`message|split(" ") has "hello"`} inline />

| Dialekt | Generiertes SQL |
|---------|-----------------|
| ClickHouse | `has(splitByChar(' ', message), 'hello')` |
| PostgreSQL | `'hello' = ANY(STRING_TO_ARRAY(message, ' '))` |
| StarRocks | `` array_contains(SPLIT(`message`, ' '), 'hello') `` |

## Typfehler

Wenn eine Transformer-Kette inkompatible Typen enthält, zeigt der Editor einen Fehler an:

<FlyqlCode code={`message|len|upper`} />

Das schlägt fehl, weil <FlyqlToken type="transformer">len</FlyqlToken> ein `int` ausgibt, <FlyqlToken type="transformer">upper</FlyqlToken> aber einen `string` als Eingabe benötigt. Der Editor zeigt: "len outputs int, upper requires string input".

## Beispiele

<FlyqlCode code={`status|upper = "ERROR"`} />
<FlyqlCode code={`message|lower|len > 100`} />
<FlyqlCode code={`host|lower != "localhost"`} />
<FlyqlCode code={`path|len > 0`} />
<FlyqlCode code={`tags|split(",") has "urgent"`} />
