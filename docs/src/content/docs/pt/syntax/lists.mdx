## Pertenca a Listas

Use <FlyqlToken type="operator">in</FlyqlToken> para verificar se um valor esta numa lista, e <FlyqlToken type="operator">not in</FlyqlToken> para verificar que nao esta:

<FlyqlCode code={`status in [200, 201, 204]`} />
<FlyqlCode code={`env not in ['prod', 'staging']`} />

## Regras de Sintaxe

- Os valores sao colocados entre parenteses rectos <FlyqlToken type="operator">[ ]</FlyqlToken> e separados por virgulas
- Os valores podem ser de tipos mistos: <FlyqlCode code={`[200, 'ok', true, null]`} inline />
- Os valores de texto devem estar entre aspas: <FlyqlCode code={`['a', 'b', 'c']`} inline />
- Os valores numericos nao levam aspas: <FlyqlCode code={`[1, 2, 3]`} inline />
- Espacos a volta das virgulas e dos parenteses rectos sao permitidos

### Listas de Strings

<FlyqlCode code={`method in ['GET', 'POST', 'PUT']`} />
<FlyqlCode code={`env not in ['dev', 'test', 'local']`} />

Tanto aspas simples como aspas duplas funcionam para valores de texto.

### Listas Numericas

<FlyqlCode code={`status in [200, 201, 204]`} />
<FlyqlCode code={`port not in [80, 443]`} />

### Listas Vazias

Listas vazias sao permitidas:

<FlyqlCode code={`status in []`} />

Uma lista <FlyqlToken type="operator">in</FlyqlToken> vazia e sempre falsa — nada corresponde a uma lista vazia.

<FlyqlCode code={`status not in []`} />

Uma lista <FlyqlToken type="operator">not in</FlyqlToken> vazia e sempre verdadeira — tudo esta "fora" de uma lista vazia.

## Tipos Mistos

As listas suportam valores heterogeneos — pode misturar strings, numeros, booleanos e null:

<FlyqlCode code={`field in [200, 'ok', true, null]`} />

Cada elemento transporta o seu proprio tipo no array `valuesTypes` da expressao analisada (por exemplo, `["integer", "string", "boolean", "null"]`). Para listas vazias, `valuesTypes` e `null`.

Quando um gerador SQL processa uma lista, valida que os tipos dos valores da lista sao compativeis com o tipo da coluna de destino. Por exemplo, passar valores de texto para uma coluna numerica produzira um erro de validacao durante a geracao de SQL.

## Parametros em Listas

Os marcadores de parametros podem aparecer dentro de listas IN, misturados livremente com valores literais:

<FlyqlCode code={`status in [$x, $y, 200]`} />
<FlyqlCode code={`env not in [$current, 'dev']`} />

Os objetos `Parameter` permanecem no array `values` ate que `bindParams()` os resolva. Consulte [Parametros](/pt/syntax/parameters/) para a referencia completa.

## Combinacao com Outros Operadores

Os operadores de lista combinam-se com operadores booleanos como qualquer outra condicao:

<FlyqlCode code={`status in [200, 201] and method in ['GET', 'POST']`} />
<FlyqlCode code={`level = 'error' or status not in [200, 201, 204]`} />
<FlyqlCode code={`(status in [500, 502, 503] or level = 'critical') and env='prod'`} />

## Exemplos

<FlyqlCode code={`status in [200, 201, 204]`} />
<FlyqlCode code={`env not in ['dev', 'test', 'staging']`} />
<FlyqlCode code={`method in ['GET', 'POST'] and status not in [401, 403, 404]`} />
