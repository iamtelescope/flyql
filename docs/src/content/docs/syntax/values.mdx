FlyQL supports 9 value types: **Integer**, **BigInt**, **Float**, **String**, **Boolean**, **Null**, **Array**, **Column**, and **Function**. Each value in the AST carries an explicit `value_type` field.

## Value Types

### Integer

Whole numbers within the int64 range:

<FlyqlCode code={`count = 42`} />
<FlyqlCode code={`status != 200`} />
<FlyqlCode code={`age >= 18`} />

AST: `value_type: "integer"`

### BigInt

Integers exceeding the int64 range (Go/Python) or `MAX_SAFE_INTEGER` (JavaScript):

<FlyqlCode code={`id = 18446744073709551615`} />

AST: `value_type: "bigint"`

### Float

Numbers with a decimal point:

<FlyqlCode code={`price > 9.99`} />
<FlyqlCode code={`ratio = 3.14`} />
<FlyqlCode code={`temperature<=-0.5`} />

AST: `value_type: "float"`

### String

Text values **must be quoted** using single or double quotes. Unquoted values are interpreted as column references, not strings.

<FlyqlCode code={`level = 'error'`} />
<FlyqlCode code={`user = "John Doe"`} />
<FlyqlCode code={`message = 'hello world'`} />
<FlyqlCode code={`count = '42'`} />

AST: `value_type: "string"`

Escape quotes with backslash:

```
user = 'John\'s'
message="said \"hello\""
```

### Boolean

The keywords <FlyqlToken type="boolean">true</FlyqlToken> and <FlyqlToken type="boolean">false</FlyqlToken> (lowercase, case-sensitive):

<FlyqlCode code={`active = true`} />
<FlyqlCode code={`debug = false`} />
<FlyqlCode code={`active != true`} />

AST: `value_type: "boolean"`, value is native boolean.

Capitalized variants (`True`, `FALSE`) are treated as column references, not booleans.

**SQL generation:**
| Dialect | FlyQL | Generated SQL |
|---------|-------|---------------|
| ClickHouse | <FlyqlCode code={`field = true`} inline /> | `field = true` |
| ClickHouse | <FlyqlCode code={`field = false`} inline /> | `field = false` |
| PostgreSQL | <FlyqlCode code={`field = true`} inline /> | `field = TRUE` |
| PostgreSQL | <FlyqlCode code={`field = false`} inline /> | `field = FALSE` |
| StarRocks | <FlyqlCode code={`field = true`} inline /> | `field = true` |
| StarRocks | <FlyqlCode code={`field = false`} inline /> | `field = false` |

### Null

The keyword <FlyqlToken type="null">null</FlyqlToken> (lowercase, case-sensitive):

<FlyqlCode code={`field = null`} />
<FlyqlCode code={`field != null`} />

AST: `value_type: "null"`, value is native null.

Capitalized variants (`Null`, `NULL`) are treated as column references, not null.

**Restrictions:** Only <FlyqlToken type="operator">=</FlyqlToken> and <FlyqlToken type="operator">!=</FlyqlToken> operators are valid with null. Using <FlyqlToken type="operator">{'>'}</FlyqlToken>, <FlyqlToken type="operator">{'<'}</FlyqlToken>, <FlyqlToken type="operator">{'>='}</FlyqlToken>, <FlyqlToken type="operator">{'<='}</FlyqlToken>, <FlyqlToken type="operator">{'~'}</FlyqlToken>, or <FlyqlToken type="operator">{'!~'}</FlyqlToken> with null produces a parse error.

**SQL generation:** <FlyqlCode code={`field = null`} inline /> emits `field IS NULL`, <FlyqlCode code={`field != null`} inline /> emits `field IS NOT NULL` (all dialects).

### Column Reference

Any unquoted value that is not a number, boolean, null, or temporal function is treated as a **column reference** — a pointer to another column for column-to-column comparisons:

<FlyqlCode code={`price > min_price`} />
<FlyqlCode code={`updated_at > created_at`} />
<FlyqlCode code={`field = otherfield`} />

AST: `value_type: "column"`

If you intend to match a string value, you **must** use quotes:

<FlyqlCode code={`level = 'error'`} />

Without quotes, <FlyqlCode code={`level = error`} inline /> treats `error` as a reference to a column named `error`.

### Array

Arrays are used with <FlyqlToken type="operator">in</FlyqlToken> and <FlyqlToken type="operator">not in</FlyqlToken> operators. Elements can be mixed types (heterogeneous):

<FlyqlCode code={`status in [200, 201, 202]`} />
<FlyqlCode code={`name in ['alice', 'bob']`} />
<FlyqlCode code={`field in [1, "hello", true, null]`} />

Each element carries its own type in the `values_types` array.

## Truthy/Falsy Expressions

A standalone key without an operator checks if the field has a truthy value:

<FlyqlCode code={`active`} />
<FlyqlCode code={`message and status`} />

A value is considered **falsy** if it is:
- `null` / `None` / missing
- Empty string `""`
- Zero `0`
- Boolean <FlyqlToken type="boolean">false</FlyqlToken>

Everything else is **truthy**.

Use <FlyqlToken type="operator">not</FlyqlToken> to check for falsy values:

<FlyqlCode code={`not archived`} />
<FlyqlCode code={`active and not debug`} />

Truthy/falsy checks are distinct from null comparisons: <FlyqlCode code={`not field`} inline /> matches when the field is null, empty, or zero. <FlyqlCode code={`field = null`} inline /> matches only when the field is specifically null.

## Whitespace

FlyQL is whitespace-tolerant. Spaces around operators are optional:

<FlyqlCode code={`status = 200`} />
<FlyqlCode code={`status =200`} />

All of the above are equivalent.

## Type Constants Module

All type constants are available from the `flyql.types` module:

**Python:**
```python
from flyql.types import ValueType
# ValueType.INTEGER, ValueType.BIGINT, ValueType.FLOAT,
# ValueType.STRING, ValueType.BOOLEAN, ValueType.NULL, ValueType.ARRAY
```

**Go:**
```go
import "github.com/iamtelescope/flyql/golang/types"
// types.Integer, types.BigInt, types.Float,
// types.String, types.Boolean, types.Null, types.Array
```

**JavaScript:**
```javascript
import { ValueType } from 'flyql/types'
// ValueType.INTEGER, ValueType.BIGINT, ValueType.FLOAT,
// ValueType.STRING, ValueType.BOOLEAN, ValueType.NULL, ValueType.ARRAY
```

## Temporal Functions

FlyQL supports temporal function calls as values on the right-hand side of comparison expressions. These enable time-relative filters without hardcoding timestamps.

AST: `value_type: "function"`, `value` is a `FunctionCall` object.

### `ago(duration)`

Returns a timestamp relative to the current time. Supports compound durations.

<FlyqlCode code={`timestamp > ago(1h)`} />
<FlyqlCode code={`created_at >= ago(7d)`} />
<FlyqlCode code={`updated_at > ago(1h30m)`} />

**Duration units:** `s` (seconds), `m` (minutes), `h` (hours), `d` (days), `w` (weeks)

### `now()`

Returns the current timestamp.

<FlyqlCode code={`expires_at < now()`} />

### `today()`

Returns today's date. Accepts an optional IANA timezone argument.

<FlyqlCode code={`date = today()`} />
<FlyqlCode code={`date = today('Europe/Berlin')`} />

### `startOf(unit)`

Returns the start of the current day, week (Monday), or month. Accepts an optional timezone.

<FlyqlCode code={`created_at > startOf('day')`} />
<FlyqlCode code={`created_at > startOf('week')`} />
<FlyqlCode code={`created_at > startOf('month', 'Asia/Tokyo')`} />

**Valid units:** `'day'`, `'week'`, `'month'`

### Operator Restrictions

Temporal functions can only be used with comparison operators: `=`, `!=`, `>`, `>=`, `<`, `<=`.

Using `~` or `!~` with a temporal function produces a parse error. Operators like `in`, `has`, `like`, and `ilike` use separate parsing paths and are not compatible with function values.

### Column Reference Without Parens

Bare function names without parentheses are treated as column references, not function calls:

<FlyqlCode code={`field = ago`} />

This produces `value_type: "column"`, not a function call.

## Migration Notes

If upgrading from a previous FlyQL version:

- **Breaking:** Go's `ValueTypeString`/`ValueTypeNumber` removed, replaced by 7-value `types.ValueType`
- **Breaking:** AST `value_type` field changed from `"string"`/`"number"` to `"integer"`/`"float"`/`"bigint"`/`"string"`/`"boolean"`/`"null"`
- **Breaking:** Mixed-type list enforcement removed — heterogeneous arrays are now valid
- **New:** <FlyqlToken type="boolean">true</FlyqlToken>, <FlyqlToken type="boolean">false</FlyqlToken>, <FlyqlToken type="null">null</FlyqlToken> are parsed as typed literals (not strings)
- **New:** `values_types` array on IN-list expressions provides per-element types
- **New:** `flyql.types` module exports canonical type constants
- **Breaking:** Unquoted non-numeric values are now parsed as `value_type: "column"` (column references), not `"string"`. To match a string, wrap the value in quotes: `field = 'value'`
- **New:** Temporal function values (`ago()`, `now()`, `today()`, `startOf()`) with `value_type: "function"`
