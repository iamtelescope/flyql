## JSON カラム

StarRocks は 2 種類の JSON カラム型をサポートし、それぞれ異なる SQL を出力します。

### ネイティブ JSON 型

ネイティブ JSON カラムでは、JSON 文字列化したキー付きの `->` 演算子を使います。

<SqlExamplesTable rows={jsonNative} />

### JSON String 型

JSON を文字列として持つカラムでは、走査前に `parse_json()` が必要です。

<SqlExamplesTable rows={jsonString} />

## ネストされた複合型

StarRocks ジェネレーターは、セグメント化されたパスを単一のコンテナ型に属するものとして扱います。現時点では `Map(K, Struct)`、`Array(Map)`、あるいはそれ自身が `Map`/`Array` である `Struct` フィールドのような混在したネストを辿りません。そのようなカラムでは、パスを平坦化するか、内側の値を別カラムとして公開してください。

## Map カラム

Map カラムでは、角括弧記法と直接比較を使います。

<SqlExamplesTable rows={mapColumns} />

## Array カラム

FlyQL の配列インデックスは 0 始まりです。StarRocks 配列は SQL レイヤーで 1 始まりであるため、StarRocks ジェネレーターは SQL を生成する際にインデックスに 1 を加算します。

### カラム設定

{/* TODO: native-speaker review */}
JSON 内容を持つ text/varchar カラムは、ClickHouse と同様に合成的な生型 `jsonstring` として宣言します。

```go
// Go
columns := map[string]*starrocks.Column{
    "message": starrocks.NewColumn(starrocks.ColumnDef{Name: "message", Type: "VARCHAR(255)"}),
    "data":    starrocks.NewColumn(starrocks.ColumnDef{Name: "data", Type: "JSON"}),      // native JSON
    "log":     starrocks.NewColumn(starrocks.ColumnDef{Name: "log", Type: "jsonstring"}),       // JSON string
}
```

```javascript
// JavaScript
const columns = {
    message: newColumn({ name: "message", type: "VARCHAR(255)" }),
    data:    newColumn({ name: "data", type: "JSON" }),      // native JSON
    log:     newColumn({ name: "log", type: "jsonstring" }),       // JSON string
}
```
