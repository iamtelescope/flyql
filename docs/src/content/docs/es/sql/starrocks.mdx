El SQL de StarRocks usa identificadores entre backticks, `regexp()` para regex y `->` con claves serializadas como JSON para el acceso a JSON.

## Ejemplos básicos

| FlyQL | SQL generado |
|-------|--------------|
| <FlyqlCode code={`message = 'hello'`} inline /> | `` `message` = 'hello' `` |
| <FlyqlCode code={`message != 'hello'`} inline /> | `` `message` != 'hello' `` |
| <FlyqlCode code={`count > 10`} inline /> | `` `count` > 10 `` |
| <FlyqlCode code={`count <= 100`} inline /> | `` `count` <= 100 `` |
| <FlyqlCode code={`message ~ 'error.*'`} inline /> | `` regexp(`message`, 'error.*') `` |
| <FlyqlCode code={`message !~ 'test.*'`} inline /> | `` not regexp(`message`, 'test.*') `` |
| <FlyqlCode code={`host = prod*`} inline /> | `` `host` LIKE 'prod%' `` |

## Operadores booleanos

| FlyQL | SQL generado |
|-------|--------------|
| <FlyqlCode code={`a = '1' and b > 10`} inline /> | `` (`a` = '1' and `b` > 10) `` |
| <FlyqlCode code={`a = '1' or b = '2'`} inline /> | `` (`a` = '1' or `b` = '2') `` |
| <FlyqlCode code={`not a = '1'`} inline /> | `` not (`a` = '1') `` |

## Pertenencia a listas

| FlyQL | SQL generado |
|-------|--------------|
| <FlyqlCode code={`status in [200, 201]`} inline /> | `` `status` IN (200, 201) `` |
| <FlyqlCode code={`env not in ['dev', 'test']`} inline /> | `` `env` NOT IN ('dev', 'test') `` |
| <FlyqlCode code={`status in []`} inline /> | `0` |
| <FlyqlCode code={`status not in []`} inline /> | `1` |

## Contención (`has` / `not has`)

El operador <FlyqlToken type="operator">has</FlyqlToken> genera SQL distinto según el tipo de columna:

| Tipo de columna | FlyQL | SQL generado |
|-----------------|-------|--------------|
| String | <FlyqlCode code={`message has 'error'`} inline /> | `` INSTR(`message`, 'error') > 0 `` |
| String con negación | <FlyqlCode code={`message not has 'error'`} inline /> | `` (`message` IS NULL OR INSTR(`message`, 'error') = 0) `` |
| Array | <FlyqlCode code={`tags has 'web'`} inline /> | `` array_contains(`tags`, 'web') `` |
| Array con negación | <FlyqlCode code={`tags not has 'web'`} inline /> | `` NOT array_contains(`tags`, 'web') `` |
| Map | <FlyqlCode code={`metadata has 'key'`} inline /> | `` array_contains(map_keys(`metadata`), 'key') `` |
| Map con negación | <FlyqlCode code={`metadata not has 'key'`} inline /> | `` NOT array_contains(map_keys(`metadata`), 'key') `` |
| JSON nativo | <FlyqlCode code={`data has 'key'`} inline /> | `` json_exists(`data`, concat('$.', 'key')) `` |

## Columnas JSON

StarRocks admite dos tipos de columnas JSON con salidas SQL distintas.

### Tipo JSON nativo

Las columnas JSON nativas usan el operador `->` con claves serializadas como JSON:

| FlyQL | SQL generado |
|-------|--------------|
| <FlyqlCode code={`data.name = 'test'`} inline /> | `` `data`->'\"name\"' = 'test' `` |
| <FlyqlCode code={`data.user.name = 'john'`} inline /> | `` `data`->'\"user\"'->'\"name\"' = 'john' `` |

### Tipo JSON string

Las columnas que almacenan JSON como string requieren `parse_json()` antes del recorrido:

| FlyQL | SQL generado |
|-------|--------------|
| <FlyqlCode code={`data.name = 'test'`} inline /> | `` parse_json(`data`)->'\"name\"' = 'test' `` |
| <FlyqlCode code={`data.user.name = 'john'`} inline /> | `` parse_json(`data`)->'\"user\"'->'\"name\"' = 'john' `` |

## Columnas Map

Las columnas Map usan notación con corchetes y comparación directa:

| FlyQL | SQL generado |
|-------|--------------|
| <FlyqlCode code={`metadata.key1 = 'value1'`} inline /> | `` `metadata`['key1'] = 'value1' `` |
| <FlyqlCode code={`metadata.key1 != 'value1'`} inline /> | `` `metadata`['key1'] != 'value1' `` |
| <FlyqlCode code={`metadata.nested.key = 'value'`} inline /> | `` `metadata`['nested']['key'] = 'value' `` |
| <FlyqlCode code={`metadata.pattern ~ 'test.*'`} inline /> | `` `metadata`['pattern'] regexp 'test.*' `` |

## Columnas Array

Las columnas Array usan indexación base 0 con corchetes:

| FlyQL | SQL generado |
|-------|--------------|
| <FlyqlCode code={`tags.0 = 'first'`} inline /> | `` `tags`[0] = 'first' `` |
| <FlyqlCode code={`tags.1 = 'second'`} inline /> | `` `tags`[1] = 'second' `` |

### Configuración de columnas

El parámetro `jsonString` controla qué modo se usa, igual que en ClickHouse:

```go
// Go
columns := map[string]*starrocks.Column{
    "message": starrocks.NewColumn("message", false, "VARCHAR(255)", nil),
    "data":    starrocks.NewColumn("data", false, "JSON", nil),      // native JSON
    "log":     starrocks.NewColumn("log", true, "STRING", nil),       // JSON string
}
```

```javascript
// JavaScript
const columns = {
    message: newColumn("message", false, "VARCHAR(255)", null),
    data:    newColumn("data", false, "JSON", null),      // native JSON
    log:     newColumn("log", true, "STRING", null),       // JSON string
}
```

## Transformers

Los transformers generan llamadas a funciones estándar de StarRocks con nombres de columnas entre backticks:

| FlyQL | SQL generado |
|-------|--------------|
| <FlyqlCode code={'status\|upper = "ERROR"'} inline /> | `` UPPER(`status`) = 'ERROR' `` |
| <FlyqlCode code={'status\|lower = "info"'} inline /> | `` LOWER(`status`) = 'info' `` |
| <FlyqlCode code={'message\|len > 100'} inline /> | `` LENGTH(`message`) > 100 `` |
| <FlyqlCode code={'message\|lower\|len > 100'} inline /> | `` LENGTH(LOWER(`message`)) > 100 `` |
| <FlyqlCode code={'tags\|split(",") has "urgent"'} inline /> | `` array_contains(SPLIT(`tags`, ','), 'urgent') `` |
