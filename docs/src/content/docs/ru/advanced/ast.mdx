Парсер FlyQL создаёт **абстрактное синтаксическое дерево (AST)** — структурированное представление запроса, которое можно обходить для генерации любого целевого вывода.

## Типы узлов AST

AST состоит из трёх основных сущностей: **Node**, **Expression** и **Key**.

### Node

`Node` может быть либо ветвью, либо листовым узлом.

| Поле | Тип | Описание |
|------|-----|----------|
| `boolOperator` | `string` | `"and"` или `"or"` |
| `expression` | `Expression` или `null` | выражение, если это листовой узел |
| `left` | `Node` или `null` | левый потомок |
| `right` | `Node` или `null` | правый потомок |
| `negated` | `boolean` | обёрнут ли узел в `not` |

### Expression

Одиночное условие: ключ, оператор и значение.

### Key

Поле запроса, представленное как путь из сегментов.

## Зачем понимать AST

Это позволяет:

- писать собственные генераторы
- делать анализ и переписывание запросов
- строить собственные механизмы сопоставления
