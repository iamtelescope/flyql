## Regex マッチ

値を正規表現に一致させるには `~`、否定するには `!~` を使います。

```
message~"error.*timeout"
path!~"^/health"
```

| 演算子 | 意味 |
|--------|------|
| `~` | 値が regex パターンに一致する |
| `!~` | 値が regex パターンに一致しない |

### Regex パターン構文

regex の構文はターゲットのバックエンドによって異なります。

- **ClickHouse**: `match()` 関数を使用（RE2 構文）
- **PostgreSQL**: `~` と `!~` 演算子を使用（POSIX regex）
- **StarRocks**: `regexp` 関数を使用
- **メモリ上の matcher**: ホスト言語の regex engine を使用（Go `regexp`、Python `re2`、JS `RegExp`）

スペースや特殊文字を含むパターンはクォートしてください。

```
message~"connection (refused|reset)"
host~"^prod-[0-9]+"
```

### 大文字小文字を区別しないマッチ

大文字小文字を区別しない regex マッチを行うには、パターン先頭に `(?i)` フラグを付けます。

```
message~"(?i)error"
level~"(?i)^(warn|error|fatal)$"
```

インラインフラグ `(?i)` は regex の機能であり、サポート状況はバックエンドごとに異なります。

| バックエンド | `(?i)` のサポート | 備考 |
|--------------|-------------------|------|
| **ClickHouse** | Yes | RE2 構文 |
| **PostgreSQL** | Yes | POSIX regex |
| **StarRocks** | Yes | RE2 構文 |
| **Go メモリ matcher** | Yes | Go `regexp`（RE2） |
| **Python メモリ matcher** | Yes | `google-re2` |
| **JavaScript メモリ matcher** | No | ECMAScript `RegExp` は `(?i)` インラインフラグをサポートしません |

JavaScript のメモリ matcher では、代わりに文字クラスを使って大文字小文字を区別しないマッチを実現してください。

```
message~"[eE][rR][rR][oO][rR]"
```

また、SQL バックエンドのみを対象にするなら、`(?i)` は 3 つの方言すべてで安定して使えます。

大文字小文字を区別しないマッチと否定を組み合わせることで、大小文字に関係なく特定パターンを除外できます。

```
message~"(?i)error" and message!~"(?i)warning"
```

## ワイルドカードマッチ

クォートされていない値の末尾に `*` を付けると、プレフィックス一致を行えます。

```
host=prod*
service=api-*
```

これは指定したプレフィックスで始まる任意の値に一致します。ワイルドカードを使えるのは値の末尾だけで、`*prod` や `pr*od` はサポートされません。

### ワイルドカードと regex の違い

| パターン | 構文 | 例 |
|----------|------|----|
| ワイルドカード（プレフィックス） | `key=value*` | `host=prod*` |
| regex（完全指定） | `key~"pattern"` | `host~"^prod-[0-9]+"` |

単純なプレフィックス一致にはワイルドカードを使い、より複雑なパターンが必要な場合は regex を使ってください。

## 例

```
# Regex: Meldungen mit „error“ oder „fail“
message~"error|fail"

# Regex: Pfade, die mit /api/v2 beginnen
path~"^/api/v2"

# Negiertes Regex: Health-Check-Pfade ausschließen
path!~"^/health"

# Wildcard: Hosts, die mit „prod“ beginnen
host=prod*

# Kombiniert
host=prod* and message~"timeout.*" and path!~"^/health"
```
