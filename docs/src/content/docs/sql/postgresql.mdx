PostgreSQL SQL uses double-quoted identifiers, <FlyqlToken type="operator">{'~'}</FlyqlToken> for regex, uppercase <FlyqlToken type="operator">and</FlyqlToken>/<FlyqlToken type="operator">or</FlyqlToken>, and JSONB operators for nested access.

## Basic Examples

| FlyQL | Generated SQL |
|-------|--------------|
| <FlyqlCode code={`message = 'hello'`} inline /> | `"message" = 'hello'` |
| <FlyqlCode code={`message != 'hello'`} inline /> | `"message" != 'hello'` |
| <FlyqlCode code={`count > 10`} inline /> | `"count" > 10` |
| <FlyqlCode code={`count <= 100`} inline /> | `"count" <= 100` |
| <FlyqlCode code={`message ~ 'error.*'`} inline /> | `"message" ~ 'error.*'` |
| <FlyqlCode code={`message !~ 'test.*'`} inline /> | `"message" !~ 'test.*'` |
| <FlyqlCode code={`host = 'prod'`} inline /> | `"host" = 'prod'` |

## Boolean Operators

PostgreSQL uses uppercase <FlyqlToken type="operator">and</FlyqlToken>/<FlyqlToken type="operator">or</FlyqlToken> (unlike ClickHouse and StarRocks):

| FlyQL | Generated SQL |
|-------|--------------|
| <FlyqlCode code={`a = '1' and b > 10`} inline /> | `("a" = '1' AND "b" > 10)` |
| <FlyqlCode code={`a = '1' or b = '2'`} inline /> | `("a" = '1' OR "b" = '2')` |
| <FlyqlCode code={`not a = '1'`} inline /> | `NOT ("a" = '1')` |

## Truthy Checks

A bare column reference (`active`, `not active`) generates type-aware SQL — there is no single "is truthy" expression, because what counts as falsy differs by column type.

| Column Type | FlyQL | Generated SQL |
|-------------|-------|--------------|
| String | <FlyqlCode code={`message`} inline /> | `("message" IS NOT NULL AND "message" != '')` |
| Int / Float | <FlyqlCode code={`count`} inline /> | `("count" IS NOT NULL AND "count" != 0)` |
| Bool | <FlyqlCode code={`active`} inline /> | `"active"` |
| Date | <FlyqlCode code={`created_at`} inline /> | `("created_at" IS NOT NULL)` |
| Negated String | <FlyqlCode code={`not message`} inline /> | `("message" IS NULL OR "message" = '')` |

## Containment (`has` / `not has`)

The <FlyqlToken type="operator">has</FlyqlToken> operator generates different SQL depending on the column type:

| Column Type | FlyQL | Generated SQL |
|-------------|-------|--------------|
| String | <FlyqlCode code={`message has 'error'`} inline /> | `position('error' in "message") > 0` |
| String (negated) | <FlyqlCode code={`message not has 'error'`} inline /> | `("message" IS NULL OR position('error' in "message") = 0)` |
| Array | <FlyqlCode code={`tags has 'web'`} inline /> | `'web' = ANY("tags")` |
| Array (negated) | <FlyqlCode code={`tags not has 'web'`} inline /> | `NOT ('web' = ANY("tags"))` |
| JSONB | <FlyqlCode code={`data has 'key'`} inline /> | `"data" ? 'key'` |
| JSONB (negated) | <FlyqlCode code={`data not has 'key'`} inline /> | `NOT ("data" ? 'key')` |
| Hstore | <FlyqlCode code={`metadata has 'key'`} inline /> | `"metadata" ? 'key'` |

## JSONB Columns

PostgreSQL uses JSONB operators with type checking via `jsonb_typeof()`:

### String Comparison

| FlyQL | Generated SQL |
|-------|--------------|
| <FlyqlCode code={`data.name = 'test'`} inline /> | `(jsonb_typeof("data"->'name') = 'string' AND "data"->>'name' = 'test')` |
| <FlyqlCode code={`data.user.name = 'john'`} inline /> | `(jsonb_typeof("data"->'user'->'name') = 'string' AND "data"->'user'->>'name' = 'john')` |

### Numeric Comparison

Numeric comparisons cast the extracted value:

| FlyQL | Generated SQL |
|-------|--------------|
| <FlyqlCode code={`data.age > 25`} inline /> | `(jsonb_typeof("data"->'age') = 'number' AND ("data"->>'age')::numeric > 25)` |

### JSONB Operators

- `->` returns JSONB (used for intermediate path traversal)
- `->>` returns text (used for the final value extraction)
- `jsonb_typeof()` checks the JSON type before comparison

## Hstore (Map) Columns

PostgreSQL uses hstore type with the `->` operator for key access:

| FlyQL | Generated SQL |
|-------|--------------|
| <FlyqlCode code={`metadata.key1 = 'value1'`} inline /> | `"metadata"->'key1' = 'value1'` |
| <FlyqlCode code={`metadata.key1 != 'value1'`} inline /> | `"metadata"->'key1' != 'value1'` |
| <FlyqlCode code={`metadata.pattern ~ 'test.*'`} inline /> | `"metadata"->'pattern' ~ 'test.*'` |

## Array Columns

PostgreSQL arrays are 1-based — FlyQL automatically converts the 0-based index:

| FlyQL | Generated SQL |
|-------|--------------|
| <FlyqlCode code={`tags.0 = 'first'`} inline /> | `"tags"[1] = 'first'` |
| <FlyqlCode code={`tags.1 = 'second'`} inline /> | `"tags"[2] = 'second'` |
| <FlyqlCode code={`tags.0 ~ 'tag.*'`} inline /> | `"tags"[1] ~ 'tag.*'` |

### Column Setup

PostgreSQL columns do not have a `jsonString` parameter — JSONB type is detected from the column type:

```go
// Go
columns := map[string]*postgresql.Column{
    "message": postgresql.NewColumn("message", "varchar", nil),
    "data":    postgresql.NewColumn("data", "jsonb", nil),
}
```

```javascript
// JavaScript
const columns = {
    message: newColumn("message", "varchar", null),
    data:    newColumn("data", "jsonb", null),
}
```
