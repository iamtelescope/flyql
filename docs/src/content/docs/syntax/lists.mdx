## List Membership

Use <FlyqlToken type="operator">in</FlyqlToken> to check if a value is in a list, and <FlyqlToken type="operator">not in</FlyqlToken> to check it is not:

<FlyqlCode code={`status in [200, 201, 204]`} />
<FlyqlCode code={`env not in ['prod', 'staging']`} />

## Syntax Rules

- Values are enclosed in square brackets <FlyqlToken type="operator">[ ]</FlyqlToken> and separated by commas
- Values can be mixed types: <FlyqlCode code={`[200, 'ok', true, null]`} inline />
- String values must be quoted: <FlyqlCode code={`['a', 'b', 'c']`} inline />
- Numeric values are unquoted: <FlyqlCode code={`[1, 2, 3]`} inline />
- Spaces around commas and brackets are allowed

### String Lists

<FlyqlCode code={`method in ['GET', 'POST', 'PUT']`} />
<FlyqlCode code={`env not in ['dev', 'test', 'local']`} />

Both single and double quotes work for string values.

### Numeric Lists

<FlyqlCode code={`status in [200, 201, 204]`} />
<FlyqlCode code={`port not in [80, 443]`} />

### Empty Lists

Empty lists are allowed:

<FlyqlCode code={`status in []`} />

An empty <FlyqlToken type="operator">in</FlyqlToken> list is always false — nothing matches an empty list.

<FlyqlCode code={`status not in []`} />

An empty <FlyqlToken type="operator">not in</FlyqlToken> list is always true — everything is "not in" an empty list.

## Mixed Types

Lists support heterogeneous values — you can mix strings, numbers, booleans, and null:

<FlyqlCode code={`field in [200, 'ok', true, null]`} />

Each element carries its own type in the `valuesTypes` array on the parsed expression (e.g. `["integer", "string", "boolean", "null"]`). For empty lists, `valuesTypes` is `null`.

When a SQL generator processes a list, it validates that the list value types are compatible with the target column type. For example, passing string values to a numeric column will raise a validation error during SQL generation.

## Parameters in Lists

Parameter placeholders can appear inside IN-lists, mixed freely with literal values:

<FlyqlCode code={`status in [$x, $y, 200]`} />
<FlyqlCode code={`env not in [$current, 'dev']`} />

The `Parameter` objects sit in the `values` array until `bindParams()` resolves them. See [Parameters](/syntax/parameters/) for the full reference.

## Combined with Other Operators

List operators combine with boolean operators like any other condition:

<FlyqlCode code={`status in [200, 201] and method in ['GET', 'POST']`} />
<FlyqlCode code={`level = 'error' or status not in [200, 201, 204]`} />
<FlyqlCode code={`(status in [500, 502, 503] or level = 'critical') and env='prod'`} />

## Examples
