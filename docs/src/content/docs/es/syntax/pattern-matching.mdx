## Coincidencia regex

Usa <FlyqlToken type="operator">{'~'}</FlyqlToken> para comparar un valor con una expresión regular y <FlyqlToken type="operator">{'!~'}</FlyqlToken> para negar la coincidencia:

<FlyqlCode code={`message ~ "error.*timeout"`} />
<FlyqlCode code={`path !~ "^/health"`} />

| Operador | Significado |
|----------|-------------|
| <FlyqlToken type="operator">{'~'}</FlyqlToken> | El valor coincide con el patrón regex |
| <FlyqlToken type="operator">{'!~'}</FlyqlToken> | El valor no coincide con el patrón regex |

### Sintaxis de patrones regex

La sintaxis regex depende del backend de destino:

- **ClickHouse**: usa la función `match()` (sintaxis RE2)
- **PostgreSQL**: usa los operadores <FlyqlToken type="operator">{'~'}</FlyqlToken> y <FlyqlToken type="operator">{'!~'}</FlyqlToken> (regex POSIX)
- **StarRocks**: usa la función `regexp`
- **Matcher en memoria**: usa el motor regex del lenguaje host (Go `regexp`, Python `re2`, JS `RegExp`)

Pon el patrón entre comillas cuando contenga espacios o caracteres especiales:

<FlyqlCode code={`message ~ "connection (refused|reset)"`} />
<FlyqlCode code={`host ~ "^prod-[0-9]+"`} />

### Flags de regex

Puedes modificar el comportamiento de la regex usando flags inline al inicio del patrón con la sintaxis `(?...)`. FlyQL soporta tres flags que funcionan en todos los backends:

#### `(?i)` — Sin distinción de mayúsculas

Hace que el patrón coincida sin importar las mayúsculas o minúsculas:

<FlyqlCode code={`message ~ "(?i)error"`} />
<FlyqlCode code={`level ~ "(?i)^(warn|error|fatal)$"`} />

`(?i)error` coincide con "Error", "ERROR", "error", etc.

#### `(?m)` — Multilínea

Cambia `^` y `$` para que coincidan con el inicio y el final de cada **línea** en lugar de toda la cadena:

<FlyqlCode code={`log ~ "(?m)^ERROR"`} />

Sin `(?m)`, `^ERROR` solo coincide si la cadena completa empieza con "ERROR". Con `(?m)`, también coincide con "INFO first\nERROR second" porque "ERROR" aparece al inicio de una línea.

#### `(?s)` — Dotall

Hace que `.` coincida con **cualquier** carácter, incluidos los saltos de línea. Por defecto, `.` no coincide con `\n`:

<FlyqlCode code={`log ~ "(?s)start.*end"`} />

Sin `(?s)`, `start.*end` no coincide con "start\nend" porque `.` no cruza el salto de línea. Con `(?s)`, sí coincide.

#### Combinar flags

Los flags se pueden combinar en un solo grupo:

<FlyqlCode code={`log ~ "(?im)^ERROR"`} />
<FlyqlCode code={`log ~ "(?ims)^START.*END$"`} />

#### Compatibilidad con backends

Los tres flags funcionan en todos los backends de FlyQL:

| Backend | `(?i)` | `(?m)` | `(?s)` |
|---------|--------|--------|--------|
| **ClickHouse** | Yes | Yes | Yes |
| **PostgreSQL** | Yes | Yes | Yes |
| **StarRocks** | Yes | Yes | Yes |
| **Go matcher** | Yes | Yes | Yes |
| **Python matcher** | Yes | Yes | Yes |
| **JavaScript matcher** | Yes | Yes | Yes |

Combina la coincidencia sin distinción de mayúsculas con negación para excluir patrones sin importar el caso:

<FlyqlCode code={`message ~ "(?i)error" and message !~ "(?i)warning"`} />

## Coincidencia LIKE

Usa <FlyqlToken type="operator">like</FlyqlToken> e <FlyqlToken type="operator">ilike</FlyqlToken> para coincidencia de patrones estilo SQL con comodines:

<FlyqlCode code={`host like 'prod%'`} />
<FlyqlCode code={`service like 'api-%'`} />
<FlyqlCode code={`message ilike '%Error%'`} />

| Operador | Significado |
|----------|-------------|
| <FlyqlToken type="operator">like</FlyqlToken> | El valor coincide con el patrón (sensible a mayúsculas) |
| <FlyqlToken type="operator">not like</FlyqlToken> | El valor no coincide con el patrón |
| <FlyqlToken type="operator">ilike</FlyqlToken> | El valor coincide con el patrón (sin distinción de mayúsculas) |
| <FlyqlToken type="operator">not ilike</FlyqlToken> | El valor no coincide con el patrón (sin distinción de mayúsculas) |

### Comodines

Hay dos caracteres comodín disponibles en los patrones <FlyqlToken type="operator">like</FlyqlToken> / <FlyqlToken type="operator">ilike</FlyqlToken>:

| Comodín | Significado | Ejemplo | Coincide con |
|---------|-------------|---------|--------------|
| `%` | Cualquier secuencia de caracteres (incluido vacío) | <FlyqlCode code={`host like 'prod%'`} inline /> | `prod`, `prod-1`, `production` |
| `_` | Exactamente un carácter | <FlyqlCode code={`code like 'E__'`} inline /> | `E01`, `Exx`, pero no `E1` ni `Error` |

Los comodines pueden aparecer en cualquier posición del patrón — al inicio, en medio, al final, o combinados:

<FlyqlCode code={`message like '%timeout%'`} />
<FlyqlCode code={`host like 'web-_-prod'`} />
<FlyqlCode code={`path like '/api/%/status'`} />

### Escapar comodines

Para buscar un `%` o `_` literal, escápalo con una barra invertida:

<FlyqlCode code={`value like '100\\%'`} />
<FlyqlCode code={`col like 'foo\\_bar'`} />

`100\%` coincide con la cadena literal "100%", no con "100" seguido de cualquier cosa.

### Coincidencia sin distinción de mayúsculas

Usa <FlyqlToken type="operator">ilike</FlyqlToken> para coincidencia de patrones sin distinción de mayúsculas:

<FlyqlCode code={`message ilike '%error%'`} />
<FlyqlCode code={`host not ilike 'TEST%'`} />

### LIKE frente a Regex

| Caso de uso | Sintaxis | Ejemplo |
|-------------|----------|---------|
| Coincidencia por prefijo | <FlyqlCode code={`key like 'prefix%'`} inline /> | <FlyqlCode code={`host like 'prod%'`} inline /> |
| Contiene | <FlyqlCode code={`key like '%word%'`} inline /> | <FlyqlCode code={`msg like '%error%'`} inline /> |
| Patrón complejo | <FlyqlCode code={`key ~ "pattern"`} inline /> | <FlyqlCode code={`host ~ "^prod-[0-9]+"`} inline /> |

Usa <FlyqlToken type="operator">like</FlyqlToken> para coincidencia simple con comodines. Usa regex (<FlyqlToken type="operator">{'~'}</FlyqlToken>) cuando necesites clases de caracteres, alternación u otros patrones avanzados.

### Compatibilidad con backends

Todos los backends SQL (ClickHouse, PostgreSQL, StarRocks) emiten operadores SQL nativos `LIKE` / `ILIKE` / `NOT LIKE` / `NOT ILIKE`. El matcher en memoria convierte el patrón a regex internamente.

:::note
StarRocks `ILIKE` requiere versión 2.5 o posterior.
:::

## Ejemplos

<FlyqlCode code={`message ~ "error|fail"`} />
<FlyqlCode code={`path ~ "^/api/v2"`} />
<FlyqlCode code={`path !~ "^/health"`} />
<FlyqlCode code={`host like 'prod%'`} />
<FlyqlCode code={`host like 'prod%' and message ~ "timeout.*" and path !~ "^/health"`} />
