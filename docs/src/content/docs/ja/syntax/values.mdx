FlyQL は 7 つの値型をサポートします。**Integer**、**BigInt**、**Float**、**String**、**Boolean**、**Null**、**Array** です。AST 内の各値には明示的な `value_type` フィールドが付きます。

## 値型

### Integer

int64 の範囲内に収まる整数です。

<FlyqlCode code={`count = 42`} />
<FlyqlCode code={`status != 200`} />
<FlyqlCode code={`age >= 18`} />

AST: `value_type: "int"`

### BigInt

int64 の範囲（Go/Python）または `MAX_SAFE_INTEGER`（JavaScript）を超える整数です。

<FlyqlCode code={`id = 18446744073709551615`} />

AST: `value_type: "bigint"`

### Float

小数点を含む数値です。

<FlyqlCode code={`price > 9.99`} />
<FlyqlCode code={`ratio = 3.14`} />
<FlyqlCode code={`temperature<=-0.5`} />

AST: `value_type: "float"`

:::caution[文字列はクォートが必須]
よくある間違い: <FlyqlCode code={`level = 'error'`} inline /> は**文字列** `error` にマッチしますが、<FlyqlCode code={`level = error`} inline /> は**カラム参照** `error`（つまり「`error` という名前のカラム」）にマッチします。FlyQL はクォート無しの識別子を黙って文字列に変換することはありません。

| FlyQL | 意味 |
|-------|------|
| <FlyqlCode code={`level = 'error'`} inline /> | `level` が文字列 `error` と等しい |
| <FlyqlCode code={`level = "error"`} inline /> | `level` が文字列 `error` と等しい（シングルクォートとダブルクォートは等価） |
| <FlyqlCode code={`level = error`} inline /> | `level` が `error` という名前のカラムの値と等しい（カラム同士の比較） |

クエリが無音で 0 件しか返さない、または `level = error_column_name` のような予期しない SQL が出ている場合、まず最初に確認すべき点です。カラム同士の比較形式については [カラム参照](#カラム参照) を参照してください。
:::

### String

テキスト値はシングルクォートまたはダブルクォートで**必ずクォート**してください。クォートのない値はカラム参照として解釈され、文字列にはなりません。

<FlyqlCode code={`level = 'error'`} />
<FlyqlCode code={`user = "John Doe"`} />
<FlyqlCode code={`message = 'hello world'`} />
<FlyqlCode code={`count = '42'`} />

AST: `value_type: "string"`

クォートはバックスラッシュでエスケープできます。

<FlyqlCode code={`user = 'John\\'s'`} />
<FlyqlCode code={`message = "said \\"hello\\""`} />

:::tip
シングルクォートとダブルクォートは互換なので、エスケープを避けたい方を選べます。値にアポストロフィが含まれる場合はダブルクォートを、ダブルクォートが含まれる場合はシングルクォートを使います。

<FlyqlCode code={`user = "John's"`} />
<FlyqlCode code={`message = 'said "hello"'`} />
:::

:::note[Date カラム型 vs. DateTime カラム型]
カラムスキーマは **`Type.Date`**（暦日のみ — `YYYYMMDD`）と **`Type.DateTime`**（ミリ秒解像度の時点）を区別します。マッチャーは宣言された型を使って、RHS の ISO-8601 文字列リテラルやネイティブ時間値をコアースします。DateTime カラムはオプションで `tz`（IANA）と `unit`（`ms` / `s` / `ns`）メタデータを宣言でき、ナイーブ文字列や数値の曖昧さを解消します。

完全な iso8601 契約、tz フォールバックルール、DST 処理、サポートされるレコード値型の一覧は [日付](/ja/syntax/dates) を参照してください。
:::

### Boolean

<FlyqlToken type="boolean">true</FlyqlToken> と <FlyqlToken type="boolean">false</FlyqlToken> という小文字のキーワードです。大文字小文字は区別されます。

<FlyqlCode code={`active = true`} />
<FlyqlCode code={`debug = false`} />
<FlyqlCode code={`active != true`} />

AST: `value_type: "bool"`。値はネイティブ boolean です。

先頭が大文字のバリエーション（`True`、`FALSE`）は文字列として扱われます。

**SQL 生成:**
| 方言 | FlyQL | 生成される SQL |
|------|-------|---------------|
| ClickHouse | <FlyqlCode code={`field = true`} inline /> | `field = true` |
| ClickHouse | <FlyqlCode code={`field = false`} inline /> | `field = false` |
| PostgreSQL | <FlyqlCode code={`field = true`} inline /> | `field = TRUE` |
| PostgreSQL | <FlyqlCode code={`field = false`} inline /> | `field = FALSE` |
| StarRocks | <FlyqlCode code={`field = true`} inline /> | `field = true` |
| StarRocks | <FlyqlCode code={`field = false`} inline /> | `field = false` |

### Null

<FlyqlToken type="null">null</FlyqlToken> という小文字のキーワードです。大文字小文字は区別されます。

<FlyqlCode code={`field = null`} />
<FlyqlCode code={`field != null`} />

AST: `value_type: "null"`。値はネイティブ null です。

先頭が大文字のバリエーション（`Null`、`NULL`）は文字列として扱われます。

**制約:** null と組み合わせて使えるのは <FlyqlToken type="operator">=</FlyqlToken> と <FlyqlToken type="operator">!=</FlyqlToken> のみです。<FlyqlToken type="operator">{'>'}</FlyqlToken>、<FlyqlToken type="operator">{'<'}</FlyqlToken>、<FlyqlToken type="operator">{'>='}</FlyqlToken>、<FlyqlToken type="operator">{'<='}</FlyqlToken>、<FlyqlToken type="operator">{'~'}</FlyqlToken>、<FlyqlToken type="operator">{'!~'}</FlyqlToken> を null と使うと parse error になります。

**SQL 生成:** <FlyqlCode code={`field = null`} inline /> は `field IS NULL`、<FlyqlCode code={`field != null`} inline /> は `field IS NOT NULL` を生成します。全方言で共通です。

### Array

Array は <FlyqlToken type="operator">in</FlyqlToken> と <FlyqlToken type="operator">not in</FlyqlToken> で使います。要素には異なる型を混在できます。

<FlyqlCode code={`status in [200, 201, 202]`} />
<FlyqlCode code={`name in ['alice', 'bob']`} />
<FlyqlCode code={`field in [1, "hello", true, null]`} />

各要素は `values_types` 配列にそれぞれの型を持ちます。

## Truthy/Falsy 式

演算子を持たない単独のキーは、そのフィールドが truthy な値を持つかどうかをチェックします。

<FlyqlCode code={`active`} />
<FlyqlCode code={`message and status`} />

次の値は **falsy** とみなされます。
- `null` / `None` / 未存在
- 空文字列 `""`
- 0
- boolean <FlyqlToken type="boolean">false</FlyqlToken>

それ以外は **truthy** です。

falsy をチェックするには <FlyqlToken type="operator">not</FlyqlToken> を使います。

<FlyqlCode code={`not archived`} />
<FlyqlCode code={`active and not debug`} />

truthy/falsy チェックは null 比較とは異なります。<FlyqlCode code={`not field`} inline /> は、フィールドが null、空、または 0 の場合に一致します。<FlyqlCode code={`field = null`} inline /> は、フィールドが明示的に null の場合にのみ一致します。

## 空白

FlyQL は空白に寛容です。演算子の前後のスペースは省略できます。

<FlyqlCode code={`status = 200`} />
<FlyqlCode code={`status =200`} />

上記はすべて同じ意味です。

## LiteralKind 定数

パースされた式の値には `LiteralKind` タグが付きます。定数は各実装のルートパッケージからエクスポートされます。

**Python:**
```python
from flyql import LiteralKind
# LiteralKind.INTEGER, LiteralKind.BIGINT, LiteralKind.FLOAT,
# LiteralKind.STRING, LiteralKind.BOOLEAN, LiteralKind.NULL, LiteralKind.ARRAY
```

**Go:**
```go
import "github.com/iamtelescope/flyql/golang/literal"
// literal.Integer, literal.BigInt, literal.Float,
// literal.String, literal.Boolean, literal.Null, literal.Array
```

**JavaScript:**
```javascript
import { LiteralKind } from 'flyql'
// LiteralKind.INTEGER, LiteralKind.BIGINT, LiteralKind.FLOAT,
// LiteralKind.STRING, LiteralKind.BOOLEAN, LiteralKind.NULL, LiteralKind.ARRAY
```

## テンポラル関数

FlyQL は比較式の右辺の値としてテンポラル関数呼び出しをサポートしています。タイムスタンプをハードコーディングせずに、時間相対フィルターを実現できます。

AST: `value_type: "function"`、`value` は `FunctionCall` オブジェクトです。

### `ago(duration)`

現在時刻からの相対タイムスタンプを返します。複合的な期間指定に対応しています。

<FlyqlCode code={`timestamp > ago(1h)`} />
<FlyqlCode code={`created_at >= ago(7d)`} />
<FlyqlCode code={`updated_at > ago(1h30m)`} />

**単位（小文字）:** `s`（秒）、`m`（分）、`h`（時間）、`d`（日）、`w`（週）。

**文法。** 期間は 1 つ以上の `<整数><単位>` のペアで構成されます。単位は **厳密な降順** で **各単位は最大 1 回まで** 出現できます：`w > d > h > m > s`。これは Prometheus / Grafana / Loki の慣例に従っています。

有効:

<FlyqlCode code={`ago(30s)`} />
<FlyqlCode code={`ago(2h15m)`} />
<FlyqlCode code={`ago(1w2d3h4m5s)`} />
<FlyqlCode code={`ago(1w30s)`} />

解析時に拒否:

- `ago(30m1h)` — 単位の順序が逆
- `ago(1h1h)` — 単位の重複
- `ago(3h1w)` — 小さい単位の後に大きい単位
- `ago(1H)` — 単位は小文字でなければなりません

:::note[なぜ月/年がないのか？]
月および年の単位は意図的に **サポートされていません**：これらの長さは可変（28〜31 日、365／366 日）であり、同じリテラルがコンテキストによって異なる秒数を意味することになります。カレンダー相対の時間ウィンドウは代わりに [`startOf('month')`](#startofunit) で表現します。
:::

### `now()`

現在のタイムスタンプを返します。

<FlyqlCode code={`expires_at < now()`} />

### `today()`

今日の日付を返します。オプションで IANA タイムゾーン引数を指定できます。

<FlyqlCode code={`date = today()`} />
<FlyqlCode code={`date = today('Europe/Berlin')`} />

### `startOf(unit)`

現在の日、週（月曜日）、または月の開始時刻を返します。オプションでタイムゾーンを指定できます。

<FlyqlCode code={`created_at > startOf('day')`} />
<FlyqlCode code={`created_at > startOf('week')`} />
<FlyqlCode code={`created_at > startOf('month', 'Asia/Tokyo')`} />

**有効な単位:** `'day'`、`'week'`、`'month'`

### 演算子の制約

テンポラル関数は比較演算子のみで使用できます: `=`、`!=`、`>`、`>=`、`<`、`<=`。

テンポラル関数に `~` や `!~` を使うと parse error になります。`in`、`has`、`like`、`ilike` などの演算子は別の解析パスを使用するため、関数値とは互換性がありません。

### 括弧なしのカラム参照

括弧のない関数名はカラム参照として扱われ、関数呼び出しにはなりません。

<FlyqlCode code={`field = ago`} />

これは `value_type: "column"` を生成し、関数呼び出しにはなりません。

## Parameter

パラメータプレースホルダ（名前付きは `$name`、位置指定は `$1`）は、実行時に `bindParams()` で解決される値の代わりになります。リテラル値が有効な場所であればどこにでも現れる可能性があり、IN リスト、テンポラル関数の引数、transformer の引数なども含まれます。

<FlyqlCode code={`status = $code`} />
<FlyqlCode code={`level in [$primary, 'warn']`} />
<FlyqlCode code={`created > ago($duration)`} />

AST: `value_type: "parameter"`、値は `name` と `positional` フィールドを持つ `Parameter` オブジェクトです。

完全なリファレンスは [パラメータ](/ja/syntax/parameters/) を参照してください。
