FlyQL 7 मान प्रकारों का समर्थन करता है: **Integer**, **BigInt**, **Float**, **String**, **Boolean**, **Null**, और **Array**। AST में प्रत्येक मान एक स्पष्ट `value_type` फ़ील्ड रखता है।

## मान प्रकार

### Integer

int64 रेंज के भीतर पूर्ण संख्याएँ:

<FlyqlCode code={`count = 42`} />
<FlyqlCode code={`status != 200`} />
<FlyqlCode code={`age >= 18`} />

AST: `value_type: "integer"`

### BigInt

int64 रेंज (Go/Python) या `MAX_SAFE_INTEGER` (JavaScript) से अधिक पूर्णांक:

<FlyqlCode code={`id = 18446744073709551615`} />

AST: `value_type: "bigint"`

### Float

दशमलव बिंदु वाली संख्याएँ:

<FlyqlCode code={`price > 9.99`} />
<FlyqlCode code={`ratio = 3.14`} />
<FlyqlCode code={`temperature<=-0.5`} />

AST: `value_type: "float"`

### String

टेक्स्ट मान, उद्धृत या अनउद्धृत:

<FlyqlCode code={`level = error`} />
<FlyqlCode code={`user = "John Doe"`} />
<FlyqlCode code={`message = 'hello world'`} />
<FlyqlCode code={`count = '42'`} />

AST: `value_type: "string"`

बैकस्लैश से उद्धरण चिह्न एस्केप करें:

```
user = 'John
message="said \"hello\""
```

### Boolean

कीवर्ड <FlyqlToken type="boolean">true</FlyqlToken> और <FlyqlToken type="boolean">false</FlyqlToken> (लोअरकेस, केस-संवेदनशील):

<FlyqlCode code={`active = true`} />
<FlyqlCode code={`debug = false`} />
<FlyqlCode code={`active != true`} />

AST: `value_type: "boolean"`, मान नेटिव बूलियन है।

बड़े अक्षर वाले रूप (`True`, `FALSE`) स्ट्रिंग के रूप में माने जाते हैं।

**SQL जनरेशन:**
| डायलेक्ट | FlyQL | जनरेटेड SQL |
|---------|-------|---------------|
| ClickHouse | <FlyqlCode code={`field = true`} inline /> | `field = true` |
| ClickHouse | <FlyqlCode code={`field = false`} inline /> | `field = false` |
| PostgreSQL | <FlyqlCode code={`field = true`} inline /> | `field = TRUE` |
| PostgreSQL | <FlyqlCode code={`field = false`} inline /> | `field = FALSE` |
| StarRocks | <FlyqlCode code={`field = true`} inline /> | `field = true` |
| StarRocks | <FlyqlCode code={`field = false`} inline /> | `field = false` |

### Null

कीवर्ड <FlyqlToken type="null">null</FlyqlToken> (लोअरकेस, केस-संवेदनशील):

<FlyqlCode code={`field = null`} />
<FlyqlCode code={`field != null`} />

AST: `value_type: "null"`, मान नेटिव null है।

बड़े अक्षर वाले रूप (`Null`, `NULL`) स्ट्रिंग के रूप में माने जाते हैं।

**प्रतिबंध:** null के साथ केवल <FlyqlToken type="operator">=</FlyqlToken> और <FlyqlToken type="operator">!=</FlyqlToken> ऑपरेटर मान्य हैं। null के साथ <FlyqlToken type="operator">{'>'}</FlyqlToken>, <FlyqlToken type="operator">{'<'}</FlyqlToken>, <FlyqlToken type="operator">{'>='}</FlyqlToken>, <FlyqlToken type="operator">{'<='}</FlyqlToken>, <FlyqlToken type="operator">{'~'}</FlyqlToken>, या <FlyqlToken type="operator">{'!~'}</FlyqlToken> का उपयोग करने पर पार्स त्रुटि होती है।

**SQL जनरेशन:** <FlyqlCode code={`field = null`} inline /> `field IS NULL` उत्पन्न करता है, <FlyqlCode code={`field != null`} inline /> `field IS NOT NULL` उत्पन्न करता है (सभी डायलेक्ट)।

### Array

ऐरे <FlyqlToken type="operator">in</FlyqlToken> और <FlyqlToken type="operator">not in</FlyqlToken> ऑपरेटर के साथ उपयोग किए जाते हैं। तत्व मिश्रित प्रकार (विषम) हो सकते हैं:

<FlyqlCode code={`status in [200, 201, 202]`} />
<FlyqlCode code={`name in ['alice', 'bob']`} />
<FlyqlCode code={`field in [1, "hello", true, null]`} />

प्रत्येक तत्व `values_types` ऐरे में अपना प्रकार रखता है।

## Truthy/Falsy अभिव्यक्तियाँ

बिना ऑपरेटर वाली स्वतंत्र कुंजी जाँचती है कि फ़ील्ड का मान truthy है:

<FlyqlCode code={`active`} />
<FlyqlCode code={`message and status`} />

कोई मान **falsy** माना जाता है यदि वह:
- `null` / `None` / अनुपस्थित
- खाली स्ट्रिंग `""`
- शून्य `0`
- बूलियन <FlyqlToken type="boolean">false</FlyqlToken>

बाकी सब कुछ **truthy** है।

Falsy मानों की जाँच के लिए <FlyqlToken type="operator">not</FlyqlToken> का उपयोग करें:

<FlyqlCode code={`not archived`} />
<FlyqlCode code={`active and not debug`} />

Truthy/falsy जाँच null तुलनाओं से भिन्न हैं: <FlyqlCode code={`not field`} inline /> तब मैच करता है जब फ़ील्ड null, खाली, या शून्य हो। <FlyqlCode code={`field = null`} inline /> केवल तभी मैच करता है जब फ़ील्ड विशेष रूप से null हो।

## व्हाइटस्पेस

FlyQL व्हाइटस्पेस-सहिष्णु है। ऑपरेटर के आसपास स्पेस वैकल्पिक हैं:

<FlyqlCode code={`status = 200`} />
<FlyqlCode code={`status =200`} />

उपरोक्त सभी समतुल्य हैं।

## प्रकार स्थिरांक मॉड्यूल

सभी प्रकार स्थिरांक `flyql.types` मॉड्यूल से उपलब्ध हैं:

**Python:**
```python
from flyql.types import ValueType
# ValueType.INTEGER, ValueType.BIGINT, ValueType.FLOAT,
# ValueType.STRING, ValueType.BOOLEAN, ValueType.NULL, ValueType.ARRAY
```

**Go:**
```go
import "github.com/iamtelescope/flyql/golang/types"
// types.Integer, types.BigInt, types.Float,
// types.String, types.Boolean, types.Null, types.Array
```

**JavaScript:**
```javascript
import { ValueType } from 'flyql/types'
// ValueType.INTEGER, ValueType.BIGINT, ValueType.FLOAT,
// ValueType.STRING, ValueType.BOOLEAN, ValueType.NULL, ValueType.ARRAY
```

## टेम्पोरल फ़ंक्शन

FlyQL तुलना अभिव्यक्तियों के दाहिने हिस्से में मान के रूप में टेम्पोरल फ़ंक्शन कॉल का समर्थन करता है। ये टाइमस्टैम्प हार्डकोड किए बिना समय-सापेक्ष फ़िल्टर सक्षम करते हैं।

AST: `value_type: "function"`, `value` एक `FunctionCall` ऑब्जेक्ट है।

### `ago(duration)`

वर्तमान समय के सापेक्ष एक टाइमस्टैम्प लौटाता है। संयुक्त अवधियों का समर्थन करता है।

<FlyqlCode code={`timestamp > ago(1h)`} />
<FlyqlCode code={`created_at >= ago(7d)`} />
<FlyqlCode code={`updated_at > ago(1h30m)`} />

**अवधि इकाइयाँ:** `s` (सेकंड), `m` (मिनट), `h` (घंटे), `d` (दिन), `w` (सप्ताह)

### `now()`

वर्तमान टाइमस्टैम्प लौटाता है।

<FlyqlCode code={`expires_at < now()`} />

### `today()`

आज की तारीख लौटाता है। एक वैकल्पिक IANA टाइमज़ोन तर्क स्वीकार करता है।

<FlyqlCode code={`date = today()`} />
<FlyqlCode code={`date = today('Europe/Berlin')`} />

### `startOf(unit)`

वर्तमान दिन, सप्ताह (सोमवार), या महीने की शुरुआत लौटाता है। एक वैकल्पिक टाइमज़ोन स्वीकार करता है।

<FlyqlCode code={`created_at > startOf('day')`} />
<FlyqlCode code={`created_at > startOf('week')`} />
<FlyqlCode code={`created_at > startOf('month', 'Asia/Tokyo')`} />

**मान्य इकाइयाँ:** `'day'`, `'week'`, `'month'`

### ऑपरेटर प्रतिबंध

टेम्पोरल फ़ंक्शन केवल तुलना ऑपरेटरों के साथ उपयोग किए जा सकते हैं: `=`, `!=`, `>`, `>=`, `<`, `<=`।

टेम्पोरल फ़ंक्शन के साथ `~` या `!~` का उपयोग करने पर पार्स त्रुटि होती है। `in`, `has`, `like`, और `ilike` जैसे ऑपरेटर अलग पार्सिंग पथ का उपयोग करते हैं और फ़ंक्शन मानों के साथ संगत नहीं हैं।

### बिना कोष्ठक के कॉलम संदर्भ

कोष्ठक के बिना नंगे फ़ंक्शन नाम कॉलम संदर्भ के रूप में माने जाते हैं, फ़ंक्शन कॉल नहीं:

<FlyqlCode code={`field = ago`} />

यह `value_type: "column"` उत्पन्न करता है, फ़ंक्शन कॉल नहीं।

## माइग्रेशन नोट्स

यदि पिछले FlyQL संस्करण से अपग्रेड कर रहे हैं:

- **ब्रेकिंग:** Go का `ValueTypeString`/`ValueTypeNumber` हटाया गया, 7-मान `types.ValueType` से प्रतिस्थापित
- **ब्रेकिंग:** AST `value_type` फ़ील्ड `"string"`/`"number"` से बदलकर `"integer"`/`"float"`/`"bigint"`/`"string"`/`"boolean"`/`"null"` हुआ
- **ब्रेकिंग:** मिश्रित-प्रकार सूची प्रवर्तन हटाया गया — विषम ऐरे अब मान्य हैं
- **नया:** <FlyqlToken type="boolean">true</FlyqlToken>, <FlyqlToken type="boolean">false</FlyqlToken>, <FlyqlToken type="null">null</FlyqlToken> टाइप्ड लिटरल के रूप में पार्स होते हैं (स्ट्रिंग नहीं)
- **नया:** IN-सूची अभिव्यक्तियों पर `values_types` ऐरे प्रति-तत्व प्रकार प्रदान करता है
- **नया:** `flyql.types` मॉड्यूल कैनोनिकल प्रकार स्थिरांक निर्यात करता है
