FlyQL은 ClickHouse, PostgreSQL, StarRocks 전반에서 인젝션에 안전한 SQL로 변환되는 가벼운 쿼리 언어이며, Go, Python, JavaScript 구현을 제공합니다.

## FlyQL이란?

FlyQL은 데이터를 필터링하기 위해 설계된 단순하고 사람이 읽기 쉬운 쿼리 언어입니다. `status >= 400 and host = prod*` 같은 쿼리를 작성하면 FlyQL이 이를 데이터베이스용 파라미터화 SQL로 변환하거나, 데이터 레코드에 대해 메모리에서 직접 평가합니다.

이 쿼리는 백엔드가 ClickHouse, PostgreSQL, StarRocks 또는 메모리 내 데이터셋인 경우에도 동일하게 동작합니다.

## 주요 기능

- **설계 자체로 인젝션 방지**: 문법상 서브쿼리, `JOIN`, `UNION`, DDL을 표현할 수 없습니다. 보안은 방어 로직이 아니라 구조 자체로 보장됩니다. 생성되는 SQL은 모두 파라미터화된 값을 사용합니다.
- **여러 SQL 방언 지원**: 하나의 쿼리 언어로 방언별 출력을 생성합니다. FlyQL은 JSON 경로, Map 접근, Array 처리 등 타입 특성을 올바르게 반영하여 ClickHouse, PostgreSQL, StarRocks용 SQL을 생성합니다.
- **다중 언어 지원**: Go, Python, JavaScript에 완전한 구현이 제공됩니다. 프런트엔드에서 parse하고, 백엔드에서 validate하고, 어디서든 SQL을 생성할 수 있으며, 같은 쿼리는 같은 결과를 냅니다.
- **의존성 없음**: 모든 언어 구현은 외부 의존성 없는 수작업 상태 머신 parser를 사용합니다. 통합이 간단하고 충돌할 요소가 없습니다.
