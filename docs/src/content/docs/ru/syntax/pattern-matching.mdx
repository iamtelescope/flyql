## Регулярные выражения

Используйте <FlyqlToken type="operator">{'~'}</FlyqlToken> для проверки соответствия регулярному выражению и <FlyqlToken type="operator">{'!~'}</FlyqlToken> для отрицания:

<FlyqlCode code={`message ~ "error.*timeout"`} />
<FlyqlCode code={`path !~ "^/health"`} />

| Оператор | Значение |
|----------|----------|
| <FlyqlToken type="operator">{'~'}</FlyqlToken> | Значение соответствует регулярному выражению |
| <FlyqlToken type="operator">{'!~'}</FlyqlToken> | Значение не соответствует регулярному выражению |

### Синтаксис регулярных выражений

Синтаксис зависит от целевого бэкенда:

- **ClickHouse** — функция `match()` (синтаксис RE2)
- **PostgreSQL** — операторы <FlyqlToken type="operator">{'~'}</FlyqlToken> и <FlyqlToken type="operator">{'!~'}</FlyqlToken> (POSIX-регулярные выражения)
- **StarRocks** — функция `regexp`
- **Сопоставление в памяти** — движок регулярных выражений языка выполнения (Go `regexp`, Python `re2`, JS `RegExp`)

Заключайте шаблон в кавычки, если он содержит пробелы или специальные символы:

<FlyqlCode code={`message ~ "connection (refused|reset)"`} />
<FlyqlCode code={`host ~ "^prod-[0-9]+"`} />

### Флаги регулярных выражений

Вы можете изменить поведение регулярного выражения с помощью инлайн-флагов в начале шаблона, используя синтаксис `(?...)`. FlyQL поддерживает три флага, которые работают во всех бэкендах:

#### `(?i)` — Регистронезависимый поиск

Шаблон сопоставляется без учёта регистра:

<FlyqlCode code={`message ~ "(?i)error"`} />
<FlyqlCode code={`level ~ "(?i)^(warn|error|fatal)$"`} />

`(?i)error` совпадает с «Error», «ERROR», «error» и т. д.

#### `(?m)` — Многострочный режим

Изменяет поведение `^` и `$` так, что они совпадают с началом и концом каждой **строки**, а не всей строки целиком:

<FlyqlCode code={`log ~ "(?m)^ERROR"`} />

Без `(?m)` выражение `^ERROR` совпадает только если вся строка начинается с «ERROR». С `(?m)` оно также совпадает с «INFO first\nERROR second», потому что «ERROR» находится в начале строки.

#### `(?s)` — Dotall

Делает так, что `.` совпадает с **любым** символом, включая перевод строки. По умолчанию `.` не совпадает с `\n`:

<FlyqlCode code={`log ~ "(?s)start.*end"`} />

Без `(?s)` выражение `start.*end` не совпадает со «start\nend», так как `.` не пересекает перевод строки. С `(?s)` — совпадает.

#### Комбинирование флагов

Флаги можно комбинировать в одной группе:

<FlyqlCode code={`log ~ "(?im)^ERROR"`} />
<FlyqlCode code={`log ~ "(?ims)^START.*END$"`} />

#### Поддержка бэкендов

Все три флага работают во всех бэкендах FlyQL:

| Бэкенд | `(?i)` | `(?m)` | `(?s)` |
|--------|--------|--------|--------|
| **ClickHouse** | Yes | Yes | Yes |
| **PostgreSQL** | Yes | Yes | Yes |
| **StarRocks** | Yes | Yes | Yes |
| **Go matcher** | Yes | Yes | Yes |
| **Python matcher** | Yes | Yes | Yes |
| **JavaScript matcher** | Yes | Yes | Yes |

Регистронезависимый поиск можно сочетать с отрицанием, чтобы исключать шаблоны независимо от регистра:

<FlyqlCode code={`message ~ "(?i)error" and message !~ "(?i)warning"`} />

## Подстановочный шаблон

Используйте <FlyqlToken type="operator">{'*'}</FlyqlToken> в конце некавыченного значения для сопоставления по префиксу:

<FlyqlCode code={`host = prod*`} />
<FlyqlCode code={`service = api-*`} />

Совпадает с любым значением, начинающимся с указанного префикса. Подстановочные знаки поддерживаются только в конце значения — `*prod` или `pr*od` не поддерживаются.

### Подстановочный шаблон и регулярные выражения

| Шаблон | Синтаксис | Пример |
|--------|-----------|--------|
| Подстановочный (префикс) | <FlyqlCode code={`key = value*`} inline /> | <FlyqlCode code={`host = prod*`} inline /> |
| Регулярное выражение | <FlyqlCode code={`key ~ "pattern"`} inline /> | <FlyqlCode code={`host ~ "^prod-[0-9]+"`} inline /> |

Используйте подстановочные шаблоны для простого сопоставления по префиксу. Используйте регулярные выражения, когда нужны более сложные шаблоны.

## Примеры

<FlyqlCode code={`message ~ "error|fail"`} />
<FlyqlCode code={`path ~ "^/api/v2"`} />
<FlyqlCode code={`path !~ "^/health"`} />
<FlyqlCode code={`host = prod*`} />
<FlyqlCode code={`host = prod* and message ~ "timeout.*" and path !~ "^/health"`} />
