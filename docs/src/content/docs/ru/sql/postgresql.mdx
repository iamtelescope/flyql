PostgreSQL использует двойные кавычки для идентификаторов, `~` для регулярных выражений, `AND`/`OR` в верхнем регистре и операторы JSONB для вложенного доступа.

## Базовые примеры

| FlyQL | Генерируемый SQL |
|-------|------------------|
| `message = 'hello'` | `"message" = 'hello'` |
| `message != 'hello'` | `"message" != 'hello'` |
| `count > 10` | `"count" > 10` |
| `count <= 100` | `"count" <= 100` |
| `message ~ 'error.*'` | `"message" ~ 'error.*'` |
| `message !~ 'test.*'` | `"message" !~ 'test.*'` |
| `host = prod*` | `"host" LIKE 'prod%'` |

## Логические операторы

PostgreSQL использует `AND`/`OR` в верхнем регистре (в отличие от ClickHouse и StarRocks):

| FlyQL | Генерируемый SQL |
|-------|------------------|
| `a = '1' and b > 10` | `("a" = '1' AND "b" > 10)` |
| `a = '1' or b = '2'` | `("a" = '1' OR "b" = '2')` |
| `not a = '1'` | `NOT ("a" = '1')` |

## Принадлежность к списку

| FlyQL | Генерируемый SQL |
|-------|------------------|
| `status in [200, 201]` | `"status" IN (200, 201)` |
| `env not in ['dev', 'test']` | `"env" NOT IN ('dev', 'test')` |
| `status in []` | `FALSE` |
| `status not in []` | `TRUE` |

## Содержимое (`has` / `not has`)

Оператор `has` генерирует различный SQL в зависимости от типа столбца:

| Тип столбца | FlyQL | Генерируемый SQL |
|-------------|-------|------------------|
| String | `message has 'error'` | `position('error' in "message") > 0` |
| String (с отрицанием) | `message not has 'error'` | `("message" IS NULL OR position('error' in "message") = 0)` |
| Array | `tags has 'web'` | `'web' = ANY("tags")` |
| Array (с отрицанием) | `tags not has 'web'` | `NOT ('web' = ANY("tags"))` |
| JSONB | `data has 'key'` | `"data" ? 'key'` |
| JSONB (с отрицанием) | `data not has 'key'` | `NOT ("data" ? 'key')` |
| Hstore | `metadata has 'key'` | `"metadata" ? 'key'` |

## Столбцы JSONB

PostgreSQL использует операторы JSONB с проверкой типа через `jsonb_typeof()`:

### Сравнение строк

| FlyQL | Генерируемый SQL |
|-------|------------------|
| `data.name = 'test'` | `(jsonb_typeof("data"->'name') = 'string' AND "data"->>'name' = 'test')` |
| `data.user.name = 'john'` | `(jsonb_typeof("data"->'user'->'name') = 'string' AND "data"->'user'->>'name' = 'john')` |

### Числовое сравнение

Числовые сравнения приводят извлечённое значение к числовому типу:

| FlyQL | Генерируемый SQL |
|-------|------------------|
| `data.age > 25` | `(jsonb_typeof("data"->'age') = 'number' AND ("data"->>'age')::numeric > 25)` |

### Операторы JSONB

- `->` возвращает JSONB (используется для промежуточного обхода пути)
- `->>` возвращает текст (используется для извлечения конечного значения)
- `jsonb_typeof()` проверяет тип JSON перед сравнением

## Столбцы Hstore (Map)

PostgreSQL использует тип hstore с оператором `->` для доступа по ключу:

| FlyQL | Генерируемый SQL |
|-------|------------------|
| `metadata.key1 = 'value1'` | `"metadata"->'key1' = 'value1'` |
| `metadata.key1 != 'value1'` | `"metadata"->'key1' != 'value1'` |
| `metadata.pattern ~ 'test.*'` | `"metadata"->'pattern' ~ 'test.*'` |

## Столбцы Array

Массивы PostgreSQL нумеруются с 1 — FlyQL автоматически преобразует индекс с нулевой базой:

| FlyQL | Генерируемый SQL |
|-------|------------------|
| `tags.0 = 'first'` | `"tags"[1] = 'first'` |
| `tags.1 = 'second'` | `"tags"[2] = 'second'` |
| `tags.0 ~ 'tag.*'` | `"tags"[1] ~ 'tag.*'` |

### Настройка столбцов

Столбцы PostgreSQL не имеют параметра `jsonString` — тип JSONB определяется по типу столбца:

```go
// Go
columns := map[string]*postgresql.Column{
    "message": postgresql.NewColumn("message", "varchar", nil),
    "data":    postgresql.NewColumn("data", "jsonb", nil),
}
```

```javascript
// JavaScript
const columns = {
    message: newColumn("message", "varchar", null),
    data:    newColumn("data", "jsonb", null),
}
```

## Трансформеры

Трансформеры генерируют стандартные вызовы функций PostgreSQL:

| FlyQL | Генерируемый SQL |
|-------|------------------|
| `status\|upper = "ERROR"` | `UPPER(status) = 'ERROR'` |
| `status\|lower = "info"` | `LOWER(status) = 'info'` |
| `message\|len > 100` | `LENGTH(message) > 100` |
| `message\|lower\|len > 100` | `LENGTH(LOWER(message)) > 100` |
| `tags\|split(",") has "urgent"` | `'urgent' = ANY(STRING_TO_ARRAY(tags, ','))` |
