## Regex-Matching

Verwende <FlyqlToken type="operator">{'~'}</FlyqlToken>, um einen Wert gegen einen regulären Ausdruck zu prüfen, und <FlyqlToken type="operator">{'!~'}</FlyqlToken>, um das Matching zu negieren:

<FlyqlCode code={`message ~ "error.*timeout"`} />
<FlyqlCode code={`path !~ "^/health"`} />

| Operator | Bedeutung |
|----------|-----------|
| <FlyqlToken type="operator">{'~'}</FlyqlToken> | Wert passt auf das Regex-Muster |
| <FlyqlToken type="operator">{'!~'}</FlyqlToken> | Wert passt nicht auf das Regex-Muster |

### Syntax von Regex-Mustern

Die Regex-Syntax hängt vom Ziel-Backend ab:

- **ClickHouse** – verwendet die Funktion `match()` (RE2-Syntax)
- **PostgreSQL** – verwendet die Operatoren <FlyqlToken type="operator">{'~'}</FlyqlToken> und <FlyqlToken type="operator">{'!~'}</FlyqlToken> (POSIX-Regex)
- **StarRocks** – verwendet die Funktion `regexp`
- **In-Memory-Matcher** – verwendet die Regex-Engine der Host-Sprache (Go `regexp`, Python `re2`, JS `RegExp`)

Setze das Muster in Anführungszeichen, wenn es Leerzeichen oder Sonderzeichen enthält:

<FlyqlCode code={`message ~ "connection (refused|reset)"`} />
<FlyqlCode code={`host ~ "^prod-[0-9]+"`} />

### Regex-Flags

Du kannst das Verhalten von Regex-Mustern mit Inline-Flags am Anfang des Musters ändern, indem du die `(?...)`-Syntax verwendest. FlyQL unterstützt drei Flags, die in allen Backends funktionieren:

#### `(?i)` — Groß-/Kleinschreibung ignorieren

Das Muster gleicht unabhängig von Groß- und Kleinschreibung ab:

<FlyqlCode code={`message ~ "(?i)error"`} />
<FlyqlCode code={`level ~ "(?i)^(warn|error|fatal)$"`} />

`(?i)error` passt auf „Error", „ERROR", „error" usw.

#### `(?m)` — Mehrzeilig

Ändert das Verhalten von `^` und `$`, sodass sie den Anfang und das Ende jeder **Zeile** abgleichen statt den Anfang und das Ende der gesamten Zeichenkette:

<FlyqlCode code={`log ~ "(?m)^ERROR"`} />

Ohne `(?m)`, `^ERROR` greift nur, wenn die gesamte Zeichenkette mit „ERROR" beginnt. Mit `(?m)` wird auch „INFO first\nERROR second" gefunden, weil „ERROR" am Anfang einer Zeile steht.

#### `(?s)` — Dotall

Lässt `.` **jedes** Zeichen einschließlich Zeilenumbrüche abgleichen. Standardmäßig gleicht `.` kein `\n` ab:

<FlyqlCode code={`log ~ "(?s)start.*end"`} />

Ohne `(?s)` matcht `start.*end` nicht auf „start\nend", da `.` den Zeilenumbruch nicht überquert. Mit `(?s)` wird es gefunden.

#### Flags kombinieren

Flags können in einer Gruppe kombiniert werden:

<FlyqlCode code={`log ~ "(?im)^ERROR"`} />
<FlyqlCode code={`log ~ "(?ims)^START.*END$"`} />

#### Backend-Unterstützung

Alle drei Flags funktionieren in jedem FlyQL-Backend:

| Backend | `(?i)` | `(?m)` | `(?s)` |
|---------|--------|--------|--------|
| **ClickHouse** | Yes | Yes | Yes |
| **PostgreSQL** | Yes | Yes | Yes |
| **StarRocks** | Yes | Yes | Yes |
| **Go Matcher** | Yes | Yes | Yes |
| **Python Matcher** | Yes | Yes | Yes |
| **JavaScript Matcher** | Yes | Yes | Yes |

Case-insensitives Matching kann mit Negation kombiniert werden, um Muster unabhängig von der Groß-/Kleinschreibung auszuschließen:

<FlyqlCode code={`message ~ "(?i)error" and message !~ "(?i)warning"`} />

## LIKE-Matching

Verwende <FlyqlToken type="operator">like</FlyqlToken> und <FlyqlToken type="operator">ilike</FlyqlToken> für SQL-artiges Muster-Matching mit Wildcards:

<FlyqlCode code={`host like 'prod%'`} />
<FlyqlCode code={`service like 'api-%'`} />
<FlyqlCode code={`message ilike '%Error%'`} />

| Operator | Bedeutung |
|----------|-----------|
| <FlyqlToken type="operator">like</FlyqlToken> | Wert passt auf das Muster (Groß-/Kleinschreibung beachten) |
| <FlyqlToken type="operator">not like</FlyqlToken> | Wert passt nicht auf das Muster |
| <FlyqlToken type="operator">ilike</FlyqlToken> | Wert passt auf das Muster (Groß-/Kleinschreibung ignorieren) |
| <FlyqlToken type="operator">not ilike</FlyqlToken> | Wert passt nicht auf das Muster (Groß-/Kleinschreibung ignorieren) |

### Wildcards

Zwei Wildcard-Zeichen stehen in <FlyqlToken type="operator">like</FlyqlToken> / <FlyqlToken type="operator">ilike</FlyqlToken>-Mustern zur Verfügung:

| Wildcard | Bedeutung | Beispiel | Trifft zu |
|----------|-----------|----------|-----------|
| `%` | Beliebige Zeichenfolge (auch leer) | <FlyqlCode code={`host like 'prod%'`} inline /> | `prod`, `prod-1`, `production` |
| `_` | Genau ein Zeichen | <FlyqlCode code={`code like 'E__'`} inline /> | `E01`, `Exx`, aber nicht `E1` oder `Error` |

Wildcards können überall im Muster erscheinen — am Anfang, in der Mitte, am Ende oder kombiniert:

<FlyqlCode code={`message like '%timeout%'`} />
<FlyqlCode code={`host like 'web-_-prod'`} />
<FlyqlCode code={`path like '/api/%/status'`} />

### Wildcards escapen

Um ein literales `%` oder `_` zu matchen, escape es mit einem Backslash:

<FlyqlCode code={`value like '100\\%'`} />
<FlyqlCode code={`col like 'foo\\_bar'`} />

`100\%` matcht den literalen String „100%", nicht „100" gefolgt von beliebigem Text.

### Groß-/Kleinschreibung ignorieren

Verwende <FlyqlToken type="operator">ilike</FlyqlToken> für Muster-Matching ohne Berücksichtigung der Groß-/Kleinschreibung:

<FlyqlCode code={`message ilike '%error%'`} />
<FlyqlCode code={`host not ilike 'TEST%'`} />

### LIKE vs. Regex

| Anwendungsfall | Syntax | Beispiel |
|----------------|--------|----------|
| Präfix-Match | <FlyqlCode code={`key like 'prefix%'`} inline /> | <FlyqlCode code={`host like 'prod%'`} inline /> |
| Enthält | <FlyqlCode code={`key like '%word%'`} inline /> | <FlyqlCode code={`msg like '%error%'`} inline /> |
| Komplexes Muster | <FlyqlCode code={`key ~ "pattern"`} inline /> | <FlyqlCode code={`host ~ "^prod-[0-9]+"`} inline /> |

Verwende <FlyqlToken type="operator">like</FlyqlToken> für einfaches Wildcard-Matching. Verwende Regex (<FlyqlToken type="operator">{'~'}</FlyqlToken>), wenn du Zeichenklassen, Alternation oder andere erweiterte Muster brauchst.

### Backend-Unterstützung

Alle SQL-Backends (ClickHouse, PostgreSQL, StarRocks) erzeugen native `LIKE` / `ILIKE` / `NOT LIKE` / `NOT ILIKE` SQL-Operatoren. Der In-Memory-Matcher konvertiert das Muster intern in eine Regex.

:::note
StarRocks `ILIKE` erfordert Version 2.5 oder höher.
:::

## Beispiele

<FlyqlCode code={`message ~ "error|fail"`} />
<FlyqlCode code={`path ~ "^/api/v2"`} />
<FlyqlCode code={`path !~ "^/health"`} />
<FlyqlCode code={`host like 'prod%'`} />
<FlyqlCode code={`host like 'prod%' and message ~ "timeout.*" and path !~ "^/health"`} />
