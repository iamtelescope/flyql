Bem-vindo à documentação do projeto **FlyQL**. Versão atual: **{flyqlVersion}** ([registo de alterações](/pt/changelog)).

O FlyQL é uma linguagem de consulta leve e legível, concebida para filtrar dados. Transpila em SQL parametrizado e à prova de injeção para **ClickHouse**, **PostgreSQL** e **StarRocks** — com implementações completas em **Go**, **Python** e **JavaScript**. Escreva uma consulta uma vez, e o FlyQL gera SQL específico do dialeto para a sua base de dados — ou avalia-a em memória contra registos de dados.

Esta consulta funciona de forma idêntica, quer o seu backend seja ClickHouse, PostgreSQL, StarRocks ou um conjunto de dados em memória.

## Funcionalidades Principais

- **À Prova de Injeção por Conceção** — A gramática não consegue expressar subconsultas, JOINs, UNIONs ou DDL. A segurança é estrutural, não defensiva. Todo o SQL gerado utiliza valores parametrizados.
- **SQL Multi-Dialeto** — Uma linguagem de consulta, saída específica por dialeto. Gera SQL correto para ClickHouse, PostgreSQL e StarRocks com tratamento adequado de tipos (caminhos JSON, acesso a Map, tratamento de Array).
- **Multi-Linguagem** — Implementações completas em Go, Python e JavaScript. Analise no frontend, valide no backend, gere SQL em qualquer lugar — mesma consulta, mesmos resultados.
- **Consultas Parametrizadas** — Marcadores `$name` e `$1` resolvidos em tempo de execução via `bindParams()`. Defina o modelo de uma consulta uma vez e ligue valores em segurança depois — sem concatenação de strings, sem escape manual.
- **Zero Dependências** — Parser de máquina de estados escrito à mão sem dependências externas em qualquer linguagem. Integração trivial, nada para conflitar.
