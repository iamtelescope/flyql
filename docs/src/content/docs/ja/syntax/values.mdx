---
title: 値と型
description: FlyQL の値型、型付きリテラル、式
---

FlyQL は 7 つの値型をサポートします。**Integer**、**BigInt**、**Float**、**String**、**Boolean**、**Null**、**Array** です。AST 内の各値には明示的な `value_type` フィールドが付きます。

## 値型

### Integer

int64 の範囲内に収まる整数です。

```
count = 42
status != 200
age >= 18
```

AST: `value_type: "integer"`

### BigInt

int64 の範囲（Go/Python）または `MAX_SAFE_INTEGER`（JavaScript）を超える整数です。

```
id = 18446744073709551615
```

AST: `value_type: "bigint"`

### Float

小数点を含む数値です。

```
price > 9.99
ratio = 3.14
temperature<=-0.5
```

AST: `value_type: "float"`

### String

クォート付きまたはクォートなしのテキスト値です。

```
level = error              # unquoted string
user = "John Doe"          # double-quoted (spaces)
message = 'hello world'    # single-quoted (spaces)
count = '42'               # quoted number is a string
```

AST: `value_type: "string"`

クォートはバックスラッシュでエスケープできます。

```
user = 'John
message="said \"hello\""
```

### Boolean

`true` と `false` という小文字のキーワードです。大文字小文字は区別されます。

```
active = true
debug = false
active != true
```

AST: `value_type: "boolean"`。値はネイティブ boolean です。

先頭が大文字のバリエーション（`True`、`FALSE`）は文字列として扱われます。

**SQL 生成:**
| 方言 | `field=true` | `field=false` |
|------|--------------|---------------|
| ClickHouse | `field = true` | `field = false` |
| PostgreSQL | `field = TRUE` | `field = FALSE` |
| StarRocks | `field = true` | `field = false` |

### Null

`null` という小文字のキーワードです。大文字小文字は区別されます。

```
field = null
field != null
```

AST: `value_type: "null"`。値はネイティブ null です。

先頭が大文字のバリエーション（`Null`、`NULL`）は文字列として扱われます。

**制約:** null と組み合わせて使えるのは `=` と `!=` のみです。`>`, `<`, `>=`, `<=`, `~`, `!~` を null と使うと parse error になります。

**SQL 生成:** `field = null` は `field IS NULL`、`field != null` は `field IS NOT NULL` を生成します。全方言で共通です。

### Array

Array は `in` と `not in` で使います。要素には異なる型を混在できます。

```
status in [200, 201, 202]
name in ['alice', 'bob']
field in [1, "hello", true, null]
```

各要素は `values_types` 配列にそれぞれの型を持ちます。

## Truthy/Falsy 式

演算子を持たない単独のキーは、そのフィールドが truthy な値を持つかどうかをチェックします。

```
active
message and status
```

次の値は **falsy** とみなされます。
- `null` / `None` / 未存在
- 空文字列 `""`
- 0
- boolean `false`

それ以外は **truthy** です。

falsy をチェックするには `not` を使います。

```
not archived
active and not debug
```

truthy/falsy チェックは null 比較とは異なります。`not field` は、フィールドが null、空、または 0 の場合に一致します。`field = null` は、フィールドが明示的に null の場合にのみ一致します。

## 空白

FlyQL は空白に寛容です。演算子の前後のスペースは省略できます。

```
status = 200
status = 200
status =200
```

上記はすべて同じ意味です。

## 型定数モジュール

型定数はすべて `flyql.types` モジュールから利用できます。

**Python:**
```python
from flyql.types import ValueType
# ValueType.INTEGER, ValueType.BIGINT, ValueType.FLOAT,
# ValueType.STRING, ValueType.BOOLEAN, ValueType.NULL, ValueType.ARRAY
```

**Go:**
```go
import "github.com/iamtelescope/flyql/golang/types"
// types.Integer, types.BigInt, types.Float,
// types.String, types.Boolean, types.Null, types.Array
```

**JavaScript:**
```javascript
import { ValueType } from 'flyql/types'
// ValueType.INTEGER, ValueType.BIGINT, ValueType.FLOAT,
// ValueType.STRING, ValueType.BOOLEAN, ValueType.NULL, ValueType.ARRAY
```

## 移行時の注意

以前の FlyQL バージョンからアップグレードする場合:

- **Breaking:** Go の `ValueTypeString` / `ValueTypeNumber` は削除され、7 値の `types.ValueType` に置き換えられました
- **Breaking:** AST の `value_type` フィールドは `"string"` / `"number"` から `"integer"` / `"float"` / `"bigint"` / `"string"` / `"boolean"` / `"null"` に変更されました
- **Breaking:** 混在型リストの制約が削除され、異種型の array が有効になりました
- **New:** `true`、`false`、`null` は文字列ではなく型付きリテラルとして parse されます
- **New:** IN リスト式の `values_types` 配列で、要素ごとの型を参照できます
- **New:** `flyql.types` モジュールから標準の型定数を export します
