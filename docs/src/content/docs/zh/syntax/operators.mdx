FlyQL 支持以下比较操作符：

## 相等比较

| 操作符 | 名称 | 示例 |
|--------|------|------|
| <FlyqlToken type="operator">=</FlyqlToken> | 等于 | <FlyqlCode code={`status = 200`} inline /> |
| <FlyqlToken type="operator">!=</FlyqlToken> | 不等于 | <FlyqlCode code={`env != 'prod'`} inline /> |

<FlyqlCode code={`status = 200`} />
<FlyqlCode code={`service != "api-gateway"`} />

操作符两侧都必须存在——<FlyqlToken type="key">字段</FlyqlToken>和<FlyqlToken type="value">值</FlyqlToken>。

## 数值比较

| 操作符 | 名称 | 示例 |
|--------|------|------|
| <FlyqlToken type="operator">{'>'}</FlyqlToken> | 大于 | <FlyqlCode code={`status > 399`} inline /> |
| <FlyqlToken type="operator">{'>='}</FlyqlToken> | 大于等于 | <FlyqlCode code={`status >= 400`} inline /> |
| <FlyqlToken type="operator">{'<'}</FlyqlToken> | 小于 | <FlyqlCode code={`duration < 1000`} inline /> |
| <FlyqlToken type="operator">{'<='}</FlyqlToken> | 小于等于 | <FlyqlCode code={`duration <= 500`} inline /> |

<FlyqlCode code={`status >= 400 and duration < 5000`} />
<FlyqlCode code={`response_time > 100 and response_time <= 2000`} />

数值比较操作符是为数字值设计的。

## 字符串比较

相同的操作符（<FlyqlToken type="operator">{'>'}</FlyqlToken>、<FlyqlToken type="operator">{'>='}</FlyqlToken>、<FlyqlToken type="operator">{'<'}</FlyqlToken>、<FlyqlToken type="operator">{'<='}</FlyqlToken>）同样可用于字符串值，此时执行字典序（按字母顺序）比较：

<FlyqlCode code={`name > 'm'`} />
<FlyqlCode code={`path <= '/api/z'`} />

## 正则匹配

| 操作符 | 名称 | 示例 |
|--------|------|------|
| <FlyqlToken type="operator">{'~'}</FlyqlToken> | 正则匹配 | <FlyqlCode code={`message ~ "error.*"`} inline /> |
| <FlyqlToken type="operator">{'!~'}</FlyqlToken> | 不匹配正则 | <FlyqlCode code={`path !~ "^/health"`} inline /> |

<FlyqlCode code={`message ~ "timeout|connection refused"`} />
<FlyqlCode code={`host !~ "^test-.*"`} />

正则语法依赖目标数据库或运行时环境。详见[模式匹配](/zh/syntax/pattern-matching/)。

:::note
`~` 和 `!~` 操作符不能与时间函数值（如 `ago(1h)`、`now()`）一起使用，否则会产生解析错误。同样，`in`、`has`、`like` 和 `ilike` 操作符也与函数值不兼容。
:::

## 列表成员检查

| 操作符 | 名称 | 示例 |
|--------|------|------|
| <FlyqlToken type="operator">in</FlyqlToken> | 在列表中 | <FlyqlCode code={`status in [200, 201]`} inline /> |
| <FlyqlToken type="operator">not in</FlyqlToken> | 不在列表中 | <FlyqlCode code={`env not in ['dev', 'test']`} inline /> |

列表语法的详细规则请查看[列表](/zh/syntax/lists/)。

## 模式匹配 (LIKE)

| 操作符 | 名称 | 示例 |
|--------|------|------|
| <FlyqlToken type="operator">like</FlyqlToken> | Like 模式 | <FlyqlCode code={`message like 'error%'`} inline /> |
| <FlyqlToken type="operator">not like</FlyqlToken> | Not like 模式 | <FlyqlCode code={`host not like 'test%'`} inline /> |
| <FlyqlToken type="operator">ilike</FlyqlToken> | 不区分大小写的 like | <FlyqlCode code={`message ilike 'Error%'`} inline /> |
| <FlyqlToken type="operator">not ilike</FlyqlToken> | 不区分大小写的 not like | <FlyqlCode code={`host not ilike 'TEST%'`} inline /> |

<FlyqlCode code={`message like '%timeout%'`} />
<FlyqlCode code={`host not like '%-staging'`} />
<FlyqlCode code={`path ilike '/API/%'`} />

SQL 风格的模式匹配，支持两个通配符：

- <FlyqlToken type="operator">%</FlyqlToken> — 匹配任意字符序列（包括空序列）
- <FlyqlToken type="operator">_</FlyqlToken> — 精确匹配一个字符

使用反斜杠转义通配符以进行字面匹配：`\%` 匹配字面 `%`，`\_` 匹配字面 `_`。

详见[模式匹配](/zh/syntax/pattern-matching/)。

## 包含检查

| 操作符 | 名称 | 示例 |
|--------|------|------|
| <FlyqlToken type="operator">has</FlyqlToken> | 包含 | <FlyqlCode code={`message has 'error'`} inline /> |
| <FlyqlToken type="operator">not has</FlyqlToken> | 不包含 | <FlyqlCode code={`message not has 'test'`} inline /> |

<FlyqlCode code={`message has 'timeout'`} />
<FlyqlCode code={`tags not has 'deprecated'`} />

<FlyqlToken type="operator">has</FlyqlToken> 会根据字段类型执行不同的检查：

- **字符串** — 子串匹配：<FlyqlCode code={`message has 'error'`} inline /> 当 <FlyqlToken type="key">message</FlyqlToken> 包含 <FlyqlToken type="string">"error"</FlyqlToken> 时为 true
- **Map** — 键是否存在：<FlyqlCode code={`metadata has 'region'`} inline /> 当 map 有 <FlyqlToken type="string">"region"</FlyqlToken> 键时为 true
- **数组/列表** — 元素是否存在：<FlyqlCode code={`tags has 'web'`} inline /> 当数组包含 <FlyqlToken type="string">"web"</FlyqlToken> 时为 true
- **JSON** — 对象是否包含某个键：<FlyqlCode code={`data has 'name'`} inline /> 当 JSON 对象有 <FlyqlToken type="string">"name"</FlyqlToken> 键时为 true
- **空值/空内容** — 对 <FlyqlToken type="operator">has</FlyqlToken> 来说恒为假，对 <FlyqlToken type="operator">not has</FlyqlToken> 来说恒为真

<FlyqlToken type="operator">has</FlyqlToken> 接收的是单个值，而不是列表。若需要判断某值是否属于多个候选值，请使用 <FlyqlToken type="operator">in</FlyqlToken>。

|--------|------|------|

| <FlyqlToken type="operator">{'>='}</FlyqlToken> | 大于等于 | <FlyqlCode code={`count >= 10`} inline /> |
| <FlyqlToken type="operator">{'<'}</FlyqlToken> | 小于 | <FlyqlCode code={`count < 100`} inline /> |
| <FlyqlToken type="operator">{'<='}</FlyqlToken> | 小于等于 | <FlyqlCode code={`count <= 100`} inline /> |
| <FlyqlToken type="operator">{'~'}</FlyqlToken> | 正则匹配 | <FlyqlCode code={`msg ~ "err.*"`} inline /> |
| <FlyqlToken type="operator">{'!~'}</FlyqlToken> | 非正则匹配 | <FlyqlCode code={`msg !~ "^ok"`} inline /> |
| <FlyqlToken type="operator">in</FlyqlToken> | 在列表中 | <FlyqlCode code={`s in [1, 2]`} inline /> |
| <FlyqlToken type="operator">not in</FlyqlToken> | 不在列表中 | <FlyqlCode code={`s not in [3]`} inline /> |
