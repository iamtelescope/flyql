FlyQL 支持 10 种值类型：**Integer**、**BigInt**、**Float**、**String**、**Boolean**、**Null**、**Array**、**Column**、**Function** 和 **Parameter**。AST 中的每个值都携带一个明确的 `value_type` 字段。

## 值类型

### Integer

int64 范围内的整数：

<FlyqlCode code={`count = 42`} />
<FlyqlCode code={`status != 200`} />
<FlyqlCode code={`age >= 18`} />

AST：`value_type: "int"`

### BigInt

超出 int64（Go/Python）或 `MAX_SAFE_INTEGER`（JavaScript）范围的整数：

<FlyqlCode code={`id = 18446744073709551615`} />

AST：`value_type: "bigint"`

### Float

带小数点的数字：

<FlyqlCode code={`price > 9.99`} />
<FlyqlCode code={`ratio = 3.14`} />
<FlyqlCode code={`temperature<=-0.5`} />

AST：`value_type: "float"`

### String

文本值，可加引号也可不加：

<FlyqlCode code={`level = 'error'`} />
<FlyqlCode code={`user = "John Doe"`} />
<FlyqlCode code={`message = 'hello world'`} />
<FlyqlCode code={`count = '42'`} />

AST：`value_type: "string"`

使用反斜杠转义引号：

<FlyqlCode code={`user = 'John\\'s'`} />
<FlyqlCode code={`message = "said \\"hello\\""`} />

:::tip
单引号和双引号可以互换使用，因此可以选择能避免转义的那一种。如果值中包含单引号，使用双引号；如果值中包含双引号，使用单引号：

<FlyqlCode code={`user = "John's"`} />
<FlyqlCode code={`message = 'said "hello"'`} />
:::

### Boolean

关键字 <FlyqlToken type="boolean">true</FlyqlToken> 和 <FlyqlToken type="boolean">false</FlyqlToken>（小写，区分大小写）：

<FlyqlCode code={`active = true`} />
<FlyqlCode code={`debug = false`} />
<FlyqlCode code={`active != true`} />

AST：`value_type: "bool"`，值为原生布尔类型。

大写变体（`True`、`FALSE`）会被当作字符串处理。

**SQL 生成：**
| 方言 | FlyQL | 生成的 SQL |
|---------|-------|--------------|
| ClickHouse | <FlyqlCode code={`field = true`} inline /> | `field = true` |
| ClickHouse | <FlyqlCode code={`field = false`} inline /> | `field = false` |
| PostgreSQL | <FlyqlCode code={`field = true`} inline /> | `field = TRUE` |
| PostgreSQL | <FlyqlCode code={`field = false`} inline /> | `field = FALSE` |
| StarRocks | <FlyqlCode code={`field = true`} inline /> | `field = true` |
| StarRocks | <FlyqlCode code={`field = false`} inline /> | `field = false` |

### Null

关键字 <FlyqlToken type="null">null</FlyqlToken>（小写，区分大小写）：

<FlyqlCode code={`field = null`} />
<FlyqlCode code={`field != null`} />

AST：`value_type: "null"`，值为原生空值。

大写变体（`Null`、`NULL`）会被当作字符串处理。

**限制：** 空值仅允许使用 <FlyqlToken type="operator">=</FlyqlToken> 和 <FlyqlToken type="operator">!=</FlyqlToken> 运算符。对空值使用 <FlyqlToken type="operator">{'>'}</FlyqlToken>、<FlyqlToken type="operator">{'<'}</FlyqlToken>、<FlyqlToken type="operator">{'>='}</FlyqlToken>、<FlyqlToken type="operator">{'<='}</FlyqlToken>、<FlyqlToken type="operator">{'~'}</FlyqlToken> 或 <FlyqlToken type="operator">{'!~'}</FlyqlToken> 会产生解析错误。

**SQL 生成：** <FlyqlCode code={`field = null`} inline /> 生成 `field IS NULL`，<FlyqlCode code={`field != null`} inline /> 生成 `field IS NOT NULL`（所有方言）。

### Array

数组与 <FlyqlToken type="operator">in</FlyqlToken> 和 <FlyqlToken type="operator">not in</FlyqlToken> 运算符搭配使用。元素可以是混合类型（异构数组）：

<FlyqlCode code={`status in [200, 201, 202]`} />
<FlyqlCode code={`name in ['alice', 'bob']`} />
<FlyqlCode code={`field in [1, "hello", true, null]`} />

每个元素在 `values_types` 数组中携带自己的类型信息。

## 真值/假值表达式

一个不带运算符的独立键名表示检查该字段是否具有真值：

<FlyqlCode code={`active`} />
<FlyqlCode code={`message and status`} />

以下值被视为**假值**：
- `null` / `None` / 缺失
- 空字符串 `""`
- 零 `0`
- 布尔值 <FlyqlToken type="boolean">false</FlyqlToken>

其他所有值均为**真值**。

使用 <FlyqlToken type="operator">not</FlyqlToken> 检查假值：

<FlyqlCode code={`not archived`} />
<FlyqlCode code={`active and not debug`} />

真值/假值检查与空值比较不同：<FlyqlCode code={`not field`} inline /> 在字段为空值、空字符串或零时匹配。<FlyqlCode code={`field = null`} inline /> 仅在字段确实为空值时匹配。

## 空白字符

FlyQL 对空白字符具有容错性。运算符周围的空格是可选的：

<FlyqlCode code={`status = 200`} />
<FlyqlCode code={`status =200`} />

以上写法完全等价。

## LiteralKind 常量

解析后的表达式值带有 `LiteralKind` 标签。每个实现都从根包导出这些常量。

**Python：**
```python
from flyql import LiteralKind
# LiteralKind.INTEGER, LiteralKind.BIGINT, LiteralKind.FLOAT,
# LiteralKind.STRING, LiteralKind.BOOLEAN, LiteralKind.NULL, LiteralKind.ARRAY
```

**Go：**
```go
import "github.com/iamtelescope/flyql/golang/literal"
// literal.Integer, literal.BigInt, literal.Float,
// literal.String, literal.Boolean, literal.Null, literal.Array
```

**JavaScript：**
```javascript
import { LiteralKind } from 'flyql'
// LiteralKind.INTEGER, LiteralKind.BIGINT, LiteralKind.FLOAT,
// LiteralKind.STRING, LiteralKind.BOOLEAN, LiteralKind.NULL, LiteralKind.ARRAY
```

## 时间函数

FlyQL 支持在比较表达式的右侧使用时间函数调用作为值。这些函数可以实现基于时间的相对过滤，无需硬编码时间戳。

AST：`value_type: "function"`，`value` 是 `FunctionCall` 对象。

### `ago(duration)`

返回相对于当前时间的时间戳。支持复合时间段。

<FlyqlCode code={`timestamp > ago(1h)`} />
<FlyqlCode code={`created_at >= ago(7d)`} />
<FlyqlCode code={`updated_at > ago(1h30m)`} />

**单位（小写）:** `s`（秒）、`m`（分钟）、`h`（小时）、`d`（天）、`w`（周）。

**语法。** 时间段由一个或多个 `<整数><单位>` 对组成。单位必须以 **严格递减顺序** 出现，且 **每个单位最多一次**：`w > d > h > m > s`。这遵循 Prometheus / Grafana / Loki 的约定。

有效:

<FlyqlCode code={`ago(30s)`} />
<FlyqlCode code={`ago(2h15m)`} />
<FlyqlCode code={`ago(1w2d3h4m5s)`} />
<FlyqlCode code={`ago(1w30s)`} />

解析时被拒绝:

- `ago(30m1h)` — 单位顺序错误
- `ago(1h1h)` — 单位重复
- `ago(3h1w)` — 较大单位出现在较小单位之后
- `ago(1H)` — 单位字母必须为小写

:::note[为什么没有月/年？]
月和年单位故意 **不** 支持：其长度是可变的（28–31 天，365/366 天），因此相同的字面量在不同上下文中会表示不同的秒数。与日历相关的时间窗口通过 [`startOf('month')`](#startofunit) 表达。
:::

### `now()`

返回当前时间戳。

<FlyqlCode code={`expires_at < now()`} />

### `today()`

返回今天的日期。接受可选的 IANA 时区参数。

<FlyqlCode code={`date = today()`} />
<FlyqlCode code={`date = today('Europe/Berlin')`} />

### `startOf(unit)`

返回当前日、周（周一）或月的起始时间。接受可选的时区参数。

<FlyqlCode code={`created_at > startOf('day')`} />
<FlyqlCode code={`created_at > startOf('week')`} />
<FlyqlCode code={`created_at > startOf('month', 'Asia/Tokyo')`} />

**有效单位：** `'day'`、`'week'`、`'month'`

### 操作符限制

时间函数只能与比较操作符配合使用：`=`、`!=`、`>`、`>=`、`<`、`<=`。

对时间函数使用 `~` 或 `!~` 会产生解析错误。`in`、`has`、`like` 和 `ilike` 等操作符使用不同的解析路径，与函数值不兼容。

### 无括号的列引用

不带括号的函数名会被视为列引用，而非函数调用：

<FlyqlCode code={`field = ago`} />

这会生成 `value_type: "column"`，而不是函数调用。

## Parameter

参数占位符（命名参数 `$name`、位置参数 `$1`）代表将在运行时由 `bindParams()` 解析的值。它们可以出现在任何字面值有效的位置 —— 包括 IN 列表、时间函数参数和 transformer 参数中。

<FlyqlCode code={`status = $code`} />
<FlyqlCode code={`level in [$primary, 'warn']`} />
<FlyqlCode code={`created > ago($duration)`} />

AST：`value_type: "parameter"`，值是带有 `name` 和 `positional` 字段的 `Parameter` 对象。

完整参考见[参数](/zh/syntax/parameters/)。
