Las consultas de FlyQL constan de una o más condiciones conectadas por operadores booleanos (<FlyqlToken type="operator">and</FlyqlToken>, <FlyqlToken type="operator">or</FlyqlToken>, <FlyqlToken type="operator">not</FlyqlToken>). Cada condición es una comparación o una comprobación truthy.

## Estructura básica de una consulta

<FlyqlCode code={`status = 200 and active and not archived`} />

- <FlyqlCode code={`status = 200`} inline /> — <FlyqlToken type="key">status</FlyqlToken> es igual a <FlyqlToken type="value">200</FlyqlToken>
- <FlyqlCode code={`active`} inline /> — <FlyqlToken type="key">active</FlyqlToken> tiene un valor truthy
- <FlyqlCode code={`not archived`} inline /> — <FlyqlToken type="key">archived</FlyqlToken> es falsy (null, vacío, cero o false)

## Más ejemplos

<FlyqlCode code={`service != api or user = "john doe"`} />

<FlyqlCode code={`message ~ "error.*" and not debug`} />

<FlyqlCode code={`(a = 1 or b = 2) and not (c = 3 and d = 4)`} />

<FlyqlCode code={`status in [200, 201] and method not in ['DELETE', 'PUT']`} />

<FlyqlCode code={`message has 'error' and tags not has 'debug'`} />

## Reglas generales

- **Claves independientes** — Una clave sin operador se interpreta como una comprobación truthy: <FlyqlCode code={`active`} inline />
- **Comparaciones** — Una clave con operador debe tener un valor correspondiente: <FlyqlCode code={`status = 200`} inline />
- **Espacios en blanco** — Los espacios alrededor de los operadores son opcionales: <FlyqlCode code={`status=200`} inline /> y <FlyqlCode code={`status = 200`} inline /> son equivalentes
- **Precedencia de operadores** — <FlyqlToken type="operator">and</FlyqlToken> tiene mayor precedencia que <FlyqlToken type="operator">or</FlyqlToken>: <FlyqlCode code={`a = 1 or b = 2 and c = 3`} inline /> se parsea como <FlyqlCode code={`a = 1 or (b = 2 and c = 3)`} inline />
- **Paréntesis** — Usa <FlyqlToken type="operator">( )</FlyqlToken> para cambiar la precedencia: <FlyqlCode code={`(a = 1 or b = 2) and c = 3`} inline />
