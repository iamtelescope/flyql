FlyQL queries consist of one or more conditions connected by boolean operators (<FlyqlToken type="operator">and</FlyqlToken>, <FlyqlToken type="operator">or</FlyqlToken>, <FlyqlToken type="operator">not</FlyqlToken>). Each condition is either a comparison or a truthy check.

## Basic Query Structure

<FlyqlCode code={`status = 200 and active and not archived`} />

- <FlyqlCode code={`status = 200`} inline /> — <FlyqlToken type="key">status</FlyqlToken> field equals <FlyqlToken type="value">200</FlyqlToken>
- <FlyqlCode code={`active`} inline /> — <FlyqlToken type="key">active</FlyqlToken> field has a truthy value
- <FlyqlCode code={`not archived`} inline /> — <FlyqlToken type="key">archived</FlyqlToken> field is falsy (null, empty, zero, or false)

## More Examples

<FlyqlCode code={`service != 'api' or user = "john doe"`} />

<FlyqlCode code={`message ~ "error.*" and not debug`} />

<FlyqlCode code={`(a = 1 or b = 2) and not (c = 3 and d = 4)`} />

<FlyqlCode code={`status in [200, 201] and method not in ['DELETE', 'PUT']`} />

<FlyqlCode code={`message has 'error' and tags not has 'debug'`} />

## General Rules

- **Standalone keys** — A key without an operator is treated as a truthy check: <FlyqlCode code={`active`} inline />
- **Comparisons** — A key with an operator must have a corresponding value: <FlyqlCode code={`status = 200`} inline />
- **Whitespace** — Spaces around operators are optional: <FlyqlCode code={`status=200`} inline /> and <FlyqlCode code={`status = 200`} inline /> are equivalent
- **Operator precedence** — <FlyqlToken type="operator">and</FlyqlToken> binds tighter than <FlyqlToken type="operator">or</FlyqlToken>: <FlyqlCode code={`a = 1 or b = 2 and c = 3`} inline /> is parsed as <FlyqlCode code={`a = 1 or (b = 2 and c = 3)`} inline />
- **Parentheses** — Use <FlyqlToken type="operator">( )</FlyqlToken> to override precedence: <FlyqlCode code={`(a = 1 or b = 2) and c = 3`} inline />
