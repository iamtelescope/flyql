FlyQL 쿼리는 불리언 연산자(`and`, `or`, `not`)로 연결된 하나 이상의 조건으로 구성됩니다. 각 조건은 비교식이거나 truthy 검사입니다.

## 기본 쿼리 구조

```
status = 200 and active and not archived
```

- `status = 200` — `status` 필드가 `200`과 같음
- `active` — `active` 필드가 truthy 값을 가짐
- `not archived` — `archived` 필드가 falsy 임 (`null`, 빈 값, 0 또는 `false`)

## 추가 예제

```
service != api or user = "john doe"
```

```
message ~ "error.*" and not debug
```

```
(a = 1 or b = 2) and not (c = 3 and d = 4)
```

```
status in [200, 201] and method not in ['DELETE', 'PUT']
```

```
message has 'error' and tags not has 'debug'
```

## 일반 규칙

- **단독 키** — 연산자 없는 키는 truthy 검사로 처리됩니다
- **비교식** — 연산자가 있는 키에는 대응하는 값이 있어야 합니다
- **공백** — 연산자 주변 공백은 허용됩니다. `status = 200`과 `status = 200`은 동일합니다
- **연산자 우선순위** — `and`는 `or`보다 더 강하게 결합합니다. 따라서 `a = 1 or b = 2 and c = 3`은 `a = 1 or (b = 2 and c = 3)`으로 parse됩니다
- **괄호** — `(`와 `)`를 사용해 우선순위를 덮어쓰고 조건을 명시적으로 그룹화할 수 있습니다
