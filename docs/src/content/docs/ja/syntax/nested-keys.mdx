## ドット記法

JSON、Map、構造化データ内のネストしたフィールドへアクセスするには、ドット区切りパスを使います。

<FlyqlCode code={`request.url = "/api/users"`} />
<FlyqlCode code={`response.headers.content_type = "application/json"`} />
<FlyqlCode code={`metadata.labels.env = 'prod'`} />

ドットで区切られた各セグメントが、ネスト構造を 1 段階ずつたどります。SQL generator は、これらのパスを各データベース方言に適した構文（JSON パスアクセス、Map lookup など）へ変換します。

## クォート付きキーセグメント

キーセグメント自体にドットが含まれる場合は、そのドットが区切りと解釈されないようにクォートで囲みます。

<FlyqlCode code={`"foo.bar".baz = value`} />

この例では:

- `"foo.bar"` は 1 つのキーセグメントであり、ドットはリテラルです
- <FlyqlToken type="key">baz</FlyqlToken> は `"foo.bar"` の子要素です

クォートがない <FlyqlToken type="key">foo.bar.baz</FlyqlToken> は、<FlyqlToken type="key">foo</FlyqlToken> → <FlyqlToken type="key">bar</FlyqlToken> → <FlyqlToken type="key">baz</FlyqlToken> の 3 セグメントとして扱われます。

### クォートルール

キーセグメントにはシングルクォートとダブルクォートのどちらも使えます。

<FlyqlCode code={`"foo.bar".baz = value`} />
<FlyqlCode code={`'foo.bar'.baz = value`} />

クォートが必要なのは、ドットを含むセグメントだけです。

<FlyqlCode code={`metadata."dotted.key".value = 123`} />

クォート付き識別子を使えば、[予約語](/ja/syntax/reserved-words/)をカラム名として使うこともできます — 例: <FlyqlCode code={`"not" = 1`} inline />。

## パスの深さ

ネストの深さに制限はありません。

<FlyqlCode code={`a.b.c.d.e = value`} />

SQL 生成時やメモリ上の評価時には、各セグメントを順にたどります。

## 演算子との組み合わせ

ネストしたキーは、すべての演算子で利用できます。

<FlyqlCode code={`request.status >= 400`} />
<FlyqlCode code={`response.body ~ "error.*"`} />
<FlyqlCode code={`metadata.tags in ['prod', 'staging']`} />
<FlyqlCode code={`request.headers.content_type != "text/html"`} />
<FlyqlCode code={`not request.authenticated`} />

## 配列要素アクセス

配列型のカラムでは、0 始まりの数値パスセグメントを使って要素にアクセスします。

<FlyqlCode code={`tags.0 = 'important'`} />
<FlyqlCode code={`tags.1 = 'second'`} />

FlyQL の配列インデックスは、**すべての SQL 方言およびインメモリマッチャーで 0 始まり** です。ジェネレーターは flyql のインデックスを各ターゲットのネイティブな SQL 規約に変換します。ClickHouse、StarRocks、PostgreSQL の配列はいずれも SQL レイヤーで 1 始まりであるため、flyql の `tags.0` はどの方言でも `tags[1]`（方言ごとの識別子クォートおよび関数形式付き）として出力されます。つまり、同じ flyql 式が方言ごとに異なる SQL 文字列を生成しますが、同じ基底データに対して **同じ結果に評価されます**。

## 例

<FlyqlCode code={`request.method = 'GET'`} />
<FlyqlCode code={`response.headers.content_type = "application/json"`} />
<FlyqlCode code={`"app.config".debug = true`} />
<FlyqlCode code={`request.status >= 400 and response.body ~ "error" and not request.internal`} />
