FlyQL は次の比較演算子をサポートしています。

## 等価比較

| 演算子 | 名称 | 例 |
|--------|------|----|
| <FlyqlToken type="operator">=</FlyqlToken> | 等しい | <FlyqlCode code={`status = 200`} inline /> |
| <FlyqlToken type="operator">!=</FlyqlToken> | 等しくない | <FlyqlCode code={`env != prod`} inline /> |

<FlyqlCode code={`status = 200`} />
<FlyqlCode code={`service != "api-gateway"`} />

演算子の両側には必ず<FlyqlToken type="key">キー</FlyqlToken>と<FlyqlToken type="value">値</FlyqlToken>が必要です。

## 数値比較

| 演算子 | 名称 | 例 |
|--------|------|----|
| <FlyqlToken type="operator">{'>'}</FlyqlToken> | より大きい | <FlyqlCode code={`status > 399`} inline /> |
| <FlyqlToken type="operator">{'>='}</FlyqlToken> | 以上 | <FlyqlCode code={`status >= 400`} inline /> |
| <FlyqlToken type="operator">{'<'}</FlyqlToken> | より小さい | <FlyqlCode code={`duration < 1000`} inline /> |
| <FlyqlToken type="operator">{'<='}</FlyqlToken> | 以下 | <FlyqlCode code={`duration <= 500`} inline /> |

<FlyqlCode code={`status >= 400 and duration < 5000`} />
<FlyqlCode code={`response_time > 100 and response_time <= 2000`} />

数値比較演算子は数値に対して機能します。文字列に対して使う場合は辞書順比較になります。

## Regex マッチ

| 演算子 | 名称 | 例 |
|--------|------|----|
| <FlyqlToken type="operator">{'~'}</FlyqlToken> | regex に一致 | <FlyqlCode code={`message ~ "error.*"`} inline /> |
| <FlyqlToken type="operator">{'!~'}</FlyqlToken> | regex に一致しない | <FlyqlCode code={`path !~ "^/health"`} inline /> |

<FlyqlCode code={`message ~ "timeout|connection refused"`} />
<FlyqlCode code={`host !~ "^test-.*"`} />

regex パターンは、対象のデータベースまたは runtime の構文に従います。regex フラグと詳細は[パターンマッチ](/ja/syntax/pattern-matching/)を参照してください。

:::note
`~` と `!~` 演算子はテンポラル関数値（例: `ago(1h)`、`now()`）と併用できません。使用すると parse error になります。同様に、`in`、`has`、`like`、`ilike` 演算子も関数値とは互換性がありません。
:::

## リスト所属

| 演算子 | 名称 | 例 |
|--------|------|----|
| <FlyqlToken type="operator">in</FlyqlToken> | リストに含まれる | <FlyqlCode code={`status in [200, 201]`} inline /> |
| <FlyqlToken type="operator">not in</FlyqlToken> | リストに含まれない | <FlyqlCode code={`env not in ['dev', 'test']`} inline /> |

リスト構文の詳細は [リスト](/ja/syntax/lists/) を参照してください。

## パターンマッチ (LIKE)

| 演算子 | 名称 | 例 |
|--------|------|----|
| <FlyqlToken type="operator">like</FlyqlToken> | Like パターン | <FlyqlCode code={`message like 'error%'`} inline /> |
| <FlyqlToken type="operator">not like</FlyqlToken> | Not like パターン | <FlyqlCode code={`host not like 'test%'`} inline /> |
| <FlyqlToken type="operator">ilike</FlyqlToken> | 大文字小文字を区別しない like | <FlyqlCode code={`message ilike 'Error%'`} inline /> |
| <FlyqlToken type="operator">not ilike</FlyqlToken> | 大文字小文字を区別しない not like | <FlyqlCode code={`host not ilike 'TEST%'`} inline /> |

<FlyqlCode code={`message like '%timeout%'`} />
<FlyqlCode code={`host not like '%-staging'`} />
<FlyqlCode code={`path ilike '/API/%'`} />

SQL スタイルのパターンマッチで、2 種類のワイルドカードが使えます。

- <FlyqlToken type="operator">%</FlyqlToken> — 任意の文字列に一致（空文字を含む）
- <FlyqlToken type="operator">_</FlyqlToken> — ちょうど 1 文字に一致

ワイルドカードをリテラルとして一致させるにはバックスラッシュでエスケープしてください。`\%` はリテラルの `%` に、`\_` はリテラルの `_` に一致します。

詳細と例は [パターンマッチ](/ja/syntax/pattern-matching/) を参照してください。

## 包含チェック

| 演算子 | 名称 | 例 |
|--------|------|----|
| <FlyqlToken type="operator">has</FlyqlToken> | 含む | <FlyqlCode code={`message has 'error'`} inline /> |
| <FlyqlToken type="operator">not has</FlyqlToken> | 含まない | <FlyqlCode code={`message not has 'test'`} inline /> |

<FlyqlCode code={`message has 'timeout'`} />
<FlyqlCode code={`tags not has 'deprecated'`} />

<FlyqlToken type="operator">has</FlyqlToken> は型に応じた包含チェックを行います。

- **String** — 部分文字列一致: <FlyqlCode code={`message has 'error'`} inline /> は、<FlyqlToken type="key">message</FlyqlToken> に <FlyqlToken type="string">"error"</FlyqlToken> が含まれていれば true
- **Map** — キー存在チェック: <FlyqlCode code={`metadata has 'region'`} inline /> は、map に <FlyqlToken type="string">"region"</FlyqlToken> キーがあれば true
- **Array/List** — 要素所属チェック: <FlyqlCode code={`tags has 'web'`} inline /> は、array に <FlyqlToken type="string">"web"</FlyqlToken> が含まれていれば true
- **JSON** — キー存在チェック: <FlyqlCode code={`data has 'name'`} inline /> は、JSON オブジェクトに <FlyqlToken type="string">"name"</FlyqlToken> キーがあれば true
- **Null/空** — <FlyqlToken type="operator">has</FlyqlToken> では常に false、<FlyqlToken type="operator">not has</FlyqlToken> では常に true

<FlyqlToken type="operator">has</FlyqlToken> は単一値を受け取ります。値集合への所属判定には <FlyqlToken type="operator">in</FlyqlToken> を使ってください。
