Добро пожаловать в документацию проекта **FlyQL**. Текущая версия: **{flyqlVersion}** ([журнал изменений](/ru/changelog)).

FlyQL — это лёгкий и удобочитаемый язык запросов, предназначенный для фильтрации данных. Он преобразуется в параметризованный SQL с защитой от инъекций для **ClickHouse**, **PostgreSQL** и **StarRocks** — с полными реализациями на **Go**, **Python** и **JavaScript**. Напишите запрос один раз, и FlyQL сгенерирует SQL для вашей базы данных — или выполнит сопоставление прямо в памяти.

<FlyqlCode code={landingIntroQuery} />

Этот запрос работает одинаково независимо от того, используете ли вы ClickHouse, PostgreSQL, StarRocks или набор данных в памяти.

## Ключевые возможности

- **Защита от инъекций на уровне дизайна**: грамматика не позволяет выражать подзапросы, `JOIN`, `UNION` или DDL. Безопасность обеспечивается структурой языка, а не дополнительными защитными мерами.
- **Поддержка нескольких диалектов SQL**: один язык запросов, но разный корректный SQL для каждой СУБД. FlyQL генерирует SQL для ClickHouse, PostgreSQL и StarRocks с учётом особенностей JSON, Map и массивов.
- **Реализации на нескольких языках**: доступны полноценные реализации на Go, Python и JavaScript. Можно парсить на фронтенде, валидировать на бэкенде и генерировать SQL где угодно.
- **Параметризованные запросы**: плейсхолдеры `$name` и `$1`, которые разрешаются во время выполнения через `bindParams()`. Шаблонизируйте запрос один раз и безопасно связывайте значения позже — никакой конкатенации строк и экранирования вручную.
- **Минимум зависимостей**: в Go и JavaScript используется вручную написанный парсер на конечном автомате без внешних библиотек. Python требует `google-re2` для безопасного regex-матчинга на RE2.
