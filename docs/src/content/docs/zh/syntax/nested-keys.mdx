## 点号语法

使用点分路径访问 JSON、Map 或结构化数据中的嵌套字段：

<FlyqlCode code={`request.url = "/api/users"`} />
<FlyqlCode code={`response.headers.content_type = "application/json"`} />
<FlyqlCode code={`metadata.labels.env = prod`} />

每一个 `.` 都表示向下进入一层嵌套结构。SQL 生成器会根据目标数据库，把这些路径转换为对应的访问语法。

## 带引号的键片段

如果某个键片段本身就包含点号，需要用引号包裹它，避免被当作路径分隔符：

<FlyqlCode code={`"foo.bar".baz = value`} />

这里：

- `"foo.bar"` 是一个完整键名，其中的点号是字面字符
- <FlyqlToken type="key">baz</FlyqlToken> 是 `"foo.bar"` 的子字段

如果不加引号，<FlyqlToken type="key">foo.bar.baz</FlyqlToken> 会被解析为三个片段：<FlyqlToken type="key">foo</FlyqlToken> → <FlyqlToken type="key">bar</FlyqlToken> → <FlyqlToken type="key">baz</FlyqlToken>。

### 引号规则

单引号和双引号都可以用于带点号的键片段：

<FlyqlCode code={`"foo.bar".baz = value`} />
<FlyqlCode code={`'foo.bar'.baz = value`} />

只有真正包含点号的片段需要加引号：

<FlyqlCode code={`metadata."dotted.key".value = 123`} />

## 路径深度

嵌套层级没有限制：

<FlyqlCode code={`a.b.c.d.e = value`} />

生成 SQL 或执行内存匹配时，所有片段都会按顺序逐层访问。

## 与操作符组合

嵌套键可以和所有操作符一起使用：

<FlyqlCode code={`request.status >= 400`} />
<FlyqlCode code={`response.body ~ "error.*"`} />
<FlyqlCode code={`metadata.tags in ['prod', 'staging']`} />
<FlyqlCode code={`request.headers.content_type != "text/html"`} />
<FlyqlCode code={`not request.authenticated`} />

## 示例

<FlyqlCode code={`request.method = GET`} />
<FlyqlCode code={`response.headers.content_type = "application/json"`} />
<FlyqlCode code={`"app.config".debug = true`} />
<FlyqlCode code={`request.status >= 400 and response.body ~ "error" and not request.internal`} />
