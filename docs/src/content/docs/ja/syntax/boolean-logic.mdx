## ブール演算子

FlyQL では、条件を組み合わせるために 3 つのブール演算子を使います。

| 演算子 | 意味 | 例 |
|--------|------|----|
| <FlyqlToken type="operator">and</FlyqlToken> | 両方の条件が true である必要がある | <FlyqlCode code={`status = 200 and active`} inline /> |
| <FlyqlToken type="operator">or</FlyqlToken> | どちらか一方が true であればよい | <FlyqlCode code={`level = 'error' or level = 'critical'`} inline /> |
| <FlyqlToken type="operator">not</FlyqlToken> | 直後の式またはグループを否定する | <FlyqlCode code={`not archived`} inline /> |

### 組み合わせた否定

<FlyqlToken type="operator">not</FlyqlToken> は <FlyqlToken type="operator">and</FlyqlToken> や <FlyqlToken type="operator">or</FlyqlToken> と組み合わせて使えます。

<FlyqlCode code={`active and not archived`} />
<FlyqlCode code={`level = 'error' or not healthy`} />

## 演算子の優先順位

FlyQL は **標準の演算子優先順位** に従います。SQL、Python、JavaScript、Go など主要なクエリ言語と同じルールです。

| 優先順位 | 演算子 | 結合 |
|---------:|--------|------|
| 1（最も強い） | <FlyqlToken type="operator">not</FlyqlToken> | 直後のオペランド（続く式または括弧で囲まれたグループ）に結合 |
| 2 | <FlyqlToken type="operator">and</FlyqlToken> | <FlyqlToken type="operator">or</FlyqlToken> より強く結合 |
| 3（最も弱い） | <FlyqlToken type="operator">or</FlyqlToken> | 最も弱く、AND の連鎖全体を結合する |

具体的には、

<FlyqlCode code={`a = 1 or b = 2 and c = 3`} />

は次のように parse されます。

<FlyqlCode code={`a = 1 or (b = 2 and c = 3)`} />

<FlyqlToken type="operator">and</FlyqlToken> が <FlyqlToken type="operator">or</FlyqlToken> より強く結合するためです。同じ優先順位の連鎖は左結合なので、<FlyqlCode code={`a = 1 and b = 2 and c = 3`} inline /> は <FlyqlCode code={`(a = 1 and b = 2) and c = 3`} inline /> として parse されます。

<FlyqlToken type="operator">not</FlyqlToken> は最も強く結合し、その直後の式またはグループのみに適用されます。前にある式全体に適用されることはありません。

## 括弧によるグループ化

<FlyqlToken type="operator">or</FlyqlToken> を <FlyqlToken type="operator">and</FlyqlToken> より先に評価したい場合は、括弧を使ってデフォルトの優先順位を上書きします。

<FlyqlCode code={`(status >= 400 or level = 'error') and service = 'api'`} />

括弧をつけない場合、同じソース

<FlyqlCode code={`status >= 400 or level = 'error' and service = 'api'`} />

は次のように parse され、

<FlyqlCode code={`status >= 400 or (level = 'error' and service = 'api')`} />

まったく別のクエリになります。<FlyqlToken type="operator">or</FlyqlToken> を優先させたいときは、明示的に括弧を使ってください。

### ネストしたグループ化

括弧はネストできます。

<FlyqlCode code={`(a = 1 or b = 2) and not (c = 3 and d = 4)`} />

### グループの否定

<FlyqlToken type="operator">not</FlyqlToken> はグループ全体に適用されます。

<FlyqlCode code={`not (status = 200 and healthy)`} />

これは複合条件全体を否定します — <FlyqlToken type="key">status</FlyqlToken> が 200 ではないか、<FlyqlToken type="key">healthy</FlyqlToken> が false のときにのみ true になります。

## 否定

<FlyqlToken type="operator">not</FlyqlToken> は、truthy チェック、比較式、グループなど、あらゆる式を否定できます。

<FlyqlCode code={`not active`} />
<FlyqlCode code={`not status = 200`} />
<FlyqlCode code={`not (a = 1 and b = 2)`} />
<FlyqlCode code={`active and not archived`} />

二重否定は打ち消し合います: <FlyqlCode code={`not not active`} inline /> は <FlyqlCode code={`active`} inline /> と同じ意味です。

## 例

<FlyqlCode code={`status = 200 and service = 'api'`} />
<FlyqlCode code={`level = 'error' or level = 'critical'`} />
<FlyqlCode code={`status >= 400 or level = 'error' and service = 'api'`} />
<FlyqlCode code={`(status >= 400 or level = 'error') and service = 'api'`} />
<FlyqlCode code={`(status >= 500 and host = 'prod') or (level = 'error' and not service = 'healthcheck')`} />
