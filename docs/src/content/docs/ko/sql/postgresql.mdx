PostgreSQL SQL은 identifier를 큰따옴표로 감싸며, regex에는 `~`, 불리언 연산에는 대문자 `AND`/`OR`, 중첩 접근에는 JSONB operator를 사용합니다.

## 기본 예제

| FlyQL | 생성된 SQL |
|-------|--------------|
| `message = 'hello'` | `"message" = 'hello'` |
| `message != 'hello'` | `"message" != 'hello'` |
| `count > 10` | `"count" > 10` |
| `count <= 100` | `"count" <= 100` |
| `message ~ 'error.*'` | `"message" ~ 'error.*'` |
| `message !~ 'test.*'` | `"message" !~ 'test.*'` |
| `host = prod*` | `"host" LIKE 'prod%'` |

## 불리언 연산자

PostgreSQL은 ClickHouse와 StarRocks와 달리 대문자 `AND`/`OR`를 사용합니다.

| FlyQL | 생성된 SQL |
|-------|--------------|
| `a = '1' and b > 10` | `("a" = '1' AND "b" > 10)` |
| `a = '1' or b = '2'` | `("a" = '1' OR "b" = '2')` |
| `not a = '1'` | `NOT ("a" = '1')` |

## 리스트 멤버십

| FlyQL | 생성된 SQL |
|-------|--------------|
| `status in [200, 201]` | `"status" IN (200, 201)` |
| `env not in ['dev', 'test']` | `"env" NOT IN ('dev', 'test')` |
| `status in []` | `FALSE` |
| `status not in []` | `TRUE` |

## 포함 검사 (`has` / `not has`)

`has` operator는 column 타입에 따라 서로 다른 SQL을 생성합니다.

| Column Type | FlyQL | 생성된 SQL |
|-------------|-------|--------------|
| String | `message has 'error'` | `position('error' in "message") > 0` |
| String (negated) | `message not has 'error'` | `("message" IS NULL OR position('error' in "message") = 0)` |
| Array | `tags has 'web'` | `'web' = ANY("tags")` |
| Array (negated) | `tags not has 'web'` | `NOT ('web' = ANY("tags"))` |
| JSONB | `data has 'key'` | `"data" ? 'key'` |
| JSONB (negated) | `data not has 'key'` | `NOT ("data" ? 'key')` |
| Hstore | `metadata has 'key'` | `"metadata" ? 'key'` |

## JSONB Columns

PostgreSQL은 `jsonb_typeof()`를 통한 타입 검사와 함께 JSONB operator를 사용합니다.

### String Comparison

| FlyQL | 생성된 SQL |
|-------|--------------|
| `data.name = 'test'` | `(jsonb_typeof("data"->'name') = 'string' AND "data"->>'name' = 'test')` |
| `data.user.name = 'john'` | `(jsonb_typeof("data"->'user'->'name') = 'string' AND "data"->'user'->>'name' = 'john')` |

### Numeric Comparison

숫자 비교는 추출한 값을 cast합니다.

| FlyQL | 생성된 SQL |
|-------|--------------|
| `data.age > 25` | `(jsonb_typeof("data"->'age') = 'number' AND ("data"->>'age')::numeric > 25)` |

### JSONB Operators

- `->` 는 JSONB를 반환합니다. 중간 경로 순회에 사용됩니다.
- `->>` 는 text를 반환합니다. 최종 값 추출에 사용됩니다.
- `jsonb_typeof()` 는 비교 전에 JSON 타입을 확인합니다.

## Hstore (Map) Columns

PostgreSQL은 key 접근에 `->` operator와 함께 hstore 타입을 사용합니다.

| FlyQL | 생성된 SQL |
|-------|--------------|
| `metadata.key1 = 'value1'` | `"metadata"->'key1' = 'value1'` |
| `metadata.key1 != 'value1'` | `"metadata"->'key1' != 'value1'` |
| `metadata.pattern ~ 'test.*'` | `"metadata"->'pattern' ~ 'test.*'` |

## Array Columns

PostgreSQL array는 1-based이므로 FlyQL이 0-based index를 자동 변환합니다.

| FlyQL | 생성된 SQL |
|-------|--------------|
| `tags.0 = 'first'` | `"tags"[1] = 'first'` |
| `tags.1 = 'second'` | `"tags"[2] = 'second'` |
| `tags.0 ~ 'tag.*'` | `"tags"[1] ~ 'tag.*'` |

### Column Setup

PostgreSQL column에는 `jsonString` 파라미터가 없습니다. JSONB 타입은 column 타입에서 감지됩니다.

```go
// Go
columns := map[string]*postgresql.Column{
    "message": postgresql.NewColumn("message", "varchar", nil),
    "data":    postgresql.NewColumn("data", "jsonb", nil),
}
```

```javascript
// JavaScript
const columns = {
    message: newColumn("message", "varchar", null),
    data:    newColumn("data", "jsonb", null),
}
```

## Transformers

Transformer는 PostgreSQL 표준 함수 호출을 생성합니다.

| FlyQL | 생성된 SQL |
|-------|--------------|
| `status\|upper = "ERROR"` | `UPPER(status) = 'ERROR'` |
| `status\|lower = "info"` | `LOWER(status) = 'info'` |
| `message\|len > 100` | `LENGTH(message) > 100` |
| `message\|lower\|len > 100` | `LENGTH(LOWER(message)) > 100` |
| `tags\|split(",") has "urgent"` | `'urgent' = ANY(STRING_TO_ARRAY(tags, ','))` |
