FlyQL supports the following comparison operators:

## Equality

| Operator | Name | Example |
|----------|------|---------|
| <FlyqlToken type="operator">=</FlyqlToken> | Equals | <FlyqlCode code={`status = 200`} inline /> |
| <FlyqlToken type="operator">!=</FlyqlToken> | Not equals | <FlyqlCode code={`env != 'prod'`} inline /> |

<FlyqlCode code={`status = 200`} />
<FlyqlCode code={`service != "api-gateway"`} />

Both sides of the operator are required — a <FlyqlToken type="key">key</FlyqlToken> and a <FlyqlToken type="value">value</FlyqlToken>.

## Numeric Comparison

| Operator | Name | Example |
|----------|------|---------|
| <FlyqlToken type="operator">{'>'}</FlyqlToken> | Greater than | <FlyqlCode code={`status > 399`} inline /> |
| <FlyqlToken type="operator">{'>='}</FlyqlToken> | Greater or equals | <FlyqlCode code={`status >= 400`} inline /> |
| <FlyqlToken type="operator">{'<'}</FlyqlToken> | Less than | <FlyqlCode code={`duration < 1000`} inline /> |
| <FlyqlToken type="operator">{'<='}</FlyqlToken> | Less or equals | <FlyqlCode code={`duration <= 500`} inline /> |

<FlyqlCode code={`status >= 400 and duration < 5000`} />
<FlyqlCode code={`response_time > 100 and response_time <= 2000`} />

Numeric comparison operators work on numeric values. When used with string values, lexicographic comparison applies.

## Regex Matching

| Operator | Name | Example |
|----------|------|---------|
| <FlyqlToken type="operator">{'~'}</FlyqlToken> | Regex match | <FlyqlCode code={`message ~ "error.*"`} inline /> |
| <FlyqlToken type="operator">{'!~'}</FlyqlToken> | Not regex match | <FlyqlCode code={`path !~ "^/health"`} inline /> |

<FlyqlCode code={`message ~ "timeout|connection refused"`} />
<FlyqlCode code={`host !~ "^test-.*"`} />

Regex patterns follow the syntax of the target database or runtime. See [Pattern Matching](/syntax/pattern-matching/) for regex flags and details.

:::note
The `~` and `!~` operators cannot be used with temporal function values (e.g., `ago(1h)`, `now()`). Doing so produces a parse error. Similarly, `in`, `has`, `like`, and `ilike` operators are not compatible with function values.
:::

## List Membership

| Operator | Name | Example |
|----------|------|---------|
| <FlyqlToken type="operator">in</FlyqlToken> | In list | <FlyqlCode code={`status in [200, 201]`} inline /> |
| <FlyqlToken type="operator">not in</FlyqlToken> | Not in list | <FlyqlCode code={`env not in ['dev', 'test']`} inline /> |

See the [Lists](/syntax/lists/) page for detailed list syntax rules.

## Pattern Matching (LIKE)

| Operator | Name | Example |
|----------|------|---------|
| <FlyqlToken type="operator">like</FlyqlToken> | Like pattern | <FlyqlCode code={`message like 'error%'`} inline /> |
| <FlyqlToken type="operator">not like</FlyqlToken> | Not like pattern | <FlyqlCode code={`host not like 'test%'`} inline /> |
| <FlyqlToken type="operator">ilike</FlyqlToken> | Case-insensitive like | <FlyqlCode code={`message ilike 'Error%'`} inline /> |
| <FlyqlToken type="operator">not ilike</FlyqlToken> | Case-insensitive not like | <FlyqlCode code={`host not ilike 'TEST%'`} inline /> |

<FlyqlCode code={`message like '%timeout%'`} />
<FlyqlCode code={`host not like '%-staging'`} />
<FlyqlCode code={`path ilike '/API/%'`} />

SQL-style pattern matching with two wildcards:

- <FlyqlToken type="operator">%</FlyqlToken> — matches any sequence of characters (including empty)
- <FlyqlToken type="operator">_</FlyqlToken> — matches exactly one character

Escape wildcards with a backslash to match them literally: `\%` matches a literal `%`, `\_` matches a literal `_`.

See [Pattern Matching](/syntax/pattern-matching/) for more details and examples.

## Containment Check

| Operator | Name | Example |
|----------|------|---------|
| <FlyqlToken type="operator">has</FlyqlToken> | Contains | <FlyqlCode code={`message has 'error'`} inline /> |
| <FlyqlToken type="operator">not has</FlyqlToken> | Does not contain | <FlyqlCode code={`message not has 'test'`} inline /> |

<FlyqlCode code={`message has 'timeout'`} />
<FlyqlCode code={`tags not has 'deprecated'`} />

The <FlyqlToken type="operator">has</FlyqlToken> operator performs a type-dependent containment check:

- **String** — substring match: <FlyqlCode code={`message has 'error'`} inline /> is true if <FlyqlToken type="key">message</FlyqlToken> contains <FlyqlToken type="string">"error"</FlyqlToken>
- **Map** — key existence: <FlyqlCode code={`metadata has 'region'`} inline /> is true if the map has a key <FlyqlToken type="string">"region"</FlyqlToken>
- **Array/List** — item membership: <FlyqlCode code={`tags has 'web'`} inline /> is true if the array contains <FlyqlToken type="string">"web"</FlyqlToken>
- **JSON** — key existence: <FlyqlCode code={`data has 'name'`} inline /> is true if the JSON object has a key <FlyqlToken type="string">"name"</FlyqlToken>
- **Null/empty** — always false for <FlyqlToken type="operator">has</FlyqlToken>, always true for <FlyqlToken type="operator">not has</FlyqlToken>

<FlyqlToken type="operator">has</FlyqlToken> takes a single value (not a list). Use <FlyqlToken type="operator">in</FlyqlToken> for checking membership in a set of values.
