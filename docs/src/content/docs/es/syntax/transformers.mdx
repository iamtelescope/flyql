Los transformers modifican los valores de las columnas antes de compararlos en las consultas o de emitirlos en SQL. Usan la sintaxis de tubería (`|`) y se aplican de izquierda a derecha.

## Sintaxis de tubería

Aplica un transformer a una columna con `|`:

<FlyqlCode code={`message|upper = "ERROR"`} />

Esto transforma el valor de <FlyqlToken type="key">message</FlyqlToken> con <FlyqlToken type="transformer">upper</FlyqlToken> antes de compararlo con `"ERROR"`.

## Encadenamiento

Encadena varios transformers con tuberías adicionales:

<FlyqlCode code={`message|lower|len > 100`} />

Los transformers se aplican de izquierda a derecha: primero se ejecuta <FlyqlToken type="transformer">lower</FlyqlToken> (de string a string) y luego <FlyqlToken type="transformer">len</FlyqlToken> (de string a int). El resultado se compara con `> 100`.

## Sistema de tipos

Cada transformer declara un tipo de entrada y un tipo de salida. Los transformers encadenados deben ser compatibles entre sí: la salida de uno debe coincidir con la entrada del siguiente.

| Tipo | Descripción |
|------|-------------|
| `string` | Valores de texto |
| `int` | Números enteros |
| `float` | Números de coma flotante |
| `bool` | Valores booleanos |
| `array` | Array de valores, normalmente usado con <FlyqlToken type="operator">has</FlyqlToken> |

## Transformers integrados

| Transformer | Entrada | Salida | Descripción |
|-------------|---------|--------|-------------|
| <FlyqlToken type="transformer">upper</FlyqlToken> | string | string | Convierte a mayúsculas |
| <FlyqlToken type="transformer">lower</FlyqlToken> | string | string | Convierte a minúsculas |
| <FlyqlToken type="transformer">len</FlyqlToken> | string | int | Obtiene la longitud del string |
| <FlyqlToken type="transformer">split</FlyqlToken> | string | array | Divide por delimitador: `split(".")` |

## Salida SQL por dialecto

Los transformers generan llamadas a funciones SQL específicas de cada dialecto. Los transformers encadenados producen llamadas anidadas, de dentro hacia fuera:

**Transformer único:** <FlyqlCode code={`status|upper = "ERROR"`} inline />

| Dialecto | SQL generado |
|---------|--------------|
| ClickHouse | `equals(upper(status), 'ERROR')` |
| PostgreSQL | `UPPER(status) = 'ERROR'` |
| StarRocks | `UPPER(\`status\`) = 'ERROR'` |

**Transformers encadenados:** <FlyqlCode code={`message|lower|len > 100`} inline />

| Dialecto | SQL generado |
|---------|--------------|
| ClickHouse | `length(lower(message)) > 100` |
| PostgreSQL | `LENGTH(LOWER(message)) > 100` |
| StarRocks | `LENGTH(LOWER(\`message\`)) > 100` |

**Split con has:** <FlyqlCode code={`message|split(" ") has "hello"`} inline />

| Dialecto | SQL generado |
|---------|--------------|
| ClickHouse | `has(splitByChar(' ', message), 'hello')` |
| PostgreSQL | `'hello' = ANY(STRING_TO_ARRAY(message, ' '))` |
| StarRocks | `` array_contains(SPLIT(`message`, ' '), 'hello') `` |

## Errores de tipo

Si una cadena de transformers tiene tipos incompatibles, el editor muestra un error:

<FlyqlCode code={`message|len|upper`} />

Esto falla porque <FlyqlToken type="transformer">len</FlyqlToken> devuelve `int`, pero <FlyqlToken type="transformer">upper</FlyqlToken> requiere una entrada `string`. El editor muestra: "len outputs int, upper requires string input".
