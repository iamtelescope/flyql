## Принадлежность списку

Используйте `in`, чтобы проверить, входит ли значение в список, и `not in`, чтобы проверить обратное:

```
status in [200, 201, 204]
env not in ['prod', 'staging']
```

## Правила синтаксиса

- значения заключаются в квадратные скобки `[ ]`
- элементы разделяются запятыми
- все значения в списке должны быть одного типа
- строки пишутся в кавычках
- числа пишутся без кавычек

### Пустые списки

```
status in []        # всегда false
status not in []    # всегда true
```

## Согласованность типов

Список должен содержать значения одного типа. Смешивание чисел и строк вызывает ошибку парсинга.

## Примеры

```
status in [200, 201, 204]
env not in ['dev', 'test', 'staging']
method in ['GET', 'POST'] and status not in [401, 403, 404]
```
